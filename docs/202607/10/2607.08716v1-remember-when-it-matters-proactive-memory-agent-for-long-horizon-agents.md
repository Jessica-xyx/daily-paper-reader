## 摘要
在长周期任务中，与决策相关的状态往往分散在不断扩展的轨迹中，而行动智能体必须将其提取出来并采取行动。随着轨迹增长，任务要求、环境事实、先前尝试、诊断和未完成的子目标可能被埋藏在上下文窗口中，甚至被推出窗口，从而在需要时无法影响决策。我们将这种失败模式称为“行为状态衰减”。我们将记忆视为一种主动干预机制，而非被动检索。一个独立的记忆智能体与未经修改的行动智能体并行运行，从最近的轨迹中更新结构化的记忆库，并决定是否注入一个基于记忆的提醒，还是保持沉默。该模块可与前沿行动智能体和现有智能体框架即插即用。在Terminal-Bench 2.0和τ²-Bench上，它提高了较弱和较强行动智能体的pass@1，在Terminal-Bench上提高了+8.3个百分点，在τ²-Bench上提高了+6.8个百分点。消融实验表明，选择性干预优于被动记忆库暴露、始终开启注入、仅作为顾问的指导以及通用检索。作为向开放权重记忆策略迈出的早期一步，我们使用SFT和GRPO在SETA上训练了Qwen3.5-27B，提高了验证奖励，并实现了向Terminal-Bench的部分迁移。

## Abstract
In long-horizon tasks, decision-relevant state is often scattered across an expanding trajectory, while the action agent must surface it and act. As trajectories grow, task requirements, environment facts, prior attempts, diagnoses, and open subgoals can be buried in the context window or pushed beyond it, failing to influence decisions when needed. We call this failure mode "behavioral state decay". We study memory as an active intervention mechanism rather than passive retrieval. A separate memory agent runs alongside an unmodified action agent, updating a structured memory bank from the recent trajectory and deciding whether to inject a memory-grounded reminder or remain silent. The module is plug-and-play with frontier action agents and existing agent harnesses. Across Terminal-Bench 2.0 and $τ^2$-Bench, it improves pass@1 for both weaker and stronger action agents, with gains of +8.3 pp on Terminal-Bench and +6.8 pp on $τ^2$-Bench. Ablations show that selective intervention outperforms passive bank exposure, always-on injection, advisor-only guidance, and general retrieval. As an early step toward open-weight memory policies, we train Qwen3.5-27B on SETA using SFT and GRPO, improving validation reward and achieving partial transfer to Terminal-Bench.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题定义**：长周期任务中，行动智能体需要很多轮工具调用和环境交互。随着轨迹增长，任务要求、环境事实、先前尝试、失败诊断、中间发现和未完成子目标等信息虽然可能仍存在于上下文窗口中，但不再可靠地影响后续决策，导致“行为状态衰减”（Behavioral State Decay）。
- **现有方案局限**：简单的记忆系统主要关注存储和检索，但长周期执行还需要决定何时让记忆重新进入行动循环。被动地暴露全部历史或粗暴地一直注入记忆可能造成延迟、token浪费甚至干扰。
- **研究目标**：将记忆视为一种主动干预机制，而非被动检索。设计一个独立并行的记忆智能体，主动维护结构化记忆库，并选择性向行动智能体注入基于记忆的提醒。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：记忆智能体与行动智能体解耦，在固定间隔（每N步）执行。记忆智能体分两阶段工作：
    - **Phase 1：记忆库管理** – 根据最近轨迹窗口和当前记忆库，通过工具调用更新三个组成部分：
        - `status`（私有状态，追踪进度、未决问题，不暴露给行动智能体）
        - `knowledge`（知识记忆：稳定事实，如任务要求、环境属性、文件路径等）
        - `procedural`（程序记忆：尝试及其结果，如失败命令、成功修复、诊断信号等）
        - 并提供`memory_delete`删除过时条目。
    - **Phase 2：主动干预选择** – 基于更新后的记忆库，决定是否注入一个简洁的提醒（`<context_for_action>`）或保持沉默（`<no_intervention/>`）。提醒内容必须是记忆相关的、具体的，而非宽泛的战略建议。如果选择沉默，行动智能体上下文不变。
- **关键技术细节**：
    - 记忆智能体使用工具调用接口，使记忆管理显式可控。
    - 触发策略：首次步骤后每隔固定步数调用一次，以隔离干预策略本身的效果。
    - 训练部分：使用SFT蒸馏Prompted记忆智能体的轨迹，再用GRPO（基于群体相对策略优化）微调以优化干预时机。
- **与现有方法区别**：不训练行动智能体来管理自己的上下文，也不做通用检索或摘要，而是让记忆智能体自主决定何时以及如何重新激活执行状态。

## 6. 主要结论与发现

- 记忆干预一致且显著提升长周期任务性能：Terminal-Bench上Sonnet 4.5提升+8.3 pp（37.6%→45.9%），Opus 4.6提升+2.4 pp；τ²-Bench上Sonnet 4.5平均提升+6.8 pp，Opus 4.6平均提升+2.5 pp。
- 较强行动智能体也能获益（虽增益较小），说明记忆不是简单弥补能力不足。
- 消融实验表明：
    - 两阶段设计（维护+选择）在平衡多领域表现上最优（宏观平均最高）。
    - 被动暴露全部记忆库（Full-bank）不如选择性干预。
    - 强制每步注入（Always inject）虽在某些微平均上接近，但宏观平均不如选择性沉默。
    - 去除持久记忆库（Injection-only）会导致不稳定，甚至在航空领域下降。
    - Mem0通用检索能改善平均分，但不如完整系统，关键区别在于是否主动决定何时干预。
- 开放权重记忆智能体训练可行：SFT学习记忆管理，GRPO优化干预时机，SETA训练的Qwen3.5-27B在迁移到Terminal-Bench时获得+3.5 pp提升。

## 7. 优点

- **方法新颖**：明确将记忆问题从“存什么/取什么”扩展到“何时干预”，提出两阶段主动干预架构。
- **即插即用**：记忆智能体与行动智能体解耦，不修改行动智能体，可与现有框架和多种模型配合。
- **实验设计全面**：覆盖两种不同性质的长周期基准、两种不同强度行动智能体、多种消融变体、以及有限但可验证的训练迁移实验。
- **定性分析深入**：揭示了六大类有效干预机制（需求/策略激活、环境扎根、失败循环避免、诊断延续、进度/实体追踪等），有助于理解如何发挥作用。
- **初步探索开放权重**：展示了将干预策略从闭源前沿模型蒸馏到开源模型的可能性，具有实际部署意义。

## 8. 不足与局限

- **训练部分证据较弱**：仅在一个数据集（SETA）上训练，迁移到Terminal-Bench提升有限（+3.5 pp），且未在其他类型任务上验证泛化性。强化学习仅聚焦于少数“关键转折步”，可能未充分学习复杂场景。

- **消融实验仅在τ²-Bench上进行**：缺少在Terminal-Bench上的消融，可能影响结论的稳健性。
