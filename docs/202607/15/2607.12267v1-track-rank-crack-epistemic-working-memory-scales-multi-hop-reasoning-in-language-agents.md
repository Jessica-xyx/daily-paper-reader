## 摘要
交错推理和工具使用的语言智能体，随着推理链的延长，其性能急剧下降，即使每个单独的步骤都很简单。我们将此归因于上下文稀释：智能体的调查状态（已确认的、怀疑的、以及仍需要的）仅隐式地存在于不断增长的上下文窗口中，早期的发现被后来的检索所淹没。我们引入了SLEUTH，通过结构化的认知工作记忆使这种状态明确且可操作：智能体维护了基于来源的已确认事实、按证据排序的活跃假设，以及直接驱动其下一步行动的未解决问题。在五个多跳基准测试和五个既有基线上，SLEUTH的优势随难度增加而增长，从HotpotQA上的+5分到4跳链上的+11分，超越了无需多次运行的Reflexion。分析剩余的差距所在，我们发现了证据充分性问题：智能体通常找到了答案但未能承诺，在无谓的验证上耗尽了预算。一个轻量级的承诺触发器解决了这个问题，但仅当智能体已经维护了结构化的状态时：将相同的触发器应用于非结构化智能体则无改进，从而将组织化的认知状态分离为有效承诺的必要条件。最后，在较弱的模型上强制执行协议遵守，在最困难的问题上恢复了高达+19分，表明智能体组织其推理的方式，而非原始模型能力，是扩展多跳推理的积极因素。

## Abstract
Language agents that interleave reasoning and tool use degrade sharply as reasoning chains lengthen, even when each individual step is easy. We trace this to context dilution: an agent's investigative state (what it has confirmed, what it suspects, and what it still needs) lives only implicitly in a growing context window, where early discoveries are buried under later retrievals. We introduce SLEUTH, which makes this state explicit and actionable through a structured epistemic working memory: the agent maintains Confirmed Facts grounded to sources, Active Hypotheses ranked by evidence, and Open Questions that directly drive its next action. Across five multi-hop benchmarks and five established baselines, SLEUTH's advantage grows with difficulty, from +5 points on HotpotQA to +11 on 4-hop chains, surpassing Reflexion without multiple episodes. Analyzing where the remaining gap lies, we identify the evidence sufficiency problem: agents often find the answer but fail to commit, exhausting their budget on needless verification. A lightweight commitment trigger fixes this, but only when the agent already maintains structured state: the identical trigger applied to an unstructured agent yields no improvement, isolating organized epistemic state as the necessary condition for effective commitment. Finally, enforcing protocol adherence on a weaker model recovers up to +19 points on the hardest problems, showing that how an agent organizes its reasoning, not raw model capability, is the active ingredient for scaling multi-hop reasoning.

- **核心问题**：语言智能体在交错推理和工具使用时，随着推理链（多跳）的延长，性能急剧下降。即使每个单跳子问题都很简单，组合起来后智能体仍表现不佳。

- **整体含义**：智能体的推理能力不仅取决于模型本身的推理能力，更取决于如何组织和管理推理过程中的认知状态。一个好的组织方式可以在不增加模型规模或训练的情况下显著提升多跳推理性能。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想：SLEUTH（Structured Layered Evidence-Updated Tracking of Hypotheses）
- SLEUTH 是一个纯提示（prompt-only）驱动的结构化认知工作记忆框架，智能体在每轮行动后维护并输出三部分结构化状态：
  - **已确认事实（Confirmed Facts Ft）**：从工具输出中提取的验证性陈述，每条事实关联具体来源（例如：`[F1] 冯·米勒赢得了超级碗50 MVP（来源：...）`）。事实单调累积，不会被撤销。
  - **活跃假设（Active Hypotheses Ht）**：候选答案，按证据支持度排序。每条假设关联支持/反对的事实ID和置信度（高/中/低）。
  - **开放问题（Open Questions Qt）**：尚未解决的不确定性，每个问题附带一个建议的动作（搜索/查询）和优先级。最高优先级的问题直接决定下一步的工具调用。

### 关键技术细节
- **更新协议**：每轮工具调用后，智能体依次执行：提取事实、更新假设（增加/降低置信度）、更新开放问题（已解决的问题移除，新问题生成）、选择最高优先级的动作执行。
- **证据单调性**：已确认事实持续保留，防止上下文稀释。
- **动作接地**：每个工具调用都由明确的开放问题驱动，消除冗余搜索。
- **承诺机制（Nudge）**：解决“证据充分性问题”（智能体找到答案但不敢提交）。当满足条件时（已消耗预算的70%且已积累至少2条事实），智能体收到一个合成提示，强制其从已确认事实和活跃假设中合成答案并提交。阈值α=0.7（70%预算）为经验最优值，控制探索-利用权衡。
- **实现方式**：完全通过系统提示（system prompt）实现，无需要训练或微调。对于较弱模型，增加格式约束和运行时的格式检查以确保协议遵循（约99%的轮次遵守）。

## 6. 论文的主要结论与发现

1. **结构化工作记忆带来显著提升**：SLEUTH在所有数据集上超过所有基线，且优势随难度增大而扩大（HotpotQA +4.7，4跳链 +10.9，超越Reflexion无需多episode）。
2. **两个关键失效模式**：
   - **信息丢失**（2/3跳为主）：事实被堆积的上下文淹没 → 通过结构化的事实跟踪解决。
   - **证据充分性麻痹**（4跳为主）：智能体找到答案但不敢提交，耗尽预算 → 通过承诺机制（nudge）解决，但该机制仅在已有结构化状态时有效（将nudge应用于ReAct无效）。
3. **组织方式比模型能力更关键**：弱模型（GLM-5）在强制遵循协议后，在4跳问题上恢复+19分，甚至超过强模型（Sonnet）的无结构ReAct。
4. **协议遵守至关重要**：即使相同模型，失去协议遵守会损失高达19分。保持活跃且持续更新的工作记忆是必要的。
5. **承诺机制阈值α=0.7最优**，且在2/3/4跳上稳定。
6. **剩余错误**主要为推理错误（32.1%），而非检索或状态管理问题。

## 7. 优点：方法或实验设计上的亮点

- **轻量且通用**：完全通过提示实现，无需训练、搜索、额外基础设施，可立即应用于任何遵循指令的模型。
- **可解释性**：每轮输出完整的三元组状态（事实、假设、问题），推理轨迹完全透明，便于人类审查。
- **优势随难度增长**：在更长的推理链上效果更显著，正好解决现有方法的瓶颈。
- **弱模型也能受益**：通过强制格式，弱模型在困难问题上的提升高达19分，表明方法论本身有效。
- **消融设计清晰**：通过逐步添加组件（无结构→事实→事实+假设→事实+问题→完整），精确分离每个组件的贡献，证明三部分协同作用而非简单加和。
- **承诺机制的隔离实验**：证明nudge本身对无结构智能体无效，必须与结构化状态结合，有力论证了结构化状态的必要性。

## 8. 不足与局限

- **Token开销**：SLEUTH的输出包含完整工作记忆，比ReAct多消耗约30-50%的token（API成本），但在最难的问题上开销比例较小（1.33× vs 1.50×）。
- **承诺阈值需调优**：α=0.7通过实验确定，可能不适用于任务结构差异极大的场景（如需要极长探索或极短预算），论文未提供自适应机制。
- **事实单调性可能固话错误**：一旦事实被确认，即使后来发现是误导性的也不会撤销，可能导致错误推理路径。论文给出了具体例子（“旧金山” vs “萨克拉门托”），并提出了可能的扩展（事实置信度、矛盾检测、时间引用）。
- **封闭上下文设定**：所有问题提供固定段落集合（distractor setting），未涉及真实开放域中的矛盾或时变信息。
- **推理错误仍是主要瓶颈**：即使SLEUTH解决了状态管理问题，仍有32.1%的4跳错误源于推理错误（从正确证据中得出错误结论），需要正交方法（如验证链、逻辑检索）来进一步解决。
- **没有与图结构或符号控制器方法对比**：论文仅与prompt-based方法对比，未与更强形式化的外部状态管理（如代码执行、知识图谱）对比，尽管这些方法代价更高。
