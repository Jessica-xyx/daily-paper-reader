## 摘要
推测执行通过使用一个更小、更便宜的模型在环境空闲时预测并预启动下一步，从而加速LLM智能体。然而，现有的推测器是无状态的，会丢弃任务之间的所有信息，导致预测质量无法随着经验提升。我们为推测器配备了三种在线记忆系统，这些系统从过去的智能体轨迹中学习：一个跟踪动作序列统计信息的对比式转移表、一个检索上下文相似片段的情景记忆，以及一个抑制重复错误的混淆追踪器。我们在六个基准测试上评估了这种方法，涵盖三种推测类型：动作预测、观察预测和链式预测。记忆增强的推测在动作预测上相对准确率提高了19-39%，在具有重复动作空间的观察预测任务上最高提升了2.5倍。随着记忆的积累，这些增益持续增长，并且在不同成本的推测器模型上具有泛化性。所有推测都是无损的，因为它在空闲期间运行，不增加挂钟时间，且执行器的轨迹与非推测执行完全相同。

## Abstract
Speculative execution accelerates LLM agents by using a smaller, cheaper model to predict and pre-launch the next step while the environment is idle. However, existing speculators are stateless and discard all information between tasks, preventing prediction quality from improving with experience. We equip the speculator with three online memory systems that learn from past agent trajectories: a contrastive transition table tracking action-sequence statistics, an episodic memory retrieving contextually similar segments, and a confusion tracker suppressing recurring errors. We evaluate this approach on six benchmarks spanning three speculation types: action prediction, observation prediction, and chained prediction. Memory-augmented speculation yields a 19--39\% relative accuracy improvement on action prediction and up to a $2.5\times$ increase on observation prediction tasks with repetitive action spaces. These gains grow continuously as memory accumulates and generalize across speculator models of varying cost. All speculation is lossless because it runs during idle time at zero added wall-clock cost, and the actor's trajectory is identical to non-speculative execution.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：LLM智能体在与外部环境交互时（如工具调用、浏览器操作、物理指令），每个步骤都会累积LLM推理延迟和环境执行延迟。现有推测执行方法使用小型模型在环境空闲时预测下一步并预执行，以隐藏延迟，但预测器是**无状态的**，每次任务独立处理，无法从历史经验中学习。
- **核心动机**：智能体任务往往存在高度规律性（如客户服务中的查用户→查订单；家庭任务中的走到货架→拿起杯子）。如果能记住这些模式，预测准确性应能随经验增长。因此，需要为推测器配备在线记忆系统，使其从过去轨迹中积累知识，实现无损加速。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：在原有推测执行架构中，为推测器（一个小型、快速的模型）集成三种互补的在线记忆系统，在环境空闲时间并行运行，更新和检索记忆，提升预测命中率。所有推测均为无损，因为错误预测被丢弃，智能体最终轨迹与非推测执行完全相同。
- **三种记忆系统**：
  1. **对比式转移表（Contrastive Transition Table）**：记录动作序列的统计信息。索引为动作类型对 (a_i → a_j)，存储成功/失败次数及参数名集合。提供置信度和成功率，优先推荐高成功率的动作序列。
  2. **情景记忆（Episodic Memory）**：存储过去完整轨迹片段，包括上下文摘要、动作列表、观察结果、参数来源、任务结果、自动生成的教训。推测时，当前上下文通过嵌入模型检索最相似的top-k片段，作为上下文示例提供给推测器（稀疏检索，延迟<50ms）。此外还记录推测失误的片段（Speculation-miss episodes），帮助从自身错误中学习。
  3. **混淆追踪器（Confusion Tracker）**：记录推测验证后的（预测动作类型, 实际动作类型）对，当某个混淆模式出现≥3次时，作为硬约束注入推测器提示，明确指示应避免该动作类型。
- **记忆更新**：所有记忆系统在每次任务完成后在线更新，不对执行器造成额外延迟。
- **推测类型**：分为三种：
  - **动作预测（Type 1）**：预测智能体下一步动作，在智能体空闲时预启动，节省环境延迟。
  - **观察预测（Type 2）**：预测环境返回的观察，在环境处理时预启动智能体的LLM调用，节省推理延迟。
  - **链式预测（Type 3）**：同时预测观察和下一步动作，节省双重延迟。
- **安全保证**：只对只读操作（如API查询、页面获取）预启动，写操作（有副作用）不执行，确保无损。

## 3. 实验设计：数据集、Benchmark 及对比方法

- **数据集与Benchmark**：涵盖六种基准，按推测类型分组：
  - Type 1（动作预测）：**WebArena**（812个网页导航任务）、**VisualWebArena（VWA）**（910个视觉网页任务）
  - Type 2（观察预测）：**ALFWorld**（134个家庭任务，13个动词）、**PDDL**（60个规划任务）
  - Type 3（链式预测）：**τ²-bench**（164个客服任务，零售+航空领域）、**HotpotQA**（300个多跳问答任务）
- **评估指标**：Type 1和Type 3使用**动作完全匹配率**；Type 2使用**观察完全匹配率**。统一报告**累计准确率**（总正确步数/总步数）。

## 6. 主要结论与发现

- **记忆增强推测在所有六个基准上一致优于无记忆基线**，且增益随记忆积累而增长（尤其Type 2任务）。
- **定量提升**：

- **在线验证**：在HotpotQA上实现1.05x加速（节省892秒/19024秒）；在ALFWorld上随环境延迟增加，节省时间单调增长，速度比峰值在10秒附近。

## 7. 优点

- **方法创新**：首次将系统化的在线记忆系统引入推测执行，克服了无状态推测器的根本局限，且设计轻量、无需额外训练。
- **无损保证**：推测错误不会影响智能体最终行为，零墙钟开销，实用性强。
- **全面评估**：覆盖三种推测类型、六个多样化基准、七种消融设置、多种模型和k值缩放，实验设计严谨、可重复。
- **在线真实数据验证**：不仅依赖重播，还在HotpotQA和ALFWorld上实测延迟与速度比，验证了理论模型。
- **可解释性**：详细分析了各记忆组件贡献、经验积累曲线、步深影响、成功/失败轨迹差异，提供了丰富洞察。
- **低资源需求**：记忆检索仅需轻量嵌入模型，更新规则化无额外LLM调用，整体算力开销很低。

## 8. 不足与局限

- **元素索引预测瓶颈**：在WebArena/VWA上，主要错误来源是页面元素索引（如click_element(7)），索引是任意且动态的，推测器难以预测，制约了绝对准确率上限。
- **记忆组合可能产生干扰**：在某些基准（如VWA）上，完整记忆配置（Full）不如仅用情景记忆（Episodic）或清晰混淆约束。不同领域最优配置不同，依赖手工选择或需要路由器。
- **重播评估的局限**：仅评估推测准确性，未考虑正确推测后实际加速可能改变智能体后续语境（实时反馈循环），需要在线部署进一步验证。
- **基准规模有限**：PDDL和τ²-bench任务数较少（60/164），部分组分析（如成功/失败分歧）可能统计力不足。
- **隐私与安全**：记忆系统存储用户轨迹数据，需匿名化和数据留存策略；推测执行本身无损，但加速可能放大高风险决策的后果。
- **未考虑写操作推测**：所有预执行仅限于只读操作，限制了潜在加速收益（如实际环境中写操作也占一定比例）。
