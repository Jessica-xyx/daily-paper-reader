## 摘要
智能体AI中的一个重大挑战是前瞻记忆：即在其他活动进行时，能够在特定的未来线索或状态下执行一个意图的能力。我们推出PM-Bench，这是一个基于文本的基准测试，用于衡量现代LLM智能体的前瞻记忆能力。受认知科学中的虚拟周范式的启发，PM-Bench评估了LLM智能体在保持用户意图、执行延迟意图以及监控潜在环境变化方面的表现。在模拟的七天一周中，智能体必须继续进行当前活动，同时判断是否有任何延迟任务到期。我们在PM-Bench上比较了八种最先进的LLM在八种不同智能体配置下的表现。PM-Bench在所有设置下都具有挑战性：最佳方法，即GPT-5.4智能体，在我们的评估中仅达到65.1%的F1分数。此外，没有任何单一的策略能在所有模型上主导提升前瞻记忆。我们发布了PM-Bench，作为一个受控测试平台，用于诊断这些失败，并开发支持可靠前瞻行为的训练或推理时干预措施。

## Abstract
A significant challenge in agentic AI is prospective memory: the ability to execute an intention at a specific future cue or state while other activities are ongoing. We introduce PM-Bench, a text-based benchmark for measuring prospective memory capabilities in modern LLM agents. Inspired by the Virtual Week paradigm from cognitive science, PM-Bench evaluates how well LLM agents maintain user intentions, execute delayed intentions, and monitor latent environment changes. Over the course of a simulated seven-day week, agents must continue an ongoing activity while deciding whether any deferred task is due. We compare eight state-of-the-art LLMs on PM-Bench under eight different agent configurations. PM-Bench proves challenging across all settings: the best method, a GPT-5.4 agent, reaches only 65.1\% F1 score under our evaluation. Furthermore, no single strategy for improving prospective memory dominates across models. We release PM-Bench as a controlled testbed for diagnosing these failures and developing training or inference-time interventions that support reliable prospective behavior.

### 论文详细中文总结

#### 1. 核心问题与整体含义
- **研究动机**：随着LLM驱动的智能体越来越多地承担长期任务（如编程、浏览、规划），一个关键失败模式不是知识不足，而是不能在**正确的未来时刻**执行已存储的意图——即**前瞻记忆（Prospective Memory）** 缺陷。认知心理学研究表明，前瞻记忆失败占日常遗忘的约50%，且在航空、医疗等领域具有重大安全影响。现有LLM评估聚焦于回顾性记忆（检索过去信息）或短期交互，缺乏对“在持续活动中何时行动”的隔离测试。
- **整体含义**：提出了一个受控的、可复现的基准测试PM-Bench，用于系统诊断LLM智能体的前瞻记忆能力，并为开发训练或推理时干预提供测试床。实验证明当前最先进模型在此任务上远未解决，且无通用支架策略。

#### 2. 方法论
- **核心思想**：受认知科学中的**Virtual Week**范式启发，构建一个为期**七天**的文本模拟世界。智能体在每个时间步（共80步，每天10-13步）必须：
  1. 从三个**持续活动选项**（A/B/C）中选择一个以推进游戏；
  2. 同时从一组匿名操作句柄（如 `task_22: 吃早餐药`）中选择当前**应该执行**的延迟任务。
- **关键技术细节**：
  - **任务类型**：事件型（基于叙事或状态通道线索）、时间型（特定时间）、通道触发型（需主动查询隐藏状态）、跨日型（多天后执行）、定期型（日常重复）等。还包括取消、重调度、覆盖更新等动态变化。
  - **状态通道**：11个可查询的隐藏通道（时钟、邮件、日历、课程门户等），智能体必须主动决定何时查询才能获取必要线索。
  - **交互协议**：每个步骤遵循“查询-行动”模式：先零次或多次查询通道，然后提交一个持续活动选择加上零个或多个任务句柄。
- **评估指标**：核心指标为**集合级F1（Set-F1）**：
  - TP：在正确步骤选择了正确任务（交集）；
  - FP：选择了当前不应执行的任务；
  - FN：未选择当前应执行的任务；
  - Set-F1 = 2TP / (2TP + FP + FN)。此外还有命中率、延迟、遗漏、错误内容、更新违反、依赖违反等细粒度错误诊断。

#### 3. 实验设计
- **基准场景**：PM-Bench本身作为固定场景，包含83个任务定义（81个可评分），57个事件型、26个时间型、28个常规、55个非常规、15个通道触发、7个跨日任务、11个更新事件等。
- **对比方法**：
  - **模型**：8个前沿LLM，包括GPT-5.4、GPT-5.3-Codex、Llama3.3-70B-Instruct、Mistral Large、Mistral Small、Qwen3-32B、Qwen3-14B、Qwen3-8B。
  - **智能体配置**（8种）：
    - 单智能体基线（无记忆支架）
    - 待办账本单智能体（in-context ledger）
    - 可选心跳（由模型决定是否启动周期性监控提醒）
    - 自动心跳（每60分钟或30分钟强制提醒）
    - 分层多智能体（1个协调器+3个专业子智能体，执行查询并建议任务）
    - 多数投票消融（重放相同查询轨迹，按多数子智能体意见选择任务）
    - 一致投票消融（仅当所有子智能体同意时才选择）
- **评估方式**：基于日志重放的评估器，累积所有步骤的TP/FP/FN。

#### 4. 资源与算力
- 论文中明确提到：Qwen3模型通过**4×H100 GPU**进行推理服务；其他所有模型（GPT、Llama、Mistral）通过**OpenRouter API**调用。
- 未提供训练或推理的总计算时间（FLOPS）或电量等详细数据。实验主要为推理评估，无模型训练。

#### 5. 实验数量与充分性
- **实验数量**：总共**64次运行**（8个模型 × 8种配置）。
- **充分性与公平性**：
  - 覆盖了不同规模（8B到~1T参数）和不同系列（闭源、开源）的模型，具有较好代表性。
  - 配置设计考虑了多种前瞻记忆增强策略（记账、心跳、多智能体），并做了详细的消融分析（投票策略）。
  - 评估固定于同一场景，且经过“完美玩法”验证（所有任务均可按时完成），保证失败归因于模型行为而非基准缺陷。
  - 指标全面，不仅报告宏观F1，还分解了跨日任务、更新敏感任务、不同监控通道的命中率，使结论具有深度。
  - 不足：仅评估了文本场景，未涉及多模态或真实世界环境；所有模型在相同场景下对比，但不同模型对提示格式敏感度未知。

#### 6. 主要结论与发现
- **前瞻记忆非常困难**：最佳配置（GPT-5.4 + 可选心跳）仅达到**65.1%宏观Set-F1**，远未饱和。
- **无通用最佳支架**：不同模型偏好不同干预策略，例如GPT-5.4在可选心跳下最好，而Llama3.3在待办账本下最好。说明支架与模型存在交互，不能一刀切。
- **监控要求高的任务最难**：需要隐藏通道监控的任务（如检查邮件、课程门户）命中率极低（多数设置<10%），而基于叙事线索或时钟的任务稍好。
- **跨日与更新敏感任务未解决**：即使最佳方法在跨日任务命中率仅50%，更新敏感任务<47.2%，说明长期意图保持和动态更新是主要瓶颈。
- **精度-召回权衡**：更积极的方法（如30分钟心跳）召回高但假阳性多（489次误报），保守方法（如待办账本）精度高但召回低。最优方法需校准这个平衡。

#### 7. 优点
- **认知科学理论基础**：借鉴成熟的心理测量范式（Virtual Week），使任务设计具有生态效度和理论支撑。
- **细粒度错误分析**：不仅报告F1，还分解多种失败类型（迟发、遗漏、更新违反等），有助于诊断具体缺陷。
- **可控可扩展**：通过种子生成管道可创建不同场景，且经过自动一致性验证，确保无基准缺陷。
- **全面评估监控维度**：设计了11个隐藏通道，并区分“无需监控”、“需时钟监控”、“需其他通道监控”，揭示不同监控类型的难度差异。
- **提供了强大的比较基线**：包括心跳、记账、多智能体等现实可行的策略，且通过投票消融分离了“证据获取”与“行动决策”两个瓶颈。

#### 8. 不足与局限
- **仅限文本环境**：当前PM-Bench为纯文本，未来需扩展到多模态或真实工具使用场景以更贴近实际应用。
- **合成场景限制**：虽然通过人工审阅确保质量，但任务和叙事为合成，可能无法覆盖真实用户意图的复杂性和多样性。
- **跨模型公平性**：不同模型对提示格式的敏感度可能影响表现（例如待办账本要求JSON输出，部分模型可能不擅长结构化输出），但论文未对此进行控制。
- **计算资源细节缺失**：除了提及Qwen3使用4xH100外，未提供推理延迟、总计算成本等，不利于资源评估。
- **未提供训练/干预方案**：论文定位为诊断基准，并未提出或测试具体的记忆增强训练方法，读者需自行探索。
- **场景规模较小**：只有一周80步，虽然足够揭示趋势，但更长期（多周、数月）的前瞻记忆挑战未覆盖。
- **潜在偏差**：所有评估在单一固定场景上进行，未来需验证基准在不同场景和随机种子下的结果稳健性。
