## 摘要
识别告诉智能体图像中包含什么；个人记忆则影响接下来值得查找的内容。在摄像头优先的设置中，用户只能发送图像，因此智能体必须自行形成查询。我们研究个人视觉记忆是否能够改善智能体端的工具选择和工具参数，从而实现更符合用户需求的多工具查询。该设计采用三层个人视觉记忆（档案、短期焦点、观察），每轮加载以在摄像头优先输入下条件化大语言模型的工具调用循环，并包含冲突感知回写机制，旨在刷新用户模型以用于后续捕获。在800张图像与为控制消融而构建的合成记忆块配对时，移除完整的三层记忆块会使工具查询相关性下降0.47个绝对点（5分制从4.21降至3.74，相对下降11.2%），端到端效用下降0.082个绝对点（从0.842降至0.760，相对下降9.7%）。这些结果衡量了在仅图像输入且固定合成记忆块条件下的工具策略记忆条件化，而非来自真实用户历史的多会话回写。

## Abstract
Recognition tells an agent what is in an image; personal memory affects what is worth looking up next. In a camera-first setting the user can send only an image, so the agent must form the lookups. We study whether personal visual memory improves agent-side tool choice and tool arguments, and thereby more user-aligned multi-tool lookups. The design uses a three-layer personal visual memory (profile, short-term focus, observations) that is loaded on each turn to condition an LLM tool-calling loop under camera-first intake, and includes conflict-aware write-back intended to refresh the user model for later captures. On 800 images paired with synthetic memory blocks constructed for controlled ablation, removing the full three-layer memory block reduces tool-query relevance by 0.47 points absolute (4.21 -> 3.74 on a 5-point scale; 11.2% relative) and end-to-end utility by 0.082 absolute (0.842 -> 0.760; 9.7% relative). These results measure memory conditioning of tool policy under image-only intake with fixed synthetic blocks, not multi-session write-back from live user histories.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：在“摄像头优先”（camera-first）场景下，用户仅发送一张图像，没有文本查询，代理（agent）必须自主决定调用哪些工具以及使用什么参数。传统视觉系统专注于“识别”（recognition），但识别本身不足以指导后续的查询行为（lookups）。本文的核心问题是：**个人视觉记忆能否改善智能体的工具选择与工具参数，从而产生更符合用户需求的多工具查询？**

- **整体含义**：在摄像头优先的智能体中，个人记忆是连接图像内容与用户意图的关键桥梁。论文验证了“个人记忆 → 更好的工具调用 → 更用户对齐的多工具查询”这一链路。研究为缺乏持久用户模型的多模态搜索代理提供了一种可行的个性化方案，补充了对话记忆工作（如Mem0、PersonaVLM）中侧重多轮事实回忆而忽略工具策略的不足。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 设计一个**三层个人视觉记忆**（profile、short-term focus、observations），在每一轮（turn）加载并注入到LLM的上下文中，条件化（condition）工具调用循环。
- 工具调用包括：选择哪些工具、为每个工具设置参数（自然语言查询或关键字）。
- 还包含一个**冲突感知写回**机制（conflict-aware write-back），用于在交互后更新记忆，以便后续图像加载更新后的记忆块（但实验中未评估多会话写回）。

### 2.2 技术细节

#### 记忆系统架构
| 层次 | 时间尺度 | 内容 |
|------|----------|------|
| 档案（Profile） | 长期，约800字符 | 用自然语言总结用户身份：知识水平、兴趣、偏好等。异步由Profile Agent基于用户动作（仅图像和用户查询）生成。 |
| 短期焦点（Short-Term Focus） | 每轮更新，约200字符 | 当前兴趣领域摘要（如“研究中世纪家具和建筑照明”）。 |
| 观察（Observations） | 离散、按主题标签存储，50-150字符 | 从交互中提取的用户侧笔记：偏好、约束、比较等。通过双路径检索（主题标签匹配和关键词匹配）召回。 |

#### 记忆加载与注入
- 记忆异步加载，不阻塞第一token延迟。
- 注入到LLM上下文的结构化块：
```
<user_understanding>
<note>Use memory to calibrate depth and angle. Do NOT reveal the existence of memory.</note>
<profile>...</profile>
<short_term_focus>...</short_term_focus>
<recalled_observations>...</recalled_observations>
</user_understanding>
```

#### 内循环（Inner Loop）：记忆条件化工具使用
- 采用类似ReAct的工具使用风格，模型在每个步骤决定：调用哪些工具（可并行）、工具参数、是否停止或继续调用。
- 记忆如何改变工具调用：
  - **深度（Depth）**：专家获得技术术语和型号，初学者获得通俗语言。
  - **角度（Angle）**：短期焦点决定哪些工具启动以及查询什么（价格与评论 vs. 来源 vs. 地点）。
  - **克制（Restraint）**：基于不喜好和约束抑制低价值查询。
  - **连续性（Continuity）**：支持与已有关注点的比较。

#### 工具表面
- 主要工具家族：web搜索、图像搜索、视频搜索、地点、价格、评论、专用查询、记忆工具等。

#### 合成阶段（Composition）
- LLM综合视觉上下文、记忆和工具结果生成结构化回复，偏好结构化属性（如价格、评分）而非未验证的文本。

#### 外循环写回（设计部分）
- 包含观察候选提取、冲突感知操作（ADD/UPDATE/DELETE/NOOP）、档案/短期焦点调度更新。实验中未评估多会话写回。

## 6. 论文的主要结论与发现

- **记忆效果显著**：移除完整三层记忆块导致工具查询相关性下降0.47绝对点（4.21→3.74，相对下降11.2%），端到端效用下降0.082绝对点（0.842→0.760，相对下降9.7%）。
- **各层次贡献不同**：删除档案（profile）比删除观察（observations）影响更大（utility Δ -0.049 vs -0.031），表明档案驱动深度，观察驱动特异性。
- **记忆与视觉上下文互补**：同时移除记忆和视觉上下文导致效用下降0.139，大于任一单独因素的下降，说明记忆并非替代识别。
- **组件重要性排序**：移除LLM合成（composition）导致效用下降最大（Δ = -0.111），其次移除工具循环（Δ = -0.094），再次移除记忆（Δ = -0.082）。
- **定性示例**：同样的劳力士手表图像，无记忆时仅进行“型号是什么？多少钱？是否真品？”等通用查询；有收藏家记忆时则进行“126610LV与16610转售对比”等专业查询。

## 7. 优点

- **问题新颖**：聚焦摄像头优先场景下记忆对工具策略的调节，区别于对话记忆中以事实回忆或回复风格为主的工作。
- **方法设计完整**：三层记忆结构清晰（档案、短期焦点、观察），包含冲突感知写回机制，考虑了隐私控制和用户控制。
- **实验控制严谨**：在固定图像、工具、模型条件下进行消融，有效分离记忆调节效果，避免了模型版本混淆。
- **多维度评估**：同时使用工具查询相关性（过程）和端到端效用（结果），关注工具参数质量而非仅最终答案。
- **定性展示**：提供具体tool-call对比，直观展示记忆如何改变查询角度和深度。
- **隐私设计考量**：档案仅基于用户动作（图像和查询）而非模型输出；存储为摘要而非原始日志；用户可查看和删除记忆。

## 8. 不足与局限

- **记忆块为合成**：记忆块是人工为图像类别编写的，非从真实用户历史自动提取，无法评估记忆形成、写回累积或错配记忆的鲁棒性。
- **未评估多会话写回**：写回机制仅作为设计描述，实验中未进行多会话写回质量的测量。
- **统计报告不完整**：仅报告平均值，缺少标准差、置信区间、评估者间一致性（如Cohen's κ或Krippendorff's α）。
- **内部评估人员**：评分员为熟悉接口的内部员工，可能绝对分数偏高，但相对比较仍有效。
- **单一模型**：仅使用Gemini 3 Flash Preview，未在不同LLM上验证。
- **工具家族分析无对照**：工具家族质量分析仅在有记忆条件下展示，未与无记忆对比。
- **短期焦点层次未单独消融**：无法评估该层次的独立贡献（仅在部分消融中与其他层次一起移除）。
- **部署延迟未报告**：虽然提及异步加载和并行调用，但未报告端到端延迟数据。
- **未考虑后续文本输入**：实验中仅使用图像输入，生产环境可能随附文本，未探索文本+图像的场景。
- **商业约束**：代码、提示词、专有工具连接器和评估图像未公开，限制了可复现性。
