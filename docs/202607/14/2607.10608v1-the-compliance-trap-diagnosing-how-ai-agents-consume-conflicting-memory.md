## 摘要
记忆正成为长周期AI代理的核心组件，使代理能够在操作浏览器、软件工具和其他交互环境时重用过去的经验。现有工作大多将记忆视为供给问题，关注写入哪些经验、如何存储以及为下一个任务检索哪个条目。然而，我们仍然缺乏对模型如何在多步动作轨迹中消费检索到的记忆的清晰描述。这一消费过程至关重要，因为它不仅决定了应检索哪些记忆，还决定了需要什么样的模型和控制策略才能安全地使用它们。为了诊断这一过程，我们提出了条目-传播-恢复（E-P-R）框架，这是一个轨迹级框架，询问记忆首次改变动作的位置、该改变是否向前传播，以及代理在偏离正确路径后能否恢复。我们在WebArena和MemTrapBench（我们构建的用于隔离这些阶段的受控基准）上实例化了E-P-R。我们发现主要失败通常始于条目阶段：即使在任务错误的情况下，代理在第一个暴露的决策点上就会接受冲突记忆。重复暴露则会放大这一早期错误，而偏离后的恢复能力较弱。这些效应共同导致了合规陷阱：跨模型来看，冲突记忆诱发了相似的合规率，但一旦代理合规，其成功率就会骤降至低点。因此，更强的代理遭受的绝对损害更大，因为每次合规事件都会抹去更多的基线能力。这些结果表明，记忆增强型代理不仅应通过检索质量或最终成功率来评估，还应通过它们在轨迹中如何消费记忆来评估。

## Abstract
Memory is becoming a core component of long-horizon AI agents, allowing agents to reuse past experience when operating web browsers, software tools, and other interactive environments. Existing work mostly treats memory as a supply problem, asking what experience to write, how to store it, and which entry to retrieve for the next task. Yet we still lack a clear account of how models consume retrieved memory across a multi-step action trajectory. This consumption process matters because it determines not only what memories should be retrieved, but also what models and control policies are needed to use them safely. To diagnose this process, we propose Entry--Propagation--Recovery (E-P-R), a trajectory-level framework that asks where memory first changes an action, whether that change carries forward, and whether the agent can recover after leaving a correct path. We instantiate E-P-R on WebArena and on MemTrapBench, a controlled benchmark we build to isolate these phases. We find that the main failure often begins at entry: agents adopt conflicting memory at the first exposed decision point even when it is task-wrong. Repeated exposure then amplifies this early error, while recovery after divergence is weak. Together, these effects create a compliance trap: across models, conflicting memory induces similar compliance rates, but once agents comply, their success rates collapse to a low floor. Stronger agents therefore suffer larger absolute damage because each compliance event erases more baseline capability. These results suggest that memory-augmented agents should be evaluated not only by retrieval quality or final success rate, but by how they consume memory throughout the trajectory.

### 1. 论文的核心问题与整体含义（研究动机和背景）

*   **研究动机**
    *   现有的AI代理记忆研究主要聚焦于“供应”问题，即**如何编写、存储和检索记忆**以用于新任务。这些研究隐含假设是，一旦记忆被放入上下文，代理策略会自动决定如何使用它。
    *   然而，代理策略本身是不完美的语言模型，检索到记忆并不能决定模型会如何使用它。**有用的记忆可能被忽略**，而**冲突的、错误的记忆**则可能被盲目遵循，导致风险。
    *   作者提出，我们缺乏一个清晰的、轨迹级的描述来解释：模型**如何消费**检索到的记忆，以及这个消费过程在何处、如何导致失败。

*   **核心问题**
    *   长周期AI代理在长期的、多步的动作轨迹中，究竟是如何**消费**已被检索到的冲突记忆的？
    *   冲突记忆是在哪个阶段对代理的行为产生决定性影响的？这种影响是暂时的还是会持续累积？代理能否纠正这种错误影响？

*   **整体含义**
    *   记忆增强型代理的安全性和可靠性不仅取决于检索到的记忆内容，更**取决于代理在轨迹中如何消费这些记忆**。因此，对代理的评估应基于其消费行为，而不仅仅是检索质量或最终成功率。

### 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想：E-P-R轨迹级诊断框架**
    *   作者将记忆消费过程分解为三个可测量的阶段：
        *   **条目 (Entry)**: 记忆是否在首次暴露的决策点（通常是第一步）改变了代理的动作？
        *   **传播 (Propagation)**: 这个初始改变是否在后续的重复暴露下持续存在或放大？
        *   **恢复 (Recovery)**: 当记忆导致轨迹偏离正确路径后，代理能否纠正并回到正确轨道上？

*   **关键技术细节**
    *   **记忆干预**: 通过固定记忆文本，仅改变其**注入时机**（早期/持久性/晚期）来探测E-P-R的三个阶段。
        *   **早期注入 (Early)**: 只在第一步暴露记忆 → **探测 Entry**。
        *   **持久注入 (Persistent)**: 在每一步都暴露记忆 → **探测 Propagation**。
        *   **晚期注入 (Late)**: 在代理执行了几步操作后才暴露记忆 → **探测 Recovery**（通过对抗已形成的动作历史）。
    *   **合规性度量**
        *   **推荐合规率 (RCR)**: 衡量代理在第一步是否执行了记忆中的错误建议。用LLM裁判判断代理的第一步动作是否执行了记忆给出的第一个指令。
        *   **每次合规的损害 (DPC)**: 在代理合规的任务子集上，衡量使用冲突记忆相对于无记忆基线的**成功率下降幅度**。公式为：`DPC(CRCR) = 1/|CRCR| * Σ(y0_i - yc_i)`，其中 `y0` 是无记忆的成功率，`yc` 是冲突记忆下的成功率。这有效分离了“合规率”和“合规代价”。
    *   **门控因果探针**：在MemTrapBench上，通过2x2析因实验，正交地控制**条目（Entry）** 和 **指示落地（Grounding）** 两个门，以隔离地测试各门控对最终损害/收益的因果贡献。

### 6. 论文的主要结论与发现

*   **发现了“合规陷阱”**
    *   **合规率相似**: 在冲突记忆下，所有模型的合规率（第一步遵循错误建议）高度相似（约63-72%），与模型能力大小无关。
    *   **合规后性能崩溃**: 一旦代理合规，其成功率骤降至一个很低且相似的水平（17-31%），远低于无记忆基线在同一子集上的表现。
    *   **强模型受损更大**: 由于更强的模型基线成功率更高，但合规后同样跌至低点，因此它们遭受的**绝对损害（DPC）更大**。例如，Qwen3.5-27B的DPC高达25.5个百分点，而较弱的Qwen3.5-9B为9.4个百分点。
*   **冲突记忆“易进难出”**
    *   **条目 (Entry)**: 冲突记忆在**早期**就进入轨迹，其首次分歧的概率在几步内就达到86-94%。
    *   **传播 (Propagation)**: **重复暴露**会显著放大早期错误。持久性冲突记忆比早期冲突记忆造成的损害大得多。
    *   **恢复 (Recovery)**: 代理从冲突记忆引发的偏离中恢复的能力**非常弱**（7-15%），远低于从有益记忆偏离中恢复的能力（27-42%）。
*   **门控机制解释了不对称性**
    *   通过因果探针实验证实，“条目”和“指示落地”是关键的门控。冲突记忆能轻易通过前序门控，但在最终环节才导致损害，因此关闭任何一个门控都能大幅减少损害。而有益记忆在每个门控都有损耗，因此总收益有限。
*   **位置无关性**: “合规陷阱”是记忆**内容驱动**的，而非传递渠道（系统提示 vs 页面底部）导致的。
*   **失败时重试是更优策略**: 基准表明，仅在代理**首次尝试失败后**再注入有益记忆，效果优于始终注入所有任务的记忆，并能接近理论上的最优调度方案。

### 7. 优点：方法或实验设计上的亮点

*   **视角新颖**: 从“供应”转向“消费”，开创性地研究了AI代理如何消化和使用记忆，而非仅仅关注记忆本身，提供了一个评估和诊断代理的新视角。
*   **框架清晰实用**: E-P-R框架将复杂的记忆消费过程分解为三个可定义、可量化的阶段，便于进行系统性诊断。
*   **实验设计严谨**: 采用成对对比、贪婪解码、自建受控基准（MemTrapBench）和多项消融实验，确保了结果的可靠性和因果的可解释性。特别是MemTrapBench的设计，通过构造而非下游结果选择任务，有效避免了数据偏差。
*   **发现深刻且反直觉**: “合规陷阱”的发现非常深刻，它揭示了一个反直觉的现象：更强的模型更容易遭受更大的损害。这挑战了简单认为“模型越强，风险管控越好”的普遍看法。
*   **多重验证**: 实验结果在两个不同的基准（WebArena和MemTrapBench）、多个模型家族（Qwen, Gemma）和不同任务Horizon（MiniWoB++）上得到验证，增强了结论的普适性。

### 8. 不足与局限

*   **应用场景局限**: 实验仅在**网页浏览**场景中进行，未验证E-P-R框架和“合规陷阱”是否能推广到其他长周期交互场景，如终端代理（shell/coding）、具身代理（机器人/模拟导航）等。这些场景的动作空间、观察模态和失败反馈不同，可能改变记忆消费的门控结构。
*   **未提出解决方案**: 论文成功地**诊断并刻画了问题**，但并未基于这些发现提出或训练一种新的记忆范式来实际解决“合规陷阱”。作者明确指出：“我们刻画了陷阱，而非构建控制器、检索器或训练时干预方案来利用它”。
*   **可能的数据偏差**: WebArena上的77-task子集是通过其对记忆的反应筛选出来的，这可能会**放大某些效应**。虽然MemTrapBench通过构造设计规避了这个问题，但WebArena子集的分析结果仍需谨慎解读。
