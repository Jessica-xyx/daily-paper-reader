## 摘要
大型语言模型（LLM）智能体会积累包括系统指令、计划、用户轮次、检索文档、工具输出和中间推理在内的异质上下文，其键值（KV）缓存可能成为主要的内存瓶颈。现有的驱逐策略通常对所有令牌应用相同的基于注意力或基于最近使用的规则，忽略了智能体编排器已经可用的语义结构。我们提出MemDecay，一种无需训练、区域感知的KV缓存驱逐策略。MemDecay为令牌分配区域特定的基础优先级和衰减率，在令牌获得注意力时刷新保留分数，并在固定缓存预算下驱逐得分最低的页面，同时允许关键区域被固定。我们还提供了一种根据测量的注意力生命周期校准衰减率的方法。我们使用Qwen2.5-1.5B和3B模型在约450和1700令牌上下文下评估MemDecay。在所有设置中，不同区域的注意力生命周期相差一个数量级：系统令牌的半衰期范围为148到189解码步，而暂存区令牌为14到16步。固定操作在所有设置下以全缓存精度保留了系统区域的事实，而没有任何基线能保留超过24个中的13个。随着上下文增长，区域感知的保留仍然有效，而基于最近使用的保留则崩溃。然而，累积注意力保留在未固定内容上表现更好，消融实验表明注意力分数归一化是当前公式的主要限制。这些结果确立了语义提示结构作为KV缓存管理的稳健信号，同时阐明了应如何将其与基于注意力的重要性相结合。

## Abstract
Large language model (LLM) agents accumulate heterogeneous context, including system instructions, plans, user turns, retrieved documents, tool outputs, and intermediate reasoning, whose key-value (KV) cache can become a major memory bottleneck. Existing eviction policies generally apply the same attention- or recency-based rule to every token, ignoring semantic structure already available to the agent orchestrator.   We introduce MemDecay, a training-free, region-aware KV-cache eviction policy. MemDecay assigns tokens region-specific base priorities and decay rates, refreshes retention scores when tokens receive attention, and evicts the lowest-scoring pages under a fixed cache budget while allowing critical regions to be pinned. We also provide a procedure for calibrating decay rates from measured attention lifetimes.   We evaluate MemDecay at approximately 450 and 1,700 token contexts using Qwen2.5-1.5B and 3B. Across all settings, attention lifetimes differ by an order of magnitude across regions: system-token half-lives range from 148 to 189 decoding steps, compared with 14 to 16 for scratchpad tokens. Pinning preserves system-region facts at full-cache accuracy in every setting, while no baseline preserves more than 13 of 24. Region-aware retention remains effective as context grows, whereas recency-based retention collapses. Accumulated-attention retention performs better on unpinned content, however, and ablations identify attention-score normalization as the main limitation of the current formulation. These results establish semantic prompt structure as a robust signal for KV-cache management while clarifying how it should be combined with attention-based importance.

# MemDecay 论文详细总结

## 1. 核心问题与整体含义

- **研究动机**：大型语言模型（LLM）越来越多地被部署为智能体（Agent），其上下文包含系统指令、计划、用户轮次、检索文档、工具输出、中间推理等多类异构内容，且随交互步数持续增长。支撑自回归解码的键值（KV）缓存因此成为主要的内存瓶颈，限制长时程推理和服务效率。
- **现有不足**：主流 KV 缓存驱逐策略（如基于累积注意力、最近使用窗口等）对所有令牌采用相同的保留规则，忽略了智能体编排器已经提供的语义结构（不同区域的信息角色和生命周期差异巨大）。例如系统指令需要持久保留，而草稿板（scratchpad）内容很快失效。
- **整体含义**：利用提示中已有的语义区域标签来指导缓存驱逐，有望在固定缓存预算下更有效地保留关键信息，从而提升长时程 Agent 推理的效率和可靠性。

## 2. 方法论

### 核心思想
MemDecay 是一种无需额外训练的、区域感知的 KV 缓存驱逐策略。它假设编排层（orchestrator）为每个令牌提供区域标签（如 system、plan、user、tool_out、retrieval、scratchpad），然后结合**区域结构先验**（基础优先级和区域特定衰减率）和**注意力重要性**（指数加权移动平均），计算每个令牌的复合保留分数。在固定缓存预算下，驱逐分数最低的页面（页级粒度），同时保护固定的关键区域。

### 关键技术细节

1. **区域配置**：每个区域 `r` 有基础优先级 `b_r`（编码插入时的价值）和衰减率 `λ_r`（控制结构性保护过期速度）。默认配置中 system 区域具有最高优先级和接近零的衰减率，被钉住（pinned）；scratchpad 区域优先级低且衰减快。
2. **保留分数公式**：
   ```
   s_i(t) = b_{r(i)} * exp(-λ_{r(i)} * Δt_i(t)) + α * a_i(t)
   ```
   - `Δ t_i(t)` 是自上次被注意力刷新（注意力超阈值 τ）以来的时间步。
   - `a_i(t)` 是注意力重要性的指数加权移动平均（EWMA），平滑因子 ρ=0.9。
   - α 控制注意力项权重（默认 α=0.5）。
3. **注意力采样**：由于融合注意力核无法直接获取每步注意力分数，MemDecay 仅在观测步（每 k 个解码步一次）读取窗口内平均注意力，仅影响约 69.8 ms vs 26.0 ms 的延迟（k=1 时）。
4. **刷新机制**：当令牌收到注意力超过阈值 τ（相对于均匀注意力 1/|K_t| 缩放）时，其衰减时钟重置，令牌重新从插入点开始衰减。
5. **页级驱逐**：生产环境中 KV 条目按固定大小分页，MemDecay 计算页面分数为页内令牌平均分数，按升序驱逐，直至满足预算。页面边界可跨区域，但跨区页面比例短上下文中约 35%，长上下文降至 17%。
6. **校准方法**：测量各区域注意力半衰期，拟合 log-linear 模型 `log A_r(Δt) ≈ log c_r - λ_r Δt`，用最小二乘法得到 `λ_r`，并用留一场景交叉验证。

### 算法流程（文字描述）
算法 1 在观测步执行：
1. 对所有缓存令牌更新注意力重要性 `a_i`（EWMA）。
2. 根据注意力是否超过阈值更新刷新时间戳 `t_ref_i`。
3. 对所有非钉住令牌计算保留分数 `s_i(t)`。
4. 计算每个页面的分数 `S_p(t)`（页内平均）。
5. 若当前缓存大小超过预算 B，则对非钉住页面建立最小堆，反复驱逐分数最低的页面直到满足预算。

## 3. 实验设计

### 场景/数据集
- 作者设计了 **8 个场景**（scenario），每个场景有 **3 个值变异版本**（variant），植入相同的事实和探针。短层（short tier）每集 405–507 令牌（5 阶段），长层（long tier）每集 1,656–1,723 令牌（8 阶段，多一次检索、工具交换和用户轮次）。
- 移动区块（工具交换、后续轮次、检索文档）在每个变异中按不同顺序到达，以确保区域身份与插入位置解耦；系统、计划和草稿板保持固定位置。

### 基准方法
- **全缓存**（无压缩）作为上限。
- **随机**：保留 4 个注意力汇点 + 16 个最近令牌 + 均匀采样，3 个种子。
- **Streaming**：保留 4 个汇点 + 最近令牌 [6]。
- **H2O-style**：基于累积注意力保留 [5] 的复现，使用与 MemDecay 相同的观测信号。
- 注意：基线不接收区域标签，且复现不代表原始方法在规模上的效果。

### 评估指标
- **召回率**：探针问题答案出现在贪婪解码的 16 个令牌中（黄金值以令牌边界出现）。每个配置单元聚合 96 个探针（24 集 × 4 探针）。置信区间基于场景级百分位自助法（8 个簇）。

### 设置
- 模型：Qwen2.5-1.5B-Instruct（短/长层），Qwen2.5-3B-Instruct（长层）。
- 硬件：1.5B 在 Apple M3 Pro（float32），3B 在 T4 GPU（float16）。
- 注意力采样间隔 k=1（每步采样），页面大小 16，衰减率通过留一场景交叉验证校准。

## 4. 资源与算力

- 实验使用了 **Apple M3 Pro**（1.5B 模型 float32）和 **T4 GPU**（3B 模型 float16）。
- 论文未明确说明训练时长或总 GPU 小时数，因为 MemDecay 是**无需训练**的（training-free），仅需单次轻量级校准（测量注意力半衰期并拟合两个参数每个区域）。校准的算力开销未单独量化。
- 测量开销：一次观测步的评分和页面选择约 0.27 ms，驱逐约 4.2 ms；但读取注意力导致每步延迟从 26.0 ms 升至 69.8 ms（k=1 时）。生产环境可通过增大观测间隔 k 来降低。

## 5. 实验数量与充分性

### 实验数量
- 总 **4,320 个评分的驱逐探针**（scored eviction probes）加上三次完整测量（三个设置）。
- 三个设置：1.5B-短层、1.5B-长层、3B-长层。
- 每个设置 8 个场景 × 3 个变异 × 4 个探针 = 96 个探针/预算水平。预算水平为 25% 和 50% 缓存。

### 消融实验
- 短层上改变注意力权重 α ∈ {0.2, 0.5, 0.8} 并禁用钉住。结果：所有配置产生几乎相同的保留集和召回率（仅 4/576 行不同），表明当前注意力项因量级太小而无法影响早期旧令牌。

### 充分性与公平性
- **充分性**：覆盖两种上下文长度（450 和 1,700 令牌）、两种模型规模（1.5B, 3B），包含移动区域位置解耦设计，且进行了校准交叉验证。但上下文长度仍远低于生产环境的数万令牌，结论向更长上下文的推广有条件性。
- **公平性**：基线复现使用了与 MemDecay 相同的观测信号（注意力仅在观测步读取），且均未获知区域标签。MemDecay 因页级驱逐导致保留令牌数略少于基准（例如 24.5% vs 25.0%），但差异极小。随机基线使用了多个种子后聚合。
- **客观性**：置信区间基于场景级百分位自助法（8 个簇），但簇数较少，区间仅近似。作者明确指出“配对场景符号比较”是政策声明的主要统计依据，而非点估计。

## 6. 主要结论与发现

1. **区域生命周期稳定且可分**：在所有设置中，系统令牌注意力半衰期（148–189 步）是草稿板（14–16 步）的 **10 倍以上**，且置信区间不重叠。检索区域的半衰期比默认预期更长（短层 48 步，长层 66–76 步），推翻了默认快速衰减的假定。
2. **钉住确保系统指令完整保留**：在 25% 和 50% 预算下，MemDecay 钉住系统区域，始终保留所有系统事实（24/24 短层，21/24 长层），而 Streaming 最多保留 2/24，H2O 也无法保证。
3. **基于文档的保留随上下文增长而崩溃**：短上下文中 Streaming 有竞争性，长上下文中召回率接近零（0.01–0.13），因为其窗口无法跨越整个轨迹。
4. **累积注意力在大模型上更强**：H2O-style 在 1.5B 长层上已超越 MemDecay 的未钉住召回，在 3B 上更显著（50% 预算非系统召回 0.60 vs 0.22）。
5. **未钉住召回损失机制**：最旧的未钉住事实（如用户轮次）因衰减先验而得分最低而被驱逐，注意力项因页面平均和 EWMA 遗忘早期注意力而无法补偿。消融表明任何 α 设置均不能挽回该损失，需要注意力项的量级归一化设计。
6. **校准有效但依赖负载**：测量出的区域半衰期与默认配置不一致（如检索比预期长寿），说明必须针对具体负载进行校准。

## 7. 优点

- **无需训练**：全部操作仅依赖推理时可获得的信号（区域标签、注意力统计），无需额外训练或辅助模块，易于部署。
- **利用已有语义结构**：直接从编排层获取区域标签（类似于 SGLang 正在开发的元数据通道），自然对齐智能体工作流。
- **可解释性**：三个参数（基础优先级、衰减率、注意力权重）各有直观含义，便于理解和调优。
- **钉住机制**：提供硬性保证，特别适合系统指令等必须常驻的内容。
- **校准方法**：通过注意力半衰期拟合衰减率，系首次在 KV 缓存领域系统化地利用区域条件注意力寿命，并验证了跨模型和场景的重复性。
- **页级兼容性**：与现有服务栈（如 PagedAttention）兼容，页面边界跨区域问题在长上下文中自然缓解。
- **低过载**：运行时状态仅 2 个标量/令牌，更新仅在观测步进行，驱逐使用最小堆，复杂度 O(P log P)。

## 8. 不足与局限

- **不可逆性**：被驱逐的令牌无法恢复，当后续查询使之重新相关时（如智体复制工具输出到草稿板后再使用），信息会永久丢失。论文提出的分层扩展（先量化再卸载再丢弃）可缓解但未实现。
- **跨区域信息流**：智体可能在不同区域间复制内容（如引用工具输出到草稿板），在默认配置下两个副本都快速衰减，可能造成信息丢失。校准降低风险，但缺乏链接机制。
- **标签质量依赖**：MemDecay 假设编排层提供准确区域标签。当前服务层元数据通道尚在开发中（SGLang 提案），若缺少编排器，需要训练轻量分类器，其对噪声标签的鲁棒性未测试。
- **头/层均匀性**：对所有头和层使用同一得分，而不同头对区域的敏感度可能不同，可与头自适应预算分配组合未来工作。
- **负载依赖性**：区域生命周期随负载变化（已有文献存在冲突），默认配置可能失效，需要校准数据代表部署负载。测量本身需要额外注意力读取开销。
- **实验规模限制**：仅覆盖 1.5B 和 3B 模型、~1,700 令牌上下文，远小于生产级（数万令牌、8B+ 模型）。置信区间基于 8 个场景簇，统计效力有限。固定位置的 system/plan/scratchpad 的生命周期估计受位置混淆。
- **未钉住召回弱点**：在累积注意力基线前失利，消融明确指向注意力项量级不足，需要设计修改（如归一化）而非参数调优，该点在论文中被作为主要改进方向。
