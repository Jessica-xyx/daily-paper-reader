## 摘要
LLM智能体基准测试衡量任务完成度、可靠性和推理成本，但不衡量智能体运行在磁盘上留下的持久化数据，包括日志、上下文快照、检查点和调试跟踪。我们引入AgentFootprint，一个跨框架的运行时智能体存储足迹基准测试。其序列化感知的度量套件衡量总保留量、通道组成、重复、增长、可压缩性和对话历史可重构性。它解决了一个测量陷阱：由于数据库分页和JSON转义掩盖了重复内容，天真的字节级测量低估了重复内容一个数量级。一个固定跟踪控制将智能体生成的逻辑卷与持久化层放大分离：通过七个持久化框架重放相同的轨迹产生6.7倍差距。在相同的模型、工具和任务下，配置准确率100%的配置在保留字节上相差15.7倍，尽管它们的默认设置支持不同的恢复和审计能力。三个完整历史配置在重复观察压力任务上呈现超线性增长。来自108个实例归一化的SWE-bench Verified提交的导出轨迹每个实例跨越三个数量级，与解决率没有可检测的相关性。一个内容寻址存储将保留量减少4.8倍至32.7倍，同时保留每个可重构性得分。这些结果将持久化存储确立为一个资源指标，应与准确性和可重构性一起报告。

## Abstract
LLM agent benchmarks measure task completion, reliability, and inference cost, but not the persistent data an agent run leaves on disk, including logs, context snapshots, checkpoints, and debug traces. We introduce AgentFootprint, a cross-framework benchmark of post-run agent storage footprint. Its serialization-aware metric suite measures total retention, channel composition, duplication, growth, compressibility, and conversation-history reconstructability. It addresses a measurement trap: naive byte-level measurement understates duplication by an order of magnitude because database paging and JSON escaping obscure repeated content. A fixed-trace control separates agent-generated logical volume from persistence-layer amplification: replaying the same trajectory through seven persisting frameworks yields a 6.7x spread. Under identical models, tools, and tasks, configurations with 100% accuracy differ by 15.7x in retained bytes, although their defaults support different recovery and audit capabilities. Three full-history configurations grow superlinearly on a repeated-observation stress task. Exported trajectories from 108 instance-normalized SWE-bench Verified submissions span three orders of magnitude per instance, with no detectable correlation with resolve rate. A content-addressed store reduces retention by 4.8x-32.7x while preserving every reconstructability score. These results establish persistent storage as a resource metric to report jointly with accuracy and reconstructability.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：现有的 LLM Agent 基准只衡量任务完成度、可靠性和推理成本，完全忽略了 agent 执行后留在磁盘上的持久数据（日志、上下文快照、检查点、调试追踪等）。这些数据在桌面部署、合规性要求或大规模集群下至关重要，但当前排行榜未包含任何存储维度。
- **动机**：持久存储（storage footprint）与推理成本不同——推理成本随执行结束而消失，而存储是存量，会累积并带来长期存储、治理和删除义务。生产环境实测中，单任务残留可达131 MB（其中框架残留79 MB，而交付产物仅2.6 MB）；整个部署累计约56 GB。因此需要使该维度可测量、可比较。
- **整体含义**：存储应作为与准确率、推理成本并列的“一等公民”指标进行报告，以便在部署前了解 agent 设计选择的操作后果。

## 2. 方法：核心思想、关键技术细节

### 核心思想
- **AgentFootprint**：一个跨框架的 agent 运行时存储足迹基准，提供序列化感知的度量套件，同时将存储与准确率、可重构性联合报告。
- **两大决定因素**：① agent 行为产生的逻辑体积；② 持久化层添加的放大倍数。通过固定轨迹控制将两者分离。

### 关键技术细节
1. **六个度量指标**：
   - **$S_{total}$**：总保留字节数（文件系统增量）。
   - **存储通道组成**：按状态/检查点、调试/事件日志、对话/动作等通道分解。
   - **$D$（重复因子）**：$D = S_{logical\_total} / S_{unique}$，通过内容定义的块（FastCDC + SHA-256）在逻辑流上计算。
   - **$\alpha$（增长指数）**：对重复观察应力任务拟合 $log S_{total} \sim \alpha log T$（$T \in \{25,50,100,200\}$），描述全历史 vs 窗口化策略。
   - **$C$（可压缩性）**：$S_{total}$ 除以 zstd -19 长窗压缩后的体积。
   - **$R$（对话历史可重构性）**：0–3 分，0 = 无保留，3 = 至少一个通道可精确重构历史。

2. **序列化感知测量（Serialization-Aware Meter）**：
   - 传统原始文件分块因 SQLite 分页和 JSON 转义而严重低估重复（如原始 chunking $D=1.01$，逻辑流 $D=12.1$）。
   - 解决方法：从 SQLite 单元格、JSONL 行等逻辑流中提取内容后再指纹；添加内容探针（原始、转义、双重转义编码）以检测跨编码的重复。

3. **边界与沙箱**：
   - 每次运行在独立沙箱中，捕获工作区和 home 目录增量；排除共享环境工件（模型权重、缓存等）。

## 3. 实验设计

### 数据集与场景
- **可控任务套件（6个）**：
  - **file-QA**（10任务）：5个顺序问题，需重读文件（读放大），精确子串匹配评分。
  - **long-horizon**：固定2 KB状态文件重复读取 $T$ 轮，隔离 $\alpha$。
  - **write-task**（5个）：读两个文档后写摘要，分离合法产物与残留。
  - **exploratory-retrieval**（5个）：自由探索，无对抗元素。
  - **edit**（5个）和 **data-analysis**（5个）：读-变换-写任务，联合评分（答案正确 + 正确交付产物）。
- **野生数据**：SVW-bench Verified 的全部 134 个提交（108个实例归一化后），导出轨迹大小 vs 解决率。

### 框架与方法
- **8个框架**：LangGraph 1.2.8, AutoGen 0.7.5, CrewAI 1.15.2, SmolAgents 1.26.0, OpenAI Agents SDK 0.18.0, LlamaIndex 0.14.23, Agno 2.7.1, InfiAgent 3.12.24。
- **公平性协议**：所有框架使用相同任务文本、相同最小工具集、相同后端模型（DeepSeek-V4-Flash，温度0，同一供应商）、3次独立重复；配置为其文档中记录的持久化默认设置。
- **固定轨迹控制**：通过模拟端点回放相同轨迹（3次读取、182.7 KB观察、1个答案）到7个持久化框架，隔离持久化层放大。

### 对比基线
- 框架间对比（8个框架的默认持久化配置）。
- 内容寻址存储（CAS）：作为缓解方法减少保留量 4.8–32.7 倍，保持可重构性。

## 6. 主要结论与发现

1. **相同准确率下存储差异极大**：file-QA 上，5个 100% 准确率的配置保留字节相差 15.7 倍（LangGraph 5.10 MB vs CrewAI 0.32 MB）；固定轨迹控制下持久化层放大 6.7 倍。
2. **全历史框架超线性增长**：在重复观察应力任务上，LangGraph、AutoGen、LlamaIndex 的 $\alpha$ 为 1.74–1.95（$T=200$ 时 LangGraph 达 323 MB），而窗口化或只追加设计保持近线性。
3. **残留远大于产物**：写任务中框架残留高达交付产物的 24,033 倍；生产部署中残留是产物的 30 倍。
4. **野生数据无存储-成功相关性**：108 个 SWE-bench Verified 系统中，导出轨迹体积跨越 1,617 倍，与解决率无可检测相关性（Kendall $\tau_b=0.080$, $p=0.222$）。
5. **内容寻址存储可大幅减量**：CAS 将保留减少 4.8–32.7 倍，同时保持所有可重构性得分（$R$ 不变）。
6. **高保真度历史重构不需要 MB 级别**：最小历史保留形式（仅保留对话历史）下，三个独立代码库收敛于 0.31–0.33 MB，等价于 OpenAI Agents 默认 0.33 MB。
