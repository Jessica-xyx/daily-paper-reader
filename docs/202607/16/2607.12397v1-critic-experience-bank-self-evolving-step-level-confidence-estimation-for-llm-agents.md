## 摘要
LLM智能体在外部环境中行动，每个动作都会改变后续决策所依赖的状态，并且单个错误步骤可能浪费交互预算或在最终失败被观察到之前很久就触发不可逆转的副作用。因此，可靠部署需要步骤级置信度估计：每个提议动作在执行之前就有一个校准的概率表示其是否富有成效。现有的LLM置信度估计器设计用于从给定提示中评分响应，但智能体置信度还取决于执行后果：类似情况下的类似动作是否在环境响应后实际推进了任务。我们引入了\method（\methodshort），一个自演进的批评框架，其中LLM批评者从其自身的过去判断及其观察到的后果中积累证据。在每个轨迹之后，一个看到完整执行反馈的后见LLM会投票每个步骤是否富有成效。得到的伪标签填充一个记忆库，当类似步骤再次出现时，从该库中检索相关的富有成效和无效经验到批评者的提示中。\methodshort无需训练，也不使用任何真实步骤标签。在三个智能体基准和三个批评骨干网络上，\methodshort在每个数据集-批评组合中达到了最佳校准（ECE和Brier）和排名（AUC），相对于最强的无训练基线，ECE降低了高达54%。

## Abstract
LLM agents act in external environments where each action changes the state that later decisions condition on, and where a single wrong step can waste interaction budget or trigger irreversible side effects long before the final failure is observed. Reliable deployment therefore requires \emph{step-level confidence estimation}: a calibrated probability that each proposed action is productive, available \emph{before} the action is executed. Existing LLM confidence estimators are designed to score a response from the given prompt, but agent confidence also depends on execution consequences: whether similar actions in similar situations actually advanced the task after the environment responded. We introduce the \method (\methodshort), a self-evolving critic framework in which an LLM critic accumulates evidence from its own past judgments and their observed consequences. After each trajectory, a hindsight LLM that sees the full execution feedback votes on whether each step was productive. The resulting pseudo-labels populate a memory bank from which related productive and unproductive experiences are retrieved into the critic's prompt whenever a similar step recurs. \methodshort requires no training and uses no ground truth step labels. Across three agent benchmarks and three critic backbones, \methodshort attains the best calibration (ECE and Brier) and ranking (AUC) in every dataset--critic combination, reducing ECE by up to $54\%$ relative to the strongest training-free baseline.

## 1. 核心问题与整体含义（研究动机和背景）

LLM 智能体（agent）在外部环境中执行任务时，每一步动作都会改变环境状态，影响后续决策。单个错误步骤可能导致整个交互预算浪费，甚至引发不可逆的副作用（如提交表单、删除文件、修改数据库）。因此，部署 LLM 智能体需要 **步骤级置信度估计**——在动作执行前给出其是否有效的校准概率。现有 LLM 置信度估计方法（如 token 对数几率、一致性、口头置信度）针对的是“从提示生成响应”的场景，仅基于当前上下文，忽略了执行后环境反馈对决策的影响。智能体置信度还依赖于“类似动作在类似情境下是否真正推进了任务”。本文提出 **Critic Experience Bank (CEB)**，让 LLM 批评者（critic）从自身过去的判断及其观察到的执行后果中积累证据，实现无需训练、无需真实步骤标签的步骤级置信度校准。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- **自演进批评框架**：利用一个固定的 LLM 批评者，结合一个动态增长的“经验银行”，在推理阶段通过检索过去轨迹中的类似经验来校准当前动作的置信度。
- **事后伪标签**：在每个完整轨迹结束后，使用一个“后见 LLM”（hindsight LLM）看到全部执行反馈，对每一步投票是否有效（productive），多数投票形成伪标签存入银行。
- **对比检索**：银行按类别分为 productive（B⁺）和 unproductive（B⁻）两部分。对新步骤，基于任务、状态、动作的复合键进行 TF-IDF 余弦相似度检索，从两个类别中各取 top-k 示例，形成对比示例加入批评者提示中。

### 关键技术细节
1. **经验记录结构**：每条记录 `e = (q', s', a', o', ŷ', ẑ', χ')`，包括任务、状态摘要、动作、观察摘要、事后伪标签 ŷ'、在线置信度 ẑ'、以及一致性标志 χ'（表示批评者原始判断与事后结果是否一致）。
2. **事后标签生成**：后见 LLM 以温度 τ>0 采样 V 次，对每一步输出 0/1，多数投票确定最终伪标签（公式 3）。
3. **检索**：复合键 `κt = task: q ∥ state: st ∥ action: at`，使用 TF-IDF 嵌入计算余弦相似度，从 B⁺ 和 B⁻ 中各取 k 个记录（若某一类不足则取全部）。此外还包含一个重复标志 ρt，提醒批评者当前动作是否与最近步骤重复。
4. **批评者提示**：包含任务、当前状态、历史、当前动作、检索到的示例、重复标志。批评者返回一个 JSON 格式的分数 rt，最终置信度 `ẑt = clip(rt, 0, 1)`。

### 算法流程（Algorithm 1）
- 初始化空银行 B。
- 对于每个任务 q：
  - 初始化轨迹 traj 和分数列表 scores。
  - 对每一步：
    - 提取状态摘要、构建复合键、从 B 中检索对比示例。
    - 批评者输出置信度 ẑt，存入 scores。
    - 执行动作，观察 ot，加入 traj。
  - 用后见 LLM 生成各步伪标签 ŷt。
  - 将所有步的记录加入银行 B。

## 3. 实验设计

### 数据集与场景
- **Mind2Web**：1,009 个网页导航轨迹，步骤标签基于参考动作匹配。
- **AMEX**：1,000 个移动 GUI 任务（随机采样），步骤标签基于触摸点匹配。
- **InterCode-Bash**：200 个 bash 任务，步骤标签基于奖励变化。

### 基准方法（Baselines）
所有基线均无需训练，且使用与 CEB 相同的冻结动作和标签进行对比：
- **PPL**（Perplexity）：基于 token 困惑度作为置信度。
- **ME**（Mean Entropy）：基于 token 平均熵。
- **PRO**（Probability Aggregation）：基于多个样本的序列对数概率 softmax。
- **Verbalization**：智能体与置信度联合生成。
- **P(True)**：批评者判断动作是否正确，取 TRUE token 的概率。
- **CoT**：思维链推理后口头置信度（rescaled）。

### 评估指标
- **ECE**（Expected Calibration Error）：校准误差。
- **Brier Score**：概率预测的均方误差。
- **AUC**：区分正负样本的排名质量。

## 6. 主要结论与发现

1. **CEB 在所有 9 个数据集-批评组合中均取得最佳 ECE、Brier 和 AUC**，相对于最强无训练基线，ECE 最高降低 54%（InterCode-Bash + GPT-4.1）。
2. **对比正负样例是 CEB 最重要的组件**：移除对比检索后 ECE 上升 0.065。
3. **经验积累带来持续改进**：随着银行增大，累积 ECE 不断下降，而静态批评者保持恒定。
4. **支持选择性执行**：CEB 在保留高置信度步骤时保持最高正确率，而 token 不确定性方法在过滤低置信度步骤后反而降低准确率（即误将正确步骤判为低置信度）。
5. **接近真实标签 Oracle 性能**：在 Mind2Web 上，CEB 可恢复 95% 的校准差距；在 AMEX 上为 61%，伪标签噪声并非主要误差来源。
6. **鲁棒性**：在 LLM-as-judge 宽松标签下 CEB 仍然最佳，说明其优势不依赖严格的参考匹配。

## 7. 优点

- **完全无训练**：无需训练模型或访问隐藏状态，适用于固定 API 模型。
- **自演进**：银行随推理过程自然增长，批评者行为因检索内容而动态适应。
- **对比性证据**：同时检索成功和失败案例，提供对比信号，帮助批评者校准。
- **无需人工标注**：伪标签来自后见 LLM 投票，仅需执行反馈。
- **广泛适用性**：在三种不同模态（Web、GUI、Shell）和三种不同 LLM 上均有效。
- **可解释性**：检索到的经验提供了置信度判断的理由。
- **组件有效性分析**：通过消融明确各设计要素的贡献。

## 8. 不足与局限

- **后见 LLM 与批评者相同**：论文未隔离独立评判者的影响（所有实验使用同一模型作为批评者和后见标签器）。
- **标签依赖参考标注或 LLM 评判**：真实的步骤标签仍需要参考动作或 LLM 重标，存在噪声（尤其 InterCode-Bash 的奖励衍生标签）。
- **银行无淘汰或压缩机制**：在大规模部署流中，银行无限增长会带来检索和提示成本问题（论文承认这是未来工作）。
- **实验覆盖有限**：未在更大规模流（如数千任务）上测试，也未测试与在线干预的闭环结合。
- **计算成本**：每个步骤需要一次批评者调用，每个轨迹需要 V 次后见 LLM 调用，成本高于简单基线。
- **超参数敏感**：检索深度 k 和投票数 V 需要调参（论文提供了 sweep，但未探讨自适应设置）。
- **公平性潜在偏差**：后见 LLM 与批评者相同可能导致自我增强的校准偏差（循环论证风险），论文未深入分析。
