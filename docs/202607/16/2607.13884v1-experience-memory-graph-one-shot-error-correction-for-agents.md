## 摘要
大型语言模型（LLM）智能体通过生成状态、动作和观察的序列轨迹，在自主决策方面展现出了显著能力。然而，在复杂的长期任务中，这些智能体经常遭遇累积错误，并且难以从失败中恢复。现有的自我修正机制依赖于基于提示的反思，这种方法本质上是脆弱的，由于迭代试错循环而耗费大量时间和API成本，并且产生难以泛化到新场景的任务特定记忆。为解决这一问题，我们提出了经验记忆图（EMG），这是一个将智能体失败恢复重新表述为图匹配问题的框架。在训练时，我们将失败的探索轨迹和成功的专家轨迹都转换为有向动作决策图。通过匹配这些图，我们提取出公共子图（成功工作流）和图编辑路径，这些路径明确指示如何修正失败（例如，在给定观察下添加、删除或重新标记哪些动作），并将其存储在具有任务内节点和跨任务边的记忆图中。在测试时，EMG检索相关见解并引导智能体进行单次、无循环的执行。在ALFWorld和ScienceWorld上的实验表明，EMG在成功率和平均奖励方面持续优于最先进的反思基线，且无需测试时的试错。

## Abstract
Large Language Model (LLM) agents have shown remarkable capabilities in autonomous decision-making by generating sequential trajectories of states, actions, and observations. However, in complex, long-horizon tasks, these agents frequently suffer from compounding errors and struggle to recover from failures. Existing self-correction mechanisms rely on prompt-based reflection, which is inherently brittle, incurs heavy time and API costs due to iterative trial-and-error loops, and produces task-specific memory that may be hard to generalize to new scenarios. To address this, we propose Experience Memory Graph (EMG), a framework that reformulates agent failure recovery as a graph matching problem. At training time, we convert both failed exploration trajectories and successful expert trajectories into directed action decision graphs. By matching these graphs, we extract common subgraphs (successful workflows) and graph edit paths that explicitly indicate how to correct failures (e.g., which actions to add, delete, or relabel under a given observation), and store them in a memory graph with intra-task nodes and cross-task edges. At test time, EMG retrieves relevant insights and guides the agent in a single, loop-free execution. Experiments on ALFWorld and ScienceWorld show that EMG consistently outperforms state-of-the-art reflection baselines in success rate and average reward, while requiring no test-time trial-and-error.

- **研究意义**：本文旨在提出一种无需测试时试错、可泛化的离线错误修正方法，将智能体从失败中恢复转化为确定性的图匹配问题，从而提升LLM agent在长程任务中的鲁棒性和效率。

## 2. 论文提出的方法论

### 2.1 核心思想

- 将失败探索轨迹和成功专家轨迹都转换为**有向动作决策图（Action Decision Graph）**，节点代表动作，边代表观察条件。
- 通过图匹配（Graph Matching）提取**公共子图**（成功工作流）和**图编辑路径**（修正指令），存储为经验记忆图（Experience Memory Graph, EMG）。
- 测试时仅需检索记忆图中的节点（任务内见解）和边（跨任务见解），单次执行即可完成任务，无需迭代反思。

### 2.2 关键技术细节

#### （1）动作决策图构建（Algorithm 1）
- 对每条轨迹，将每个动作归一化为三元组 `(type, object, receptacle)`，作为节点标签；将观察文本归一化为边标签。
- **节点重用**：如果同一个归一化动作在不同位置出现，复用节点，允许多个不同观察边指向同一节点，反映“在不同状态下可执行同一动作”。
- **并行化无效动作**：若某动作导致无信息观察（如“nothing happens”），保持当前有效状态不变，将所有后续无效动作从同一前驱节点并行连接，避免产生无意义的编辑路径。
- 图中包含虚拟初始节点 `v_init`，标签为0。

#### （2）图匹配提取结构知识
- 使用**融合Gromov-Wasserstein（FGW）距离**进行无监督节点匹配，同时考虑节点标签和边结构。
- 通过线性分配将软匹配矩阵离散化为硬匹配矩阵。
- 基于匹配结果，用**Algorithm 2**提取公共子图（保留标签相同的边和节点）；用**Algorithm 3**提取图编辑路径（包括节点删除/插入/重标记、边删除/插入/重标记）。
- 编辑路径明确指明：在特定观察下应保留、删除或替换哪个动作。

#### （3）经验记忆图构建
- **节点**：每个训练任务对应一个节点，存储任务查询、嵌入、对应的探索和专家图、奖励，以及由LLM总结的**任务内见解**（成功工作流或修正指令）。
- **边**：对相似任务（余弦相似度大于阈值）的专家图进行图匹配，提取公共子图，生成**跨任务见解**，存储于边。
- 记忆图构建完全在训练集上离线完成。

#### （4）测试时检索
- 对新任务查询，计算与所有节点嵌入的相似度，检索Top-K节点及其关联的跨任务边见解。
- 将检索到的见解作为提示直接传给agent，单次交互完成任务。

- **创新性**：提出用图匹配替代LLM自我反思，将错误修复转化为确定性的图编辑路径，避免了LLM推理的脆弱性和不确定性。
- **高效性**：记忆图离线构建，测试时单次执行，完全消除迭代试错，极大降低时间和API成本。
- **泛化能力**：通过跨任务边提取可迁移的行为模式，使记忆能在相似任务间复用，克服了传统方法任务特定记忆的限制。

## 8. 不足与局限

- **实验环境有限**：仅在两个模拟环境（ALFWorld和ScienceWorld）上验证，未在真实 Web 任务、数学推理、软件工程等更复杂场景评估，泛化性有待验证。
- **资源信息缺失**：未提供训练/推理的具体算力消耗（如GPU型号、时长、API调用次数），难以复现或对比效率。
