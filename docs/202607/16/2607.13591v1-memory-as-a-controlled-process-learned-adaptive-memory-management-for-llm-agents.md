## 摘要
大型语言模型（LLM）智能体越来越依赖外部记忆系统跨任务积累经验。然而，几乎所有现有方法，从图结构记忆到反思性洞察存储，都通过固定的手工设计启发式方法访问记忆。我们认为这种静态的记忆观是智能体学习的核心瓶颈，因为最优的记忆行为本质上是依赖上下文的。任务的早期阶段因记忆稀疏而受益于最少检索；重复的目标类型受益于计划复用而非通用最近邻查找；陷入困境的智能体受益于使用替代查询的重新检索；在长任务流中，记忆存储本身必须被整合和修剪以保持有用。我们提出“记忆作为受控过程”（MemCon），一个将记忆操作建模为马尔可夫决策过程的框架，学习一个在线策略，自适应地决定何时、检索什么以及检索多少，何时注入精炼计划，何时整合或遗忘。MemCon是后端无关的：它封装任何现有记忆实现，通过逐任务二元反馈学习，无需预训练和额外LLM调用，并使用带有UCB探索的轻量级表格上下文赌博机，在数十个任务内收敛。在6个基准测试、3个智能体框架和3个LLM骨干网络上，MemCon在任务成功率上持续优于多个记忆基线多达15.2个百分点，同时减少5-20%的令牌消耗。

## Abstract
Large Language Model (LLM) agents increasingly rely on external memory systems to accumulate experience across tasks. Yet nearly all existing approaches, from graph-structured memories to reflective insight stores, access memory through fixed, hand-designed heuristics. We argue that this static view of memory is a core bottleneck for agentic learning because optimal memory behavior is fundamentally context-dependent. The early stages of the tasks, benefit from minimal retrieval because memory is sparse; recurring goal types benefit from plan reuse rather than generic nearest-neighbor lookup; stuck agents benefit from re-retrieval with alternative queries; and across long task streams, the memory store itself must be consolidated and pruned to remain useful. We present Memory as a Controlled Process (MemCon), a framework that models memory operations as a Markov Decision Process and learns an online policy that adaptively decides when, what, and how much to retrieve, when to inject a distilled plan, and when to consolidate or forget. MemCon is backend-agnostic: it wraps any existing memory implementation, learns from task-by-task binary feedback with no pretraining and no additional LLM calls, and uses a lightweight tabular contextual bandit with UCB exploration that converges within tens of tasks. Across 6 benchmarks, 3 agent frameworks, and 3 LLM backbones, MemCon consistently outperforms multiple memory baselines by up to 15.2 points in task success while reducing token consumption by 5--20%.

-   **研究动机**：现有的大语言模型（LLM）智能体在执行任务时，普遍依赖一个外部记忆系统来存储和复用过往经验。然而，几乎所有主流方法，无论是图结构记忆、反思型记忆还是向量存储，都采用**固定、手工设计的启发式规则（如固定的 top-k 检索、固定的查询模版）**来访问记忆。
-   **核心问题**：这种“静态记忆”观是智能体学习的核心瓶颈。因为最优的记忆访问策略是**高度依赖上下文的**：
    -   任务早期：记忆稀疏，检索过多反而会稀释有效信息。
    -   重复型任务：复用成功计划，比简单的最邻近检索更有效。
    -   卡住时：需要改变查询思路进行重新检索，而非重复阅读已失效的信息。
    -   长任务流：记忆需要被整理和遗忘，否则会变得臃肿无用。
-   **整体含义**：本文提出，LLM智能体的记忆不应该是一个固定的管道，而应该是一个可以根据状态（如任务进度、记忆状态）进行动态调整的**受控过程**。

### 2. 方法论：核心思想与关键技术

-   **核心思想**：将记忆管理建模为一个马尔可夫决策过程（Memory MDP），并学习一个**轻量级的在线策略**来决定何时、检索什么、检索多少、何时注入计划或进行遗忘操作。
-   **关键技术**：
    1.  **Memory MDP 形式化**：定义了状态（$S$）、动作（$A$）和奖励（$R$）。
        -   **状态（$S$）**：融合任务进度信号（目标类型、步骤阶段、是否卡住）和记忆状态信号（记忆大小、计划是否可用、学习阶段），并离散化为一个紧凑的哈希键 $\phi(s)$。
        -   **动作（$A$）**：包括 `RETRIEVE`（检索）、`PLAN_INJECT`（注入计划）、`RE-RETRIEVE`（重新检索）、`CONSOLIDATE`（整合）、`FORGET`（遗忘）和 `NOOP`（无操作），每个动作可附带参数（如 `top_k`）。
        -   **奖励（$R$）**：任务结束时的二元反馈（成功/失败），并辅以效率奖励（步数越少奖励越高）。奖励通过**反向贴现**的方式分配给该任务中的每个记忆决策。
    2.  **在线策略学习（Contextual Bandit with UCB）**：
        -   使用一个轻量级的**表格化上下文赌博机**模型，通过**UCB（上置信界）**算法进行动作选择，平衡探索与利用。
        -   利用**热启动先验**（如 `RETRIEVE` 比 `NOOP` 有更高的初始Q值）来加速早期收敛。
        -   **无需预训练**和**零额外LLM调用**，策略更新仅基于任务结束时的二元成功/失败信号。
    3.  **后端无关的封装器（Backend-Agnostic Wrapper）**：
        -   MemCon 不是一个新的记忆存储系统，而是一个**薄薄的封装层**，可以包裹在任何现有的记忆后端（如向量存储、技能库）之上。
        -   它拦截后端提供的 `retrieve` 和 `store` 接口，并根据学习到的策略动态调整调用方式，从而实现“即插即用”。
    4.  **增强的记忆操作**：针对长程代理任务的两个常见失败模式进行了优化：
        -   **通用计划注入（Plan Injection）**：将成功的任务轨迹泛化成一个模板（如将“去 shelf 3”泛化为“去 [shelf]”），在遇到类似任务时注入，指导LLM。
        -   **目标分解（Goal Decomposition）**：针对复合型任务（如放置两个物体），自动分解为两个子任务并分别检索计划。
