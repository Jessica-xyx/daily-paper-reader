## 摘要
最近统一的的多模态模型表明，单一架构可以同时执行视觉/语言理解和图像生成/编辑。然而，它们反复将所有历史视觉和文本输入馈送到共享上下文窗口中，由于视觉令牌爆炸和不可靠的跨轮引用，限制了长期的多模态对话。我们提出了一种认知结构化多模态智能体，将视觉信息外部化到情景视觉记忆中，并在推理过程中选择性地重新激活相关情景。该智能体由用于结构化视觉抽象的感知抽象引擎、用于跨轮记忆检索的认知检索引擎以及用于自主任务推理和动作规划的多模态执行控制器组成。为了解决现有数据集中缺乏轮级检索监督的问题，我们开发了一个统一的情景引擎，通过程序生成带有细粒度检索注释的结构化多轮对话，使得强化学习能够优化抽象和检索策略。我们还构建了一个按难度分层的长期视觉对话基准，用于评估情景视觉回忆。我们的8B智能体在20轮会话中实现了91.4%的检索准确率，超过32B基线8.2%，同时将每轮推理时间几乎减半（23.1秒 -> 12.7秒）。我们进一步提出了认知结构化多模态智能体工具包（CMA-Harness），这是一种工具增强的部署，具有相同的认知结构，集成了持久多模态记忆、网络访问、图像生成/编辑/合成工具以及兼容OpenAI的服务。结构化记忆和模块化决策为长期多模态智能体提供了比单一参数扩展更可扩展、更高效的范式。代码：https://github.com/caseclose/cma-harness；项目页面：https://caseclose.github.io/cma-harness/

## Abstract
Recent unified multimodal models show a single architecture can jointly perform vision/language understanding and image generation/editing. However, they repeatedly feed all historical visual and textual inputs into a shared context window, limiting long-horizon multimodal dialogue due to visual token explosion and unreliable cross-turn referencing. We propose a Cognitive-structured Multimodal Agent that externalizes visual information into an Episodic Visual Memory and selectively reactivates relevant episodes during reasoning. The agent consists of a Perceptual Abstraction Engine for structured visual abstraction, a Cognitive Retrieval Engine for cross-turn memory retrieval, and a Multimodal Executive Controller for autonomous task inference and action planning. To address the lack of turn-level retrieval supervision in existing datasets, we develop a Unified Scenario Engine that programmatically generates structured multi-turn conversations with fine-grained retrieval annotations, enabling reinforcement learning to optimize abstraction and retrieval policies. We also construct a long-horizon visual-dialogue benchmark stratified by difficulty to evaluate episodic visual recall. Our 8B agent achieves 91.4% retrieval accuracy over 20-turn sessions, surpassing 32B baselines by +8.2% while nearly halving per-turn inference time (23.1s -> 12.7s). We further present the Cognitive-structured Multimodal Agent Harness (CMA-Harness), a tool-augmented deployment of the same cognitive structure integrating persistent multimodal memory, web access, image generation/editing/composition tools, and OpenAI-compatible serving. Structured memory and modular decision-making offer a more scalable, efficient paradigm for long-horizon multimodal agents than monolithic parameter scaling. ### 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：现有的统一多模态模型（如 BAGEL、Chameleon 等）虽然能够在一个架构中同时处理视觉/语言理解和图像生成/编辑，但在长程多轮对话中面临两个严重缺陷：
  - **视觉 token 爆炸**：每轮对话的所有历史图像以高维 token 形式重复输入，随着轮次增加，上下文窗口被大量无关视觉 token 挤占，推理成本激增。
  - **跨轮视觉引用不可靠**：模型依赖隐式注意力机制对长上下文进行跨轮检索，当轮次超过 8 轮、话题多次切换或存在近重复图像时，检索准确率急剧下降，出现语义漂移和幻觉。
- **研究动机**：参数规模的单纯扩大（如从 8B 到 32B）无法从根本上解决上述结构性问题。亟需一种显式的记忆管理与模块化决策范式，将视觉信息外部化，仅在需要时选择性召回，从而在保证性能的同时大幅降低推理成本。
- **整体含义**：本文提出了一种认知结构化的多模态智能体框架，通过分解为感知抽象、记忆检索、任务控制三个专用模块，配合强化学习优化记忆构建与检索策略，使得 8B 参数的智能体在 20 轮对话中检索准确率超越 32B 基线，推理时间减半。这一工作表明结构化记忆与模块化推理相比单纯参数扩展更具可扩展性和效率，为长程多模态交互提供了新范式。

### 2. 论文提出的方法论：核心思想、关键技术细节

#### 核心思想
将长程多模态对话中的视觉上下文选择问题形式化为一个检索策略学习问题：每一轮只选择过去图像的最小相关子集，然后在该子集上产生响应。通过三个专门化模块实现该策略：

- **感知抽象引擎 (Perceptual Abstraction Engine, PAE)**：将每张新输入的图像压缩成结构化记忆条目（标签、自然语言描述、缩略图），存入外部情景视觉记忆（EVM）。
- **认知检索引擎 (Cognitive Retrieval Engine, CoRE)**：根据当前对话状态，从 EVM 中检索出与当前查询最相关的图像索引列表。若无需历史图像则输出空列表。
- **多模态执行控制器 (Multimodal Executive Controller, MEC)**：将检索到的记忆与对话历史融合，自主推断任务类型（生成/编辑/合成/理解/纯聊天），并调度下游模型（Qwen-Image 用于生成，Qwen-Image-Edit 用于编辑和合成）产生最终输出。

#### 关键技术细节

1. **联合优化 PAE 和 CoRE**：不是孤立优化字幕生成质量，而是以检索成功率为最终目标。采用三阶段训练：
   - **阶段 1**：在统一场景引擎生成的数据上对 CoRE 进行监督微调（SFT），学习从记忆条目和查询到真实检索集的映射。
   - **阶段 2**：对 CoRE 应用强化学习（DAPO 算法），奖励函数为难度加权的 Jaccard 相似度（困难/极难轮次权重 1.2，其余 1.0），并惩罚格式错误；训练样本侧重基线错误预测、困难轮次和负样本（无需检索）。
   - **阶段 3**：冻结 CoRE，对 PAE 进行强化学习，奖励为 CoRE 在 PAE 生成的记忆条目上的检索准确率（Jaccard 相似度减去格式惩罚），使记忆表达直接为下游检索服务。

2. **统一场景引擎 (Unified Scenario Engine)**：为解决现有数据集缺乏轮级检索标注的问题，程序化生成 20 轮多模态对话，每轮附有 ground-truth 检索集、难度标签和挑战类型。流程包括：人类模拟模块（Gemini）生成查询和检索目标 → 零样本多智能体系统（Qwen-VL + Qwen-Image-Edit）产生候选回答 → GT-justify 验证并修正 → 反馈给模拟模块生成下一轮。数据按随机种子和主题划分为训练集（270 会话）、验证集（30 会话）和持出评测集 M2CA-Bench（100 会话 × 20 轮 = 2000 标注轮次）。

3. **检索难度分类**：如表 1 所示，根据话题转移、时间跨度、多图像交互和指代模糊性分为 easy / medium / hard / very hard。并设计了高相似度混淆图像和无需检索的负样本来防止取巧。

4. **模块解耦**：PAE、CoRE、MEC 各自独立可训练、可替换。MEC 进一步将任务分类为五类后分别处理，每轮只传入检索到的子集图像，限制了视觉 token 数量，实现 1.8 倍加速。

5. **CMA-Harness 部署框架**：相同的认知结构扩展为工具增强型智能体，MEC 可调用 17 种工具（图像生成/编辑/合成、搜索、记忆操作等），并支持多会话持久记忆和异步抽象，提供 OpenAI 兼容接口。

### 6. 论文的主要结论与发现

1. **统一模型在长程检索上严重不足**：BAGEL 在 Full 子集上仅 23.3% 准确率，极其难轮次 <3%，表明隐式注意力无法可靠地跨轮引用。
2. **结构化记忆过滤显著提升上下文选择**：Multi-Agent Baseline（8B）比 All-Context 基线在 Full 上高出 +2.9%~+5.8%，差距在中难轮次更大（+8.2%~+11.6%）。
3. **任务优化胜过单纯参数扩大**：将 Multi-Agent Baseline 从 8B 扩展到 32B 仅提升约 1.4%~2.0%，而本文 SFT+RL 优化的 8B 模型超越 32B 基线 +8.2%（英文）。优势随难度增大而扩大（Full +9.6% → Hard +18.4%）。
4. **推理时间减半且生成质量提升**：每轮推理时间从 23.1s 降至 12.7s，同时生成质量从 7.93 提升至 8.77（表 6）。
5. **三阶段训练逐步带来显著增益**：SFT 提升 Hard 准确率 7.3%，RL(CoRE) 再提升 6.5%，RL(PAE) 再提升 4.6%。
6. **纯文本记忆不足以支撑细粒度视觉回忆**：文本记忆（MemoryLLM、ReasoningBank）在 Hard 子集上仅 24.4% 和 39.0%，而完整 EVM（含缩略图）达 82.0%，说明视觉线索对于区分近重复图像不可或缺。
7. **CMA-Harness 验证了同一认知结构在开放工作流中的实用性**，表明结构化记忆与工具调用的结合能支持持久多模态助手。

### 7. 优点：方法或实验设计上的亮点

- **创新的记忆-决策解耦**：将传统的“统一模型处理一切”分解为感知抽象、检索、任务规划三个独立模块，每个模块可单独训练、替换或缩放，实现了性能与效率的帕累托最优。
- **闭环数据生成引擎**：使用 Gemini 模拟用户行为+零样本多智能体联合验证，自动产生带有检索标注的高质量训练数据，弥补了现有公开数据集的缺失。
- **强化学习直接优化记忆构建**：不是优化传统字幕指标，而是以下游检索准确率为奖励，使 PAE 的记忆表达真正为任务服务，这一思路具有通用性。
- **严格的困难级别与负样本设计**：基准测试包含高相似度混淆图像和无需检索的负样本，防止模型通过取巧策略（如始终选择最新图像）获得高分，使评测更具区分度。
- **多语言评估（英语和中文）**，验证了方法的语言无关性。
- **显式效率分析**：提供每轮推理时间对比，量化了结构化记忆在降低延迟方面的优势。
- **提供 CMA-Harness 开源部署框架**，包含 17 种工具和持久记忆，增强实用性和可复现性。

### 8. 不足与局限

- **训练与评测数据同源**：M2CA-Bench 和训练数据均来自统一场景引擎（Gemini+Qwen-Image-Edit 构建），存在风格和分布上的一致性偏差，可能高估模型在真实用户对话上的泛化能力。
