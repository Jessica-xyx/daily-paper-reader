## 摘要
大语言模型（LLMs）将事实知识和领域特定模式隐式存储在密集的Transformer参数中，这使得通过预训练、微调、检索增强或更长上下文来扩展知识变得成本高昂。Engram式记忆提供了一种紧凑的隐藏状态注入途径，但现有的GPU驻留设计通常依赖基于哈希的压缩，导致不相关的短语在共享槽位中发生冲突，从而削弱了短语级别的语义保真度。我们提出TF-Engram，一种无训练的Engram系统，该系统从外部语料库离线构建短语特定的语义记忆，将大型记忆表存储在GPU-DRAM-SSD层次结构中，并使用早停引导的预测预取技术来隐藏自回归解码期间的外部内存延迟。在Qwen3-0.6B上，TF-Engram将平均下游得分从57.6提升至59.4，优于冻结主干网络和参数匹配的LoRA基线。系统评估表明，大型TF-Engram表可以以适度的离线成本构建，基于SSD的存储大幅降低了GPU内存需求，而预测预取技术恢复了由外部内存访问引起的吞吐量损失。这些结果表明，静态短语记忆可以作为可扩展、无训练且低开销的系统组件集成到LLM推理中。

## Abstract
Large Language Models (LLMs) store factual knowledge and domain-specific patterns implicitly in dense Transformer parameters, making knowledge expansion costly through pretraining, fine-tuning, retrieval augmentation, or longer contexts. Engram-style memory offers a compact hidden-state injection pathway, but existing GPU-resident designs often rely on hash-based compression, causing unrelated phrases to collide in shared slots and weakening phrase-level semantic fidelity. We present TF-Engram, a train-free Engram system that constructs phrase-specific semantic memory offline from external corpora, stores large memory tables across a GPU--DRAM--SSD hierarchy, and uses Early-Exit Guided Predictive Prefetching to hide external-memory latency during autoregressive decoding. On Qwen3-0.6B, TF-Engram improves the average downstream score from 57.6 to 59.4, outperforming both the frozen backbone and a parameter-matched LoRA baseline. System evaluation shows that large TF-Engram tables can be built with moderate offline cost, SSD-backed storage substantially reduces GPU memory demand, and predictive prefetching recovers much of the throughput loss caused by external memory access. These results demonstrate that static phrase memory can be integrated into LLM inference as a scalable, train-free, and low-overhead system component.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：大语言模型（LLM）将知识隐式存储在密集的Transformer参数中，导致知识扩展（如增加新事实、领域术语）需要通过预训练、微调、检索增强或延长上下文等方式，成本高昂且效率低。
- **现有Engram记忆的局限**：Engram式记忆虽然提供紧凑的隐藏状态注入途径，但现有设计多依赖哈希压缩将大量短语映射到固定大小的GPU驻留表中，导致不相关短语冲突共享槽位，削弱了短语级别的语义保真度（例如“New York City”可能与不相关短语共享同一记忆值）。
- **研究动机**：需要一种无需训练、可扩展、语义保真度高且推理时低开销的静态短语记忆系统，将知识存储与Transformer计算解耦。
- **整体含义**：TF-Engram将Engram从紧凑的哈希压缩学习表转变为存储管理的分层外部记忆系统，实现了可扩展、无训练、低开销的LLM记忆增强。

## 2. 方法论

### 核心思想
- **无训练离线构建短语语义记忆**：从外部语料库中挖掘短语，使用冻结的语义编码器（如Qwen3-Embedding）生成记忆向量，构建精确的短语级键值表，避免哈希冲突。
- **SSD支持的GPU-DRAM-SSD层次记忆**：热条目存GPU缓存，温条目存主机DRAM，冷条目存NVMe SSD，以支持大容量精确记忆。
- **早停引导的预测预取**：在LLM的中间层（如L- r层）附加辅助预取头，提前预测下一个token及其可能短语，异步从SSD/DRAM预取记忆，覆盖剩余层的计算时间以隐藏外部存储延迟。

### 关键技术细节
1. **离线构建**：
   - 短语挖掘：从FineWeb-Edu、Wikipedia、C4、arXiv、PubMed等语料提取二元组、三元组、命名实体等。
   - 短语编码：使用冻结的Qwen3-Embedding生成语义向量。
   - 建立两层索引：token前缀→短语ID→内存地址。
   - 匹配策略：最长匹配优先，可聚合多个匹配向量。

2. **层次存储**：
   - GPU缓存：存放热点条目。
   - DRAM缓存：存放温条目和元数据索引。
   - SSD存储：存放完整记忆表，采用连续数组布局，并可能将相关短语（共享前缀或领域）就近存储以提高空间局部性。

3. **早停预测预取**：
   - 在层L-r处附加预取头，产生近似下一token分布。
   - 选择Top-K候选token，结合历史序列映射到可能的短语条目。
   - 异步预取：根据预测概率、短语有用性、存储层级收益计算优先级分数，在剩余层计算期间完成传输。
   - 预取头可以是无训练的（重用冻结投影）或轻量校准的（单独训练预取头，不更新主干）。

## 7. 优点

- **无需训练**：记忆完全离线构建，不更新主干参数，可跨模型复用（依赖于相同tokenizer），适应新领域只需重新挖掘编码。
- **语义保真度高**：避免哈希冲突，为每个短语保留独立记忆向量，支持精确匹配和聚合。
- **可扩展性强**：利用SSD容量支持海量短语，GPU内存仅存放热点，通过层次存储和预取缓解延迟瓶颈。
- **早停预取创新**：利用中间层预测，利用剩余层计算窗口隐藏外部访问延迟，设计合理（优先级评分、Top-K预算）。
- **实验全面**：覆盖效果、开销、敏感性、消融多个维度，结果清晰展示权衡。
