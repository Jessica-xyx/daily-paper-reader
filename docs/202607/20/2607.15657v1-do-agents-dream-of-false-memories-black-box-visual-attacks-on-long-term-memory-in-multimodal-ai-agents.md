## 摘要
多模态AI智能体越来越依赖持久的长期记忆来将生成结果锚定于过去的视觉和文本片段。我们表明，对视觉数据的无条件信任造成了一个关键漏洞。我们提出Lucid，一种黑盒对抗框架，在严格限制图像的威胁模型下破坏多模态记忆管道，无需访问目标MLLM、目标检索编码器或文本通道。Lucid通过制造不可察觉的扰动，基于历史上下文的可用性实现两种不同的失败模式：（1）记忆投毒——一种上下文内攻击，对抗图像替换一个其内容由先前文本上下文加强的良性图像，可靠地破坏视觉回忆并引导智能体走向攻击者选择的叙述；（2）记忆注入——一种上下文外攻击，对抗图像替换一个在缺乏先前文本基础的对话轮次中的良性图像，导致智能体生成受攻击者影响的响应，且无来自记忆的纠正信号。我们在各种对话领域和五种黑盒记忆架构（包括图结构、LLM总结和商业部署系统）上评估了Lucid。Lucid在投毒上达到61.6%的攻击成功率，在注入上达到58.4%的攻击成功率，暴露了多模态记忆管道中的结构性漏洞。

## Abstract
Multimodal AI agents increasingly rely on persistent long-term memory to ground generation in past visual and textual episodes. We show that unconditional trust in visual data creates a critical vulnerability. We propose Lucid, a black-box adversarial framework that compromises multimodal memory pipelines under a strictly image-bounded threat model, requiring no access to the target MLLM, target retrieval encoder, or the text channel. Lucid crafts imperceptible perturbations to enable two distinct failure modes based on the availability of historical context: (1) Memory poisoning, an in-context attack where the adversarial image replaces a benign one whose content is reinforced by prior textual context, reliably corrupting visual recall and steering the agent toward attacker-chosen narratives; (2) Memory injection, an out-of-context attack where the adversarial image replaces a benign one in a conversation turn devoid of prior textual grounding, causing the agent to generate attacker-influenced responses with no corrective signal from memory. We evaluate Lucid across various conversation domains and five black-box memory architectures, including graph-structured, LLM-summarized, and commercially deployed systems. Lucid achieves 61.6% ASR on poisoning and 58.4% ASR on injection, exposing a structural vulnerability in multimodal memory pipelines.

## 2. 论文提出的方法论

### 核心思想
- 攻击者仅能修改用户上传图像的像素值（ℓ∞ 扰动），不可访问目标MLLM、检索编码器、记忆内容或文本通道。
- 利用**对抗性转移性**：在公开的替代模型（surrogate ensemble）上优化对抗扰动，使扰动后的图像在目标系统的视觉编码器中产生与目标概念一致的嵌入，从而在存储和检索时误导智能体。
- 设计两种互补的攻击模式：
  - **记忆投毒（Memory Poisoning）**：攻击已存在于对话中的图像轮次，保留原始文本上下文（锚定真实事件），但替换图像使其嵌入指向攻击者选择的矛盾概念，导致智能体**错误回忆一个真实事件**（语义方向被篡改）。
  - **记忆注入（Memory Injection）**：攻击用户新上传的图像（无先前对话上下文），用扰动图像替换源图像，使系统**记住一个凭空创造的事件**，且存储时无任何不一致信号。

### 关键技术细节
- **三阶段管道 LUCID**：
  1. **目标设计（Stage 1）**：从ShareGPT4V-100K候选池中，基于三个指标评分选择与真实答案语义矛盾的目标：(a) 检索位移得分 ΦR（评估候选图像与查询的嵌入对齐度）；(b) 槽位矛盾得分 Φcontra（奖励接近错误答案、远离正确答案的文本描述）；(c) 真实答案对齐惩罚 ΦC。使用CLIP和OpenAI嵌入双重打分，并施加词汇门和主题门过滤器。
  2. **载荷构建（Stage 2）**：根据攻击模式生成源-目标图像对。投毒模式直接标记现有图像轮次；注入模式构建一个中性文本的新用户轮次，随机选取源图像。
  3. **对抗扰动（Stage 3）**：优化 ℓ∞ 约束下的损失函数 \( L = L_{OT} + \lambda_{txt} L_{txt} \)。LOT 采用FOA-Attack（特征最优对齐），在替代模型集成（CLIP-B/16, CLIP-B/32, LAION-CLIP）上组合全局余弦相似度和最优传输簇分配；Ltxt 扩展为对齐目标文本描述，确保下游MLLM生成的描述与目标语义一致。使用投影I-FGSM，步长α=0.5/255，噪声预算ε=16/255，迭代T=1000步。

## 6. 论文的主要结论与发现

- **投毒攻击普遍有效**：平均ASR (VS) 为61.6%，接近Oracle的67.9%。检索率同样高（如UniversalRAG达89.2%）。攻击主要破坏视觉依赖的轮次，对文本主导轮次影响较小。
- **注入攻击跨架构泛化**：平均ASR 58.4%，其中NGMemory最脆弱（81.3%），Mem0Memory最抵抗（37.5%）。条件ASR（仅当检索成功时）普遍更高或接近ASR，表明MLLM生成阶段也易受影响。
- **跨MLLM转移性强**：GPT-4.1最脆弱（ASR 81.3%），GPT-4o-mini最抵抗（48.8%），但检索率均高（>90%）。部分模型（如Claude-Haiku）存在检索-生成差距：检索成功但生成时不采纳虚假信息，揭示了生成阶段抵抗是一个独立的防御维度。
- **编码器消融**：CLIP优化的扰动可转移至GME、SigLIP等，表明漏洞源于共享的语义对齐属性，而非编码器特定缺陷。UniversalRAG始终最脆弱。
- **防御效果**：图像预处理（高斯模糊、JPEG压缩）最有效，可显著降低ASR（从35.9%降至15.6%）。文本侧过滤（LLM Judge、语义漂移）效果有限，因为对抗生成的描述流畅且合理。检索时异常得分无效，且损失F1。建议采用图像级防御与跨模态一致性检查。

## 7. 优点

- **新颖的威胁模型**：提出首个严格图像有限、无触发词、无文本辅助的持久记忆攻击，攻击能力弱于所有先前工作（表1），揭示了被忽视的视觉管道脆弱性。
- **架构多样性**：覆盖5种显著不同的记忆架构（从简单列表检索到图结构、事实提取、商业系统），实验设计全面。
- **鲁棒的方法设计**：目标设计阶段结合检索位移、语义矛盾、真实答案惩罚三重得分，并引入词汇门/主题门，确保选择高质量矛盾目标。对抗扰动中引入文本对齐损失Ltxt，提升转移性。
- **深入的消融与诊断**：对编码器、对话主题、注入数量、时间距离、防御策略等进行了系统消融，提供可操作见解（如单张图像即有效、时间距离反而增强攻击等）。
- **发现检索-生成分离的防御维度**：不同MLLM的生成阶段抵抗差异（如Claude），为分层防御设计提供了方向。

## 8. 不足与局限

- **实验局限**：仅使用Mem-Gallery单一基准，未在其他多模态对话数据集（如MMRember、VideoChat等）上验证。防御实验仅测试了少量经典预处理和过滤方法，未评估对抗训练、鲁棒嵌入或图像净化等更先进防御。
- **攻击条件假设**：攻击者必须能够获取用户上传的图像并进行扰动，实际场景中可能受传输层加密或图像完整性验证限制。论文中列出了三个现实入口点（CDN、SDK、网页），但未讨论端到端加密或用户端校验的阻碍。
- **转移性机制解释**：尽管实验显示攻击可在不同编码器间转移，但未深入分析为什么共享语义对齐会导致这种转移，缺乏对特征空间扰动本质的理论剖析。
- **随机性与可重复性**：源图像在注入模式中随机采样，不同随机种子可能影响结果，论文未报告多次重复实验统计（如标准误差），仅呈现单次结果。
- **防御评估不完整**：图像级防御（高斯模糊、JPEG）降低了ASR，但作者指出这些方法可能无法抵抗更隐蔽的对抗攻击（如PSI）。未测试端到端的防御系统（如视觉-文本一致性验证模块或检索后验证）。
- **应用限制**：主要针对以GPT-4o系列为后端的MLLM，未覆盖开源模型（如LLaVA、Qwen-VL），也未分析这些模型上的攻击效果。商业接口（如Mem0）在受限设置下（描述模型调用成本）可能进一步限制攻击效率。
