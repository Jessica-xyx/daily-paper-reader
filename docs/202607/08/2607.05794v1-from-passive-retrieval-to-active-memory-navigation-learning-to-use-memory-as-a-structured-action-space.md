## 摘要
长期用户记忆对于个性化对话代理至关重要，然而许多记忆系统仍通过被动检索接口暴露记忆，使模型成为预选证据的消费者。我们提出NapMem框架，一种学习将长期用户记忆用作结构化行动空间而非被动检索上下文的方法。NapMem将用户历史组织成关联的多粒度记忆金字塔，其中原始对话、类型化记忆记录、主题追踪和用户档案通过溯源关系连接，并通过记忆工具暴露这些层级。代理被训练根据查询和中间证据选择记忆，使其在回答前能检查不同记忆粒度。在PersonaMem-v2、LongMemEval和LoCoMo上的实验表明，使用记忆工具强化学习训练的NapMem代理在多种记忆密集型任务上具有竞争力，而在非记忆任务上的评估表明，学习到的策略在很大程度上保留了通用推理和工具使用能力。额外分析考察了存储、推理成本、工具使用行为以及对导航、记忆粒度和强化学习训练的消融研究。我们的结果表明，长期用户记忆受益于将结构化存储与学习到的在适当粒度上使用记忆的策略相结合。

## Abstract
Long-term user memory is essential for personalized conversational agents, yet many memory systems still expose memory through passive retrieval interfaces, making the model a consumer of pre-selected evidence. We introduce NapMem, a framework for learning to use long-term user memory as a structured action space rather than passively retrieved context. NapMem organizes user history into a linked multi-granularity memory pyramid, where raw conversations, typed memory records, topic tracks, and user profiles are connected through provenance relations, and exposes these levels through memory tools. The agent is trained to select memory according to the query and intermediate evidence, allowing it to inspect different memory granularities before answering. Experiments on PersonaMem-v2, LongMemEval, and LoCoMo show that a NapMem agent trained with memory-tool reinforcement learning is competitive across diverse memory-intensive tasks, while evaluations on non-memory tasks suggest that the learned policy largely preserves general reasoning and tool-use abilities. Additional analyses examine storage, inference cost, tool-use behavior, and ablations over navigation, memory granularity, and RL training. Our results suggest that long-term user memory benefits from coupling structured storage with a learned policy for using memory at the appropriate granularity.

## 1. 核心问题与整体含义（研究动机和背景）

长期用户记忆对于个性化对话代理至关重要，但现有记忆系统大多采取“被动检索”模式：系统将记忆存储为固定结构，通过检索接口向代理提供预选上下文。这种设计限制了代理对记忆的主动控制能力——当检索到的上下文不完整时，代理无法进一步探查更细粒度或更高层级的证据。论文提出应将长期用户记忆的使用从“系统级被动检索”转变为“代理原生记忆导航”，让代理能够根据查询和已收集的中间证据，主动决定是否需要记忆、选择哪层抽象、以及判断证据是否足够。

## 2. 方法论：核心思想、关键技术细节与算法流程

### 核心思想
NapMem 框架将长期用户记忆组织成一个**多粒度记忆金字塔**，并通过**记忆工具**暴露给代理。代理使用强化学习（GRPO）训练，学习如何主动在金字塔中导航，以选择适当粒度的记忆来回答问题。

### 关键技术细节
- **记忆金字塔结构**（自底向上）：
  - **原始对话层**：存储用户与代理的原始消息，保留最高保真度证据。
  - **记忆记录层**：将对话转化为紧凑的、类型化的记忆记录（事实、事件、指令、偏好），通过两阶段增量更新（提取+调和）维护。
  - **主题追踪层**：将相关记忆记录组织成跨会话演化的用户中心主题轨迹，提供中等粒度抽象。
  - **用户档案层**：维护全局用户概要，总结稳定属性、长期偏好和交互模式。
- **层级间通过溯源链接连接**，支持自顶向下或自底向上的导航。
- **记忆工具**（5个）：
  - `search_records` / `search_conversation`：混合检索候选记录/对话。
  - `get_records` / `get_conversation`：通过ID精确获取。
  - `read_file`：读取主题轨迹和用户档案文件。
- **强化学习训练**：使用 **GRPO**（Group Relative Policy Optimization），以组相对优势优化策略。奖励函数由三个二进制准则组成：格式有效性（$F$）、答案正确性（$C$）和记忆工具使用（$U$）。奖励定义如下：
  - $F=0$：$-1$
  - $F=1, C=1, U=1$：$+1$
  - $F=1, C=1, U=0$：$0$
  - $F=1, C=0, U=1$：$-0.5$
  - $F=1, C=0, U=0$：$-1$
- **算法流程**：对于每个查询，代理顺序执行动作（调用工具或直接回答），形成轨迹 $\tau = (a_1, o_1, \dots, a_k, o_k, y)$。GRPO目标包含组相对优势和KL散度惩罚，对每个令牌统一应用轨迹级优势。

## 3. 实验设计：数据集、基准与对比方法

### 记忆密集型任务
- **LoCoMo**：长多会话对话记忆，转换为说话者特定的用户-代理记忆库，测试长期回忆。训练/验证/测试按用户6:2:2分割，最终测试集1315个问题。
- **LongMemEval**：评估长程交互记忆（单会话回忆、多会话推理、时间推理、知识更新、偏好问题），作为**分布外测试**基准，100个问题。
- **PersonaMem-v2**：评估基于隐式用户偏好和人格信号的个性化响应选择，200个用户6:2:2分割，测试集911个问题。

### 非记忆任务（测试泛化能力）

- **BFCL-v3**：文本函数调用（工具使用）。
- **V*Bench**：视觉工具使用问答。
