## 摘要
潜在内存（latent memory）通过将过去的知识片段存储为逐层隐藏状态，已成为大型语言模型（LLMs）中长期记忆的一种有前景的范式（例如 MemoryLLM 和 M+）。然而，该范式在内存更新期间会遭受显著的性能下降，原因在于位置编码错位以及缺乏任何追踪机制来区分目标记忆片段与无关片段。为了发现这样一种追踪机制，我们探测了存储的记忆片段上的逐层注意力密度，发现少量中间Transformer层始终在目标片段上集中了最高的密度——暴露了一个内在的追踪信号。基于此，我们提出了 MemDefrag，一个无需训练且模型无关的框架，该框架（1）使用中间层追踪信号来进行内存碎片整理（对记忆进行排序、重新排序和过滤），以及（2）当容量超出时应用基于信息量的比例遗忘机制。实验表明，MemDefrag 在知识保留（例如，50次内存更新后为43.0% vs. 17.4%/17.6%）和长上下文基准测试上显著优于 MemoryLLM 和 M+，并且在各种LLMs和潜在内存变体上具有良好的泛化能力。

## Abstract
Latent memory, which stores past knowledge fragments as per-layer hidden states, has emerged as a promising paradigm (e.g., MemoryLLM and M+) for long-term memory in large language models (LLMs). However, the paradigm suffers from significant performance degradation during memory updates, due to positional encoding misalignment and the absence of any tracing mechanism to distinguish target memory fragments from irrelevant ones. To discover such a tracing mechanism, we probe the layer-wise attention density over stored memory fragments, and find that a small set of middle transformer layers consistently concentrates the highest density on the target fragment - exposing an inherent tracing signal. In light of this, we propose MemDefrag, a training-free and model-agnostic framework that (1) uses a middle-layer tracing signal to conduct memory defragmentation (rank, reorder, and filter memories), and (2) applies an informativeness-guided proportional forgetting mechanism once capacity is exceeded. Experiments show that MemDefrag substantially outperforms MemoryLLM and M+ on knowledge retention (e.g., 43.0% vs. 17.4%/17.6% after 50 memory updates) and long-context benchmarks, and generalizes well across various LLMs and latent-memory variants.

### 1. 论文的核心问题与整体含义（研究动机和背景）
- **研究动机**：大型语言模型（LLM）的长期记忆范式——潜记忆（如 MemoryLLM、M+）——将知识片段存储为逐层隐藏状态，避免重复编码完整上下文。但该范式在多次记忆更新后性能严重下降，主要源于两个障碍：
  1. **位置编码错位**：新知识片段拼接后，片段副本的原始位置索引与实际推理时位置索引不匹配，扭曲注意力计算。
  2. **缺乏追踪机制**：现有方法将所有记忆片段视为扁平前缀，无法区分目标相关片段与无关噪声。
- **背景**：现有 LLM 记忆分三类：token-level（显式文本，受限于上下文窗口）、parametric（编码进参数，需重训且易灾难性遗忘）、latent（隐式表示如隐藏状态）。本文聚焦后者。

### 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：发现中间层Transformer的注意力密度可作为**内在追踪信号**，据此提出**无需训练、模型无关**的 MemDefrag 框架，在推理时执行内存碎片整理，并在容量超限时实施信息量引导的比例遗忘。
- **关键技术细节**：
  - **注意力密度定义**：对给定提示 p 和记忆片段 xi，在层 l 计算提示最后一个 token（或所有token平均）对 xi 所有位置的注意力得分均值 ρli。该指标与问答精度高度相关（Pearson/Spearman >0.95）。
  - **追踪信号发现**：通过层间探测，发现**特定中间层（如Llama-3.1-8B的层13）**一贯性地对目标片段分配最高注意力密度（Top-1准确率85.6%，Top-3准确率95.7%）。
  - **推理时内存碎片整理**（Algorithm 1）：
    1. 根据追踪层 l* 的注意力密度对所有记忆片段升序排序。
    2. 按排序重新排列各层隐藏状态，使最相关片段靠近提示（利用因果注意力的近因偏置）。
    3. 仅保留 Top-K 个最相关片段作为推理时的临时记忆 θpn。
  - **基于自信息的比例遗忘**（Algorithm 2 & 3）：
    1. 当总记忆长度超过最大容量 Nmax 时，计算需遗忘的 token 总数 Nforget。
    2. **比例分配**：按各片段当前保留长度占总长度的比例分配遗忘配额。
    3. **自信息选择**：在每个片段内，丢弃自信息 I(x(t)i) 最低（即最冗余）的 token。自信息由模型预测概率计算：I = -log pϕ(x(t)i | x(<t)i)。

### 6. 论文的主要结论与发现
- **核心发现**：注意力密度（尤其是中间层）可作为可靠的追踪信号，且与准确性高度相关。
- **MemDefrag 显著优于基线**：
  - 知识保留：50步后 NaturalQA 准确率 43.0%（Top-1），MemoryLLM 17.4%，M+ 17.6%。

- **最佳 K 保持小值**（Top-1或Top-2）优于大K，因为集中注意力在目标片段上更有效。

- **Prompt 压缩兼容**：适当压缩可延缓遗忘触发，提升长期保持。
