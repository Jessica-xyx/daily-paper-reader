## 摘要
记忆专长是一种习得的技能：知道编码什么、何时检索以及如何组织知识——这种能力在认知科学中被称为元记忆。我们将这一视角引入大语言模型，将记忆管理视为一种可训练的技能。我们将文件系统操作提升为一等记忆动作，与任务动作并列，让模型自行决定如何管理其记忆。这种记忆技能沿着两个维度提升：支撑它的结构（提示、文件模式、动作词汇），以及运用它的模型的熟练程度。这两个维度都难以手动优化：长周期任务中的轨迹片段运行数千步，而一个记忆错误可能在暴露之前很久就隐藏起来，使得人工审查完整轨迹不切实际。我们引入了AutoMem，一个自动化这两个维度的框架。在第一个循环中，一个强LLM审查完整的智能体轨迹，并迭代修订塑造智能体与其记忆文件交互方式的记忆结构。在第二个循环中，从众多轨迹片段中识别出智能体自身良好的记忆决策，并将其用作训练信号，直接提升模型的记忆熟练度。在三个程序生成的长周期游戏（Crafter、MiniHack和NetHack）中，仅优化记忆——不修改模型的任务动作行为——就将基础智能体的性能提升了约2至4倍，使一个320亿参数的开源模型与Claude Opus 4.5和Gemini 3.1 Pro Thinking等前沿系统竞争。我们的结果表明，记忆管理是一种独立可学习的技能，并且是一个在长周期任务上产生巨大收益的高杠杆目标。

## Abstract
Memory expertise is a learned skill: knowing what to encode, when to retrieve, and how to organize knowledge--a capacity known in cognitive science as metamemory. We bring this perspective to LLMs by treating memory management as a trainable skill. We promote file-system operations to first-class memory actions alongside task actions, letting the model itself decide how to manage its memory. This memory skill improves along two axes: the structure that supports it (prompts, file schemas, action vocabulary), and the proficiency of the model exercising it. Both axes resist manual optimization: episodes in long-horizon tasks run for thousands of steps, and a single memory mistake can hide long before it surfaces, making human review of full trajectories impractical. We introduce AutoMem, a framework that automates both axes. In the first loop, a strong LLM reviews complete agent trajectories and iteratively revises the memory structure that shapes how the agent interacts with its memory files. In the second loop, the agent's own good memory decisions are identified from many episodes and used as training signal to sharpen the model's memory proficiency directly. Across three procedurally generated long-horizon games (Crafter, MiniHack, and NetHack), optimizing memory alone--without modifying the model's task-action behavior--improved the base agent's performance ~2x-4x, bringing a 32B open-weight model competitive with frontier systems such as Claude Opus 4.5 and Gemini 3.1 Pro Thinking. Our results show that memory management is an independently learnable skill, and a high-leverage objective yielding large gains on long-horizon tasks.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在长时任务（如复杂游戏）中，大语言模型（LLM）智能体需要管理超出工作记忆（上下文窗口）的信息。认知科学中的**元记忆**描述了人类如何学会决定该记忆什么、何时检索以及如何组织知识。本文将这一视角引入LLM，提出记忆管理是一项可独立训练的认知技能，而非固定的结构模块。
- **背景**：现有外部记忆方法（如检索增强生成、记忆流、总结缓冲区）大多将记忆视为预定义架构，而非可学习的技能。手动优化记忆策略在长时任务中不可行，因为单个回合可长达数万步，且记忆错误的后果可能延迟很久才显现。
- **整体含义**：作者认为通过自动化记忆技能的结构优化和模型熟练度训练，可以显著提升智能体在长时任务中的表现，甚至缩小开源模型与前沿专有模型之间的差距。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将文件系统操作（读、写、搜索、追加、创建）提升为与任务动作同等地位的**一等记忆动作**，让模型自行决定如何管理外部记忆。记忆技能沿两个维度提升：**结构**（提示词、文件模式、动作词汇）和**熟练度**（模型参数能力）。AutoMem通过两个外部循环自动化这两个维度。

- **内循环智能体**：每个回合中，智能体使用文件系统作为外部记忆。每一步执行两个例程：
  - **LOG例程**：记录刚发生的事（追加、创建新文件或重写条目）。
  - **PLAN例程**：搜索/读取记忆文件，然后提交下一个世界动作。
  - 记忆操作与任务动作共享同一动作空间，使每个记忆决策成为轨迹中可追踪的动作。

- **外循环1：记忆结构优化**
  - 一个强语言模型（meta-LLM，Claude Opus 4.6）审查完整回合轨迹（每步日志、记忆目录、智能体代码），识别结构导致的失败模式。
  - 迭代修订智能体代码、提示词、文件模式（例如引入坐标去重格式的`<|UPSERT_MAP|>`操作、自动同步库存/状态文件、预填充策略参考等）。
  - 每次修订通过固定种子评估，仅当平均进展率提升时才保留。

- **外循环2：记忆熟练度训练**
  - 在结构优化完成后，meta-LLM（Claude Opus 4.7）作为训练引擎，从大量随机回合轨迹中筛选好的记忆决策作为监督训练数据。
  - 训练仅针对**记忆专家模型**（通过LoRA微调），任务模型保持冻结，避免遗忘任务动作格式。
  - 推理时部署两个模型实例：微调的记忆专家处理LOG和记忆检索部分，基础任务模型提交世界动作，通过简短交接保持一致历史。

- **关键设计**：两个循环共享同一内循环智能体；结构优化先推高上限，熟练度训练再朝上限逼近。两个循环均由强LLM驱动，无需人工审查完整轨迹。

## 7. 优点：方法或实验设计上的亮点

- **认知科学启发**：将人类元记忆概念明确转化为LLM的可训练技能，理论框架新颖。
- **双向优化**：同时优化结构（代码级）和熟练度（参数级），且分离训练避免干扰任务能力。
- **自动化闭环**：无需人工审查完整轨迹，利用强模型进行轨迹分析、代码修订、数据筛选和训练配置，可扩展至任何长时任务。
- **统一动作空间**：记忆操作与任务动作同等地位，使所有行为可观察、可分析，便于诊断失败模式。
- **强大的消融设计**：通过逐步迭代（v0→v5等）展示记忆结构演化的具体效果，并提供行为指标定量分析。

## 8. 不足与局限

- **环境限制**：仅在游戏环境（Crafter、MiniHack、NetHack）上评估，未验证于现实世界任务（如机器人、软件开发等），游戏可能不完全代表真实记忆密集型场景。
- **记忆非持久化**：文件系统每个回合重新初始，无跨回合记忆（如长期知识积累），限制了应用范围。
- **独立优化**：每个环境单独优化结构（不同迭代版本）和训练记忆专家，未探索一个共享结构或专家能否泛化到多个环境。
- **模型族单一**：仅使用了Qwen2.5-32B作为内循环模型，未测试其他架构（如Llama、Mistral）的适用性。
- **安全与部署**：作者指出当前方法未经过安全审查，直接用于高安全任务需谨慎。
- **meta-LLM成本**：依赖强大且昂贵的meta-LLM（Claude Opus 4.6/4.7）进行审查和训练引擎，可能限制可部署性。
- **训练数据规模**：每个环境仅数百至上千条训练示例，可能不足以完全发挥微调潜力。
