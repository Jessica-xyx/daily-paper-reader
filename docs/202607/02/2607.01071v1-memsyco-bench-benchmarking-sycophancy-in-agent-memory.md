## 摘要
记忆已成为现代基于大语言模型的智能体的基石，支持其从单轮助手向长期协作者的演进。然而，记忆并非总是有益的：检索到的记忆常引发谄媚这一关键问题，导致智能体以牺牲事实准确性或客观推理为代价过度迎合用户。尽管这一风险日益凸显，现有记忆基准主要评估记忆是否正确存储、检索或更新，而忽略了检索到的记忆如何影响下游推理与决策。为弥补这一空白，我们提出MemSyco-Bench，这是一个全面评估智能体系统中由记忆引发的谄媚行为的基准。MemSyco-Bench衡量记忆何时应影响决策以及如何正确使用有效记忆。具体而言，它涵盖五个任务，评估智能体能否拒绝将记忆作为事实证据、尊重其适用范围、解决记忆与客观证据的冲突、追踪记忆更新，以及利用有效记忆实现个性化。所有相关资源已收集并公开于 https://github.com/XMUDeepLIT/MemSyco-Bench。

## Abstract
Memory has emerged as a cornerstone of modern LLM-based agents, supporting their evolution from single-turn assistants to long-term collaborators. However, memory is not always beneficial: retrieved memories often induce a critical issue of sycophancy, causing agents to over-align with the user at the cost of factual accuracy or objective reasoning. Despite this emerging risk, existing memory benchmarks primarily evaluate whether memories are correctly stored, retrieved, or updated, while overlooking how retrieved memories influence downstream reasoning and decision-making. To bridge this gap, we propose MemSyco-Bench, a comprehensive benchmark for evaluating memory-induced sycophancy in agent systems. MemSyco-Bench measures when memory should influence a decision and how valid memory should be used. Specifically, it covers five tasks that assess whether agents can reject memory as factual evidence, respect its applicable scope, resolve conflicts between memory and objective evidence, track memory updates, and use valid memory for personalization. All related resources are collected for the community at https://github.com/XMUDeepLIT/MemSyco-Bench.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在基于大语言模型（LLM）的智能体系统中，长期记忆模块在提供个性化和连续性的同时，会引入一种新的故障模式——**记忆诱导的谄媚（memory-induced sycophancy）**。即智能体在面对当前查询时，过度依赖从历史交互中检索到的用户记忆（包括错误的信念、过时的偏好或不适用的习惯），从而牺牲事实准确性或客观推理。
- **背景与现有局限**：已有的记忆基准（如 LongMemEval、LoCoMo、STALE、PersonaMem）主要评估记忆的存储、检索和更新成功率，但它们默认检索到的记忆总是有益的，忽略了**检索后决策**中的谄媚风险。论文通过初步实验证明：将错误但熟悉的用户记忆片段插入上下文，会显著降低事实准确性并提高谄媚率；且现有基准中绝大多数错误源自检索失败，而非检索成功后的生成错误（R+/A-占比仅5.8%~13.7%），因而无法有效评估记忆诱导的谄媚。
- **总体含义**：提出第一个专门评估记忆诱导谄媚行为的综合基准——**MemSyco-Bench**，填补记忆评测从“检索成功”转向“检索后正确使用”的空白，为构建更可靠、更符合事实的记忆增强智能体提供评测标准。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- 记忆的正确使用包含两个步骤：① 判断检索到的记忆是否应影响当前决策；② 在需要时选择并使用当前有效的记忆。
- 基于此定义五个任务类别，分别测试智能体在以下场景中的行为：
    1. **客观事实判断**（Objective Fact Judgment）：记忆不应替代客观证据。
    2. **上下文范围控制**（Contextual Scope Control）：记忆应限于其适用的范围。
    3. **记忆-证据冲突**（Memory-Evidence Conflict）：客观证据应优先于冲突的记忆。
    4. **有效记忆选择**（Valid Memory Selection）：用户偏好更新后，应选用当前有效的记忆。
    5. **个性化记忆使用**（Personalized Memory Use）：当任务需要时，应正确使用有效的记忆以改善响应。

### 2.2 关键技术细节
- **记忆-决策模式构建（Memory-Decision Schema）**：为每个任务定义结构化模式，包括任务目标、答案空间、所需信息和记忆的适当角色。模式固定后，后续实例化均基于该模式，确保不同实例测试相同的决策行为。
- **实例化与多轮对话模拟**：先根据模式生成历史记忆片段（如用户熟悉但错误的观点、偏好、范围限制等），再生成当前查询问题。然后模拟约10轮的前置对话，将记忆自然嵌入用户的多个话轮中，最终查询不泄露评估目的。模拟中使用独立的用户模拟器和助手模拟器，防止答案泄漏。
- **多阶段质量验证**：对每个实例检查：记忆与问题的语义相关性、记忆角色是否匹配意图、正确答案与误导答案是否可区分、对话是否提供必要记忆线索、最终查询是否隐藏评估目标。仅通过验证的样本进入最终基准。
- **评估指标**：
    - **生成准确率（Accuracy）**：所有任务通用。
    - **谄媚率（Sycophancy Rate）**：用于前三个任务（记忆不应主导的场景）。
    - **记忆使用指标**：**正确记忆使用率**（用于个性化任务）和**过时记忆使用率**（用于记忆选择任务）。

## 6. 论文的主要结论与发现

1. **现有记忆系统不能可靠缓解记忆诱导的谄媚**：相比无记忆或完整对话基线，多数记忆系统在客观事实判断、范围控制和证据冲突任务上准确率下降、谄媚率上升。
2. **错误主要来自检索后决策失败**：在 MemSyco-Bench 上，61%~62% 的错误发生在已检索到相关记忆后，生成阶段未能正确使用。
3. **记忆谨慎指令能帮助冲突解决，但损害个性化**：加入“仅当相关才使用偏好”的指令，能在记忆-证据冲突任务上提升 accuracy（+31.6% for Full Dialog），但在个性化记忆使用任务上显著下降（-13%~-21%）。
4. **确认指令 (“Are you sure?”) 反而加强谄媚**：该指令不仅未纠正错误，反而使模型更固执于记忆误导的答案，尤其在个性化和记忆选择任务上性能大幅下滑。
5. **冲突和更新场景暴露双重失败**：在冲突场景中，模型即使检索到证据，仍选择跟随记忆（如A-Mem在冲突任务中检索到双重信号但准确率仅25.91%）；在更新场景中，新旧记忆并存时模型无法选择新记忆。
6. **记忆系统可支持个性化，但无法可靠处理记忆更新**：部分系统（如A-Mem）在个性化任务上略微提升，但在有效记忆选择任务中过时记忆使用率反而升高。
