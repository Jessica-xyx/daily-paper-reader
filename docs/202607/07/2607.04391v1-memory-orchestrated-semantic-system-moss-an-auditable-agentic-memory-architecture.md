## 摘要
长时记忆仍是AI智能体的结构性弱点。主流方法——检索增强生成（RAG）依赖基于嵌入的相似性搜索，但这种方式本质不透明、难以审计，且受限于向量表示的理论上限。我们提出记忆编排语义系统（MOSS），一种由智能体驱动结构化关系数据库检索的智能体记忆架构。MOSS与模型、存储和API无关：它可运行于任何关系引擎，连接任何大语言模型提供商（或确定性非大语言模型过程），并部署于任何基础设施（本地或云端）。其检索执行是符号化且可复现的（一旦查询公式化，检索循环中不再涉及大语言模型），从索引到答案生成的每一步都可记录和检查，使得MOSS本质可审计。MOSS并非强加外部本体，而是从语料库自身推导概念词汇。我们报告了记忆智能体文献中独一无二的纵向部署：在一位学者工作语料库上的连续一年生产运行——该会话语料库可追溯至2024年10月（约4400万词元，已回溯索引），包含110,183个片段，以及163,494个编目文档、569个归纳推导的概念、322,662个概念注释和总计约500万关系的十一个元数据图——跨越连续四代基础设施。虽然本案例为单个研究者，但该架构绝非特化于个人：它服务于团队、机构或任何随时间积累知识的实体。我们认为，可审计、自主、结构无界的长时记忆是旨在陪伴个人或组织数年而非数轮会话的AI智能体的先决条件。

## Abstract
Long-term memory remains a structural weakness of AI agents. The dominant approach, retrieval-augmented generation (RAG), relies on embedding-based similarity search, which is opaque by construction, difficult to audit, and bounded by the theoretical limits of vector representations. We present the Memory-Orchestrated Semantic System (MOSS), an agentic memory architecture in which the agent drives retrieval over a structured relational database. MOSS is model-agnostic, storage-agnostic, and API-agnostic: it runs on any relational engine, connects to any LLM provider (or to deterministic non-LLM processes), and deploys on any infrastructure, local or cloud. Its retrieval execution is symbolic and reproducible (once a query is formulated, no LLM participates in the retrieval loop) and every step of the system, from indexing to answer formulation, is logged and inspectable, making MOSS auditable by construction. Rather than imposing an external ontology, MOSS derives its conceptual vocabulary from the corpus itself. We report on a longitudinal deployment unique in the agentic-memory literature: a year of continuous production over an individual scholar's working corpus--a conversational corpus reaching back to October 2024 (some 44 million tokens, retroactively indexed) comprising 110,183 segments, alongside 163,494 catalogued documents, 569 inductively derived concepts, 322,662 concept annotations, and eleven metadata graphs totaling approximately five million relations--across four successive infrastructure generations. While the present case is that of a single researcher, the architecture is in no way specific to one person: it serves a team, an institution, or any entity that accumulates knowledge over time. We argue that auditable, sovereign, structurally unbounded memory is a precondition for AI agents intended to accompany a person or an organization over years rather than sessions.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前AI智能体的长期记忆依赖检索增强生成（RAG），即通过嵌入向量相似性搜索来检索信息。这种方法存在三大结构性缺陷：**不透明**（高维空间中的余弦相似度无法被人类验证）、**难审计**（系统行为无法追溯至具体查询和逻辑）、**理论受限**（单向量检索有形式极限，随语料增长相关文档被“淹没”）。
- **研究背景**：大语言模型本质上是无状态的（stateless），智能体的连续性必须由外部基础设施提供。主流范式 `Agent = Model + Harness` 中，记忆组件决定了智能体能否陪伴用户数月乃至数年。现有方法（如RAG、GraphRAG、MemGPT、Mem0等）要么是检索管道而非记忆系统，要么是黑盒合成，要么局限于会话级规模。
- **整体含义**：作者主张**可审计、主权化、结构无界**的长期记忆是人工智能智能体能够长期陪伴个人或组织的先决条件。论文提出的MOSS系统试图填补“终身语料+智能体导航器+可审计关系型基础”的交集空白。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **数据库是地图，文档是领土**：原始语料（对话转录和文档）以普通文件形式完整保留，不被分块或替换。其上使用关系型数据库存储密集的元数据地图（摘要、时间戳、参与者、情感状态、概念标注等）。大多数查询仅从地图中回答，无需触达原始文件。
- **查询主权**：相关性不是存储时预计算为静态权重，而是**查询时由智能体根据意图动态决定权重**（例如时间性问题优先时间戳，情感性问题优先效价/激活值）。这颠倒了RAG假设单一嵌入空间适用于所有查询的假设。
- **审计一切，不隐藏任何东西**：每次调用实时记录，每个检索是显式SQL查询，可重现和检查。对话档案保存为纯文本+时间戳+词元计数。任何回答都可追溯至具体查询和片段。

1. **系统架构**（图1）：分为**编排器**（Orchestrator）和**记忆结构**（Memory Structure）。编排器包含REST API/MCP端点、LLM连接器、QueryProfiler和SQL检索层；记忆结构是关系型数据库（当前部署SQLite，亦可PostgreSQL等）。系统是模型无关、存储无关、API无关的。
2. **结构化记忆**：
   - 片段（Segments）：110,183个对话片段，每个携带时间戳、词元计数、摘要、提及人物、情感状态（Russell环状模型：效价×激活）、概念标注。片段是主要检索单元。
   - 文档（Documents）：163,494个编目文件，其中约21,000个文本文件被丰富（摘要、关键词、结构大纲、概念标注）。大纲允许智能体分层导航。
   - 归纳式本体（Inductive Ontology）：569个概念从语料自底向上推导（结合少量手工种子），共322,662个片段级标注。类似于定性研究中的编码簿主题分析。

3. **查询生命周期**（图2）：
   - 意图分析（智能体层）：分析查询意图（时间性、主题性、情感性等）。
   - 概要分析（智能体→结构化）：智能体自然语言请求给QueryProfiler，携带结构化权重；QueryProfiler翻译为SQL。
   - 执行（确定性的）：SQL检索层运行查询，无LLM参与；结果可重现、可解释。
   - 评估与迭代（智能体层）：智能体检查结果，若不满意则调整权重或策略重新查询（最多15次迭代）。
   - 答案形成：引用找到的信息，生成最终回答。
4. **实时索引与归档**：专用scribe智能体拦截离开滑动窗口（~35K词元）的内容，立即进行索引（分段、摘要、情感和概念标注）并归档到当天对话文件中。使有限窗口实际上解除了记忆上限。
5. **审计性**：检索是显式SQL、每次调用日志实时记录、语料为纯文本、本体为关系表——答案“为何如此”始终可重构。

## 6. 论文的主要结论与发现

1. **审计性改变关系**：因为每次检索可查看实时日志、每个答案可追溯至查询和片段，系统失败时可诊断和修复，用户对系统的信任源于其可检查性而非绝对准确率。
2. **地图通常足够**：承载摘要和丰富元数据的数据库能独立回答大部分日常问题（如“第一次运行MOSS是什么时候？”“自我1995年以来关于X的工作总结”），无需读取原始文件。这与RAG的分块填充方式形成结构性token经济对比。
3. **情感索引有价值**：编码每个片段的效价和激活值（情感状态）是工程驱动系统中不常见的选择，但在实践中成为最常用的检索维度之一（关于个人历史的问题经常带有情感性）。
4. **系统无关性经检验**：MOSS的核心架构（地图>领土、查询主权、全日志）在跨越三个LLM供应商、两个云提供商、一个本地工作站和一个专用服务器的四代迁移中存活下来。
5. **主权化为架构特性**：因为记忆是标准SQL，语料是普通文件，整个系统可以随意搬迁（跨云、到笔记本）而无需转换，避免供应商锁定。
6. **个体知识工作者是最难案例**：企业部署继承现有本体（客户、病例等），而个体语料混合研究、教学、行政、家庭、金融等，无既定模式。解决个体案例的方法论可以向下转移到组织。

## 7. 优点：方法或实验设计上的亮点

- **架构设计亮点**：
  - **三库无关性**（模型、存储、API）：摆脱任何形式的锁定，提升系统迁移性和可扩展性。
  - **查询时加权（Query-time Sovereignty）**：反转RAG静态权重假设，让智能体根据意图动态决定相关性，更灵活。
  - **确定性检索+智能体迭代**：将非确定性限制在LLM边缘（意图分析和答案形成），检索核心为纯SQL，实现可重现、可审计。
  - **情感索引纳入**：将情感作为第一类元数据，满足个人记忆中对情感维度的自然需求。
  - **归纳式本体**：自底向上从语料中抽取概念，比外部强加本体更贴合个体。
  - **Métacalque原则（已部分实现）**：将文学理论中的多层阅读概念操作性转化为可执行检索层，具有原创性。
- **实验亮点**：
  - **长期现场部署**：约一年连续生产，穿越四代基础设施，提供了业界缺乏的操作性经验。
  - **透明度**：详细报告语料规模、索引数量、技术演进，并诚实地声明局限性。

## 8. 不足与局限

- **缺乏标准基准对比**：未与Mem0、Zep、GraphRAG等竞争系统进行定量比较，结论仅基于自身体验。
- **单用户偏差**：设计者和使用者为同一人（第一作者），可能引入确认偏误；系统提示和本体设计隐含个人偏好。
- **不可复现**：语料为个人私密数据，无法用于独立验证。
- **归纳本体构建成本未量化**：569个概念如何从语料自动或半自动提取？成本（标注时间、LLM调用量）未报告。
- **检索确定性不等于全管道确定性**：智能体意图分析和迭代仍依赖LLM，存在随机性。作者承认但将该随机性视为特征。
- **多模态尚未完成**：音频、视频、图像的富化仍在开发中；当前系统仅文本完备。
- **多租户隔离与记忆投毒防御未处理**：目前单用户部署，将来的多用户场景需要额外安全措施。
- **纵向证据的概括性有限**：一个学者的工作模式与其他职业（律师、医生、客服）差异巨大，架构通用性仅在论证层面，未经验证。
- **情感模型依赖Russell模型及Mattek等人修正，可能简化复杂情感状态**。
