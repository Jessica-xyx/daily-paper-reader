## 摘要
长期存在的人工智能体需要在交互之间保持连续性，但连续性无法通过简单扩展提示窗口获得。智能体必须保留有用的先前经验、选择性检索、区分个人背景与外部证据，并在底层情境变化时修正记忆。我们提出了一种沿两个正交轴组织的架构性记忆基板：表示轴涵盖结构化记录、向量表示和图关系；时间轴涵盖短期痕迹、中期抽象和长期语义承诺。其关键设计约束是同步的结构化-向量-图记忆：结构化记录决定资格，向量表示支持回忆，图关系在门控上下文投影之前裁决支持、矛盾和取代。其核心主张是，可靠的个性化是一个记忆设计问题：有用的记忆是结构化的、选择性暴露的、持续巩固的，并且带有认识论标签，而不是存储为无差别的对话历史。除框架外，我们将MRMS实例化为一个轻量级原型，实现了结构化记录、向量检索、时间策略和基于图的修正。该原型通过预生成记忆选择、修正、边界强制和证据归因，在具有明确证据要求的受控长期交互场景下，演练了核心基板机制。

## Abstract
Long-lived AI agents require continuity across interactions, but continuity cannot be obtained by simply extending the prompt window. An agent must preserve useful prior experience, retrieve it selectively, distinguish personal context from external evidence, and revise memory when the underlying situation changes. We propose an architectural memory substrate organized along two orthogonal axes: a representational axis spanning structured records, vector representations, and graph relations; and a temporal axis spanning short-term traces, medium-term abstractions, and long-term semantic commitments. Its key design constraint is synchronized structured-vector-graph memory: structured records govern eligibility, vector representations support recall, and graph relations adjudicate support, contradiction, and supersession before gated context projection. Its central claim is that reliable personalization is a memory design problem: useful memory is structured, selectively exposed, continuously consolidated, and epistemically labeled rather than stored as undifferentiated conversation history. Beyond the framework, we instantiate MRMS as a lightweight prototype implementing structured records, vector retrieval, temporal policies, and graph-based revision. The prototype exercises the core substrate mechanisms through pre-generation memory selection, revision, boundary enforcement, and evidence attribution under controlled long-lived interaction scenarios with explicit evidence requirements.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：长期存在（long-lived）的AI代理需要跨交互保持连续性，但简单扩展语言模型的提示窗口（prompt window）无法实现真正可靠的记忆。代理必须能够保留有用先验经验、选择性检索、区分个人背景与外部证据，并在底层情境变化时动态修订记忆。
- **研究动机**：记忆错误的代价随时间累积——假阴性（遗忘）会丢失连续性，但假阳性（误用过时或错误记忆）危害更大，可能反复引入错误推理。因此，可靠记忆需要**控制影响力**，而非仅仅最大化召回率。
- **关键洞察**：连续性本质上是**记忆基板（memory substrate）**问题。基板应结构化、选择性暴露、持续巩固，并带有认识论标签（如来源、可信度、时效性），而非存储无差别的对话转录。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 2.1 两轴记忆基板（Two-Axis Memory Substrate）
- **表示轴（Representational Axis）**：每个记忆对象同时具有三种视图：
  - **结构化记录（Structured Record）**：控制平面（如记忆ID、状态、来源、作用域、置信度、修订历史），负责授权和精确过滤。
  - **向量表示（Vector Representation）**：用于语义相似性检索。
  - **图关系（Graph Relations）**：记录记忆间的支持（supports）、矛盾（contradicts）、取代（supersedes）、衍生（derived-from）等关系，负责关系推理。
- **时间轴（Temporal Axis）**：
  - **短期痕迹（Short-term）**：当前交互原始记录，低承诺，易丢弃。
  - **中期抽象（Medium-term）**：会话摘要、开放决策，压缩可复用状态。
  - **长期语义承诺（Long-term）**：稳定偏好、事实、约束，需要强证据支撑。
- **四种记忆状态**：原始（raw）、临时（provisional）、活跃（active）、被取代（superseded）、退休（retired）。修订不等于删除——过时记忆仍可保留作为历史记录，但不可引导当前生成。

### 2.2 同步结构化-向量-图记忆（Synchronized Structured-Vector-Graph Memory）
- **跨视图同步不变量（4条）**：
  1. 稳定跨视图标识符：所有视图共享相同记忆ID作为联结键。
  2. 图节点有效性：图遍历不能引用无效结构化记录。
  3. 高影响力图边需有证据记录支持。
  4. 结构化状态主导选择：退休或取代的记忆不能被选择，即使向量或图视图仍可到达。
- **选择操作的四个阶段**：
  - `x_t -> A_t -> N_k(x_t) -> H_t -> C_t`
  - `A_t`：结构化门控后的授权记忆集（基于用户、作用域、状态过滤）。
  - `N_k(x_t)`：在授权集上的top-k向量近邻搜索。
  - `H_t`：在图视图上展开证据子图（支持、矛盾、取代边）。
  - `C_t`：最终组装的结构化上下文包（含当前任务状态、情景证据、语义主张、关系注释、外部证据、负面上下文）。

### 2.3 关键公式
- 候选记忆评分：`s_i = αR(m_i, x_t) + βC(m_i) + γF(m_i, t) + δU(m_i, π_t) - ηN(m_i, C_t)`
  - R: 语义相关性；C: 置信度；F: 新鲜度；U: 任务效用；N: 冗余或干扰成本。
- 巩固提升条件：`promote(T) = I[φ_support(T) + φ_utility(T) - φ_risk(T) > κ]`

### 2.4 上下文包（Context Packet）
- 包含：局部状态、情景证据、语义主张、关系注释、外部证据、负面上下文。包的设计强制生成器区分不同记忆类，并保持简洁（避免无控制地重放转录）。

## 6. 主要结论与发现

- **完整MRMS达到98.8%总体准确率**（95% bootstrap区间98.0–99.4）。
- **各机制贡献明确**：
  - 仅向量检索（21.2%）因无法区分过时/越界记忆而差。
  - 结构化门控将边界控制提升至100%。
  - 时间策略修复过时抑制和弃权（从0到100%）。
  - 图展开贡献矛盾处理（100%）和证据归因（95%）。
- **简洁性**：完整MRMS平均只选择1.11个记忆对象，候选评分后约1.8个，说明投影紧凑。
- **核心结论**：可靠个性化是记忆设计问题，应通过**结构化状态主导选择、跨视图同步、时间分层巩固和认识论标签**来解决，而非靠无差别提示窗口扩展。
