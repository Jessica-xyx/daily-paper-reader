## 摘要
智能体框架是一种外部控制层，通过管理上下文、工具、编排、记忆、解码和输出处理，将基础大语言模型转化为可执行的智能体。虽然框架设计对智能体行为有很大影响，但大多数自动改进方法优化的对象较为狭窄，例如提示、流水线或工作流，且部署的智能体通常对所有情况重用单一的全局框架。我们提出了MemoHarness，一种自适应框架优化方法，能从自身的执行中学习。MemoHarness将框架分解为六个可编辑的控制维度，在双层经验库中存储每个案例的诊断结果和提炼出的全局模式，并利用检索到的经验（无需测试时标签、反馈或额外搜索）将学习到的框架适应到每个测试案例。在针对shell智能体、代码生成和分析推理基准的评估中，MemoHarness相对于我们比较的固定框架有所改进，并显示出对未见过的测试集和基础模型的选择性迁移能力。当大部分检索到的经验可缓存时，其额外上下文也能保持成本竞争力。这些结果证明，执行经验是构建比单一静态配置更具适应性的智能体框架的实用基础，而更广泛的关于统计稳健性和组件归因的结论则留给未来工作。

## Abstract
An agent harness is the external control layer that turns a base LLM into an executable agent by managing context, tools, orchestration, memory, decoding, and output handling. While harness design strongly affects agent behavior, most automatic improvement methods optimize narrower artifacts such as prompts, pipelines, or workflows, and deployed agents usually reuse a single global harness for all cases. We introduce MemoHarness, an adaptive harness optimization framework that learns from its own executions. MemoHarness decomposes the harness into six editable control dimensions, stores per-case diagnoses and distilled global patterns in a dual-layer experience bank, and adapts the learned harness to each test case using retrieved experience without test-time labels, feedback, or additional search. In our evaluation across shell-agent, code-generation, and analytical-reasoning benchmarks, MemoHarness improves over the fixed harnesses we compare against and shows selective transfer to unseen suites and base models. Its additional context can also remain cost-competitive when much of the retrieved experience is cacheable. These results provide evidence that execution experience is a practical substrate for building agent harnesses that are more adaptive than a single static configuration, while leaving broader claims about statistical robustness and component attribution to future work.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想

### 关键技术细节
- **六维框架空间**：沿推理时间流将框架分解为6个功能控制面（表1）：
  1. D1 上下文组装（prompt构造、演示添加、上下文压缩）
  2. D2 工具交互（检索启用、top-k设置、证据重排序）
  3. D3 生成控制（解码配置：max tokens、temperature、候选采样）
  4. D4 编排（工作流拓扑：从单次调用到计划/执行/精炼）
  5. D5 记忆管理（跨调用状态持久化、摘要、丢弃陈旧上下文）
  6. D6 输出处理（提取答案、验证模式、回退选择）
- **双层经验库**：由两部分构成：
  - **案例级经验（E_t）**：每次搜索迭代记录执行轨迹、奖励、成本、诊断结果（成功/失败、主要失败维度、次要贡献维度、自然语言分析）。
  - **全局模式（G_t）**：每N轮迭代通过“蒸馏操作”从失败聚类中提取跨案例的规律，包括现象描述、支持证据和预期修复效果。
- **训练时优化**：
  - 从极简框架W0开始（无演示、无工具、单次解码、无记忆、原始输出通过）。
  - 控制器根据当前框架和累积经验构建查询，检索相关切片（包含全局模式、匹配历史），提出新候选框架。
  - 对所有搜索案例执行候选框架，计算奖励（基于参考输出）和成本（token使用量），并生成诊断。
  - 使用**正确性优先的字典序选择**：首先最大化平均任务奖励，仅在奖励相同时才使用更低的平均token成本作为次要排序标准。
- **测试时案例适应**：
  - 对于无标签测试案例，控制器从冻结的经验库中检索：最相似的**成功案例**、**失败案例**、特征匹配的历史切片和全局模式。
  - 利用这些证据，将训练得到的全局框架W*适应为案例特定框架W(x)，然后执行得到最终输出。
  - **无需测试时标签、反馈或额外搜索**，实现了零成本案例级适应。

## 6. 论文的主要结论与发现

1. **RQ1**：自适应框架优化显著提升任务成功率。MemoHarness在Terminal-Bench上达到0.806，比最强基线Codex（0.722）提升+0.084，比其他基线提升+0.250~+0.445。
2. **RQ2**：搜索过程中框架质量稳步提升，尤其在长周期分析任务（FinanceAgent）上持续改善，而接近饱和的单次代码生成（LiveCodeBench）改善幅度较小。
3. **RQ3**：学习到的框架展示出**选择性迁移**能力。Terminal-Bench来源框架在MMMLU（+0.030）、StrongReject（+0.030）和SWE-Bench Pro（+0.059）上提升；某些饱和套件（HumanEvalFix、Reasoning-Gym-Easy）无变化。证明迁移非通用但非局限。
4. **RQ4**：框架跨基础模型迁移有效。6个额外模型平均提升+0.098，从GPT-4.1的+0.038到GLM-5的+0.233。说明学习到的控制决策具有一定的模型无关性。
5. **RQ5**：成本具有竞争力。MemoHarness输入token较多（14.18M），但大部分可缓存（13.32M），最终成本$6.89低于Codex（$10.28）和Claude Code（$9.51），同时获得更高精度。更便宜的选项（Terminus、OpenCode）精度显著更低。
