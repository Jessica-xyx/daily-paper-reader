## 摘要
多模态大语言模型必须适应不断变化的任务和领域，但在有限的部署足迹下持续改进仍然困难，因为重复的参数更新或不断增长的回放存储会随时间累积适应状态。我们研究固定足迹持续适应：部署的适应状态保持在固定的内存预算下，而骨干模型保持不变，任务特定的更新被外部化。我们提出了InduceKV，这是一种基于检索的方法，它将每个选定的训练前缀存储为一个注意力就绪的记忆条目，由一个冻结的检索键和紧凑的逐层键值（KV）负载组成，这些负载可以附加到模型的自注意力缓存上。在严格的内存预算下，InduceKV通过双层选择构建紧凑的诱导集：为检索拟合轻量级校准，而选定的记忆平衡当前任务似然、基于锚点的保留以及冻结检索空间中的覆盖。在任务增量指令调优、持续VQA、域增量适应和终身多模态指令调优中，InduceKV在匹配的内存预算下始终优于PEFT、MoE、回放和提示检索基线。我们进一步报告了骨干匹配、阶段1 CoIN、计算匹配和可扩展性诊断，表明增益并非来自更强的骨干、仅回放或无限的候选池。

## Abstract
Multimodal large language models must adapt to evolving tasks and domains, yet continual improvement under bounded deployment footprint remains difficult because repeated parameter updates or growing replay stores can accumulate adaptation state over time. We study fixed-footprint continual adaptation: the deployed adaptation state is kept under a fixed memory budget, while the backbone model is left unchanged and task-specific updates are externalized. We propose InduceKV, a retrieval-based method that stores each selected training prefix as an attention-ready memory entry, consisting of a frozen retrieval key and compact layerwise key--value (KV) payloads that can be appended to the model's self-attention cache. Under a strict memory budget, InduceKV constructs a compact inducing set through bilevel selection: a lightweight calibration is fit for retrieval, while the selected memory balances current-task likelihood, anchor-based retention, and coverage in the frozen retrieval space. Across task-incremental instruction tuning, continual VQA, domain-incremental adaptation, and lifelong multimodal instruction tuning, InduceKV consistently improves over PEFT, MoE, replay, and prompt-retrieval baselines under matched memory budgets. We further report backbone-matched, stage-1 CoIN, compute-matched, and scalability diagnostics, showing that the gains are not due to a stronger backbone, replay alone, or an unbounded candidate pool.

## 1. 核心问题与研究动机
- **问题**：多模态大语言模型（MLLM）在持续适应新任务时，部署足迹（deployment footprint）通常受限。重复的参数更新或不断增长的重放存储会累积适应状态，导致预算难以控制、遗忘加剧。
- **目标**：在固定内存预算下，实现高效的持续适应，同时保持骨干模型参数冻结，将任务特定更新外部化到紧凑的KV记忆中。
- **意义**：为多模态持续学习提供一种无需反复更新骨干、可控内存消耗的新范式，适用于计算/隐私敏感的部署场景。

## 2. 方法论：InduceKV
### 核心思想
- **外部化KV记忆**：将每个训练前缀（prefix）提取为注意力就绪的KV记忆条目，包括一个归一化的检索键和逐层的压缩KV负载。
- **双层选择（Bilevel Selection）**：在固定预算下，通过内层校准拟合检索参数，外层联合优化当前任务似然、历史锚点保留和检索空间的谱覆盖，构建紧凑的诱导集。
- **推理时KV注入**：检索到的记忆条目通过加权和注入到模型各层的自我注意力缓存中，影响生成而不修改骨干参数。

### 关键技术细节
1. **条目提取**：
   - 检索键：对前缀的最终层隐藏状态取平均再归一化。
   - KV负载：对每一层的前缀key/value进行固定长度平均池化（长度m=8），存储压缩后的张量。
2. **检索加权与门控**：
   - 检索权重：基于余弦相似度与可学习的温度τ。
   - 值门控：每层有一个可学习的标量λℓ∈(0,1)，控制记忆注入强度。
3. **双层优化**：
   - 内层：拟合校准参数φ（温度、门控），最小化当前任务负对数似然+L2正则。
   - 外层：优化选择权重w（连续松弛），目标函数包含当前任务损失、锚点保留损失、谱覆盖正则项（log-determinant惩罚），投影到预算单纯形。
   - 最终通过Top-B离散化得到选中的记忆条目。

### 算法流程（文字描述）
- 对每个新任务：从当前数据中提取候选条目，与旧记忆合并为候选池。
- 运行I次双层迭代：每轮执行J步内层梯度更新φ，然后更新一次外层权重w，并投影到预算约束。
- 取前B个权重对应的条目作为新记忆，存储校准参数φ。
- 推理时：先计算查询键，检索记忆，加权后作为额外KV token注入自注意力。

## 8. 不足与局限
1. **推理开销**：需要额外一次前缀前向计算检索键，导致预填延迟高于无记忆的PEFT方法（约2倍）。作者定位为提示检索的替代而非零开销方法，但某些实时场景可能不适用。
2. **依赖注意力接口**：需要访问骨干模型的隐藏状态和KV缓存，对封闭API或非标准注意力架构的支持受限。
3. **锚点假设**：理论要求锚点能代表性近似历史风险，虽然文中验证了高相关性，但在分布漂移极大的场景下该假设可能不成立。
4. **预算上限**：实验最大预算为512条目（约578MB），未测试更大预算下的饱和现象；条目数爆炸时双层优化的候选池大小可能成为瓶颈。
5. **任务顺序**：虽然做了顺序敏感性实验，但未覆盖所有可能的组合（CoIN仅16种随机顺序），极端顺序下的表现未充分探讨。
6. **安全与隐私**：记忆条目直接存储训练数据衍生物，可能引入隐私泄露风险，文中仅提及但未提供防御措施。
7. **固定长度载荷**：使用固定m=8的平均池化，对于极长或极短的前缀可能信息损失，未探索自适应池化方案。
