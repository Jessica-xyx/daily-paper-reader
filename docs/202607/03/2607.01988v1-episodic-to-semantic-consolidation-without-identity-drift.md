## 摘要
长期运行的自适应智能体面临知识整合与信息完整性之间的结构性张力。记忆整合传统上被视为一种改变智能体的操作：模型微调、提示重写、策略蒸馏、或对指导未来行为的上下文添加反思。在受监管的自主部署中，这是一种负担，因为智能体在约束和审计合同下运行，这些合同绑定到特定的、经过加密认证的身份。我们提出将整合视为不是规划器或身份清单的突变，而是作为情景记忆上的确定性函数f: M^ep -> M^sem，其输出是一个单独可寻址的语义知识层；身份哈希不读取M^sem，因此整合在不改变智能体认证身份的情况下更新知识。我们给出了智能体表示的形式化描述，通过一个关于清单哈希输入集的结构引理证明了身份不变性，指定了一个确定性聚合算法，其输出是可审计的数据库行，具有明确的置信度和支持事件来源，并通过合成实验验证了该结构，展示了字段级别的正确性、跨整合轮次的字节一致身份，以及相比校准的贝叶斯收缩基线，平均减少79.82%的低效规划器尝试（10个种子，95% BCa CI [78.02%, 81.49%]）。该结构是一种用于自主智能体的知识更新纪律，其中经验积累为可查询的事实，而智能体的认证身份在其运行生命周期内保持字节一致，以具体服务智能体作为运行案例研究。

## Abstract
Long-running adaptive intelligent agents face a structural tension between knowledge consolidation and information integrity. Memory consolidation is conventionally treated as an agent-changing operation: a model is fine-tuned, a prompt rewritten, a policy distilled, or a reflection appended to the context that governs future behaviour. In regulated autonomic deployment this is a liability because the agent operates under commitments and audit contracts that bind to a specific, cryptographically certified identity. We propose to treat consolidation not as a mutation of the planner or the identity manifest, but as a deterministic function f: M^ep -> M^sem over episodic memory whose output is a separately addressable semantic knowledge layer; the identity hash does not read M^sem, so consolidation updates knowledge without changing the agent's certified identity. We give a formal account of the agent representation, prove identity invariance through a structural lemma on the manifest's hash-input set, specify a deterministic aggregation algorithm whose outputs are auditable database rows with explicit confidence and supporting-event provenance, and validate the construction with synthetic experiments demonstrating per-field correctness, byte-equal identity across consolidation passes, and a mean 79.82% reduction in unproductive planner attempts (95% BCa CI [78.02%, 81.49%] across 10 seeds) against a calibrated Bayesian-shrunk baseline. The construction is a knowledge-update discipline for autonomic agents in which lessons accumulate as queryable facts while the agent's certified identity remains byte-equal across its operational lifetime, with an embodied service agent as the running case study.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心矛盾**：长期运行的自主智能体需要在受监管环境中不断进行知识整合（从情景经验中学习），但其加密认证身份必须保持字节不变（用于合规审计等）。传统记忆整合方法（如模型微调、提示重写、策略蒸馏、反思追加）会改变智能体本身，导致每个整合步骤都成为重新认证事件，无法同时满足知识更新与身份不变性的需求。
- **研究目标**：解决稳定性-可塑性难题在自主治理下的特殊约束——智能体身份由加密摘要定义，而非其权重或提示。论文提出一种“身份稳定的整合”框架，使智能体在不改变认证身份的情况下积累可查询的事实知识。
- **应用背景**：自主体机器人在医院、制造、仓储等受监管环境中的部署，需要同时满足自适应学习与信息完整性（审计、合同）的要求。

## 2. 方法论：核心思想、关键技术细节、算法流程
- **核心思想**：将记忆整合视为**确定性函数 f : M^ep → M^sem**，即从智能体的情景事件日志（仅追加记录）到独立可寻址的语义知识层（可查询的事实）的映射。身份清单的哈希输入集不包含 M^sem，因此更新语义层不改变认证身份。
- **关键技术细节**：
  - **智能体状态形式化**：表示为元组 (M, M^ep, M^sem, π)，其中 M 为身份清单（固定字段数据类），M^ep 为附加型情景日志（按 SHA256(M) 分区），M^sem 为派生语义记忆表，π 为规划器。身份哈希 h = SHA256(M)。
  - **结构身份不变性引理（Lemma 1）**：任何只写入语义存储的操作 f ∈ F_sem 都不会改变 M 的字节内容，因此 h 保持不变。该证明基于类型签名和清单字段集合的编译时检查，非运行时断言。推论（Corollary 1）扩展到任意有限序列整合。
  - **确定性 v1 整合算法**：
    1. 读取上次检查点后的情景行
    2. 提取结构化字段（技能 ID、目标类、环境、参数、成功/失败标志）
    3. 按稳定键分组（如 skill_id+target+environment）
    4. 聚合（计数、成功率、参数分布 IQR、最常见失败原因）
    5. 更新（upsert）语义表，记录置信度、观测数、最后支持事件 ID、规则版本
    6. 审计追加：记录 consolidation_run 事件（规则版本、处理行范围、影响行数）
  - 算法**确定、幂等、顺序无关**：相同输入+相同规则版本 → 字节相等输出。
  - **置信度计算**：结合观测数与组内方差，使少量观测的事实影响力小于大量观测的事实。
  - **规划器只读接口**：通过 HTTP GET 端点公开语义事实（按置信度排序）；规划器可选使用，但事实仅作为信息表面，不具权威性；消费仍通过运行时门控（准入、策略、合同、审计）。

## 3. 实验设计
- **使用场景/数据集**：利用合成数据（非真实生产数据，因为真实长周期数据尚未累积）进行确定性正确性和规划器接地效用验证。
  - V1（确定性正确性）：1000 条 manipulation.grasp 执行结果行（800 成功，200 失败）；验证输出语义事实匹配预期。
  - V2（身份不变性）：PersistentAgent 等效 IdentityManifest 快照，整合后重新计算哈希是否不变；以及通过修改清单字段验证哈希变化。
  - V3（规划器接地效用）：200 条 glass_cup 抓取行（80% 隐藏成功率）和 50 条 unknown_object 抓取行（20% 成功率），在合成场景中运行 1000 次决策。
- **基准/对比方法**：
  - no_memory：无记忆控制，尝试每次抓取。
  - uniform：固定置信度为 1.0，仅测试“事实存在”结构价值。
  - raw：直接使用整合器的成功率作为置信度。
  - calibrated（主要比较对象）：对每个技能的速率应用贝叶斯收缩（先验均值 0.5，先验权重 10），在 held-out 20% 历史切片上校准。
- **对比指标**：低效规划器尝试次数（即尝试抓取但实际无成功可能的行为）的减少百分比。

## 4. 资源与算力
- 论文**未提及 GPU**；所有实验在单 CPU 核心上运行。
- 每个种子每次控制约需 4 秒（单个核心）；完整 10 种子 × 3 控制（无 memory、uniform、calibrated）约需 20 秒墙钟时间。

## 6. 主要结论与发现
- **身份不变性成立**：结构化引理得以证明且通过测试；跨整合轮次 SHA-256 哈希字节相等。
- **规划器效用显著**：相比无记忆基线，校准控制平均减少 79.82% 低效规划器尝试（95% BCa CI [78.02%, 81.49%]）。原始单种子值 74.58% 是该分布中的最小值，并非人为有利。
- **确定性整合开销极低**：在 10^5 条情景行下整合时间 < 310ms，内存增量 ~15 MB，适合每次写入时触发。
- **Uniform 控制置信区间跨越零** ([−4.7%, +0.4%])，证明仅靠“事实存在”结构无改善，置信度信号是关键。
- 算法设计选择（阈值 0.5、先验均值 0.5 权重 10）位于参数空间的稳定内部，附近无急剧退化。

## 7. 优点：方法或实验设计上的亮点
- **创新性**：首次将记忆整合与加密身份绑定问题分离，通过结构（编译时）而非运行时断言保证身份不变性，为受监管自主代理提供新的知识更新范式。
- **形式化严谨**：给出清晰的状态元组定义、身份不变性引理及输入集检查证明，使论点可靠。
- **可审计性**：每条语义事实显式携带置信度、观测数、支持事件 ID 和规则版本，审计链完整（从提议动作→事实→证据→规则）。
- **确定性 + 幂等 + 顺序无关**：确保输出可复现、可认证，避免非确定性导致的合规难题。
- **低开销、可部署**：实现为单一模块（约 1040 行 Python），运行时开销可忽略，可在每次情景写入时触发。
- **敏感性分析全面**：系统扫描了阈值、先验参数、决策数，证明了报告结果在合理默认参数周围稳健。
- **开放代码与复现性**：提供参考实现（MIT 许可）并附带验证脚本与 SHA256SUMS。

## 8. 不足与局限
- **仅支持结构化聚合**：v1 只能处理已有结构化字段的聚合，无法从自由文本事件摘要中提取叙事性经验（LLM 辅助 v2 仅作为未来方向讨论）。
- **无遗忘或衰减机制**：置信度和观测数不随时间衰减，旧证据与新证据权重相同，可能导致过时事实长期影响决策。
- **假定可信的情景存储**：无对抗防御；攻击者可通过写入伪造情景行操纵语义事实。
- **情景载荷格式假设**：依赖载荷带有足够结构化字段；仅输出自由文本的能力不可见。
- **未使用真实部署数据**：所有验证基于合成数据；真实长期运行中的语义积累和规划器行为漂移尚未研究。
- **跨身份迁移未支持**：语义表严格按身份哈希分区，无法直接共享或合并跨机器人经验（但有意识留作未来工作）。
- **自适应倾向超出范围**：语义记忆不得成为暗中改变风险偏好、伦理下限或人工检查阈值的途径；这些属于独立的治理调节领域。
- **LLM 辅助扩展存在审计挑战**：非确定性、模型依赖的语义事实难以像 SQL 聚合一样审计，需额外纪律（来源签名、人类审查门、稳定比较）后方可部署。
