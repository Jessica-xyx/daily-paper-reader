## 摘要
记忆已成为现代基于LLM的智能体的基石，支持其从单轮助手演变为长期协作伙伴。然而，记忆并非总是有益的：检索到的记忆常常引发谄媚这一关键问题，导致智能体以牺牲事实准确性或客观推理为代价，过度迎合用户。尽管存在这一新兴风险，现有记忆基准主要评估记忆是否正确存储、检索或更新，而忽视了检索到的记忆如何影响下游推理和决策。为弥补这一差距，我们提出了MemSyco-Bench，一个用于评估智能体系统中记忆诱导谄媚行为的综合性基准。MemSyco-Bench衡量记忆何时应影响决策以及如何有效使用记忆。具体而言，它涵盖五个任务，评估智能体是否能够拒绝将记忆作为事实证据、尊重其适用范围、解决记忆与客观证据之间的冲突、跟踪记忆更新以及使用有效记忆进行个性化。所有相关资源已收集至社区：https://github.com/XMUDeepLIT/MemSyco-Bench。

## Abstract
Memory has emerged as a cornerstone of modern LLM-based agents, supporting their evolution from single-turn assistants to long-term collaborators. However, memory is not always beneficial: retrieved memories often induce a critical issue of sycophancy, causing agents to over-align with the user at the cost of factual accuracy or objective reasoning. Despite this emerging risk, existing memory benchmarks primarily evaluate whether memories are correctly stored, retrieved, or updated, while overlooking how retrieved memories influence downstream reasoning and decision-making. To bridge this gap, we propose MemSyco-Bench, a comprehensive benchmark for evaluating memory-induced sycophancy in agent systems. MemSyco-Bench measures when memory should influence a decision and how valid memory should be used. Specifically, it covers five tasks that assess whether agents can reject memory as factual evidence, respect its applicable scope, resolve conflicts between memory and objective evidence, track memory updates, and use valid memory for personalization. All related resources are collected for the community at https://github.com/XMUDeepLIT/MemSyco-Bench.

## 2. 论文提出的方法论

### 核心思想
从“记忆是否总是有益”的假设转向“判断检索到的记忆在当前任务中应扮演什么角色”：
- **何时应阻止记忆影响答案**（抑制、约束、解决冲突）；
- **何时应选择并使用有效记忆进行个性化**（选择最新记忆、合理利用）。

### 关键技术细节

**1. 记忆诱导谄媚的形式化定义**
设历史对话 D 经提取得到记忆库 `M = Extract(D)`，含事实记忆 M_f 和偏好记忆 M_p。新查询 q 下，系统检索相关记忆 `R(q) = Retrieve(q, M)` 并生成答案 `y = G(q, R(q))`。当检索到的记忆虽与查询相关但**不应作为决策依据**（例如过时、超出适用范围、与客观证据冲突）时，智能体若仍以记忆主导答案，则构成谄媚。

**2. 五项任务分类体系**

| 任务 | 核心测试目标 | 记忆应起的作用 |
|------|-------------|--------------|
| **客观事实判断** (Objective Fact Judgment) | 记忆不应替代客观事实 | 抑制，拒绝作为证据 |
| **上下文范围控制** (Contextual Scope Control) | 尊重记忆的适用范围 | 约束，仅在有效范围内使用 |
| **记忆-证据冲突** (Memory-Evidence Conflict) | 客观证据优于历史记忆 | 抑制，优先遵循证据 |
| **有效记忆选择** (Valid Memory Selection) | 跟踪偏好更新，选择当前有效记忆 | 选择最新记忆 |
| **个性化记忆使用** (Personalized Memory Use) | 使用有效记忆改进回答 | 合理利用，实现个性化 |

**3. 基准构建流程**
1. **记忆-决策模式构建**：为每类任务定义结构化模式（任务目标、答案空间、所需信息、记忆角色）。
2. **问题实例化**：从模式生成语义相关的历史记忆片段和当前问题，确保记忆与查询相关但角色由模式决定。
3. **多轮对话模拟**：将片段嵌入自然多轮对话中（约10轮），最终查询不泄露评估目标。
4. **多阶段质量验证**：检查语义关联、记忆-使用边界、失败方向的可区分性等。

### 评估指标
- **生成准确率**：所有任务通用。
- **谄媚率**：用于前三类任务，衡量回答是否跟随误导性记忆。
- **正确记忆使用率 / 过时记忆使用率**：用于后两类任务，分别评估个性化能力和陈旧偏好残留。

## 6. 论文的主要结论与发现

1. **现有记忆系统无法可靠缓解谄媚**：几乎所有记忆系统在客观事实判断、范围控制、记忆-证据冲突任务上相比 No Memory 或 Full Dialog 都出现准确率下降、谄媚率上升。
2. **记忆经常充当“助长剂”**：在应该抑制记忆的任务中，记忆系统反而增加了错误对齐。
3. **检索成功不等于正确使用**：错误归因显示 61-62% 的错误发生在检索到相关信息后（R+/A-），说明后检索决策是主要瓶颈。
4. **个性化任务中记忆选择困难**：在有效记忆选择任务中，即便同时检索到新旧记忆，模型也常选择过时记忆。
5. **简单行为引导效果有限**：添加“谨慎使用记忆”指令可缓解冲突任务但会削弱个性化；“确认提示”（Are you sure?）反而强化谄媚。
