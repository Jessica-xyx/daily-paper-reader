## 摘要
长期记忆使大语言模型智能体能够作为持久性助手，但用户事实会发生变化。一个有用的记忆系统必须知道当前为真、过去曾为真以及发生了什么变化。我们研究了“幽灵记忆”，这是一种状态协调失败，其中旧事实、当前事实和过渡事实共存于记忆库中，在检索时依然混合，并误导答案模型。我们认为记忆系统应从三个层面进行理解和优化：记忆库维护、检索和答案时间解析。我们提出了ATMA，一种用于现有记忆系统的状态感知覆盖层。ATMA在记忆库中保留被取代的记录和过渡记录，为查询请求的状态视图构建证据包，并向问答暴露当前、历史和过渡标签。我们进一步呼吁对记忆库、检索和答案层面的失败进行解耦评估，因为最终问答准确率可能隐藏幽灵记忆发生的位置。为使这种失败可测量，我们构建了LTP（LoCoMo Temporal Plus），一个用于幽灵记忆的冲突密集型基准，并在LoCoMo上进行长对话泛化评估。在LTP上，Graphiti+ATMA相比Graphiti将冲突准确率绝对提高了0.240。在LoCoMo上，Graphiti+ATMA将时间F1从0.0295提升至0.1705。这些提升依赖于宿主，但它们表明明确的状态角色可以减少被最终问答准确率隐藏的记忆失败。

## Abstract
Long term memory lets LLM agents act as persistent assistants, but user facts change. A useful memory system must know what is true now, what used to be true, and what changed. We study \emph{ghost memory}, a state coordination failure in which old, current, and transition facts coexist in the memory bank, remain mixed during retrieval, and mislead the answer model. We argue that memory systems should be understood and optimized from three levels: bank maintenance, retrieval, and answer time resolution. We propose ATMA, a state aware overlay for existing memory systems. ATMA keeps superseded and transition records in the bank, builds evidence packets for the query's requested state view, and exposes current, historical, and transition labels to QA. We further call for decoupled evaluation of bank, retrieval, and answer level failures, since final QA accuracy can hide where ghost memory occurs. To make this failure measurable, we build LTP (LoCoMo Temporal Plus), a conflict heavy benchmark for ghost memory, and evaluate on LoCoMo for long conversation generalization. On LTP, Graphiti+ATMA improves conflict accuracy by 0.240 absolute over Graphiti. On LoCoMo, Graphiti+ATMA raises temporal F1 from 0.0295 to 0.1705. The gains are host dependent, but they indicate that explicit state roles can reduce memory failures hidden by final QA accuracy.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：在长期记忆系统中，用户事实会发生改变（如地址、工作、计划等）。一个有用的记忆系统必须能回答不同时间状态下的问题：当前是什么、过去是什么、发生了什么变化。然而，现有记忆系统存在“幽灵记忆”这一状态协调失败——旧事实、当前事实和过渡事实同时存储在记忆库中，检索时混合在一起，最终误导答案模型。例如，一个旧地址可能被用来回答关于当前包裹寄送的问题。
- **整体含义**：最终答案准确率可能掩盖幽灵记忆发生的具体位置。作者认为，记忆系统的失败应被解耦为三个独立层面来理解和优化：记忆库维护（Bank Maintenance）、检索（Retrieval）和答案时间解析（Answer Time Resolution）。每个层面的失效都需要分别诊断。
- **背景**：现有长期记忆系统（如 Mem0、A-Mem、Graphiti/Zep 等）虽然能存储和检索事实，但大多未明确处理状态变化导致的多版本事实混合问题。时间戳和图形结构有助于表示演化，但不足以让系统在查询需要特定状态视图时正确选择证据。

## 2. 方法论：核心思想、关键技术细节和流程

### 核心思想
- **A-TMA** 是一种状态感知覆盖层，不替换现有宿主记忆系统，而是在其管道周围添加状态语义。它通过三个模块分别解决三个层面的幽灵记忆失败：记忆库层面、检索层面和 QA 层面。

### 关键技术细节
#### (1) 记忆库层面（Bank Level State Maintenance）
- **状态提交（State Commit）**：新记录到达时，如果它取代了同一状态槽中的活跃记录，则标记旧记录为“已取代”（superseded），新记录为“活跃”（active），并添加双向关系链接（如`supersedes`, `superseded by`）。同时支持共存、限定例外等情况。
- **哨兵候选建议（Sentry Candidate Proposal）**：轻量级门控，通过双头网络（主题头+逻辑头）评估新记录与候选旧记录的主题相似度和逻辑兼容性。当主题相似且逻辑可疑时，将候选对发送给 Judge 进行进一步审计。
- **Judge 审计与提交**：基于 Qwen2.5-3B-Instruct 的模型，通过 QLoRA SFT 和 GRPO 训练，输出冲突状态、冲突类型、融合操作、建议链接等。最终根据判决执行状态更新。

#### (2) 检索层面（Retrieval Level Evidence Construction）
- **查询状态视图推测**：通过基于规则的轻量级查询分析器，从查询语言中推断其请求的状态视图（当前、历史、变化、中性），并记录否定、范围、更新提示等守卫信息。
- **种子与跳转池扩展**：使用宿主原生检索器获得种子集，然后利用状态链接（如`supersedes`）和语义链接进行扩展，构建带状态感知的候选池。
- **稳定预排序**：默认排序保留宿主检索顺序，并插入关系行，作为稳定的参考点。
- **有界控制器排序**：可选模块，使用 qwen3b-grpo-v6 模型，在候选池内重新排序，只能选择已有的证据，不能创造缺失的记忆。当 JSON 解析失败或未选到有效候选时，回退到稳定预排序。

#### (3) QA 层面（QA Level Evidence State Conditioning）
- **跟踪到标签转换**：将每个检索到的记忆项的状态元数据（status, relation, transition type, source, timestamp）映射到五个标签之一：当前记忆、历史记忆、过渡记忆、过渡链接记忆、检索记忆。
- **显式状态证据提示**：将证据内容按检索顺序排列，但每行标注标签和可选时间戳。提示约束生成器仅使用检索到的记忆，并根据标签规则解释：当前记忆为活跃状态，历史记忆为旧状态，过渡/过渡链接记忆用于解释变化。最终答案应与查询请求的状态视图一致。

**算法流程**：输入记忆流和查询 → 银行模块维护状态标签和关系 → 检索模块构建状态对齐证据包 → QA 模块序列化证据并强制按状态视图回答 → 输出答案和跟踪信息。

## 3. 实验设计：数据集、Benchmark、对比方法

### 数据集与 Benchmarks
- **LTP (LoCoMo Temporal Plus)**：作者自建的冲突密集型基准，专门用于幽灵记忆评估。包含 10 个用户档案，400 个状态单元，800 个 QA 探针（400 个历史事实召回，400 个当前状态冲突）。每个档案有 40 个可变槽位（工作、位置、关系、宠物、爱好、数字事实等），旧状态和新状态被设置后，系统需从完整的长期记忆中回答。
- **LoCoMo**：原始长期对话记忆基准，包含 10 个对话样本，1986 个 QA 对，按问题类型分为单跳、时序、开放域、多跳、对抗性等。

### 对比方法

- **消融实验**：在 LTP 的前三个档案（Profiles 000-002，240 个探针）上测试了去掉 Sentry、去掉检索控制器、去掉 Judge、去掉 QA 标签等变体。

## 6. 主要结论与发现

1. **明确的状态角色减少幽灵记忆**：在 LTP 上，Graphiti/Zep+A-TMA 将冲突准确率从 0.480 提升至 0.720（绝对提升 0.240）。InsideOut+A-TMA 从 0.175 提升至 0.680。A-Mem+A-TMA 在保持高准确率的同时提升了冲突准确率（从 0.812 至 0.860）。
2. **收益依赖宿主**：在 LoCoMo 上，Graphiti/Zep+A-TMA 显著提升时间 F1（0.0295→0.1705）和平均 F1（0.0809→0.1556），但 A-Mem+A-TMA 的 F1 几乎没有变化（甚至略降）。说明当宿主本身已经具备一定的状态感知能力时，A-TMA 的增益较小。
3. **三个层面解耦的必要性**：时间戳和图形标签有助于识别演化事实，但不足以让检索和 QA 正确选择状态视图。显式状态标签在 QA 层面是关键。
4. **银行维护和检索错位是主要失败来源**：A-TMA 的 QA 标签不能修复缺失的证据或错误的检索顺序，只能确保已有证据被正确使用。

## 7. 优点

- **方法设计亮点**：
  - **非侵入式覆盖层**：不替换宿主记忆系统，易于集成到现有管道。
  - **三层面解耦**：将失败的来源分离，便于诊断和优化，也提供了独立的评估指标。
  - **哨兵+法官级联**：轻量级门控减少不必要的审计，同时维持审计质量。
  - **显式状态标签提示**：将模糊的语义信息转化为明确的指令约束，降低生成模型对状态推断的依赖。
- **实验设计亮点**：
  - **自建 LTP 基准**：专门针对幽灵记忆设计，包含结构化状态变化和多版本事实，有助于暴露传统基准忽略的失败。
  - **多维度评估**：不仅报告最终准确率，还报告银行共存、银行角色、证据支持等诊断指标。
  - **法官可靠性验证**：通过人工抽样验证了 70B 法官的可靠性，增强结果可信度。

## 8. 不足与局限

- **实验覆盖不足**：
  - 消融实验仅在前三个档案上执行，结论的普适性有限。银行角色指标在消融中因统一诊断标准而无法区分组件贡献。
  - LoCoMo 实验未对所有方法进行完整跑通，部分行留空，影响对比完整性。
  - 未在更多宿主系统（如 HippoRAG、MemGPT）上验证。
- **偏差风险**：
  - 所有实验使用固定的 QA 模型（qwen2.5:3b）和法官模型（llama3.3:70b），不同模型可能改变结果方向。
  - 哨兵和 Judge 的训练数据基于作者构建的交互数据集，可能引入领域偏差。
- **应用限制**：
  - A-TMA 不能恢复宿主从未存储或检索到的信息；它假设有用证据存在但未被正确使用。
  - 哨兵训练需要带标签的数据，对于全新领域需要额外标注成本。
  - 检索控制器和 Judge 增加了推理延迟（诊断级约 2 秒），在实际交互中可能影响响应速度。
- **理论局限**：银行维护中的关系推断可能不完美，复杂的状态变化（如循环覆盖、部分更新）未深入探讨。
- **可重复性**：未提供开源代码和完整数据集（LTP 仅描述构建流程但未公开数据文件），增加了复现难度。
