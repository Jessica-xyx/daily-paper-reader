## 摘要
连接人类飞行员意图与自主飞行操作之间的差距对于现实世界的电动垂直起降（eVTOL）飞机部署至关重要。飞行规划传统上依赖于难以融入灵活人类偏好的经典算法。我们提出FRAMe，一种端到端的大型语言模型（LLM）飞行规划工具，具有基于RAG的记忆和多模态教练智能体。我们的系统将规划LLM与多模态教练智能体以及基于检索增强生成（RAG）的记忆相结合，生成满足任务约束同时符合人类飞行操作员偏好的飞行计划。我们在各种难度级别的真实世界场景中演示了该系统。在四个LLM中，完整的FRAMe系统（RAG和教练）为每个规划器产生了最高的有效性（最佳规划器的综合有效性高达93.8%，在简单场景中为99%），并将偏好相关指标向操作员偏好的方向移动，其中指标有提升空间。FRAMe标志着先进LLM如何被用于以人为中心的使命规划，将自然语言指令转化为安全、高效和灵活的飞行路线。代码可在github.com/amin-tabrizian/FlightPlanningLLMs获取。

## Abstract
Bridging the gap between human pilot intent and autonomous flight operation is critical for real-world electric vertical takeoff and landing (eVTOL) aircraft deployment. Flight planning traditionally relies on classic algorithms that struggle to incorporate flexible human preferences. We present FRAMe, an End-to-End Large Language Model (LLM) Flight Planning tool with RAG-based Memory and Multi-modal Coach Agent. Our system integrates a planner LLM with a multi-modal coach agent and retrieval augmented generation (RAG)-based memory to generate flight plans that satisfy mission constraints while aligning with human flight operator preferences. We demonstrate the system in a range of real-world-inspired scenarios of varying difficulty levels. Across four LLMs, the full FRAMe system (RAG and coach) yields the highest validity for every planner (up to 93.8% aggregate, 99% on Easy scenarios for the strongest planner) and shifts preference-relevant metrics in the operator-favored direction where the metric has headroom. FRAMe signifies how advanced LLMs can be deployed for human-centric mission planning, translating natural language instructions into safe, efficient, and flexible flight routes. The code is available at: github.com/amin-tabrizian/FlightPlanningLLMs

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：传统飞行规划算法（如 A*、RRT）依赖数学化定义的约束和代价函数，难以灵活融入人类驾驶员/操作员的**自然语言偏好**（例如“尽量远离危险区域”“尽可能减少航点”）。特别是在电动垂直起降（eVTOL）航空器大规模部署的背景下，需要一种能够理解模糊、上下文依赖的飞行意图的自动化规划系统。
- **整体含义**：研究提出 **FRAMe 系统**，将大型语言模型（LLM）与检索增强生成（RAG）记忆、多模态教练智能体相结合，实现了从自然语言指令到安全、高效且符合人类偏好的飞行路线的端到端转化。这项工作标志着 LLM 在以人为中心的自主飞行规划中的实际应用。

## 2. 论文提出的方法论

- **核心思想**：FRAMe 包含三个主要组件：
  1. **规划 LLM**：接收场景描述（起点、终点、可飞空域、禁飞多边形）和操作员的自然语言偏好，生成一系列航点。
  2. **RAG 记忆模块**：将过去成功的飞行计划及其评估结果存入向量数据库；对新请求时，通过精确场景匹配保证几何一致性，再通过偏好文本的余弦相似度检索最相关的 K 条历史记录，作为上下文加入提示。
  3. **多模态教练智能体**：对生成的计划进行三阶段审查：① 几何验证（航点是否在可飞区、是否经过禁飞区、起终点是否正确）；② 利用 o4-mini 的视觉能力分析渲染的飞行路径图像，评估与偏好的对齐程度；③ 操作员可提供可选的反馈。
- **算法流程**（文字描述）：
  1. 解析 KML 文件获取场景几何数据，构建系统消息和用户消息。
  2. 若启用 RAG，对偏好文本做嵌入，从同一几何场景的记忆中检索前 K 个最相似的历史记录，拼接至提示。
  3. 调用规划 LLM 生成航点序列和推理说明。
  4. 若启用教练，先通过几何工具检查有效性；若有效，再让多模态 LLM 基于路径图像评价对齐度；记录操作员反馈。
  5. 将完整记录（场景、嵌入、有效性、对齐度、反馈、航点、推理）存入数据库，供后续检索。
- **关键公式**：余弦距离用于评估偏好向量相似度，\( \text{CosineDistance}(e_p, e_h) = 1 - \frac{e_p \cdot e_h}{\|e_p\|\|e_h\|} \)，其中 \(e_p\) 是当前偏好嵌入，\(e_h\) 是历史偏好嵌入。

## 3. 实验设计

- **场景与数据集**：使用 Dallas–Fort Worth 地区的合成多边形禁飞区，分为三个难度级别（Easy: 2个禁飞区；Medium: 4个；Hard: 7个）。每个难度组合固定起点和终点，产生多种配置。
- **基准（Benchmark）**：经典 A* 算法作为几何最优基线（100% 有效性但偏好盲目）。
- **对比方法**：
  - **Baseline**：仅 LLM 规划器，无 RAG 或教练。
  - **+RAG**：规划器 + RAG 记忆（K=2），无教练。
  - **+RAG+Coach**：完整 FRAMe 系统。
  - **A***：经典 A* 规划器。
- **评估的 LLM**：OpenAI o3-mini、o4-mini、GPT-5.4 和 DeepSeek-R1。
- **偏好目标**：三种可计算的定量偏好：最小化总飞行距离、最小化航点数量、最大化与禁飞区的最小距离。对齐度通过比较偏好激活时指标是否向有利方向移动来评估。

## 6. 论文的主要结论与发现

- 完整 FRAMe 系统（+RAG+Coach）在所有 LLM 上取得了最高的规划有效性（o3-mini 88.9%，o4-mini 90.7%，DeepSeek-R1 93.8%，GPT-5.4 55.6%），其中 DeepSeek-R1 在简单场景中达到 99% 有效性。
- **RAG 单独使用不一定提升有效性**（如 o3-mini 的 Baseline 79.6% → +RAG 79.1%），教练的审查起到关键恢复和提升作用。
- **偏好捕获效果依赖于模型的“提升空间”**：对于距离偏好，各模型基线已接近最优，改进有限；航点偏好在 o3-mini 和 GPT-5.4 上效果明显；清除偏好在 o4-mini 和 DeepSeek-R1 上提升显著。
- 与 A* 相比，FRAMe（甚至仅 Baseline）在清除距离和航点数量上显著优于 A*，代价是飞行距离略微增加（6 km 内）；A* 完全偏好盲目。
- 不同模型对场景复杂度敏感性不同：o4-mini 在 Hard 场景表现最好，DeepSeek-R1 在 Easy/Medium 优势明显但 Hard 下降，o3-mini 随难度单调下降。

## 7. 优点

- **端到端自然语言规划**：直接接受操作员自然语言偏好，无需繁琐的数学建模。
- **模块化架构**：RAG 和教练可独立消融，便于评估每个组件的贡献。
- **无标注偏好评估**：通过几何可计算的三个定量指标自动评估对齐度，避免了人工标注的主观性和高成本。
- **多模态教练**：结合规则几何验证和视觉 LLM 分析，从多个维度检查计划的有效性和偏好匹配。
- **RAG 记忆设计**：采用精确几何场景匹配 + 偏好余弦相似度排序，保证了检索内容的几何相关性和意图相关性。
- **实验结果全面**：跨四个先进 LLM、三种难度、三种偏好及消融条件，结果具有参考价值。

## 8. 不足与局限

- **仅模拟环境**：试验使用合成多边形禁飞区，未考虑真实天气、动态交通流、无人机/有人机冲突、电池能量约束、飞行器动力学等安全关键因素。
- **头号空间有限**：某些偏好（如距离）因基线已近乎最优，难以观测到有意义的改进，导致偏好捕获的证据不均衡。
- **LLM 裁判偏差**：教练中的对齐评估依赖 o4-mini 的视觉判断，存在位置、冗长、自我偏好等偏见风险；缺乏人类裁判验证。
- **GPT-5.4 性能极差**：该模型在所有条件下有效性大幅落后于其他三个原因模型（仅 37-56%），可能与其指令跟随能力较差有关，但论文未深入探讨原因。
- **静态禁飞区**：未考虑随时间变化的（动态）限制或临时空中交通管制，与真实空管环境有差距。
- **无真实飞行验证**：仅通过网页原型和模拟测试，缺乏实际 eVTOL 或无人机飞行数据。
- **可能不平衡的检索**：RAG 仅从同一几何场景中检索，若历史库中缺乏足够多样或高质量的计划，可能影响效果。
