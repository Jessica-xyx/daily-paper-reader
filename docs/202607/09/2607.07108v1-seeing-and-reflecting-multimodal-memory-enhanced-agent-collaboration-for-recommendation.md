## 摘要
基于大语言模型的智能体推荐系统通过自然语言推理在用户偏好建模方面展现出潜力，但仍受限于以文本为中心的输入和粗粒度的记忆更新，导致智能体容易遗漏视觉证据、受到语义噪声干扰以及无法捕捉偏好漂移。为解决这些局限，我们提出MMEACR，一个面向推荐的多模态记忆增强型智能体协作框架。MMEACR引入双轨记忆架构，将可解释的智能体推理与细粒度多模态匹配相分离。在推理轨道中，协作的用户记忆智能体和物品记忆智能体维护持久的多模态记忆，并通过属性引导的强化与反思机制进行更新。在匹配轨道中，从原始交互叙述和物品图像构建解耦的多模态嵌入记忆，以保留超出结构化记忆更新的详细跨模态信号。两轨道通过加权倒数排名融合进行整合，产生鲁棒且可解释的排序结果。在三个真实世界领域的实验表明，MMEACR相较于基于LLM和基于智能体的强基线方法取得了整体优异性能，在视觉导向的推荐场景中尤为显著。

## Abstract
Large language model (LLM)-based agentic recommender systems show promise in modeling user preferences through natural-language reasoning, yet they remain limited by text-centric inputs and coarse-grained memory updates, making agents prone to missing visual evidence, semantic noise, and preference drift. To address these limitations, we propose MMEACR, a Multimodal Memory-Enhanced Agent Collaboration framework for recommendation. MMEACR introduces a dual-track memory architecture that separates interpretable agent reasoning from fine-grained multimodal matching. In the reasoning track, collaborative User and Item Memory Agents maintain persistent multimodal memories and update them through an attribute-guided reinforcement-and-reflection mechanism. In the matching track, a decoupled multi-modal embedding memory is built from raw interaction narratives and item images to preserve detailed cross-modal signals beyond structured memory updates. The two tracks are integrated through weighted Reciprocal Rank Fusion to produce robust and interpretable rankings. Experiments on three real-world domains show that MMEACR achieves strong overall performance against competitive LLM-based and agent-based baselines, with notable gains in visually grounded recommendation scenarios.

- **本文目标**：提出一种多模态记忆增强的智能体协作框架，通过“看见（Seeing）”和“反思（Reflecting）”两个核心机制，使智能体能够利用多模态证据，并结构化地进化记忆，从而实现更鲁棒、可解释的推荐。

## 2. 方法论：核心思想、关键技术细节

- **整体框架（MMEACR）**：采用双轨（dual-track）架构，两个轨道互补融合：
  - **推理轨（Reasoning Track）**：包含用户记忆智能体（User Memory Agent）和物品记忆智能体（Item Memory Agent），维护可解释的多模态语言记忆，并通过属性引导的强化与反思机制进行更新。
  - **匹配轨（Matching Track）**：构建解耦的多模态嵌入记忆（Multimodal Embedding Memory），利用预训练的多模态嵌入模型（如GME）编码原始交互叙述和物品图像，保留细粒度的跨模态信号。
  - **融合**：通过加权倒数排名融合（Weighted Reciprocal Rank Fusion, RRF）将两轨产生的排名整合为最终推荐列表。

- **关键步骤与算法流程**：
  1. **记忆初始化**：
     - 物品记忆：使用多模态LLM（如GPT-4o）生成图像描述，结合标题构建多模态文本记忆。
     - 用户记忆：用领域特定模板初始化，如对音乐、电子或时尚的通用偏好陈述。
     - 同时维护原始叙述（raw narratives）用于嵌入匹配。
  2. **属性引导的记忆演化**：
     - 每步构造三元组（用户，正物品，负物品），智能体选择匹配项并生成推理。
     - 基于预定义的语义属性空间（如风格、功能、外观），提取属性级偏好信号。
     - 若选择正确→强化（reinforce）记忆；若错误→反思（reflect）纠正。
     - 用户和物品记忆按规则更新，并异步更新原始叙述。
  3. **推理轨排名**：使用演化后的用户和物品记忆，通过LLM排序生成描述式排名（π_des）。
  4. **匹配轨排名**：用户和物品嵌入分别基于原始叙述和图像，通过余弦相似度排序（π_emb）。
  5. **融合**：对每个候选物品计算RRF分数：S(i) = w_des/(k_des+rank_des(i)) + w_emb/(k_emb+rank_emb(i))。

- **关键创新点**：
  - 双轨分离：避免将多模态信息直接注入语言记忆导致冗余和不稳定。
  - 属性引导的强化与反思：通过属性空间约束，减少语义噪声和偏好漂移。
  - 嵌入记忆保留原始细节：弥补结构化记忆更新可能丢失的细粒度信息。
