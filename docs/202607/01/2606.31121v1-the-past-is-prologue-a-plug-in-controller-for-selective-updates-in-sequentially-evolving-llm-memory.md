## 摘要
顺序演进的LLM记忆使智能体能够复用过往经验，但现有系统通常直接部署每个局部生成的记忆更新，而不检查该更新是否改进了未来行为。结果，有助于当前任务的更新可能覆盖有用知识、引入过度特化的规则，或使最终记忆偏向近期样本。我们提出Janus，一个即插即用的记忆控制器，用于决定是接受候选记忆更新还是保留先前的记忆。为了高效做出决策，Janus使用记忆动量触发器识别记忆更新轨迹中的可疑偏差，并在覆盖、边界和新鲜任务的紧凑混合评估集上比较新旧记忆，而非重放全部历史。Janus是方法无关的，可包裹现有更新器而不改变其更新规则。在六个数据集、两个基座LLM和两个记忆更新器上，Janus相比对应基更新器平均准确率提升+2.7至+4.6个百分点。

## Abstract
Sequentially evolving LLM memory enables agents to reuse past experience, but existing systems usually deploy each locally generated memory update without checking whether it improves future behavior. As a result, updates that help the current task may overwrite useful knowledge, introduce over-specific rules, or bias the final memory toward recent examples. We propose Janus, a plug-in memory controller that decides whether to accept a candidate memory update or retain the previous memory. To make this decision efficient, Janus uses a Memory Momentum Trigger to identify suspicious deviations in the memory-update trajectory, and compares old and new memories on a compact hybrid evaluation set of coverage, boundary, and fresh tasks instead of replaying the full history. Janus is method-agnostic and wraps existing updaters without changing their update rules. Across six datasets, two backbone LLMs, and two memory updaters, Janus improves average accuracy by +2.7 to +4.6 points over the corresponding base updaters.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出Janus，一个即插即用的记忆控制器，包裹现有基础记忆更新器（如ExpeL、DC-RS），决定是接受候选记忆 `cM_t` 还是保留先前记忆 `M_{t-1}`。Janus不修改基础更新器的更新规则，而是作为部署环节的过滤器。
- **关键技术细节**：
  - **记忆动量触发器（Memory Momentum Trigger, MMT）**：
    - 目标：决定“何时”比较新旧记忆，避免每次任务都触发重放，也不采用固定间隔的启发式方法。
    - 实现：利用文本编码器 `ϕ(·)` 将记忆状态映射为向量，定义候选更新方向 `z_t = ϕ(cM_t) - ϕ(M_{t-1})`。维护动量向量 `m_t = β·m_{t-1} + (1-β)·z_t`（β=0.9）。当 `z_t` 与 `m_{t-1}` 的余弦相似度低于阈值 τ（默认0.0）时，认为候选更新偏离了近期轨迹，触发比较。
  - **混合触发时评估集（Hybrid Trigger-Time Evaluation Set）**：
    - 目标：决定“用什么任务”进行比较，以近似完整历史重放但成本极低。
    - 组成：`E_t = S^cov_t ∪ S^bdry_t ∪ F_t`
      - **覆盖集（Coverage set, S^cov_t）**：对已见任务嵌入进行聚类，选取离簇中心最近的代表性任务，保证全局覆盖。
      - **边界集（Boundary set, S^bdry_t）**：存储那些在不同记忆下正确性发生翻转（flip）的任务，即对记忆变化敏感的任务。每次触发后更新，去除已选入选覆盖集的任务，并保留历史边界集中不冲突的任务。
      - **新鲜集（Fresh set, F_t）**：从最近一次触发后到当前步之间的任务中采样，防止决策过拟合到固定支持集。
    - 默认大小：总支持集 K=20（覆盖集12个+边界集8个），新鲜集 K_F=5。
- **决策流程**：
  1. 基础更新器提出候选记忆 `cM_t`。
  2. MMT检测：若 `cos(z_t, m_{t-1}) < τ`，触发比较；否则直接接受候选记忆。
  3. 触发后，在 `E_t` 上评估 `M_{t-1}` 和 `cM_t` 的性能（相同LLM），部署表现更好的记忆。
  4. 根据触发结果更新覆盖集、边界集和新鲜集。

## 6. 论文的主要结论与发现

- **主要结论**：顺序记忆系统需要部署控制器，因为最新的记忆并非总是最好的记忆。Janus作为即插即用的选择性更新控制器，能在不改变基础更新器的情况下，显著提升最终记忆对未来任务的泛化性能。
- **关键发现**：
  - 结构化记忆（DC-RS, ExpeL）并不总是优于轻量级经验重用（ExpRAG），说明记忆抽象可能丢失有用细节或引入噪声。
  - Janus在6个数据集和2个LLM上稳定提升准确率：DC-RS+Janus相比DC-RS平均提升+2.7~+4.6个百分点；ExpeL+Janus相比ExpeL平均提升+2.7~+3.2个百分点。
  - MMT能有效识别需要比较的关键时刻，在相似触发预算下优于随机和周期性触发，且接近“始终触发”的性能但比较次数更少。
  - 混合评估集的三个组成部分（覆盖、边界、新鲜）均不可或缺，移除任何一部分都会导致性能下降，其中移除新鲜集带来的下降最大。
  - 流顺序敏感性实验表明Janus能显著降低最终记忆对任务顺序的依赖，提高稳定性。
  - 定性案例显示Janus能拒绝正确但狭窄的局部更新，接受引入可复用能力的更新。
