## 摘要
语言体在交互中依赖于记忆。然而，大型语言模型的有限上下文窗口及其推理成本限制了单次可使用的记忆量。现有系统主要遵循两种策略：记忆保留和记忆整合。保留方法保留原始记录并保存精确细节，但在预算紧张时相关证据可能无法容纳；整合方法压缩并合并记录，提高每个token的覆盖率，但存在丢失查询关键细节的风险。两种策略均非普遍最优。这引出了两个核心问题：何时应该用整合替代保留，以及应选择哪种算子——合并、抽象还是重写？我们通过将每个算子的效用分解为对保留遗漏证据的覆盖效应和对已适合的原始证据的有符号替代效应来形式化这一决策。它们的平衡解释了为何偏好行为随相对预算压力而变化。我们通过离线抽象-安全（OAS）实现该机制，这是一个轻量级学习器，利用保留伤害校准从预生成特征估计行动效用。公开的LongMemEval和LoCoMo基准测试显示了相同的预算依赖模式。在LongMemEval上，整合在预算紧张时将绝对准确率提高多达48%，而在宽松预算下保留更优；LoCoMo在较小预算下再现了这种交叉，与其较短证据一致。两个数据集上，当需要压缩时，跨笔记抽象和合并通常优于局部重写。

## Abstract
Language agents depend on memory across interactions. However, the limited context windows of large language models (LLMs) and their inference costs constrain how much memory can be used at once. Existing systems mainly follow two strategies: memory retention and memory consolidation. Retention keeps raw records and preserves exact details, but relevant evidence may not fit under a tight budget; consolidation compresses and combines records, improving coverage per token but risking the loss of query-critical details. Neither strategy is universally preferable. This raises two central questions: when should consolidation replace retention, and which operator -- Merge, Abstract, or Rewrite -- should be selected? We formalize this decision by decomposing each operator's utility into a coverage effect on evidence omitted by retention and a signed replacement effect on raw evidence that already fits. Their balance explains why the preferred action changes with relative budget pressure. We implement this mechanism with Offline Abstraction-Safety (OAS), a lightweight learner that estimates action utilities from pre-generation features with held-out harm calibration. The public LongMemEval and LoCoMo benchmarks show the same budget-dependent pattern. On LongMemEval, consolidation improves absolute accuracy by up to 48% under tight budgets, whereas retention is preferable under loose budgets; LoCoMo replicates this crossover at a smaller budget, consistent with its shorter evidence. On both datasets, cross-note abstraction and merging generally outperform local rewriting when compression is necessary.

## 2. 方法论：核心思想、关键技术细节与算法流程

### 核心思想
- **效用分解**：将每个整合算子的预期效用分解为两项：
  - **覆盖效应（Coverage Effect）**：整合对保留遗漏的证据提供支持的增益。
  - **替代效应（Replacement Effect）**：整合替换已能容纳的原始证据时可能带来的质量损失（符号可变）。
- **预算依赖**：当预算紧张时，覆盖效应占主导，整合有利；当预算宽松时，替代效应（损失）占主导，保留更好。

### 关键技术细节
1. **问题形式化**：
   - 动作集 \( \mathcal{A} = \{ \text{ret}, \text{Merge}, \text{Abstract}, \text{Rewrite} \} \)
   - 每个实例 \(z = (M, C, q, B)\)，条件效用 \(\mu_a(z) = \mathbb{E}[u_a(z) \mid z]\)
   - 最优动作：\( a^\star(z) = \arg\max_a \mu_a(z) \)，当 \(\max_o \mu_o(z) \leq \mu_{\text{ret}}(z)\) 时选择保留。
2. **离线抽象-安全（OAS）学习器**：
   - **特征**：11维，包括归一化预算、证据成本/预算比、保留拟合分数、会话数、嵌入不一致性、余弦相似度、查询类型独热编码。
   - **效用估计**：对每个动作，使用岭回归（ridge regression）或MLP（非线性对照）从离线数据学习 \(\hat{\mu}_a(z)\)。
   - **决策**：选择预测效用最大的动作；安全版本（Calibrated OAS）在验证集上设置阈值，仅当预测优势超过阈值时才允许整合，否则保留，以减少有害替代。
3. **机制代理（Surrogate）**：
   - 引入理想化代理效用函数 \(U_B(M) = \mathbb{E}_F[s_B(q, M)]\)，其中 \(s_B\) 是带符号的证据贡献。
   - 代理效用变化 \(\Delta U_{B,o} = C_{B,o} + R_{B,o}\)，严格推导出预算依赖性质。

### 算法流程（文字描述）
1. 离线收集：对每个训练实例，执行所有四个动作（保留、Merge、Abstract、Rewrite），获取答案准确率作为效用标签。
2. 特征提取：从实例中提取预生成特征（11维）。
3. 训练：对每个动作训练一个回归模型（岭回归或MLP），输入特征预测效用值。
4. 校准（可选）：在验证集上选择每个预算对应的阈值，使有害整合次数满足约束（如零）。
5. 部署：对新实例，计算各动作预测效用，取最大者；若启用安全机制，则仅当预测优势超过阈值时才执行整合。

## 6. 主要结论与发现

1. **预算交叉（Budget Crossover）**：整合在预算紧张时显著优于保留（LongMemEval最紧预算下Abstract提升48%准确率），而保留在宽松预算下优于整合（256 token时保留领先8-11%）。LoCoMo以更小预算值复制该模式，证实是相对预算压力而非绝对token值决定。
2. **算子选择**：在需要压缩时，跨笔记抽象（Abstract）和合并（Merge）显著优于局部重写（Rewrite）。但Abstract与Merge之间无显著差异。
3. **机制验证**：分层分析显示，当保留不能容纳全部证据时整合带来帮助，当所有证据已保留时整合带来伤害，支持覆盖-替代解释。
4. **OAS路由器**：在受控交叉验证下，OAS Direct在紧张预算下优于保留和固定策略；Calibrated OAS减少了有害整合但牺牲了部分增益。全历史迁移测试中OAS未超过最优固定策略，表明当前方法在端到端迁移中尚不成熟。
5. **跨模型稳健性**：GLM-5.2复制了相同的预算交叉模式。

## 7. 优点

- **问题形式化创新**：首次统一建模保留与整合的联合决策，提出效用分解机制（覆盖与替代），提供可检验的预算依赖假设。
- **轻量级可学习的方案**：OAS从离线标注中学习预生成特征与效用之间的关系，避免生成所有替代的高昂成本，且采用低容量模型（岭回归）减少过拟合。
- **实验设计严谨**：严格控制检索、答案模型、评判器不变，只改变记忆表示；所有结果经过统计检验（bootstrap、p值）；公开基准验证；包含跨模型、跨数据集复制。

- **安全校准**：引入Calibrated OAS，通过约束有害替代来提升实际部署安全性。

## 8. 不足与局限

- **受控设置简化了现实**：实验中固定了证据簇、检索器、答案模型等。实际语言体需要联合学习检索、整合和打包，且记忆更新是序列性的，本文未涉及。
- **迁移性未建立**：全历史端到端评估中OAS未超越最优固定策略（如固定Merge），表明所学路由难以迁移到有检索噪声的真实场景。作者承认“boundary of transferable routing”。
- **特征设计依赖领域知识**：11维特征中包含了查询类型（单会话、多会话、时间、知识更新）等，需要预分类器或人工标注，影响可迁移性。
- **线性假设可能不足**：虽然MLP对照未显著改善，但ridge regression的线性约束仍可能限制对复杂非线性关系的捕获。实验仅在小数据集上验证，更大规模数据下线性可能不足。
- **仅两个基准**：LongMemEval和LoCoMo，缺乏更多样化的领域（如多模态、工具交互等）的验证。
- **未讨论长期影响**：多次整合可能导致错误累积或记忆漂移，本文仅考虑单步决策。
- **算力成本未透明**：虽使用API，但离线标注需大量生成（4动作×4预算×多次重复×数百问题），实际成本可能较高，文中未提及。
