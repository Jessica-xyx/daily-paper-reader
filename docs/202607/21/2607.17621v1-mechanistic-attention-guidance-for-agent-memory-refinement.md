## 摘要
现有的自进化记忆系统主要基于文本输出（如任务轨迹和反思）来改进智能体记忆。然而，这种基于文本的范式很少纳入内部机制信号，导致检索到的记忆在任务执行中实际如何被利用这一问题尚未充分探索。这一限制可能导致不可靠的错误归因和幻觉式的记忆修改。在本工作中，我们表明检索头注意力提供了一种机制信号，用于揭示片段级别的记忆利用情况。通过对记忆片段和决策步骤的注意力进行聚合，我们构建了一个上下文利用矩阵，该矩阵暴露了重复出现的记忆使用模式，并指示了相应的精炼策略。基于这一观察，我们提出了注意力引导记忆精炼（AGMR），这是一个利用注意力揭示的使用模式来指导目标片段级记忆更新的框架。AGMR纠正或增强失败执行中的记忆，简化成功执行中的记忆，并通过重新执行验证每次更新。在交互式决策基准上的实验表明，与仅基于文本的记忆精炼基线相比，AGMR提高了任务性能和记忆效率。代码可在https://anonymous.4open.science/r/AGMR_code-3262/获取。

## Abstract
Existing self-evolving memory systems mainly improve agent memory based on textual outputs, such as task trajectories and reflections. However, this text-based paradigm rarely incorporates internal mechanistic signals, leaving how retrieved memory is actually utilized during task execution underexplored. This limitation can lead to unreliable error attribution and hallucinated memory modifications. In this work, we show that retrieval-head attention provides a mechanistic signal for revealing segment-level memory utilization. By aggregating attention over memory segments and decision steps, we construct a context utilization matrix that exposes recurring memory-use patterns and indicates corresponding refinement strategies. Building on this observation, we propose Attention-Guided Memory Refinement (AGMR), a framework that uses utilization patterns revealed by attention to guide targeted segment-level memory updates. AGMR corrects or enhances memory for failed executions, simplifies memory for successful executions, and verifies each update through re-execution. Experiments on interactive decision-making benchmarks show that AGMR improves both task performance and memory efficiency over text-only memory refinement baselines. ## 2. 论文提出的方法论

### 2.1 核心思想
利用检索头（retrieval heads）的注意力权重作为记忆利用的机制信号，构建上下文利用矩阵（Context Utilization Matrix），揭示记忆片段在决策步骤中的实际被利用模式，并据此指导目标性的片段级记忆更新（修正、增强、简化）。

### 2.2 关键技术细节

#### （1）检索头选择
- 构建检索监督数据集 \( D_{\text{ret}} = \{ (\text{key\_context}^{(i)}, \text{formatted\_prompt}^{(i)}) \}_{i=1}^{N} \)，其中key_context是对应关键决策步骤应检索到的正确记忆片段。
- 对每个头，计算从prompt最后一个token到key_context token的注意力总和作为检索得分：
  \[
  \text{score}_{l,h} = \sum_{i} \sum_{t \in \text{key\_context}^{(i)}} \text{Attention}_{l,h}^{(i)}(\text{last\_token} \to t)
  \]
- 选择得分最高的5个头作为检索头集合 \( H_{\text{ret}} \)。（对Qwen-2.5-7B-Instruct和Llama-3.1-8B-Instruct分别选择。）

#### （2）上下文利用矩阵构建
- 将检索到的轨迹记忆 \( m^* \) 分割为有序片段列表 \( m^* = [s_0, \dots, s_L] \)。
- 智能体执行任务产生步骤序列 \( \tau = [u_0, \dots, u_T] \)。
- 对每个决策步骤 \( t \) 和片段 \( \ell \)，计算步骤级利用得分：
  \[
  \tilde{a}_{t,\ell} = \frac{n_t}{|H_{\text{ret}}|} \sum_{h \in H_{\text{ret}}} \sum_{j \in \bar{s}_\ell} \text{Attention}_h(q_t \to j)
  \]
  其中 \( q_t \) 是生成步骤 \( u_t \) 前的最后一个prompt token，\( \bar{s}_\ell \) 排除观测token以减小长度影响，\( n_t \) 为当前prompt长度。
- 堆叠所有步骤和片段得到矩阵 \( C \in \mathbb{R}^{(L+1) \times (T+1)} \)。

#### （3）记忆利用模式发现
通过分析C，归纳出三种失败模式和一种成功模式：
- **聚焦但错误（Attended but wrong）**：相关片段被高注意力，但内容有误导性 → 修正该片段。
- **遗漏相关记忆（Missed relevant memory）**：相关片段未被注意力覆盖 → 增强该片段的显著性。
- **被无关记忆干扰（Distracted by irrelevant memory）**：无关片段吸引异常注意力 → 双重干预：增强目标片段 + 重写干扰片段。
- **低利用率冗余（Low-utilization redundancy）**：成功任务中，某些片段在整个轨迹中低利用率 → 压缩或删除这些低价值片段。

#### （4）两阶段记忆精炼框架（AGMR）
- **语义反射器（Semantic Reflector）**：通过语义推理定位关键错误步骤/冗余候选。
- **注意力引导精炼器（Attention-Guided Refiner）**：利用C确定利用模式并执行目标性更新。
- **重新执行验证**：对每个候选更新，用精炼后的记忆重新执行任务，只有通过验证才提交到记忆库。
