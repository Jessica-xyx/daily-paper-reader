## 摘要
多轮反思使得基于大语言模型的智能体能够在单个任务中从失败中恢复，但每个任务仍然是孤立的片段：在某个任务上经过多轮反思获得的经验会在下一个任务开始前被丢弃。我们在一个代码生成任务上研究这一差距：智能体根据科学论文的章节，使用开源的Manim库编写Python代码来渲染数学动画。我们提出了ManimAgent，一种自进化的多模态智能体，它通过完全从其自身的任务流中成长的双通道情境记忆库，跨任务携带反思经验，无需权重更新和人工种子。每次动画收敛后，视觉语言模型对渲染的关键帧进行评分；得到的信号填充正通道M+（将成功理由存储为软参考示例）和负通道M-（将验证过的失败模式存储为硬已知陷阱）。在针对无记忆、相同预算的检索增强生成以及记忆打乱基线的固定探针评估中，随着记忆库规模增长，盲测Pass@1上升，反思轮数下降。我们将发布代码、固定记忆快照和任务流。

## Abstract
Multi-round reflection lets agents built on large language models recover from failures within a single task, but each task remains an isolated episode: lessons learned across many reflection rounds on one task are discarded before the next begins. We study this gap on a code-generation task: from a scientific paper section, the agent writes Python in the open-source Manim library to render a mathematical animation. We present ManimAgent, a self-evolving multimodal agent that carries reflection experience across tasks through a dual-channel Episodic Memory Bank grown entirely from its own task stream, with no weight updates and no human seeds. After each animation converges, a vision-language model scores the rendered keyframes; the resulting signals populate a positive channel M+ that stores success rationales as soft Reference Examples, and a negative channel M- that stores validated failure patterns as hard Known Pitfalls. On a fixed-probe evaluation against no-memory, matched-budget retrieval-augmented generation, and shuffled-memory baselines, blind human Pass@1 rises and reflection rounds fall as memory size grows. We will release the code, frozen memory snapshots, and the task stream.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：现有基于大语言模型（LLM）的反射型智能体（如Self-Refine, Reflexion）能在单个任务内通过多轮反思从失败中恢复，但每轮任务结束后，之前积累的经验（如布局技巧、公式渲染规则）立即被丢弃。这种**跨任务遗忘**导致同一类错误在不同任务上重复出现，智能体无法持续进步。作者将这种结构性缺陷称为跨任务遗忘。
- **研究背景**：具体任务场景是从科学论文章节生成Manim动画（Python开源数学动画库）。多智能体系统需要一个反思管线来处理崩溃和视觉质量差的问题，但缺乏跨任务记忆。作者类比人类教师：有经验的教师会携带正面模板（该怎么做）和负面经验（不该做什么）跨主题迁移。
- **整体含义**：提出ManimAgent，通过构建一个**双通道情节记忆库**（Episodic Memory Bank, EMB），使智能体能够自发积累跨任务经验，无需模型权重更新、无需人工种子，实现自我进化。这是首次在视觉教育代码生成场景中解决跨任务遗忘问题。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 在标准反思管线基础上添加两个组件：
  1. **视觉语言模型（VLM）作为结构化奖励源**：对渲染关键帧打分，驱动每个任务内的反思循环，并控制记忆写入。
  2. **双通道情节记忆库（EMB）**：自动从自身任务流中生长，包含正通道（M+）存储成功理由作为软参考示例，负通道（M-）存储已验证的失败模式作为硬已知陷阱。

### 2.2 系统架构与流程
- **任务定义**：每个任务 τ = (s, r, d)，其中s是论文节选文本，r是场景角色（背景/方法/实验/结论），d是领域标签。
- **智能体管线**：
  - **Storyboarder（故事板规划器）**：将任务分解为有序场景。
  - **Coder（代码生成器）**：根据场景计划、从EMB检索的正负记忆生成Manim代码。
  - **文本反射循环**：处理渲染崩溃（最多2次重试），由文本Reviewer给出提示。
  - **视觉反射循环**：处理可渲染但视觉质量差的结果（最多2次修订），由VLM Reviewer对四个关键帧评分（三个维度：逻辑流、布局/遮挡、准确性），驱动视觉修订。
  - **Best-of-N交付**：从所有可渲染候选中选择得分最高的版本。
  - **记忆写入**：任务收敛后，根据VLM分数将高质量结果写入M+，根据成功的因果归因（崩溃→成功或视觉分数提升≥5分）将失败模式写入M-。蒸馏方式：由LLM Distiller蒸馏为可迁移的课程描述，而非字面diff。

### 2.3 记忆读取与注入
- 对每个新场景，使用Sentence-BERT（all-MiniLM-L6-v2）嵌入查询，从M+检索top-2记录作为“参考示例”（软指引），从M-检索top-3记录作为“已知陷阱”（硬约束）。
- 正负通道分离：M+作为软先验，M-作为硬排除规则，模仿人类教师的双重经验形式。
- 写入阈值：M+要求最佳得分≥85（低于自动通过阈值90，确保可记忆）；M-要求相邻尝试得分提升≥5分或文本修复成功。

## 3. 实验设计

### 3.1 数据集与场景
- 自建论文节选动画数据集：从科学论文中提取112个任务，按论文级别分为：
  - **记忆构建流**（39个任务）：用于增长EMB。
  - **固定探针集**（33个任务）：用于评估每个冻结EMB快照（只读模式，不写入探针输出）。
  - **交叉测试集**（40个任务）：探索性，仅用于定性检查。
  - **人类评分子集**（25个任务）：用于盲测。
- 领域覆盖：计算机科学、数学、物理学、量子、经济学等5个领域，4种场景角色。
- 确保论文级分离：记忆构建和探针集无重叠论文，防止数据泄漏。

### 3.2 基线方法
- **VLM reflection only**：无记忆检索的标准反思管线。
- **EMB@0**：空记忆（结构相同但插槽为空，隔离提示结构效应）。
- **EMB@50 / 100 / 200**：冻结的EMB快照，分别包含50/100/200条记录。
- **Manim-code RAG**：使用相同编码器、索引和预算从固定的Manim文档和社区示例库中检索（匹配公共RAG基线）。
- **Random EMB**：记录内容打乱，保持计数和格式，破坏内容-查询对齐。
- **消融实验**：包括单通道（仅M+或仅M-）、无门控（存储所有诊断）、top-1检索、仅理由无代码、弱VLM替代、仅文档RAG等。

### 3.3 评估指标
- **人类Pass@1**：盲测人工判断首次尝试视频是否可作为论文节选动画使用（二进制）。
- **反思轮数**：任务收敛所需的平均修复轮数（更少更好）。
- **人类质量**：五个平均Likert维度（视觉设计、关键主张覆盖、视觉鲁棒性、动画流、首次尝试可用性），每个1-5分。
- **VLM分数**：仅作为辅助诊断，非主要证据（避免评估者泄露风险）。

### 3.4 人类评价协议
- 54名盲测评分者（研究生，CS/AI为主），每位视频获得2-3份独立评分。
- 评分前有校准环节，评分者看不到条件标识。
- 所有人类评分存储在模型不可见的文件中。

- **模型**：所有文本智能体和VLM评审器使用GPT-5.5系列；弱VLM消融使用Step-3.7-Flash。
- **渲染**：Manim Community v0.20.1，Cairo后端；沙箱限制：墙上时间180s，CPU 120s，RAM 4GB。

## 5. 实验数量与充分性

- **主要实验**：7个头部条件（VLM-only, EMB@0/50/100/200, Manim-code RAG, Random EMB）在固定探针集上比较，每个条件使用相同的人类评分协议（25个任务，每个3票）。
- **消融实验**：共6组消融，分别测试通道分离、门控、检索深度、内容类型、VLM强度、RAG内容等。每个消融同样在固定探针集上评估。
- **定性分析**：一个BERT案例的轨迹分析，展示视觉修订前后改进；检索记忆的定性分类。
- **充分性评价**：实验设计较为充分：固定探针快照设计隔离了任务顺序和测试时间学习；随机EMB基线控制了格式和长度效应；消融实验覆盖关键设计选择；盲测人类评分避免了评估者偏差。但也有一些局限（见下文不足部分）。

## 6. 主要结论与发现

- EMB@K在所有三个指标上单调改善：Pass@1从62.0%升至84.9%，人类质量从3.26升至3.88，反思轮数从12.2降至6.5（EMB@0 vs EMB@200）。
- EMB@200在人类指标上略优于Manim-code RAG（84.9% vs 83.3% Pass@1，3.88 vs 3.65质量），且反思轮数大幅减少（6.5 vs 11.4）。
- Random EMB表现与VLM-only接近，证明记忆内容的对齐是关键，而非格式。
- 消融实验显示：正通道M+主要提升首次成功率，负通道M-主要减少反思轮数；移除M+导致Pass@1大幅下降（-14.9%），移除M-导致反思轮数膨胀（+5.1轮）。门控机制有效：无门控M-导致Pass@1下降（-3.1%）。
- VLM与人类评分的一致性很弱（Pearson r=-0.17, Cohen's κ=-0.07），因此VLM仅作为内部信号，不能替代人类评估。
- EMB@200位于成本-质量帕累托前沿：以稍高的墙上时间（+4分钟 vs RAG）换取+0.23质量的提升。

## 7. 优点

- **方法创新**：双通道记忆（M+软指引+M-硬排除）的设计直观且有效，模仿人类教学中的正向模板和负面教训；蒸馏为可迁移描述而非字面diff，增强泛化性。
- **实验严谨**：固定探针快照设计隔离了任务顺序效应；随机EMB基线控制格式/长度混淆；盲测人类评分避免评估者泄露；多组消融明确各组件贡献。
- **实用价值**：无需权重更新、无需人工种子，即可实现跨任务知识迁移，对资源有限的场景有吸引力；代码和记忆快照将开源，利于复现。
- **广泛的评估**：涵盖5个领域4种角色，人类评分者多样（CS、数学、物理、经济），协议详细规范。

## 8. 不足与局限

- **评估范围有限**：仅使用Manim渲染器，仅测试英文科学文献；未验证迁移到其他渲染器（如D3.js, Three.js）或非英语文献；未做下游学习效果研究（学生使用动画的实际学习成果）。
- **规模与成本**：EMB仅验证了~200条记录规模，生产级百万级需要去重和分层索引；每任务成本较高（19-31分钟墙上时间），且随EMB增长渲染时间增加；记忆可能随LLM/VLM升级而“腐烂”（因写入门控依赖当前VLM）。
- **奖励与提示敏感性**：VLM-人类一致性弱，弱VLM消融仅部分缓解；检索查询措辞和提示格式为固定工程选择，未经消融。
- **数据污染**：虽然按论文级别分离了记忆构建和探针集，但记忆构建流内部的近似重复未做句级去重；VLM作为评估器可能存在自我确认偏见（虽使用人类评估缓解）。
- **评估统计**：人类评分者间一致性很低（Fleiss' κ=-0.01 for binary, ICC ≤0.33），表明主观性强，但作者直接聚合未做裁决，可能引入噪声；Pass@1的统计显著性未报告。
- **未与最相关系统对比**：虽然提及Code2Video、Manimator、LLM2Manim等，但论文声称“输出模态和终止条件不能映射到单个Manim场景”而拒绝直接基准测试，削弱了相对优势的证明。
