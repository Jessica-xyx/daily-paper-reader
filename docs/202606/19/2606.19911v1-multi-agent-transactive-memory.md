## 摘要
在不同任务中部署具有多样能力的LLM智能体，激发了跨异构智能体群体的知识共享基础设施需求。正如搜索引擎索引人类生成的制品以支持人类问题解决，检索系统可以组织智能体生成的制品以便在智能体群体中重用。我们将检索增强生成（展示了人类撰写的制品对个体智能体的价值）扩展到检索支持智能体群体的智能体生成制品。特别地，智能体轨迹编码了可重用的程序性知识，但这些制品通常在一次使用后被丢弃或仅由生成智能体保留，迫使新实例化的智能体反复重新发现现有解决方案。我们提出多智能体交互记忆（MATM），一种用于智能体生成轨迹的群体级存储与检索框架，其中生产者智能体将轨迹贡献到共享存储库，消费者智能体检索它们以改进任务执行。我们专注于交互环境（ALFWorld和WebArena），其中的轨迹较长且编码了特别丰富的程序性结构。我们的实验表明，从MATM检索轨迹能提高下游任务性能并减少交互步骤，而无需协调或联合训练。这些结果将MATM定位为开放智能体生态系统中群体级经验共享的设计模式。

## Abstract
The decentralized deployment of LLM agents with diverse capabilities across diverse tasks motivates infrastructure for knowledge sharing across heterogeneous agent populations. Just as search engines index human-generated artifacts to support human problem solving, retrieval systems can organize agent-generated artifacts for reuse across agent populations. We extend retrieval-augmented generation - which demonstrates the value of human-authored artifacts to individual agents - to retrieval of agent-generated artifacts supporting a population of agents. In particular, agent trajectories encode reusable procedural knowledge, yet these artifacts are typically discarded after a single use or retained only by the producing agent, forcing newly instantiated agents to repeatedly rediscover existing solutions. We propose Multi-Agent Transactive Memory (MATM), a framework for population-level storage and retrieval of agent-generated trajectories, where producer agents contribute trajectories to a shared repository and consumer agents retrieve them to improve task execution. We focus on interactive environments (ALFWorld and WebArena), where trajectories are long and encode especially rich procedural structure. Our experiments demonstrate that retrieving trajectories from MATM improves downstream task performance and reduces interaction steps without coordination or joint training. These results position MATM as a design pattern for population-level experience sharing in open agent ecosystems.

# 多智能体交互记忆（Multi-Agent Transactive Memory）论文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：LLM智能体在不同任务中广泛部署，产生了大量包含程序性知识的交互轨迹，但这些轨迹通常被丢弃或仅由产生者保留，导致新智能体需要反复“重新发明轮子”。
- **核心动机**：现有方法（如RAG）支持智能体检索人类文档，但缺乏对智能体自身生成制品的群体级共享机制；个体级别的经验复用（如Buffer of Thoughts）局限于同一智能体，无法跨群体受益。
- **研究目标**：提出一种群体级别的共享记忆框架，让智能体生产者贡献轨迹、消费者检索轨迹，从而提升任务性能与效率，无需协调或联合训练。

## 2. 方法论：核心思想、关键技术、算法流程

- **核心思想**：基于“交互记忆”（transactive memory）概念，将智能体群体视为一个社会系统，通过共享存储与检索机制分布知识。
- **框架名称**：Multi-Agent Transactive Memory (MATM)
- **关键组件**：
  - **生产者（Producer）**：在完成任务后，将成功轨迹贡献到共享仓库。
  - **消费者（Consumer）**：在遇到新任务时，从仓库检索相关轨迹作为参考，辅助决策。
  - **共享仓库**：存储按“键-值”模式分块的轨迹，键为最近历史步骤，值为后续步骤。
  - **检索机制**：采用两阶段流水线——第一阶段：稠密检索（E5-Base嵌入模型）获取候选；第二阶段：学习排序（LTR）重排序，使用SVMRank、LambdaMART或FFN等模型。
- **重要创新**：LTR重排序利用六类特征（生产者元数据、消费者元数据、第一阶段得分、查询特征、轨迹特征、查询-轨迹交互特征），实现信任建模与个性化。
- **算法流程**：先预填充初始索引（公开轨迹），再增量更新：每个训练问题分配生产者，先生成无检索基线，再在随机分支点检索top-K，并评估每条结果轨迹的边际效用（与基线得分差），并将成功轨迹加入仓库，同时收集标签训练LTR。

## 6. 主要结论与发现

1. **MATM提升效果与效率**：两个环境中，检索均提高成功率并减少步数（ALFWorld SR +8%p，WebArena +2%p，步数分别减少0.6和1.7步）。
2. **LTR重排序进一步增强**：SVMRank在ALFWorld上最优（SR 64.3% vs 基线47.1%），FFN在WebArena最优（SR 20.5% vs 基线18.2%），RPP显著提升。
3. **收益广泛分布**：消费者受益不依赖于生产者能力更强，弱生产者也能提供有用轨迹；重排序进一步提升所有配对的收益。
4. **跨任务泛化**：即使限制检索来源为不同任务类型轨迹，仍显著优于无检索基线（ALFWorld SR 59.9% vs 47.1%），说明轨迹包含可转移模式。
5. **规模正效应**：索引越大，效果越好（ALFWorld单调提升，WebArena非单调但最终最高）。

## 7. 优点

- **创新性**：首次提出群体级别的智能体轨迹共享机制，区别于个体记忆或RAG，定位精准。
- **实用性**：无需协调或联合训练，兼容异构智能体群体，即插即用。
- **系统性实验**：多个消融、规模效应、能力差距分析，验证了方法鲁棒性和泛化性。
- **重排序设计巧妙**：利用生产者元数据实现信任建模，利用消费者元数据实现个性化，示例丰富。

## 8. 不足与局限

- **环境覆盖有限**：仅两个交互环境（ALFWorld、WebArena），未覆盖更复杂或真实世界场景（如编程、对话、机器人控制）。
