## 摘要
长期运行的LLM智能体积累的交互历史远超任何上下文窗口，迫使一个持续存在的问题：在固定记忆预算下，应深度编码什么、遗忘什么、检索什么。生产系统通过语义相似性或近期性来回答——这两者对于遗忘决策而言都是错误设定的，因为遗忘决策是在合并时刻、在未知未来查询之前作出的。我们提出了一个多因子记忆价值函数 V(m)=∑ᵢ wᵢ fᵢ(m)，该函数基于从认知心理学中提取的七个可解释因子（情绪强度、目标相关性、价值一致性、自我/用户相关性、任务效用、可靠性和使用历史），其权重通过无梯度优化器从下游目标学习而来，其单一标量统一控制编码深度、遗忘风险和检索排名。我们提出了一个方法论观点：在LongMemEval上，针对保留评估问题对目标相关性进行评分会使黄金证据的保留率饱和到约0.98——这衡量的是检索而非遗忘。在现实盲设场景中，学习得到的多因子价值模型在479个可用案例中保留了0.770±0.011的黄金证据，而均匀权重为0.657，最佳单因子为0.518，近期性为0.368；每对差距的95%自助置信区间均高于零，且基于相同因子的神经网络与线性模型持平。学习得到的权重是可解释的——可靠性、情绪强度和自我/用户相关性占主导，而查询时的目标相似性在遗忘决策中被正确降权。一个带有植入混淆因素的受控合成任务证实，学习器恢复出了分离性权重（保留率为1.00），而均匀权重则失败（保留率为0.62）。该基础框架是开源的；所有实验在单个CPU上运行，无需API调用。

## Abstract
Long-running LLM agents accumulate interaction histories far larger than any context window, forcing a standing decision: what to encode deeply, what to forget, and what to retrieve under a fixed memory budget. Production systems answer with semantic similarity or recency -- both mis-specified for the forgetting decision, which is made at consolidation time before the future query is known. We propose a multi-factor memory value function V(m)=\sum_i w_i f_i(m) over seven interpretable factors (emotional intensity, goal relevance, value alignment, self/user relevance, task utility, reliability, and usage history) drawn from cognitive psychology, whose weights are learned from a downstream objective by a gradient-free optimiser, and whose single scalar uniformly controls encoding depth, forget risk, and retrieval rank. We make a methodological point: on LongMemEval, scoring goal relevance against the held-out evaluation question saturates gold-evidence retention at \approx 0.98 -- this measures retrieval, not forgetting. In the realistic blind regime, a learned multi-factor value retains 0.770 \pm 0.011 of gold evidence across 479 usable cases, versus 0.657 for uniform weights, 0.518 for the best single factor, and 0.368 for recency; every paired gap's 95% bootstrap CI is above zero, and a neural network over the same factors ties the linear model. The learned weights are interpretable -- reliability, emotional intensity, and self/user relevance dominate, while query-time goal similarity is correctly down-weighted for the forgetting decision. A controlled synthetic task with planted confounds confirms the learner recovers a separating weighting (1.00 retention) where uniform weighting fails (0.62). The substrate is open-source; all experiments run on a single CPU with no API calls.

## 1. 论文的核心问题与整体含义（研究动机和背景）

长期运行的LLM智能体（LLM agent）会积累远超任何上下文窗口长度的交互历史。系统必须在固定记忆预算下做三件事：**编码什么（编码深度）、遗忘什么（遗忘风险）、检索什么（检索排序）**。现有生产级记忆系统通常采用两种单因子策略：  
- **语义相似性**：通过当前查询的嵌入相似度排序；  
- **时间近因性**：优先保留最近的信息。

**核心问题**：这两种策略都不适用于“遗忘”决策。遗忘决策在**合并（consolidation）时刻**作出，而此时未来查询尚不可知。语义相似性只在检索时（查询已知时）才有定义；近因性虽然与查询无关，但会牺牲旧但重要的事实（如用户过敏、项目约束）而保留近期无价值的闲聊。  
**整体含义**：人类记忆不是单因子驱动的。认知心理学表明，记忆保留受**多因子价值估计**引导（价值导向记忆、加工水平、情绪唤醒、自适应遗忘等）。本文将这一原则引入智能体记忆，提出一个**多因子记忆价值函数**，其权重从下游任务目标中学习，统一控制三叉戟决策，从而解决遗忘决策在信息未知时的根本性缺陷。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
定义一个记忆价值函数 \(V(m)=\sum_i w_i f_i(m)\)，其中 \(f_i(m)\) 是七个可解释因子，\(w_i\) 是从下游目标（如黄金证据保留率）通过**无梯度优化**学习的权重。单个标量 \(V(m)\) 统一驱动三个操作：编码深度、遗忘风险、检索排序。

### 七个因子（均为 \([0,1]\) 归一化）
| 因子 | 定义（API-free 形式） | 本文实验中是否激活 |
|------|----------------------|-------------------|
| 情绪强度（emotional intensity） | 基于词汇/结构提取器的 \(|\text{Vaff}| \cdot A\) | 是 |
| 目标相关性（goal relevance） | \(\frac12 (1+\cos(e_m, e_{\text{goal}}))\) | 是（含 oracle 和 blind 两种锚点） |
| 自我/用户相关性（self/user relevance） | \(\frac12 (1+\cos(e_m, \mu_{\text{user}}))\)，\(\mu_{\text{user}}\) 为用户话语质心 | 是 |
| 可靠性（reliability） | 来源启发式（用户陈述 > 模型陈述） | 是 |
| 价值一致性（value alignment） | 匹配价值档案（SOUL） | 否（置0） |
| 任务效用（task utility） | 边际任务成功贡献（LLM 裁判） | 否（置0） |
| 使用历史（usage history） | \(r/(1+r)\)，\(r\)=检索次数 | 否（置0） |

### 三个操作的统一控制
- **编码深度**：将归一化值 \(\bar V(m)\) 按阈值 {0.20, 0.45, 0.70} 分为四个层级（浅层/语义/图式/元），高层级分配更深编码。
- **遗忘分数**：\(\text{forget}(m) = (\Delta t)^{0.7} \cdot \frac{1}{1+r} \cdot \frac{1}{1+\beta V(m)}\)，其中 \(\Delta t\) 是天数，\(r\) 是检索次数。高 \(V(m)\) 提供乘性遗忘抵抗。
- **检索排序**：直接按 \(V(m)\)（使用相应场景下的目标锚点）排序，然后挑选预算内的记忆。

### 权重学习算法（A2）
- 目标函数 \(R(w)\)：平均黄金证据保留率（固定保留预算下）。
- 优化器：**随机重启爬山法**（gradient-free stochastic hill-climb，算法1）。由于编码→遗忘→检索→回答管道包含离散决策和非可微的外部回答器，使用无梯度优化。
- 初始化 \(w = 1\)（均匀），每轮对每个因子以概率0.5添加高斯噪声 \(\mathcal{N}(0,\sigma)\)，若新权重 \(w'\) 产生更高 \(R\) 则接受，并退火衰减 \(\sigma\)。
- 限制：仅搜索实际有因子值的维度（避免赋予恒为0的因子非零权重）。

## 3. 实验设计：数据集、基准、对比方法

### 数据集
- **LongMemEval-S**：500 个多会话聊天历史问题（约 500 轮/案例），每个问题有标注的黄金证据（gold-evidence）回合，位于大量的干扰会话（haystack）中。实际使用 479 个至少包含一个黄金证据的问题。
- **合成混淆实验**：每个案例包含 4 个黄金项和 16 个干扰项。黄金项携带真实信号（高目标相关性、任务效用、可靠性）；干扰项植入人为混淆（高价值一致性、高自我相关性、高情绪噪声）。

### 基准（对比方法）
- **单因子策略**：仅使用单一因子评分（目标相关性、情绪强度、自我相关性、可靠性、最近性）。
- **均匀权重（uniform_V）**：所有因子等权重。
- **随机（random）**：随机保留预算比例的记忆（仅作为底线）。
- **神经网络（MLP）**：以相同四个活因子输入（16×16隐藏层），通过成对逻辑排序损失训练，用于验证线性模型是否损失性能。
- **Oracle vs. Blind 对比**：
  - **Oracle**：目标锚点使用**保留问题的嵌入**（窥视未来查询，测量检索上限）。
  - **Blind**：目标锚点使用**当前会话中用户回合的质心**（整合时刻实际可知的信息，测量遗忘）。

### 评估指标
- 主要：在固定保留预算（κ=30%）下，**黄金证据保留率**（Gold-evidence retention，即被保留的记忆中包含的黄金回合比例）。
- 辅助：合成实验中的黄金保留率；κ 从 0.1 到 0.5 的 sweep。

## 6. 论文的主要结论与发现

1. **多因子学习价值远超单因子和均匀权重**：在盲设（realistic forgetting）下，learned_V 保留 0.770±0.011 黄金证据，而均匀权重 0.657，最佳单因子（自我相关性）0.518，最近性 0.368。每对差距的95%自助置信区间均高于零。
2. **Oracle 与 Blind 的差异至关重要**：在 oracle 下（目标锚点窥视未来查询），单因子目标相关性即可达到 0.979（learned_V 0.993），无法体现多因子优势。盲设下此因子塌陷至 0.286（接近随机水平），凸显“检索 vs 遗忘”的混淆问题。
3. **学习到的权重可解释**：在盲设中，可靠性权重最高（0.64），情绪强度次之（0.55），自我相关性 0.23，目标相关性被降为 0.00。表明 LongMemEval 黄金证据主要由可靠、用户陈述、自我相关的事实贡献，会话主题相似性对预测未来查询无帮助。
4. **线性模型足矣**：MLP 与线性模型表现持平（+0.003±0.013），说明因子以接近加法方式组合。
5. **优势随预算收紧增大**：在 κ≤0.4 的所有激进预算下，learned_V 始终领先；仅在 κ=0.5（几乎松散的预算）时所有策略饱和。
6. **合成实验确认学习器能恢复分离权重**：当均匀权重被混淆因子误导（保留率0.62）时，学习器成功聚焦真实信号因子，保留率 1.00。

## 7. 优点：方法或实验设计上的亮点

- **认知动机清晰**：每个因子均来自认知心理学确立的记忆决定因素，非随意组合。
- **统一三操作**：单个标量同时控制编码深度、遗忘和检索，替换之前分开调优的规则，简化系统设计。
- **学习权重而非手工设定**：权重从下游目标中优化，适应具体任务，避免了 Generative Agents 那样手动设定1-10重要性分数的不确定性。
- **盲测 vs Oracle 的对比**：方法论贡献显著，明确指出常用记忆评估基准的混淆——评估“遗忘”时不应使用未来查询信息，否则实际上测量的是“检索”的上限。这对未来的性能评估有指导意义。
- **完全离线、无API、单CPU可复现**：实验廉价且可重复，降低了科研门槛，避免了 API 成本波动影响结果。
- **权重可解释作为工作负载诊断**：学习到的权重可直接解读该任务中最值得记忆的信号，可用以指导系统设计。
- **统计严谨性**：使用多次随机分割、自助法置信区间、贝叶斯式验证，结果稳定可靠。
