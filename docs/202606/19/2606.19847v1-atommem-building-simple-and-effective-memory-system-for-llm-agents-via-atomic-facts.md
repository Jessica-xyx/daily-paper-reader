## 摘要
大型语言模型（LLM）展现出强大的推理和生成能力，但其固定的上下文窗口限制了跨多会话交互中的长时信息积累与复用。现有记忆增强系统通常以粗粒度且不稳定的方式构建记忆，依赖于低效的记忆表示或不可控的无约束更新。为应对这些挑战，我们提出AtomMem，一种面向价值密集型存储与稳定记忆演化的长时记忆系统。AtomMem引入事实执行器（Fact Executor），从长程交互中选择性提取高价值原子事实，作为高效记忆表示。随后，AtomMem将这些事实组织为层次化事件结构与时间轮廓，捕获连贯的情境上下文并追踪随时间动态演化的用户属性。在检索时，系统激活关联记忆图以连接碎片化记忆。在LoCoMo基准上的实验证实，AtomMem在各种推理任务中均达到最先进性能，为部署智能个性化智能体提供了可扩展且经济可行的解决方案。

## Abstract
Large language models (LLMs) demonstrate strong reasoning and generation abilities, but their fixed context windows limit long-term information accumulation and reuse across multi-session interactions. Existing memory-augmented systems often construct memory in a coarse and unstable manner, relying on inefficient memory representations or unstable unconstrained updates. To address these challenges, we propose AtomMem, a long-term memory system designed for value-dense storage and stable memory evolution. AtomMem introduces a Fact Executor, which selectively extracts high value atomic facts from long form interactions to serve as highly efficient memory representations. Subsequently, AtomMem organizes these facts into hierarchical event structures and temporal profiles, capturing coherent episodic contexts and tracking dynamically evolving user attributes over time. During retrieval, the system activates an associative memory graph to connect fragmented memories. Experiments on the LoCoMo benchmark confirm that AtomMem achieves state-of-the-art performance across various reasoning tasks, offering a scalable and economically viable solution for deploying intelligent personalized agents.

# 论文总结：AtomMem：通过原子事实为LLM智能体构建简单有效的记忆系统

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：LLM 的固定上下文窗口限制了其在多轮、跨会话交互中长期信息的积累与复用。现有记忆增强系统存在两大缺陷：
  - **记忆表示粗粒度**：原始对话冗余噪声多，压缩表示则丢失细节，且LLM自身生成噪声随更新累积。
  - **更新不稳定**：依赖频繁的 LLM 重写，易引发幻觉或错误编辑，导致记忆发散、原始事实被破坏。
- **意义**：需要一种信息密度高、更新稳定、跨会话可关联的长期记忆系统，以支撑智能体的持续个性化服务。

## 2. 方法论

### 核心思想
以 **原子事实** 为基本记忆单元，通过层次化结构（事实 → 事件 → 时间画像）组织，并利用关联图在检索时激活碎片化记忆。

### 关键技术细节

- **步骤 1：原子事实提取**
  - 采用 **SFT 微调**的 Fact Executor（基于 Qwen3-14B），从原始对话中提取自包含、高价值的原子事实（解决指代消解、时间锚定）。
  - 每个原子事实结构化为：`{id, 文本c, 向量v, 参与者P, 关键词K, 时间T, 事件列表E}`。
  - 设计**事实验证**机制：通过混合相似度（语义嵌入 Jaccard + 关键词）检索候选事实，再由 LLM 判断冗余/冲突，生成剩余内容或更新。

- **步骤 2：层次化事件记忆**
  - 将相关原子事实聚合为 **事件（Event）**，包含摘要S、事实列表Fids、参与者/关键词/时间跨度。
  - 算法：新事实经验证后，LLM 判断应吸收到现有事件、创建新事件，或独立存储（具体见 Algorithm 1）。

- **步骤 3：时间画像建模**
  - **Profile** 条目记录用户稳定属性（偏好、习惯），支持历史版本回溯（`H`）。
  - 采用**会话级批处理**：对话结束后，候选特征经检索+ LLM 判定，决定冗余、更新、创建新画像或修改历史版本。

- **步骤 4：关联检索**
  - 构建**记忆图**，节点为原子事实，边分三类：
    - **实体边**（关键词重叠，用 IDF 加权避免高频词噪声）
    - **事件边**（同事件内事实，惩罚大事件）
    - **时间边**（邻近对话回合，指数衰减）
  - **检索流程**：
    1. **主检索**：基于参与者/时间过滤 + 混合相似度 `Sh` 得到种子集合 `Rseed`。
    2. **补偿检索**：从匹配事件中提取事实，融合事件级别与事实级别分数。
    3. **关联检索**：以 `Rseed` 为中心扩展局部图，采用**随机游走带重启（RWR）** 传播激活分数，取 top-k 事实。
  - 若查询需画像，额外检索相关 profile 并选择对应时间版本。

- **整体生成**：将检索到的事实集和画像集拼接，输入 LLM 生成最终回答。

## 7. 优点

- **创新性**：提出“原子事实”作为记忆基元，兼具信息密度与上下文自包含性，且通过 SFT 训练提升抽取质量。
- **系统设计完整**：从提取、验证、层次组织（事件/画像）到基于图的关联检索，形成闭环。
- **可扩展性**：压缩冗余对话，大幅降低 token 消耗，经济性突出。
- **实验严谨**：统一基线设置、多指标（含 LLM 评判）、严格消融、多基准测试。

## 8. 不足与局限

- **对 LLM 依赖性**：多个阶段（提取、验证、路由、生成）均依赖底层 LLM 能力，性能受制于基模型稳定性。
- **模态局限**：当前仅支持文本交互，不支持图像、音频等多模态输入。
- **数据集偏差**：训练数据基于 LoCoMo 风格模拟对话，可能与真实用户交互存在差异；LongMemEval 虽为独立测试，但同样为合成数据。
- **评估覆盖**：未在真实部署场景中测试延迟、长程退化等问题；未与其他检索增强方法（如 Self-RAG）对比上游事实抽取模块。
- **优化空间**：尽管 token 效率较好，但全管线仍有一定开销（平均 3.6s 在线延迟），对于实时性要求极高的场景可能需进一步优化。
