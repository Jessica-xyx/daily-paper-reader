## 摘要
跨会话持久存在的AI智能体需要可检索、审计、更新和清除的记忆。现有记忆系统常将来源证据、提取的事实、检索的上下文和回答策略合并为一条不透明的提示路径，导致故障难以诊断：错误回答可能源于缺失证据、不支持的提取、过时状态、检索丢失或回答模型行为。我们提出Eywa，一种围绕“证据先于信念”构建的基于来源的记忆架构。Eywa在推导规范事实之前存储不可变的来源证据，根据类型化信号和来源支持验证提取的记忆，并通过确定性多路由读取路径（检索内部零LLM调用）检索有界记忆上下文。检索到的上下文与回答指令分开返回，使得同一记忆基底可跨前沿、预算和本地回答模型进行评估。在冻结的、工件记录的检索配置下，Eywa在LoCoMo C1-C4分割上使用Claude Sonnet 4.6写入和QA角色达到90.19%的评判准确率。在LongMemEval-S上达到88.2%的检索充分性准确率。在BEAM（一个700问题技术记忆压力基准）上达到81.45%的平均nugget分数和85.29%的pass@score >= 0.5。完整的每问题工件，包括问题、黄金答案、模型答案、检索上下文和标签，已在https://eywa.to/research发布。

## Abstract
AI agents that persist across sessions need memory they can retrieve, audit, update, and erase. Existing memory systems often collapse source evidence, extracted facts, retrieved context, and answer policy into one opaque prompt path, making failures difficult to diagnose: a wrong answer may come from missing evidence, unsupported extraction, stale state, retrieval loss, or answer-model behavior. We present Eywa, a provenance-grounded memory architecture built around evidence before belief. Eywa stores immutable source evidence before deriving canonical facts, validates extracted memories against typed signals and source support, and retrieves bounded memory context through a deterministic multi-route read path with zero LLM calls inside retrieval. Retrieved context is returned separately from answer instructions, allowing the same memory substrate to be evaluated across frontier, budget, and local answer models. Under a frozen, artifact-recorded retrieval configuration, Eywa reaches 90.19% judge accuracy on the LoCoMo C1-C4 split with Claude Sonnet 4.6 write and QA roles. On LongMemEval-S, it reaches 88.2% retrieval-sufficiency accuracy. On BEAM, a 700-question technical-memory stress benchmark, it reaches 81.45% mean nugget score and 85.29% pass@score >= 0.5. Full per-question artifacts, including questions, gold answers, model answers, retrieved context, and labels, are published at https://eywa.to/research.

- **问题**：现有AI智能体记忆系统通常将来源证据、提取的事实、检索的上下文和回答策略糅合在一条不透明的提示路径中，导致错误难以诊断。一个错误回答可能源于：缺失证据、不支持的提取、过时状态、检索丢失或回答模型行为，但系统无法定位具体故障层。
- **背景**：人类记忆支持自我连续性、社会协调和面向未来的行动，而AI智能体跨会话交互需要功能连续性。长期上下文窗口（如Gemini 1.5）不足以解决记忆管理问题（存储什么、如何修订、如何遗忘、如何审计和擦除）。已有工作如Generative Agents、Reflexion、MemoryBank、MemGPT等建立了主动记忆，但缺乏对故障层的可审计性。
- **整体含义**：Eywa提出“证据先于信念”原则，构建可审计、可调试的记忆架构，将不可变来源证据与派生信念分离，确定性读取（零LLM调用）与答案策略分离，使记忆系统成为可独立评估和改进的基础设施。

## 2. 论文提出的方法论
- **核心思想**：证据先于信念（Evidence Before Belief）。记忆系统应优先存储不可变的来源证据，在此基础上推导、验证和链接规范事实（beliefs）。检索阶段不依赖LLM，而是通过确定性多路由返回有界上下文，并与答案指令分开传递给回答模型。
- **关键技术细节**：
  - **写路径（Write Path）**：
    - **Tier 0 不可变捕获**：存储原始对话作为不可变证据行，并通过规则（无LLM）提取类型化信号（日期、实体、金额、版本号、URL等）。
    - **Tier 1 验证提取**：LLM从证据窗口中提取候选事实，通过验证函数 $V$ 检查：源支持（词汇/模糊重叠）、硬锚（精确匹配）、主体存在、动作保留（否定/不确定性）。验证通过后写入规范事实，并保留来源链接。
  - **读路径（Read Path）**：零LLM调用。确定性查询规划器分析问题形状（如精确、推荐、推理、计数等），以规则权重激活多路由：
    - 向量检索 $R_v$（余弦相似度）
    - 关键词检索 $R_k$（BM25）
    - 时间检索 $R_t$（日期窗口过滤）
    - 实体/图检索 $R_g$（从知识图遍历）
    - 支持通道（原始剧情救援、推理支持）
    - 融合：加权互惠排名融合（RRF），公式：$\text{RRF}(f) = \sum_{c \in C} \frac{w_c}{k + \text{rank}_c(f)}$
    - 后融合过滤器：人物降级（非目标实体权重×0.05）、保留下限（前25名不跌落）。
    - 上下文打包：按实体分组、按排名顺序填充至预算大小。
  - **答案策略分离**：`ReadResult`包含`context`和`answer_instructions`（策略模式：strict / balanced / advanced），使得同一记忆基底可被不同回答模型消费。
  - **记忆对象模型**：证据（不可变）、信号（追加）、候选（临时）、信念（可修订）、链接（追加）。来源证据是权威基底，提取的事实是可查询的索引而非替代品。

## 6. 论文的主要结论与发现
- **性能**：Eywa在LoCoMo C1-C4上达90.19%（Sonnet 4.6），LongMemEval-S上88.2%检索充分性，BEAM上81.45%平均nugget分数。
- **模型分离验证**：同一记忆基底可服务前沿、预算和本地模型（Qwen3 32B读取Sonnet写记忆后准确率从69.09%提升至79.68%）。

- **拒绝感知评估**：指出官方LoCoMo F1可能因模型弃权而膨胀，建议报告拒绝率和拒绝感知指标。
- **零LLM读取**：读取路径无LLM调用，延迟约150-200ms（6k事实规模），适合产品化。

## 7. 优点
- **架构创新**：证据与信念分离、确定性读取、答案策略分离，使记忆系统可审计、可调试、可独立改进。
- **可审计性与透明性**：每问题工件公开（问题、黄金答案、检索上下文、模型回答、裁判结果），支持细粒度诊断。
- **模型无关性**：同一记忆基底可服务于不同回答模型（从本地3B到前沿API），降低部署切换成本。
- **成本效率**：零LLM读取设计使读取频繁的场景下推理成本大幅降低。
- **局部优先设计**：基于SQLite + LanceDB等自托管组件，无需外部依赖，适合本地部署。
- **拒绝感知指标**：暴露非对抗性弃权对准确率的影响，推动更严谨的评估。
