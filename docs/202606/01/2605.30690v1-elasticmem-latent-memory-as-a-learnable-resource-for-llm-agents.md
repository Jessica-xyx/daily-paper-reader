## 摘要
长期记忆对于LLM智能体在扩展交互中保持连贯推理、个性化响应和复用过往经验至关重要。然而，现有的记忆增强方法通常将记忆视为固定资源：文本空间方法将检索到的记忆拼接至上下文窗口，导致大量token开销和对噪声证据的敏感性，而潜在空间方法虽降低了文本成本，但仍依赖于刚性检索或固定容量的记忆接口。这在查询依赖的记忆效用与固定记忆分配之间造成了不匹配。我们提出ElasticMem，一种学习将记忆作为弹性潜在资源使用的记忆增强LLM框架。ElasticMem构建了一个包含检索键和内容缓存的离线潜在记忆库，从推理器的隐藏状态中自适应地检索记忆，通过习得的策略为每条检索到的记忆分配可变潜在预算，并将选中的潜在状态作为软记忆token注入生成过程。整个记忆使用过程通过组相对策略优化，利用下游任务奖励进行优化。我们在MemorySuite上评估ElasticMem，涵盖记忆密集型问答和具身智能体控制。在Qwen2.5-3B-Instruct和Qwen2.5-7B-Instruct骨干网络上，ElasticMem相较于最强基线，分别提升了加权平均QA准确率26.2%和24.6%，ALFWorld成功率提升66.3%和27.2%，同时实现了最低的ALFWorld token成本。消融和定性分析进一步表明，自适应检索和弹性预算分配帮助ElasticMem优先关注有用证据和可迁移计划，超越了刚性余弦相似度。我们的ElasticMem代码将在https://github.com/ulab-uiuc/ElasticMem发布。

## Abstract
Long-term memory is essential for LLM agents to reason coherently across extended interactions, personalize responses, and reuse past experience. However, existing memory-augmented methods typically treat memory as a fixed resource: text-space approaches concatenate retrieved memories into the context window, causing substantial token overhead and sensitivity to noisy evidence, while latent-space approaches reduce textual cost but still rely on rigid retrieval or fixed-capacity memory interfaces. This creates a mismatch between query-dependent memory utility and fixed memory allocation. We propose ElasticMem, a memory-augmented LLM framework that learns to use memory as an elastic latent resource. ElasticMem builds an offline latent memory bank with retrieval keys and content caches, retrieves memories adaptively from the reasoner's hidden state, assigns each retrieved memory a variable latent budget through a learned policy, and injects selected latent states as soft memory tokens for generation. The full memory-use process is optimized with downstream task rewards through group-relative policy optimization. We evaluate ElasticMem on MemorySuite, covering memory-intensive QA and embodied agent control. Across Qwen2.5-3B-Instruct and Qwen2.5-7B-Instruct backbones, ElasticMem improves weighted average QA accuracy by 26.2% and 24.6%, and improves ALFWorld success rate by 66.3% and 27.2%, respectively, over the strongest baselines, while achieving the lowest ALFWorld token cost. Ablations and qualitative analyses further show that adaptive retrieval and elastic budget allocation help ElasticMem prioritize useful evidence and transferable plans beyond rigid cosine similarity. Our code for ElasticMem will be released at https://github.com/ulab-uiuc/ElasticMem.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有记忆增强的LLM智能体将记忆视为“固定资源”，导致以下不匹配：
  - 文本空间方法（如MemoryBank、A-MEM）将检索到的记忆直接拼接到上下文窗口，造成大量token开销，且对噪声证据敏感。
  - 潜在空间方法（如AutoCompressor、MemGen）虽降低了文本成本，但仍依赖**刚性检索**（固定余弦相似度）和**固定容量记忆接口**（每条记忆分配相同数量的潜在token）。
  - 记忆的**效用是查询依赖和任务依赖的**，但检索和容量分配却是固定的，这导致有用的记忆可能被压缩、无用的记忆却占据空间。
- **整体含义**：提出一种新的视角——将记忆视为**可学习的弹性潜在资源**，让智能体根据下游任务效用**自适应地决定检索哪些记忆、以及每条记忆分配多少潜在容量**。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
ElasticMem通过四个阶段将记忆使用建模为一个可学习的资源分配问题：
1. **离线潜在记忆库构建**：预处理所有记忆块，存储为检索键（key）+内容缓存（content cache）。
2. **查询条件检索**：使用LoRA适配的推理器的隐状态（而非独立检索器）进行检索，检索方向由“检索控制令牌”引导。
3. **弹性潜在预算分配**：一个轻量级Transformer策略网络为每条检索到的记忆分配0~B_max个可变潜在token数量。
4. **软记忆令牌注入与生成**：将选中的缓存状态通过投影器映射到推理器的输入嵌入空间，作为软token参与生成。

### 关键技术细节
- **记忆库构建**：每个记忆块用冻结的基座LLM编码器编码，取最后隐状态作为检索键，最后N_c个隐状态作为内容缓存。
- **检索控制令牌**：推理器先生成一个二元或分类令牌a（表示是否检索、如何检索），然后基于(a, query)得到查询状态h_q。
- **检索机制**：h_q与所有记忆键做余弦相似度，选出Top-Z个记忆块。
- **预算策略网络**：为每个检索到的块构造特征（包含h_q、键、相似度、排序嵌入），通过Transformer自注意力得到上下文感知的表示，然后采样离散预算动作b_j ∈ {0,...,B_max}。b_j=0表示完全抑制该记忆。
- **潜在投影器**：两层MLP+GELU+层归一化，将选中的隐状态映射到推理器输入嵌入空间。
- **优化**：使用GRPO（Group-Relative Policy Optimization），奖励函数是任务特定的（如准确率、任务成功）。轨迹包括检索控制令牌、预算分配、最终生成。优化LoRA推理器、预算策略网络、潜在投影器，记忆库冻结。

### 算法流程（文字说明）
1. 离线阶段：遍历所有记忆块，用冻结基座编码器产生键和缓存，存入固定记忆库。
2. 在线训练阶段：对每个训练样本，采样G条轨迹：
   - 推理器采样检索控制令牌a → 计算查询状态h_q → 余弦检索Top-Z块。
   - 预算策略网络基于h_q和检索到的块特征，为每个块采样预算b_j。
   - 根据b_j从内容缓存中取最后b_j个隐状态，投影为软token。
   - 拼接待查询、控制令牌、软token、任务指令，推理器生成答案。
   - 根据奖励计算优势，用GRPO更新参数。

- **模型范围有限**：仅在Qwen2.5-3B/7B上验证，未在其他LLM系列（如Llama、Mistral）上测试，泛化性待确认。
