## 摘要
大语言模型（LLMs）在融入新知识时面临遗忘或代价高昂的重新训练问题。我们提出DYNA，一个轻量级框架，通过时间知识图谱增强冻结的LLM，其中事件作为节点，时间关系作为带时间戳的有向边。该图作为外部可更新的记忆。在查询时，DYNA通过随机游走和中心性度量检索相关节点，然后增强LLM的响应。在三个时间回忆任务上的评估表明，DYNA相比微调降低了约7%的灾难性遗忘，并在时间顺序上比标准RAG提升了约5%。更高的图聚类系数与更好的检索性能相关，表明图结构的重要性。贡献：（1）将情景记忆作为时间知识图谱，（2）无需重新训练的LLM增强方法，（3）图属性作为检索性能的预测因子。

## Abstract
Large Language Models (LLMs) struggle to incorporate new knowledge without forgetting or costly retraining. We propose DYNA, a lightweight framework that augments a frozen LLM with a temporal knowledge graph where events are nodes and temporal relations are directed, timestamped edges. The graph serves as an external, updatable memory. At query time, DYNA retrieves relevant nodes via random walks and centrality measures, then augments the LLM's response. Evaluated on three temporal recall tasks, DYNA reduces catastrophic forgetting by ~7% compared to fine-tuning and improves temporal ordering by ~5% over standard RAG. Higher graph clustering coefficients correlate with better retrieval, showing that graph structure matters. Contributions: (1) episodic memory as temporal KG, (2) retraining-free LLM augmentation, (3) graph properties as predictors of retrieval performance.

- **核心问题**：大型语言模型（LLMs）在完成初始训练后，难以高效地整合新知识。传统的微调方法会导致**灾难性遗忘**（新知识覆盖旧知识），而全面重训练则计算成本过高。同时，检索增强生成（RAG）方法虽然可以引入外部知识，但将记忆视为无结构文本，缺乏对事件间**时序关系**（如先后顺序、共现）的显式建模，因此难以处理需要时序推理的任务。
- **整体含义**：本文旨在为LLMs提供一种**持续学习**的轻量级外部记忆机制，使其能够在不修改模型参数的前提下，不断吸收新事件，同时保留原有能力，并增强对事件时间顺序的理解。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：受认知科学中“情景记忆”（Episodic Memory）启发，将事件及其时序关系存储在一个**动态时序知识图谱**中，作为冻结LLM的外部记忆模块。查询时通过图探索检索相关事件，增强LLM的响应。
- **关键技术细节**：
    - **时序知识图谱构建**：
        - 每个事件 `e_i` 作为节点 `v_i`，存储文本描述。
        - 节点间根据时间顺序添加三种**有向、带时间戳的边**：`before`（前序）、`after`（后序，由逆向推理得到）、`co-occurs`（共现，双向边）。
        - **边权重**采用指数衰减函数：`w(v_i, v_j) = exp(-λ * |t_i - t_j|)`，其中 `λ` 为衰减超参数，时间上越接近的事件权重越高。
    - **图更新与剪枝**：
        - 持续添加新节点和边。
        - **时间剪枝**：移除早于阈值 `T_max` 的节点。
        - **度剪枝**：移除连接度低的节点（贡献较小）。
    - **检索机制**：
        - **种子节点选择**：将查询中的关键词/实体与节点描述匹配，选出 top-k 最相似节点。
        - **随机游走**：从种子节点出发进行多次随机游走，沿权重边探索，捕获时序相邻事件。
        - **中心性评分**：为每个访问到的节点计算综合得分：
          `score(v) = α * degree(v) + β * closeness(v) + γ * visit_freq(v)`
          其中 degree 为节点度，closeness 为接近中心性，visit_freq 为随机游走中的访问频率。
    - **查询增强与响应生成**：将检索到的事件（带时间戳）与原始查询拼接，输入冻结的LLM，按标准RAG范式生成答案。

## 6. 主要结论与发现

- DYNA在**时序顺序准确率（TOA）**上达到 **71.42%**，优于最强基线DynaGen（68.14%），提升约 **3.28个百分点**。

- **敏感性分析**：DYNA在所有配置下均优于DynaGen，但优势大小受超参数影响（平均+1.15%到+2.99%）。最差情况发生在短游走长度（L=2）和严格剪枝（T_max=500）时，对TimeQA Hard优势仅+0.4个百分点。
- **失败场景**：隐式推理（TimeQA Hard）优势最小（+2.68%）；短序列（<5个事件）与RAG无显著差异；稀疏图（前300个事件）检索精度低于LDCL。

## 7. 优点

- **免重训练持续学习**：冻结LLM，通过外部知识图谱动态更新，从根本上避免了灾难性遗忘。
- **显式时序建模**：通过有向、带时间戳的边编码事件间关系，优于无结构的RAG。
- **轻量高效**：检索仅依赖简单图遍历（随机游走）和中心性计算，计算复杂度随图大小线性增长。
- **可解释性**：图结构属性（如聚类系数、节点度）可作为检索效果的预测指标，有助于系统调优。
- **模块化设计**：可无缝集成到任意LLM中，无需修改模型参数。

## 8. 不足与局限

- **检索精度低**：P@1仅21.28%，顶部结果正确率低，影响最终回答质量。

- **短序列和冷启动问题**：对于少于5个事件或前300个事件的稀疏图，DYNA并不优于RAG或LDCL，说明图积累期效果不佳。
- **速度限制**：在10,000个节点时检索时间达0.9秒，对实时应用（需<500ms）可能不可接受。
- **模型单一**：仅使用LLaMA 3 (8B) 评估，未在不同架构或规模上验证泛化性。
- **数据集简化**：所用基准虽为标准，但可能无法完全代表真实世界噪声事件流。
- **改进幅度有限**：相比强基线（DynaGen），整体提升约3%，属于增量改进而非突破性进展。
