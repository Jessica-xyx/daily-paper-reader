## 摘要
检索增强生成（RAG）使代理能够访问累积的知识，但缺乏时间模型。当事实发生变化（例如函数重命名或API重构）时，RAG会检索到过时和当前值，且嵌入相似度几乎相同。代理要么弃权，要么提供已被取代的事实。我们表明这是一个结构性问题：在一个校准数据集上，余弦相似度区分矛盾事实与重复事实的AUROC为0.59（接近随机），因为矛盾事实与原始事实的嵌入相似度通常高于改写的重复事实。我们提出了MemStrata，一种维护时间有效性的检索记忆。它像RAG一样存储事实，保留静态召回，但当事实值被矛盾时，一种确定性的（主体、关系、客体）取代规则会在双时账本中淘汰过时值——无需相似度阈值，无需调用LLM。在本地使用7B模型运行的六个基准测试中，MemStrata在静态知识上与RAG持平，在演化知识上达到0.95-1.00的准确率（RAG为0.20-0.47）。核心结果是无过时事实错误率：当需要回答时，RAG在15-40%的情况下提供被取代的值；MemStrata将此降至约0%，这是RAG无法避免的失败类别。MemStrata在检索延迟（约2.1秒）方面优于LLM重排序基线（约16-18秒）。我们公开了测试框架、数据集以及一种用于知识演化下记忆的无标记评估协议。

## Abstract
Retrieval-augmented generation (RAG) gives agents access to accumulated knowledge, but has no model of time. When a fact changes (e.g., a function is renamed or API restructured), RAG retrieves both the stale and current value with near-identical embedding similarity. The agent then either abstains or serves the superseded fact. We show this is a structural problem: on a calibrated dataset, cosine similarity distinguishes a contradicted fact from a duplicated one with AUROC 0.59 (near chance), as contradictions are often more embedding-similar to the original than rephrased duplicates.   We present MemStrata, a retrieval memory maintaining temporal validity. It stores facts like RAG, preserving static recall, but when a fact's value is contradicted, a deterministic (subject, relation, object) supersession rule retires the stale value in a bi-temporal ledger - with no similarity threshold and no LLM call. Across six benchmarks run locally with a 7B model, MemStrata ties RAG on static knowledge and reaches 0.95-1.00 accuracy on evolving knowledge (where RAG reaches 0.20-0.47). The central result is the stale-fact-error rate: when required to answer, RAG serves superseded values 15-40% of the time; MemStrata drives this to ~0%, a failure class RAG cannot avoid. MemStrata achieves this at retrieval latency (~2.1s) versus ~16-18s for LLM-reranking baselines. We release the harness, datasets, and a marker-free evaluation protocol for memory under knowledge evolution.

- **问题**：检索增强生成（RAG）作为LLM代理的主流记忆机制，在存储和检索知识时缺乏对**时间维度**的建模。当知识发生演变（如函数重命名、配置值变更、API升级）时，旧事实和新事实的嵌入表示几乎相同，RAG无法区分哪个是当前有效的，导致代理要么拒绝回答，要么以较高置信度提供已过时的事实。
- **关键发现**：过时事实错误不是参数调优或相似度阈值可以解决的，而是一个**结构性失败**——因为在嵌入空间中，矛盾事实（变化了一个值）比真正的改写重复事实与原始事实的余弦相似度更高（平均值0.812 vs 0.800），分类AUROC仅0.59（接近随机），因此任何基于相似度的规则都无法安全地自动更新。
- **整体含义**：该论文提出，对于需要持续跟踪演变知识的AI代理（如代码助手、研究助理），记忆系统必须内建**时间有效性**机制，否则将始终存在15-40%的过时事实错误，这是RAG无法规避的。

## 2. 论文提出的方法论：核心思想与关键技术细节

### 核心思想
MemStrata 是一种位于代理与LLM之间的本地记忆层，通过**确定性（主体, 关系, 客体）取代规则**维护事实的当前有效性，无需相似度阈值或LLM调用。

### 关键技术细节

- **写路径**：
  - 精确重复短路：归一化文本哈希，零成本去掉逐字重复。
  - 确定性断言路径：若输入可提取为（主体, 关系, 客体）三元组，则归一化（主体, 关系）作为键，检查是否存在活跃断言。
    - 若存在不同对象 → 新断言取代旧断言：关闭旧行的有效区间（设valid_to、链接superseded_by），开启新行。
    - 相同对象 → 重复（加强）。
    - 无先前键 → 新事实（存储）。
    - **完全不依赖余弦相似度或LLM判断**。
  - 文本门控回退：非三元组散文通过“惊喜门”分类（相似度+LLM判断），但仅用于保留非矛盾的近似重复，算法不在此路径处理矛盾（已在上层排除）。

- **双时态账本**：事实被淘汰而非删除。存储valid_from、valid_to、superseded_by，使过时事实仍可用于未来的“按时间点查询”。

- **“保留后取代”设计**：早期变体过分压缩导致静态召回下降，最终设计像RAG一样保留所有非矛盾事实，仅在矛盾时替换，因此静态知识无损。

- **读路径**：嵌入查询 → 用余弦检索活跃事实 → 应用确定性过滤（丢弃已经淘汰的行） → 将存活事实的原始源句子打包（而非压缩后的三元组）。无LLM参与，延迟约2.1秒（嵌入耗时）。

- **无标记基准协议**：演化基准中，旧版和新版事实除了变化的值外文本完全相同，无任何“过期”“旧版”等标记，确保唯一可供利用的信号是摄入顺序，防止基线利用文本标记作弊。

## 6. 论文的主要结论与发现

1. **结构性的不可能结果**：余弦相似度无法区分矛盾事实与重复事实（AUROC 0.59），矛盾事实甚至比重复事实嵌入更相似。任何基于相似度的安全自动更新规则都不可行。
2. **MemStrata在静态知识上持平RAG，在演化知识上显著超越**：静态准确率temporal_v6 vs RAG（0.82 vs 0.86, 0.30 vs 0.30）；演化准确率0.95-1.00 vs RAG的0.20-0.47，提升2-5倍。
3. **过时事实错误率降至~0%**：允许弃权时，RAG的过时错误被弃权隐藏；强制回答时，RAG在15-40%的情况下给出过时值，MemStrata始终接近0%。这是RAG因结构设计无法避免的失败类别。
4. **延迟优势**：MemStrata读取延迟约2.1秒（嵌入耗时），与naive RAG相同；而LLM重排序/验证基线延迟16-18秒（8倍）。
5. **无标记基准协议的重要性**：移除文本标记导致基线准确率下降最多14-18个点，证实了该协议的必要性，建议所有演化知识记忆评估采用此协议。

  - **“保留后取代”设计**：在保留全部静态召回能力的同时，仅淘汰矛盾事实，实现零代价的时间有效性。
  - **读取路径无LLM**：保证了低延迟和确定性。

## 8. 不足与局限

- **演化的基准仅限结构化单值模板**：所有演变基准为“状态A → 状态B”的单元素变化；对于更复杂的自然语言矛盾（多值句子、模糊表述），三元组提取准确率从97%骤降至44%，因此该部分实验被隔离，并未作为正式结果。即**提取质量是瓶颈**，而非取代机制本身。
- **摄入顺序代理时间**：当前使用顺序（先A后B）表示时间，未来需要引入显式时间戳和“按时间点查询”模式。
- **单7B模型、小规模基准**：所有结论基于Qwen2.5-Coder-7B和数十个样本的基准，更大模型或更大规模数据上的表现未验证。作者坦诚这是“隔离机制”的测试，而非排行榜，后续需要扩展到真实纵向数据。
- **单模型噪音**：3B判断器偶尔对过时值判定为正确，但不影响时间层自身（已将过时事实从上下文中移除）。
- **未涉及训练**：方法完全基于规则和prompt，没有学习组件，可能难以适应需要隐式时间推理的场景。
