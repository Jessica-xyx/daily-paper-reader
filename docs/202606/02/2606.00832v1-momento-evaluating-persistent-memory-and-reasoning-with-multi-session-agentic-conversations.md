## 摘要
近期智能体AI的进步使得智能体能够通过工具使用、推理和多步规划来完成复杂任务。然而，现有基准测试仅评估单会话中的智能体，忽略了智能体必须整合的过去行为、明示偏好和先前决策，以实现个性化用户目标。我们提出了Momento，一个用于多会话服务环境中持久智能体任务完成的基准测试，要求智能体在解决跨会话的时间依赖和演变用户目标的同时，执行具有后果的工具中介动作。实验结果表明，当前智能体的主要失败原因是对用户状态的错误估计，将会话历史视为当前上下文的可靠代理，而非需要重新验证的过时信息，凸显了当前智能体能力与现实长期人机交互之间的巨大差距。

## Abstract
Recent advances in agentic AI have enabled agents to complete complex tasks through tool use, reasoning, and multi-step planning. Yet existing benchmarks evaluate agents within a single session, ignoring past actions, stated preferences, and prior decisions that agents must integrate to fulfill personalized user goals. We introduce Momento, a benchmark for persistent agentic task completion in multi-session service environments, requiring agents to take consequential, tool-mediated actions while resolving temporal dependencies and evolving user goals across sessions. Experimental results reveal that current agents fail primarily through misestimation of user state, treating prior session history as a reliable proxy for current context rather than stale information requiring re-validation, highlighting a substantial gap between current agent capabilities and realistic long-horizon human-agent interaction.

## 6. 主要结论与发现

- **性能排名**：Gemma 4-31B-IT在所有模型中获得最高的PASS@3和PASS^3，表现最优；而MoE模型（Qwen 3.6-35B-A3B、Gemma 4-26B-A4B-IT）明显落后于同参数量级稠密模型。
- **失败模式**：主要失败原因是**工具召回错误**（Tool Recall errors），而非不知道工具存在。智能体倾向于基于历史会话形成对用户当前状态的错误假设，从而省略了验证或检索工具（如`get_membership_benefits`、`get_membership_status`），导致状态误判。
- **关键发现**：当前智能体将历史会话视为当前上下文的可靠代理，而非需要重新验证的过时信息。这种“虚假的连续性感知”在多会话场景中尤为突出，是阻碍长期交互的瓶颈。

## 7. 优点

- **新颖性**：首次提出系统性地评估跨会话、持续的智能体任务完成基准，填补了现有单会话基准的空白。
- **生态设计**：包含26个真实服务工具（预订、订单、会员等），覆盖多种实用场景；任务设计包含偏好演变、隐式引用、时间依赖等复杂现象，更接近真实部署。
- **方法论严谨**：明确区分用户代理和助理代理，设计了混合检索（SQL+语义）和上下文压缩机制，并提供了详细的提示模板和超参数。
- **开源贡献**：公开数据集和代码，促进后续研究。

## 8. 不足与局限

- **模拟器局限**：依赖LLM角色扮演生成用户行为，可能无法完全模拟真实人类的多变性和非理性（作者在“Limitations”中已承认），存在从模拟到现实的差距（Sim2Real gap）。
- **场景单一**：仅基于餐厅服务平台，缺乏其他领域（如旅行、医疗、金融）的验证，泛化性待考。
- **模型覆盖**：仅评估了开源VLM和GPT-5.4-MINI，未包含更多闭源模型（如GPT-4o、Claude等）或更小的开源模型，可能错过更全面的性能洞察。
- **实验量较小**：161个任务实例虽然经过人工验证，但数量相对较少；重复次数K=3可能不足以完全刻画模型稳定性（虽然报告中PASS@K和PASS^K差异较大，说明存在不稳定性）。
- **未进行消融实验**：没有对记忆模块的不同设计（如无长时检索、不同压缩比率）进行消融，难以量化各组件贡献。
- **伦理考量**：作者指出所有交互为合成数据，不涉及真实用户数据，但模拟数据仍可能继承底层模型的偏见与幻觉风险。
