## 摘要
在边缘服务器上部署轻量级大语言模型（LLM）代理可以降低延迟，并将代理服务靠近用户，但资源受限的边缘模型在处理需要持久记忆、子目标跟踪和反思的长时程任务时往往力不从心。部署后对边缘模型进行微调成本高昂且难以跨异构节点扩展，而纯本地记忆则使代理拥有孤立经验且提示上下文不断增长。我们提出\textsc{CoMIC}，一种无需参数更新的云-端协作记忆与洞察循环框架。\textsc{CoMIC}遵循\textit{集中反思、分散执行}的设计：边缘代理使用面向子目标的层次化记忆和选择性扩展相关历史记录进行本地执行，而云端的LLM评审器异步评估完成的轨迹，过滤可复用经验，并通过语义子目标标识符聚合跨代理的指导。在涵盖符号规划和文本交互的五项长时程代理任务中，\textsc{CoMIC}提升了弱边缘代理的进度率和行动基础，并在不更新模型参数的情况下实现了任务相关的成功率增益。

## Abstract
Deploying lightweight Large Language Model (LLM) agents on edge servers can reduce latency and move agentic services closer to users, but resource-constrained edge models often struggle with long-horizon tasks that require persistent memory, subgoal tracking, and reflection. Fine-tuning edge models after deployment is costly and difficult to scale across heterogeneous nodes, while purely local memory leaves agents with isolated experience and growing prompt context. We propose \textsc{CoMIC}, a parameter-update-free cloud-edge framework for Collaborative Memory and Insights Circulation. \textsc{CoMIC} follows a \textit{Centralized Reflection, Decentralized Execution} design: edge agents execute locally using subgoal-oriented hierarchical memory and selective re-expansion of relevant histories, while a cloud-side LLM critic asynchronously evaluates completed trajectories, filters reusable experience, and aggregates cross-agent guidance keyed by semantic subgoal identifiers. Across five long-horizon agent tasks spanning symbolic planning and text interaction, \textsc{CoMIC} improves progress rate and action grounding for weak edge agents and yields task-dependent success-rate gains without updating model parameters.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：资源受限的边缘 LLM 代理在执行长时程任务（long-horizon tasks）时面临三重挑战：
  - 持久记忆不足：边缘模型参数量小（如 Mistral-7B），上下文窗口有限，难以维持整个任务历史。
  - 子目标追踪困难：复杂任务需要分解为多个子目标，但弱模型缺乏结构化推理能力。
  - 缺乏跨代理经验共享：纯本地执行导致每个边缘代理只有孤立经验，无法从其他代理的成功/失败中学习。
- **现有方案缺陷**：
  - 传统记忆方法（如拼接全部历史到提示）导致上下文爆炸和高 token 消耗。
  - 部署后微调成本高，难以扩展到异构边缘节点。
  - 纯本地记忆无法利用跨代理知识。
- **整体含义**：本文提出一种无需参数更新的云-端协作框架 **CoMIC**，通过将计算密集的跨轨迹反思和长时记忆管理卸载到云端，边缘仅维护轻量级层次记忆，实现“集中反思、分散执行”的范式，从而在不更新模型参数的前提下显著提升弱边缘代理的长时任务能力。

## 2. 方法论：核心思想、关键技术细节、公式与算法

### 核心思想
- **集中反思、分散执行**：边缘代理负责在线子目标驱动执行，云端 LLM 批评者异步评估已完成轨迹，筛选高质量经验，并通过语义子目标标识符聚合跨代理指导，返回给边缘。
- **无需参数更新**：所有增强仅通过文本级记忆交互实现，不改变模型权重。

### 关键技术细节

#### 2.1 边缘代理框架
- **子目标驱动执行**：全局任务 \( g \) 被分解为有序的子目标轨迹 \( \tau_i^g = \{ \hat{\tau}_i^1, \hat{\tau}_i^2, ..., \hat{\tau}_i^{T_g} \} \)，每个子目标轨迹 \( \hat{\tau}_i^t = (g, g_i^t, o_i^t, a_i^t, r_i^t) \)。
- **层次化记忆与选择性重扩**：
  - 活跃子目标保留详细动作-观察对。
  - 完成的前序子目标压缩为摘要（符号 PDDL 域保留谓词级状态）。
  - 历史上下文 \( H_i^t(\mathcal{I}) = \{ \text{Summary}(\hat{\tau}_i^t) \}_{t \notin \mathcal{I}} \cup \{ \hat{\tau}_i^t \}_{t \in \mathcal{I}} \)，其中 \(\mathcal{I}\) 是当前决策相关的前序子目标索引集。
- **提示组装函数**：
  \[
  P_i^{t+1} = \mathcal{A}(g, g_i^{t+1}, o_i^{t+1}, H_i^{t+1}(\mathcal{I}_{t+1}), \mathcal{G}(u_{t+1}))
  \]
  其中 \(\mathcal{G}(u_{t+1})\) 是从云端全局知识库选出的针对当前子任务标识符 \(u_{t+1}\) 的指导。

#### 2.2 云端 LLM 批评者框架
- **单轨迹规范化与评估**：
  - 轨迹序列化： \( x_i^t = \text{Serialize}(\hat{\tau}_i^t, \tilde{m}_i^t) \)，其中 \(\tilde{m}_i^t\) 为规范化特征集。
  - 批评者输出包含摘要、洞见、建议和自报告置信度 \(s_{\text{self}}\)。
- **基于门的经验提炼**：
  - 准入分数： \( s_{\text{adm}}(x_i^t) = \text{clip}_{[0,1]}( s_{\text{self}} - \lambda_{\text{ctx}} I_{\text{retry}} - \lambda_{\text{short}} I_{\text{short}} ) \)，其中 \(I_{\text{retry}}\) 和 \(I_{\text{short}}\) 分别表示是否重试或轨迹过短。
  - 经验库准入： \( M_{\text{exp}}^+ = \{ m \in M_{\text{exp}} \mid s_{\text{adm}}(m) \geq \gamma_{\text{kb}} \} \)。
- **记忆组织**：
  - 短期记忆 \( M_{\text{STM}} = T_{\text{STM}} \cup E_{\text{STM}} \)。
  - 云端知识库：经验记忆 \( M_{\text{exp}} \) 和全局指导 \( M = M_{\text{manual}} \cup M_{\text{global}} \)。

#### 2.3 云-端协作机制
- **跨边缘聚合**：按语义子目标标识符 \(u\) 分组经验，定期调用聚合函数 \( \bar{\mathcal{G}}(u) = f_{\text{agg}}(G(u)) \)，其中 \( G(u) = \{ m \in M_{\text{exp}}^+ \mid \kappa(m) = u \} \)。
- **协作上下文组装**：边缘代理仅在必要时从云端拉取全局指导，并组装到提示中；异步上传轨迹不阻塞执行。

### 算法流程（文字说明）
- **云端准入路径**：边缘完成子目标后异步上传轨迹 → 云端序列化并评估 → 计算准入分数 → 若通过则加入经验库。
- **云端聚合路径**：定期按标识符分组经验 → 聚合生成全局指导（摘要、洞见、建议、可信度得分）→ 提交到 \(M_{\text{global}}\)。
- **边缘运行时路径**：当需要指导时，从云端选择匹配的指导 → 接地到当前观察 → 组装提示并执行（以最新观察为准）。
