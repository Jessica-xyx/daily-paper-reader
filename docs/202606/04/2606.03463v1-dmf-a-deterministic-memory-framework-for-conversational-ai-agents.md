## 摘要
对话式AI智能体需要能够在长时间交互中兼具可扩展性和语义一致性的记忆系统。现有方法主要依赖大语言模型（LLM）在写入时进行摘要生成，这引入了非确定性、不断攀升的令牌成本以及剪枝决策的不透明性。我们提出确定性记忆框架（DMF），这是一种以CPU为核心的方法，通过基于经典NLP分析、向量几何和数学评分的全确定性流程替代生成式记忆压缩。DMF为每次对话交互分配一个生存分数Ω，该分数由确定性内容信号、对话线索和结构化来源通过逻辑投影组合计算得出。一个基于交互次数的衰减律记为Ω_eff(Δn)，用于控制随着新轮次出现而演变的相关性，其中Δn是较新交互的数量而非实际时间，从而保持完全确定性。我们给出了DMF的数学公式、结构化召回流程、剪枝决策过程以及评估协议。实验在基于LoCoMo和LongMemEval数据集专门构建的基准上进行。我们将DMF与AI智能体常用的记忆层Mem0进行比较。DMF在实现相当准确性的同时，在准备记忆上下文时使用零令牌，在整个对话过程中使用的令牌数减少了5倍到242倍。这些结果表明，可以从记忆管理循环中消除LLM调用，将令牌成本降至几乎为零，并为对话式AI智能体实现确定性记忆系统。

## Abstract
Conversational AI agents require memory systems that are both scalable and semantically coherent across long interaction horizons. Existing approaches rely predominantly on large language model (LLM)-based summarisation at write time, which introduces non-determinism, escalating token costs, and opacity in pruning decisions. We present the Deterministic Memory Framework (DMF), a CPU-first approach that replaces generative memory compression with a fully deterministic pipeline grounded in classical NLP analysis, vector geometry, and mathematical scoring. DMF assigns each conversational interaction a Survival Score $Ω$ computed from deterministic content signals, conversational cues, and structured provenance, combined through a logistic projection. An interaction-count decay law, denoted as $Ω_{\mathrm{eff}}(Δn)$, governs how relevance evolves as new turns arrive, where $Δn$ is the number of newer interactions rather than wall-clock time, preserving full determinism. We present the mathematical formulation of DMF, its structured recall pipeline, the pruning decision procedure, and the evaluation protocol. Experiments are conducted on a purpose-built benchmark using the LoCoMo and LongMemEval datasets. We compare DMF against Mem0, a popular memory layer for AI agents. DMF achieves comparable accuracy while using zero tokens to prepare the memory context and 5x to 242x fewer tokens over the entire conversation. These results show that it is possible to eliminate LLM calls from the memory-management loop, reducing token costs to nearly zero and enabling deterministic memory systems for conversational AI agents.

### 2. 论文提出的方法论

**核心思想**：不依赖LLM进行记忆压缩，而是利用确定性信号计算每个交互的**生存分数Ω**，并基于交互计数（而非物理时间）进行衰减，通过数学规则决定记忆保留与剪枝。

**关键技术细节**（文字说明）：
- **NLP特征提取**：对每条对话文本，使用spaCy提取信息密度ID（内容性词占比）、实体数量E（经截止归一化）、VADER情感强度|S|；通过滑动窗口嵌入中心计算语义发散度D（余弦距离，高发散为惩罚项）。
- **对话信号层**：基于规则适配器提取操作信号（约束、偏好、当前状态、纠正、替换等）及主题标识对，用于后续分数计算。
- **生存分数Ω**：
  - 内容通道：`z_content = α·ID + β·|S| + γ·E_norm + δ·D`（δ<0为惩罚）。
  - 操作通道：`z_op = λ_op·(η_con·1_constraint + ...)`（对约束、纠正等信号赋予正权重）。
  - 来源通道：`z_prov = κ_uc·1_user_correction + ...`（来源元数据）。
  - 总激活：`z_total = z_content + z_op + z_prov`。
  - 通过逻辑斯蒂函数映射到(0,1)：`Ω = σ(z_total - x_0)`。
- **时间衰减**：基于**交互计数Δn**（较新交互的数量），而非墙钟时间。
  - 基础指数衰减：`Ω_eff(0) = Ω`，`Ω_eff(Δn) = Ω·exp(-λ·Δn)`。
  - 引入**分数依赖惯性**：`μ(Ω)=1-η·Ω`，有效衰减率`λ_eff = λ·(1-ηΩ)`，高分数条目衰减更慢。
  - 完整公式：`Ω_eff(Δn) = Ω·exp(-λ·(1-ηΩ)·Δn)`。
- **剪枝机制**：
  - **周期性硬杀**：当`Ω_eff < Ω_kill`（默认0.05）时直接移除并归档。
  - **预算压力剪枝**：当活跃队列token超过预算（默认4096）时，仅从`Ω_eff ≤ τ_H`（非健康）条目中，根据综合剪枝分数`S_prune = Ω_eff + B_ret - P_sup`选出最低者移除，其中`B_ret`为操作性信号（约束、偏好等）提供的保留奖金，`P_sup`为主题被更新的惩罚。
- **长期记忆（LTM）归档**：被驱逐条目保存为原始文本记录（RawLTMRecord），不含分数等衍生字段；可选生成确定性卡片（MemoryCard）作辅助检索，但始终回退至原始文本。
- **召回管线**：查询嵌入→查询帧解析→多通道候选（语义+词法+卡片语义+卡片符号）→答案可回答性重排序（结合多种特征和惩罚）→证据组装→上下文渲染（召回内容置于活跃对话之前）。

### 3. 实验设计

**数据集与场景**：
- **LoCoMo**：多会话长对话记忆评测数据集，含1540个问题，分为四类：单跳、多跳、时间推理、开放域。
- **LongMemEval-10**：从LongMemEval随机抽取每类别10个问题，共60个，六类别：知识更新、多会话、单会话助手、单会话偏好、单会话用户、时间推理。

**基准方法**：对比**Mem0**（流行记忆层，基于LLM事实提取和语义记忆）。

**评估指标**：
- **Avg. Judge Score**：LLM-as-judge（GPT-5-mini）判断回答是否正确（二元）。
- **Exact Match**：标准化精确匹配。
- **Token F1**：答案与标准答案的token级F1。
- **Token消耗**：各类token数量（端到端输入、记忆内部prompt/completion等）。

**对比结果**：
- **LoCoMo**：DMF总体准确率0.775（Mem0 0.688），+12%；Exact Match 0.104 vs 0.075；Token F1 0.372 vs 0.314。时间推理类别DMF准确率0.707，Mem0仅0.153（约4.6倍）。DMF总token消耗（Rec. E2E）约1060万，Mem0约5665万，减少5.3倍。
- **LongMemEval-10**：DMF总体准确率0.717，Mem0为0.767（-7%）。但在“单会话助手”类别DMF 0.900 vs 0.600（+50%）。DMF总token约57万，Mem0约1.39亿，减少约242倍。

### 6. 论文的主要结论与发现

1. **确定性可行**：DMF在不需要任何LLM调用记忆管理的前提下，取得了与Mem0相当或更优的准确率（LoCoMo上超越，LongMemEval-10略低但差距小）。
2. **极致Token效率**：记忆管理循环使用零token，总token消耗比Mem0减少5～242倍，极大降低了端到端成本。
3. **时间推理优势**：DMF因保留原始时间戳和交互顺序，时间推理准确率远优于Mem0（0.707 vs 0.153）。
4. **透明与可审计**：所有分数、衰减、剪枝均基于数学公式，决策可复现、可解释。
5. **可扩展性**：交互计数衰减使记忆状态仅依赖于消息序列，与实时时钟无关，便于回放和调试。

### 8. 不足与局限

**实验覆盖层面的不足**：

- **缺乏消融实验**：未验证社交地板、惯性项、操作信号权重、卡片投影等各组件贡献。
- **数据集规模有限**：LongMemEval-10样本太少，结果可能受随机性影响。
- **未测试多语言**：当前仅支持英语，POS标签集和关键词列表需为其他语言定制。
- **未测试极端长对话**：LoCoMo虽然长，但未专门评估数十万token场景。

**方法本身的局限**：
- **参数固定**：评分权重、衰减率、阈值等均为手动设定，未实现自适应优化；不同任务可能需要重新调参。
- **嵌入模型固定**：未进行不同嵌入模型（如bi-encoder vs cross-encoder）的系统消融，召回质量对嵌入选择的敏感性未知。
- **静态Token预算**：预算为全局固定值，无法根据对话复杂度或任务需求动态调整。
- **复杂语义处理能力有限**：确定性规则无法处理需要深层推理的对话（如反事实、幽默、多步意图消歧），可能仍需LLM辅助。
- **英文独占**：英文规则适配器（如“I prefer”“currently”等短语匹配）无法直接迁移至其他语言。

**偏差与风险**：
- **准确性上限**：在LongMemEval-10上低于Mem0（相差5个百分点），提示部分需要LLM级语义理解的场景下DMF可能不足。
- **可能过度剪枝**：社交地板仅提升分数至0.25，长时间对话中社交性话语仍可能快速衰减，导致上下文连贯性损失。
- **卡片投影保守**：仅捕获可确定性规范化的模式，复杂关系（如因果、条件）无法表示，限制了检索可能性。
