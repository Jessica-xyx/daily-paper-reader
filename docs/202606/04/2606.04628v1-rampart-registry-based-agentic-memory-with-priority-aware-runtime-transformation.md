## 摘要
RAMPART是一种编译时记忆模型和纯内存块注册表，适用于基于LLM的智能体。上下文组装是一种可编程的运行时操作，其中内容按照显式的排序、包含和驱逐策略从结构化注册表中编译而成。五个可组合的原语（提升、门控、写入、驱逐、回滚）在编译前对命名的可寻址块进行操作，且无需提示令牌成本。来源标签和不可驱逐的作者标志实现了具有块级所有权的权限记忆模型。使用Qwen3-8B Q4进行的受控探针实验表明，编译时放置以及块与任务查询之间的结构关系会影响任务成功率，当任务跟随注册表时，成功率大约在第七个块位置急剧下降，而当任务先于注册表时，则大约在第十二个块位置。将关键块与内容相邻的邻居分组，并将该组作为一个单元提升，在单块放置失败的位置上，能将任务成功率提升数十个百分点。在Qwen2.5-7B、Llama-3.1-8B、Mistral-7B-v0.3和Qwen3-14B上的跨模型复制显示，内容启动效应在不同系列的模型中出现在相同的绝对位置，其幅度随模型强度变化。在最难的注册表大小下，块分组使Mistral的平均通过率提高了大约五倍，并且在中等注册表区域，采用干预的较小模型可以超越未采用干预的较大模型。相关性门控将提示成本降低了67.8%，同时恢复了提升条件下成功率的83%。模式驱逐导致0%的调用（而模式存在时为100%），基于策略的方法在结构上无法保证这一特性。共享注册表协调将智能体间通信减少为一次方法调用，协调令牌成本为零。

## Abstract
RAMPART is a compile-time memory model and pure in-RAM block registry for LLM-based agents. Context assembly is a programmable runtime operation where content is compiled from a structured registry under explicit policy for ordering, inclusion, and eviction. Five composable primitives (promote, gate, write, evict, rollback) act on named addressable blocks before compilation at zero prompt-token cost. Provenance tags and non-evictable authorship flags implement a permissioned memory model with block-level ownership. Controlled probes with Qwen3-8B Q4 show that compile-time placement and the structural relationship between blocks and the task query affect task success, with the cliff falling at roughly the seventh block position when the task follows the registry and the twelfth when it precedes. Grouping the critical block with content-adjacent neighbours and promoting the group as a unit lifts task success by tens of percentage points at positions where single-block placement fails. Cross-model replication on Qwen2.5-7B, Llama-3.1-8B, Mistral-7B-v0.3, and Qwen3-14B shows the content-priming effect appears at the same absolute positions across families, with magnitude varying with model strength. Block grouping raises Mistral's mean pass rate roughly fivefold at the hardest registry size, and a smaller model with the intervention can outperform a larger model without it in the mid-registry zone. Relevance gating reduces prompt cost by 67.8\% while recovering 83% of the promoted-condition success rate. Schema eviction produces 0% invocations against 100% with the schema present, a property policy-based approaches cannot guarantee by construction. Shared-registry coordination reduces inter-agent communication to a method call at zero coordination token cost.

  - **上下文位置敏感性**：已有研究（Liu et al., Du et al.）表明，LLM的输入位置存在U型注意力分布，且上下文长度增加本身会降低性能，即使无关标记被屏蔽。静态提示无法根据任务动态调整块的位置和包含关系。

  - **无结构化内存管理**：现有系统没有将上下文组装视为可编程的运行时操作；要么是全量拼接，要么是相似性检索，缺乏对块顺序、选择性包含、驱逐和权限控制的显式支持。

- **核心问题**：如何为LLM智能体设计一种编译时的、纯内存的、可编程的上下文组装机制，使得上下文内容和排列能在推理前精确控制，同时最小化提示令牌开销、实现零I/O、支持代理自写和权限化内存，并降低多智能体协调成本。

- **整体含义**：提出RAMPART——一个基于注册表的编译时内存模型，将上下文组装从静态拼接转变为带策略的编译步骤，赋予系统操作者对块位置、分组、门控、驱逐和相互引用的直接控制。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- **编译时内存模型**：上下文组装是一个可编程的运行时步骤，内容从结构化注册表中按照显式的排序、包含和驱逐策略编译而成。
- **纯内存注册表**：无数据库、无磁盘I/O（冷启动后），延迟边界为方法调用。每个注册表实例维护一个有序字典（`OrderedDict[UUID → InstructionBlock]`）。
- **可组合的原语**：五个零提示令牌成本的操作——`promote`（提升）、`gate`（门控）、`write`（写入）、`evict`（驱逐）、`rollback`（回滚），作用于命名的可寻址块。
- **权限化内存模型**：通过来源标签（seed/agent/orchestrator）和不可驱逐的作者标志，实现块级所有权，类似于操作系统的能力访问控制。

### 2.2 关键技术细节
- **指令块（Instruction Block, IB）**：原子单元，包含id、content、priority、source、author、removable、run、created、accesses、embedding（384维稠密向量，冷启动时由all-MiniLM-L6-v2计算）。
- **注册表与种子库**：
  - 工作注册表（Working Registry）：从种子库（Seed Library）选择性加载，只包含当前任务需要的块。
  - 种子库：一个独立的纯内存存储，按命名空间标识符索引，应用启动时填充。
- **种子文件格式**：YAML frontmatter + Markdown正文，每个块为命名节。现有SKILL.md/CLAUDE.md可直接导入（无frontmatter视为单匿名块）。
- **上下文编译（compile()）**：四阶段过程：
  1. 相关性门控（可选余弦相似度过滤）
  2. 按优先级有序遍历注册表
  3. 令牌预算截断（并记录被排除的块名）
  4. 将被包含块的访问计数递增
  - 返回编译后的提示字符串、编译顺序名称列表、被截断的块名、总令牌数。
  - 支持“干运行”（dry-run），不修改状态。
- **排序与提升**：唯一通过注册表顺序控制位置，`promote`是O(k)重链接操作。基于注意力U型曲线，提升到位置0或指定k。
- **写入与来源标记**：`write_block()`追加或插入块，自动标记调用方UUID。`removable`标志控制其他实例是否可删除。
- **驱逐**：用户可配置的评分函数（默认加权priority和inverse access count），不可驱逐块和他人作者块自动排除。驱逐是显式操作，不随编译触发。
- **共享注册表协调**：一个注册表实例持有对另一个的引用，直接调用其方法（如写入、提升），零令牌协调开销。通过写时复制（copy-on-write fork）支持多进程分布。
- **ASLR风格地址随机化**：每个会话加载时块获得新的UUID4作为运行时ID，防止通过语义名称的间接提示注入攻击。

## 6. 论文的主要结论与发现

1. **位置敏感性**：
   - 任务查询后跟随注册表时，关键块在位置0–6附近保持高成功率，之后急剧下降；任务查询先于注册表时，可用区域延伸到位置0–10左右。这个“悬崖”位置大致常数，不随注册表大小比例变化。
   - 内容相邻（如输出格式块）会产生显著的启动效应：关键块在位置23–24附近出现残留信号。

2. **块分组干预**：
   - 将关键块与其两个输出格式邻居作为三块组一起提升，并且在组内将关键块放在**组后**（即最后），可以在单块放置失败的位置上恢复成功率数十个百分点。
   - 控制实验移除输出格式块后残留信号减少但未消失，说明该启动效应主要由内容相邻驱动，但可能还有其他因素。

3. **跨模型普遍性**：
   - 内容启动效应在5个模型家族中出现在相同的绝对位置（23–26左右），但幅度随模型强度变化：Qwen2.5-7B有宽而高的凸起（峰值90%），Llama-3.1-8B较窄，Mistral-7B几乎无凸起（崩溃到底），Qwen3-14B则饱和（天花板效应）。
   - 簇干预对弱模型效果显著：Mistral的每个位置平均通过率从4%提升至20%（5倍）；更小模型（Qwen3-8B+分组）在中注册表区域（位置10–19）超越更大模型（Qwen3-14B）10–50个百分点。

4. **相关性门控**：
   - 自动选择关键块+语义相邻块将平均编译上下文从4367 tokens降至1405 tokens（减少67.8%），同时恢复提升条件下成功率的83%，得到3.1倍的每成功任务提示令牌效率提升。

5. **模式驱逐**：
   - 模式存在时100%调用，驱逐后0%调用且0%自由文本提及。该访问控制通过编译时结构实现，无需策略指令，且模型无法感知曾被移除的内容（透明擦除）。

6. **共享注册表协调**：
   - 在一个四桌、每桌四智能体的研讨会式工作流中，RAMPART的协调令牌开销为0，而朴素消息传递基线约需35万令牌。

4. **工具访问控制**：通过编译时模式驱逐实现结构性的、不可绕过的工具调用防护，强于任何基于策略的方法。
5. **透明擦除**：模型不知道被驱逐内容的存在，不会产生拒绝或遗忘认知冲突。
6. **兼容性**：现有SKILL.md/CLAUDE.md可直接导入，无需迁移。
7. **诊断-干预工作流**：利用位置扫描自动发现启动邻居，然后部署分组干预，可在弱模型上调试后用于强模型。

3. **任务基准缺陷**：
   - 虚构令牌避免模型先验污染，但任务本身极其简单（仅需复制一个令牌），无法代表现实多步骤推理。
   - 基准检查条件要求精确字面量匹配，可能低估模型正确执行但轻微形式偏差的情况。

4. **实验设计未控制所有变量**：

   - 门控实验的“语义相邻”定义依赖余弦相似度（短文本嵌入），可能存在误判。
   - 共享注册表协调实验是说明性而非实证性（无实际模型推理，仅理论对比）。

5. **理论分析深度**：论文声明不涉及注意力机制分析，仅报告观测行为。对位置敏感性背后的因果机制（注意力头、层等）未做探测。
