## 摘要
长上下文任务要求LLM从大规模上下文中识别并保留与答案相关的信息。分块记忆智能体通过顺序读取文档块、更新紧凑记忆、并从积累的记忆中生成最终答案来解决这一问题。然而，现有的基于强化学习的分块智能体要么依赖稀疏的最终答案奖励，要么使用基于词法的中间奖励来指导记忆和检索动作。这些信号监督任务成功或局部重叠，但并未直接评估最终记忆是否支持真实答案。我们提出InfoMem，一种用于训练分块记忆智能体的奖励机制，该机制利用答案条件信息来评估最终记忆的效用。InfoMem衡量最终记忆在多大程度上增加了模型对真实答案的逐词对数似然。为了稳定强化学习优化，InfoMem仅将此信号应用于成功的轨迹，并在奖励组合前对其进行归一化。在相同的GRPO框架和训练预算下，与可比的记忆智能体强化学习基线相比，InfoMem提升了长上下文记忆智能体的性能。分析表明，有效的最终记忆奖励应作用于成功轨迹，在奖励组合前进行归一化，并以答案而非查询为条件。我们的代码可在https://github.com/GenSouKa1/InfoMem获取。

## Abstract
Long-context tasks require LLMs to identify and preserve answer-relevant information from large contexts. Chunk-wise memory agents address this issue by sequentially reading document chunks, updating a compact memory, and generating the final answer from the accumulated memory. However, existing RL-based chunk-wise agents either rely on sparse final-answer rewards or use lexical intermediate rewards for memory and retrieval actions. These signals supervise task success or local overlap, but do not directly evaluate whether the final memory supports the ground-truth answer. We propose InfoMem, a reward mechanism for training chunk-wise memory agents that evaluates final-memory utility using answer-conditioned information. InfoMem measures how much the final memory increases the model's per-token log-likelihood of the ground-truth answer. To stabilize RL optimization, InfoMem applies this signal only to successful trajectories and normalizes it before reward composition. Under the same GRPO framework and training budget, InfoMem improves long-context memory-agent performance over comparable memory-agent RL baselines. Analyses show that effective final-memory rewards should operate on successful trajectories, be normalized before reward composition, and be conditioned on the answer rather than the query. Our code is available at https://github.com/GenSouKa1/InfoMem.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在长上下文任务（如长文档问答）中，基于分块记忆的智能体（chunk-wise memory agents）需要从大量文档块中提取并压缩信息到紧凑的记忆中。然而，现有的基于强化学习（RL）的训练方法主要使用两类监督信号：
    - **稀疏的最终答案奖励（Sparse Outcome Reward）**：仅根据最终答案正确与否给予奖励，无法区分成功轨迹中不同记忆的质量（例如，记忆是否包含冗余信息）。
    - **基于词法的中间奖励（Lexical Intermediate Reward）**：使用词汇重叠等指标来评估记忆或检索动作，但不能直接衡量记忆对最终答案的语义支持程度。
- **整体含义**：当前方法缺乏一个能够**直接评估最终记忆是否“支持”了正确答案**的奖励信号。这使得优化过程无法有效地指导模型形成高质量、聚焦于答案关键证据的记忆，从而限制了长上下文任务的性能。

### 2. 论文提出的方法论（InfoMem）

- **核心思想**：从信息论角度出发，一个有用的最终记忆**应当增加模型对真实答案的支持度**。InfoMem将此概念具体化为一个奖励信号，即**答案条件信息增益**。
- **关键技术细节**：
    - **点级信息增益代理（Pointwise Surrogate）**：使用一个计算上可行的代理来近似条件互信息。具体来说，比较模型**在拥有最终记忆**和**在没有记忆（即空记忆）** 两种情况下，对真实答案 \( y^* \) 的**逐词平均对数似然（log-likelihood）**。
    - **奖励函数定义**：
        \[ r_{gain}(x, M, y^*) = \frac{1}{|y^*|} \log P_\theta(y^* | x, M) - \frac{1}{|y^*|} \log P_\theta(y^* | x, \emptyset) \]
        其中 \( M \) 是最终记忆，\( \emptyset \) 代表空记忆。该值越高，表明最终记忆越能帮助模型预测正确答案。
    - **稳定训练策略**：
        1.  **成功轨迹限制（Success-side Supervision）**：该信息增益奖励**仅应用于那些最终答案正确的（成功）轨迹**。错误轨迹仅使用稀疏的最终答案奖励。这是因为成功轨迹提供了“正样本”，能更可靠地反映记忆质量。
        2.  **奖励归一化（Pre-composition Normalization）**：由于不同问题的原始 \( r_{gain} \) 值范围差异巨大，在将其与稀疏奖励组合前，InfoMem在**同一步（rollout group）内的成功轨迹之间**进行 Z-score 归一化，以平衡不同奖励信号的尺度，避免训练不稳定。
    - **最终奖励组合**：
        \[ R_i = R_{outcome, i} + \beta \cdot \tilde{r}_i, \quad \text{for } i \in \text{successful trajectories} \]
        其中 \( R_{outcome} \) 是二元最终答案奖励，\( \tilde{r}_i \) 是归一化后的 \( r_{gain} \) 值，\( \beta \) 是平衡系数。
    - **训练框架**：该方法基于**组相对策略优化（GRPO）** 框架实现。

### 3. 实验设计

- **使用的数据集/场景**：
    - **主训练集**：**RULER-HotpotQA**，一个混合了答案相关文档和干扰文档的长上下文问答数据集（包含200个文档）。原始训练集有32,768个样本，论文为控制实验并降低计算成本，**下采样**至512个样本。
    - **评估基准（Benchmarks）**：四个覆盖不同长上下文能力的**分布外**基准：
        - **MRCR-8needle**：评估多关键点检索能力。
        - **RULER synthetic QA**：评估稀疏检索式问答。
        - **CorpusQA**：评估语料库级别的证据聚合能力。

- **对比方法**：
    - **主基线**：**Outcome-only GRPO**，即仅使用最终答案奖励进行强化学习训练。
    - **对比基线**：**ReMemR1**，一个使用了中间词法奖励的可比记忆智能体强化学习框架。
    - **初始模型**：未经过强化学习优化的基础模型（Qwen2.5-1.5B-Instruct）。
    - **消融实验变体**：
        - 不同监督侧（Success-side vs. Wrong-side vs. Both-side）。
        - 无 \( r_{gain} \) 归一化（InfoMem w/o normalization）。
        - 以查询（Query）而非答案为条件（InfoMem with QueryPMI）。

    - **客观性好**：ReMemR1因其框架差异（回调检索）而分开对比，并展示了其训练曲线。对于有评估者主观性的基准（CorpusQA和LongMemEval），论文进行了人工校准，确认了LLM评估的高可靠性（>96.8%）。

### 6. 论文的主要结论与发现

1.  **InfoMem方法性能更优**：在大多数基准测试中，InfoMem持续优于Outcome-only GRPO和ReMemR1，证明了答案条件信息增益是一种更有效的记忆监督信号。
2.  **成功轨迹使用信息增益效果最好**：信息增益奖励仅应用于成功轨迹比应用于所有轨迹或仅失败轨迹效果更好，训练更稳定。
3.  **归一化对稳定性至关重要**：在将信息增益奖励与稀疏奖励组合前对其进行归一化是必要的，否则会导致训练不稳定和性能下降。
4.  **奖励应基于答案而非查询**：以答案为条件的奖励效果远好于以查询为条件的奖励。基于查询的条件化奖励可能导致模型记忆重复查询内容而非支持性证据。
5.  **三大关键设计原则**：有效的最终记忆奖励应具备三个特征：**作用于成功轨迹**、**组合前归一化**、**以答案而非查询为条件**。

### 7. 优点

- **方法创新性**：从信息论角度出发，为分块记忆智能体的训练提供了一个原理清晰、可直接优化的视角。其奖励设计直观地衡量了“记忆是否有用”。
- **分析深入**：不仅提出新方法，还通过精巧的消融实验和合成诊断实验，深入剖析了有效记忆奖励的构成要素，提供了具有指导意义的设计原则。
- **实验设计严谨**：严格控制所有变量（数据、模型、训练预算）进行公平对比，并对潜在的评估偏差（如LLM评估）进行了人类校准，增强了结论的可靠性。

### 8. 不足与局限

- **计算成本与规模限制**：论文使用了受限的训练子集和相对较小的1.5B模型，这主要是出于控制实验和计算成本的考虑。将InfoMem扩展到更大的模型和更大规模的数据集是未来研究的方向。每个440 GPU-hours的算力需求对于大模型调优来说依然不菲。
- **方法适用范围有限**：该方法专门为“分块记忆智能体”设计，其奖励是基于最终记忆和最终答案的。对于其他长上下文范式（如纯检索增强系统、单次全上下文模型）的适用性尚未探索。
- **仅监督最终状态**：当前的奖励只评估了最终记忆和最终答案。虽然通过GRPO的优势函数可以回传到所有token，但并未直接对中间记忆更新过程提供细粒度的监督。论文也提到将信息增益扩展到中间记忆是未来工作。
- **潜在风险**：奖励机制可能被过度优化，导致模型记忆变得高度关联于“期望答案”而忽略原文的关键限定或细节。在高风险领域（如法律、医疗），这种压缩记忆不应替代源文档验证。
- **评估指标有限**：主实验中的评估指标主要关注答案的准确性和匹配度，可能未能全面捕捉模型对长上下文的理解深度、推理复杂性或忠实度。
