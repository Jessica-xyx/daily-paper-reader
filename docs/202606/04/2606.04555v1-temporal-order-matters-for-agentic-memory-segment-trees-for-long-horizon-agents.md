## 摘要
长时域对话智能体需要通过不断演变的事件、任务和意图与用户交互。这类历史记录自然是时间性的，然而许多现有记忆系统主要按主题相似性组织信息，可能忽略事件发生的顺序。我们提出分段树记忆（Segment Tree Memory，简称SegTreeMem），这是一种将对话历史表示为按时间顺序排列的话语分段树的记忆架构。SegTreeMem通过在线最右前沿更新规则逐步插入新的话语，保留时间顺序的同时形成层次化的记忆分段。在检索时，SegTreeMem通过树传播相关性分数，将局部语义匹配与层次化时间上下文相结合。在三个长时域记忆基准测试和两个LLM主干模型上，SegTreeMem在答案质量上优于平面检索、图结构记忆和树结构记忆基线。额外的时间顺序排列分析表明，性能提升依赖于在记忆构建过程中保留时间顺序，支持了时间顺序是智能体记忆关键结构的观点。

## Abstract
Long-horizon conversational agents need to interact with users through evolving events, tasks, and goals. Such histories are naturally temporal, yet many existing memory systems organize information primarily by topical similarity and may ignore the order in which events occur. We introduce Segment Tree Memory, or SegTreeMem, a memory architecture that represents conversation history as a temporally ordered Segment Tree over utterances. SegTreeMem incrementally inserts new utterances through an online rightmost-frontier update rule, preserving chronological order while forming hierarchical memory segments. For retrieval, SegTreeMem propagates relevance scores through the tree to combine local semantic matching with hierarchical temporal context. Across three long-horizon memory benchmarks and two LLM backbones, SegTreeMem improves answer quality over flat retrieval, graph-structured memory, and tree-structured memory baselines. Additional temporal-order permutation analysis shows that the performance gain depends on preserving temporal order during memory construction, supporting the claim that temporal order is a key structure for agentic memory.

## 2. 方法论：核心思想、技术细节

- **核心思想**：使用经典数据结构**分段树**（Segment Tree）来表示对话历史。每个节点对应一个连续的话语区间（即时间上相邻的一组话语），内部节点递归地将其区间划分为有序、不重叠的子区间。叶节点对应单个话语。节点带有一个由LLM生成的摘要（annotation）作为该区间内容的浓缩表示。

- **关键技术细节**：
  - **在线构建**：每次新话语\(x_{t+1}\)到达时，通过**最右前沿（rightmost frontier）**机制进行插入。前沿定义为当前树中所有右端点等于\(t\)的节点（从叶到根的一条路径）。新话语以叶节点形式创建，然后通过兼容性模型选择前沿中某个非叶节点作为父节点，将新叶子（或一串内部节点）作为最右子节点附加。若没有兼容节点，则创建新根。更新仅影响O(log|Mt|)个节点，满足在线高效更新需求。
  - **兼容性模型**：三种设计：
    - **余弦相似度**：选择前沿节点摘要嵌入与新话语最相似的节点。
    - **点式LLM**：LLM独立评分每个前沿节点与新话语的兼容性。
    - **批次式LLM**：LLM同时比较所有前沿节点与话语，选择最兼容者。
  - **内部节点摘要生成**：插入后，被影响的节点区间和摘要重新计算，摘要由LLM基于有序子节点内容生成。
  - **结构感知检索**：给定查询\(q\)，首先计算每个节点摘要的局部相关性得分（使用嵌入余弦相似度），归一化为概率分布\(s^{(0)}_q\)。然后通过**传播矩阵\(W_P\)**进行有限步（\(H\)步）传播，支持两种策略：
    - **自上而下（top-down）**：将父节点得分向其子节点传播。
    - **自下而上（bottom-up）**：将子节点得分向其父节点传播。
  - 最终检索得分是初始分布与各步传播分布的加权组合（衰减因子\(\alpha\)），选择Top-K节点作为证据。
  - 公式：\( \tilde{s}_q = \frac{\sum_{k=0}^H \alpha^k s^{(k)}_q}{\sum_{k=0}^H \alpha^k} \)，其中\(s^{(k)}_q = W_P^k s^{(0)}_q\)。

## 6. 主要结论与发现

- SegTreeMem在三个基准上均优于所有基线，LLM-judge准确率最高提升约20%（相对提升）。

- **构建策略**：批次式LLM兼容性模型质量最高，但余弦相似度在性价比上更优（更低延迟与token消耗）。

- **树结构分析**：SegTreeMem产生的树在深度上稳定（3-4层），分支因子介于MemTree（窄）与RAPTOR（宽）之间。

## 7. 优点

- **创新性**：将经典数据结构“分段树”引入智能体记忆，解决时间顺序与层次化结构的双重需求。
- **在线构建**：支持实时更新，更新范围局限在最右前沿，复杂度低（对数级别），满足交互场景需求。
- **结构感知检索**：提出基于传播矩阵的分数传播方法，能够灵活调整检索粒度（自上而下细化，自下而上扩展），且具有可解释性。

## 8. 不足与局限

- **实验覆盖**：仅评估了文本对话记忆，未涉及多模态、工具调用、动作轨迹等更广泛的智能体场景。

- **检索策略固定**：传播方向、步数、衰减因子对所有查询使用同一设置，未实现查询自适应选择。不同问题可能需要不同粒度，未来可探索自适应策略。
