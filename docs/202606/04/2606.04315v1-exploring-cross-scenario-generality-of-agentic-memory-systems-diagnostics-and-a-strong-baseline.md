## 摘要
LLM智能体积累的历史超出其上下文窗口，催生了关于记忆系统的日益增多的文献。然而，现有大多数设计均针对单一场景（多会话聊天或单轨迹格式）调整，几乎没有证据表明它们能够泛化到智能体部署中遇到的异质轨迹。我们重新审视了八个记忆系统以及一个用于搜索问题的智能体框架，在五个场景上进行了评估：单轮问答、多会话聊天、智能体轨迹问答、记忆压力测试和长期智能体任务。该框架通过工具调用自行管理纯文本文件存储，实现了最佳跨任务排名，表明记忆性能取决于赋予智能体对存储和检索的主动控制，而非固定管道背后的被动存储。我们将这一见解实例化为AutoMEM，一个具有自管理工具接口的智能体记忆框架，在我们评估的系统中实现了最佳的跨场景通用性。

## Abstract
LLM agents accumulate histories that outgrow their context windows, motivating a growing literature on memory systems. Yet most existing designs are tuned to a single scenario (multi-session chat or a single trajectory format), and there is little evidence that they generalize across the heterogeneous trajectories agents encounter in deployment. We revisit eight memory systems plus an agentic harness for search problems, on five scenarios: single-turn QA, multi-session chat, agentic-trajectory QA, memory stress tests, and long-horizon agentic tasks. The harness, which self-manages flat text-file storage via tool calls, achieves the best cross-task ranking, suggesting that memory performance hinges on giving the agent active control over storage and retrieval rather than on a passive store behind a fixed pipeline. We instantiate this insight in AutoMEM, an agentic memory harness with a self-managed tool interface that achieves the best cross-scenario generality among the systems we evaluate.

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程
- **核心思想**：记忆性能的关键在于**赋予智能体主动控制存储与检索的能力**，而非采用固定管道的被动存储。作者提出**AutoMEM**，一个基于自管理工具接口的智能体记忆框架。
- **关键技术细节**：
  - AutoMEM采用**agentic loop**（规划-执行-判断循环），每一步智能体从4种动作中选择：`rg`（原始记录的正则匹配）、`read`（按ID获取记录）、`dump`（将短语料发送给答案LLM）、`index_query`（对预构建索引执行类型化查询）。

  - 默认`index_query`使用基于LightMem摘要索引构建的Cypher属性图，并带有SAID_BY、OF_TOPIC、FOLLOWS边；可选PlugMem图形变体使用PlugMem的标签/语义/情节存储。
- **算法流程（文字说明）**：
  1. 初始化空轨迹trace和空提示hint。
  2. 循环（最多Kmax步）：

## 6. 论文的主要结论与发现
1. **没有方法在所有场景上占优**：每种基于索引的方法（预先构建图、笔记等）至少在一个基准上落后于长上下文；DCI-Lite（智能体框架）获得了最佳通用性排名。
2. **长上下文比通常假设的更强**，且成本具有竞争力：重索引方法如HippoRAG预计算成本高，仅在大量查询击中同一存储时才具性价比。
3. **索引方法在智能体轨迹QA上存在两个失败模式**：
   - 表示级失败：构建时模式（如HippoRAG的实体-关系KG）丢弃了步骤索引和动作标签，导致信息永久丢失。
   - 检索级失败：被动检索无法从存储中表面保留的证据（如PlugMem保留了原始轨迹但路由器仅返回紧凑笔记，无法通过后续调用获取详细步骤）。
4. **智能体框架（DCI-Lite）获胜的原因**：将模式承诺延后到查询时，LLM根据问题实时决定检索哪些原始证据，而非在构建时固定模式。
5. **AutoMEM结合了智能体框架和选择性索引**：在LoCoMo上比DCI-Lite提升22.3 pp，比长上下文提升5.8 pp，平均排名领先所有方法。
6. **在动态智能体任务（如ALFWorld）上，记忆设计的改进空间有限**：与长上下文的差异不显著，需要参数级后训练（如GRPO）才能有效提升。
