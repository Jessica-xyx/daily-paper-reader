## 摘要
MeMo提出了具有显式多层关联矩阵记忆（CMMs）的语言模型，其中记忆、检索和遗忘是架构性操作。本文探讨当知识发生变化时，此类记忆如何减少重新训练的需求。对于可表示为MeMo记忆关联的变化，可以通过编辑显式记忆而非重新训练整个模型来更新模型的可访问知识。我们提出了一个版本感知操作层，在该层中，高级操作（如替换、废弃、保留历史、回滚和追溯）被编译为针对序列和标记的MeMo原生原语调用。关键观察是，版本感知操作很少是单一的MeMo关联；它是一个有序的原语编辑事务，例如遗忘一个序列-标记链、记忆另一个链、保留一个历史链以及记录一个逆程序。该框架引入了两个辅助CMM：版本CMM（V-CMM）用于将版本转换映射到事务句柄，以及事务CMM（T-CMM）用于存储可重用的变更内容和逆程序。它支持直接的序列级编辑和结构化的差异级输入，并概述了针对更新成功、回滚、可追溯性、局部性和事务重用的评估路径。

## Abstract
MeMo proposes language models with explicit multi-layer correlation matrix memories (CMMs), where memorization, retrieval, and forgetting are architectural operations. This paper asks how such memories can reduce the need for retraining when knowledge changes. For changes expressible as MeMo memory associations, the model's accessible knowledge can be updated by editing explicit memories rather than retraining the whole model. We propose a version-aware operation layer in which high-level operations such as replace, obsolete, keep-history, rollback, and trace are compiled into MeMo-native primitive calls over sequences and tokens. The key observation is that a version-aware operation is rarely a single MeMo association. It is an ordered transaction of primitive edits, for example forgetting one sequence-token chain, memorizing another, preserving a historical chain, and recording an inverse program. The framework introduces two auxiliary CMMs: a Version CMM (V-CMM) for mapping version transitions to transaction handles, and a Transaction CMM (T-CMM) for storing reusable change contents and inverse programs. It supports both direct sequence-level edits and structured diff-level inputs, and outlines an evaluation route for update success, rollback, traceability, locality, and transaction reuse.

## 2. 论文提出的方法论

- **核心思想**：高层知识变更不应被视为单一的MeMo关联（序列-值），而应视为一个有序的原语编辑事务（transaction）。每个事务由一系列原语操作（记忆或遗忘一个序列-值对）组成，并通过逆程序支持回滚。引入两个辅助CMM：
  - **版本CMM（V-CMM）**：将版本转换（从版本v_a到v_b）映射到事务句柄列表，用于索引版本演化。
  - **事务CMM（T-CMM）**：将事务句柄映射到可执行的编辑字段（序列、值、符号、层），支持可重用模板和参数化实例化。

- **关键技术细节**：
  - **原语操作**：`memo(S, y)`、`forget(S, y)`、`retrieve(S)`，其中S是输入序列，y是关联的token。
  - **事务表示**：`τ = ⟨e1; e2; ...; em⟩`，每个编辑`e_j = (S_j, y_j, λ_j)`，λ_j=+1表示记忆，λ_j=-1表示遗忘。
  - **多token连续体处理**：通过`chain(S, Y, λ)`将连续体Y=[y1,...,yn]分解为一系列原语编辑。
  - **结构化键**：将知识图谱三元组（版本，主体，关系，客体）序列化为输入序列`σ(v,s,r) = [VERSION, v, SUBJECT, s, RELATION, r]`，然后通过memo/forget操作管理。
  - **高层操作示例**：`replace(S, Y_o, Y_n)`扩展为`chain(S, Y_o, -1); chain(S, Y_n, +1)`，同时可附加历史版本保存。
  - **V-CMM构建**：使用外积存储版本转换、槽位和事务句柄之间的关联，支持确定性检索。
  - **T-CMM构建**：存储事务标识符、编辑索引、字段名和字段值的映射，解码后生成有序可执行程序。
  - **回滚机制**：将原事务程序逆序并将符号取反，支持直接回滚、日志回放或视图切换三种方式。
  - **模板与重用**：T-CMM可存储可复用模板（如替换模板）和参数句柄，避免重复存储完整扩展程序。

- **公式/算法流程（文字描述）**：
  1. 用户发起高层操作（如replace(sequence, old_value, new_value)）。
  2. 操作层将操作编译为有序事务τ（由多个memo/forget原语组成）。
  3. V-CMM将当前版本转换(va→vb)与事务句柄τ关联。
  4. T-CMM存储事务的详细编辑字段（序列、值、符号、层），或存储模板+参数句柄。
  5. 执行时，解码T-CMM得到原语程序，依次调用MeMo原生memo/forget。
  6. 若需回滚，获得逆程序并执行；若需追溯，通过V-CMM找到事务句柄，再查询T-CMM和日志。
