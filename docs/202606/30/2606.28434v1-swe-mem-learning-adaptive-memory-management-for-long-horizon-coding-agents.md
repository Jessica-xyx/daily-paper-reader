## 摘要
长周期软件工程智能体通常需要在有限的上下文预算下管理冗长且嘈杂的交互历史。现有的内存管理方法通常依赖静态压缩工作流，或对压缩时机和粒度施加严格约束。此外，这些方法未能联合优化内存管理和问题解决能力，以在提升性能的同时减少令牌使用。我们提出SWE-MeM，一种用于软件工程智能体中主动和按需内存管理的训练框架。SWE-MeM提供灵活的内存工具，使智能体能够根据轨迹状态、任务进度和剩余上下文预算决定何时、何地以及如何压缩。我们通过合成的主动内存管理轨迹和内存感知GRPO训练智能体，该方法通过内存感知轨迹拆分和步骤级信用分配联合优化内存管理和问题解决。在SWE-Bench Verified上，SWE-MeM在4B和30B模型上分别达到43.4%和60.2%的解决率，在性能和效率上均优于现有内存管理基线。

## Abstract
Long-horizon software engineering agents often need to manage lengthy and noisy interaction histories under limited context budgets. Existing memory management methods typically rely on static compression workflows or impose rigid constraints on compression timing and granularity. Moreover, these approaches fail to jointly optimize memory management and issue resolution capabilities to improve performance while reducing token usage. We present SWE-MeM, a training framework for proactive and on-demand memory management in software engineering agents. SWE-MeM provides a flexible memory tool that lets agents decide when, what, and how to compress based on trajectory state, task progress, and remaining context budget. We train agents with synthesized proactive memory-management trajectories and Memory-aware GRPO, which jointly optimizes memory management and issue resolution through memory-aware trajectory splitting and step-level credit assignment. On SWE-Bench Verified, SWE-MeM achieves 43.4% and 60.2% resolve rate with 4B and 30B models, respectively, outperforming existing memory management baselines in both performance and efficiency.

## 1. 核心问题与整体含义（研究动机和背景）

长周期软件工程（SWE）智能体在执行任务时需要不断累积交互历史，导致上下文快速膨胀，超出LLM的上下文窗口限制，且长而嘈杂的轨迹会降低任务相关信息的使用效率。现有内存管理方法（如阈值触发压缩、子任务折叠）存在两个主要缺陷：**灵活性不足**——固定压缩规则无法适应轨迹中信息的异质性（如冗长的执行日志可以随时压缩，而早期检查的代码可能对后续推理至关重要）；**优化目标单一**——仅关注上下文长度缩减，而忽略了任务解决效率（导致总token使用或交互步骤反而增加）。SWE-MeM旨在让智能体学会**主动、按需的内存管理**，联合优化内存压缩与任务性能，在降低token消耗的同时提升问题解决率。

## 2. 方法论

### 核心思想
- 设计一个灵活的内存管理工具 `compress(analysis, start_step_number, end_step_number, content, remaining_work)`，让智能体基于当前轨迹状态、任务进度和剩余上下文预算自行决定**何时压缩、压缩哪些步骤、如何总结**。
- 通过**轨迹合成**（使用专有模型GPT-5.1生成记忆管理动作）和**课程微调**（两阶段SFT）教会基础模型使用工具。
- 引入**Memory-aware GRPO**强化学习，通过**轨迹拆分**（在压缩点处切分子轨迹，保持训练-推理一致性）和**步骤级信用掩码**（根据记忆质量、溢出失败等策略掩码）联合优化任务解决与内存管理。

### 关键技术细节
1. **内存工具设计**：压缩操作用一个调用来实现，它接收分析、起始/结束步数、压缩后的总结内容和剩余工作计划。压缩后，轨迹中相应范围被替换为总结，剩余工作计划追加到末尾。
2. **轨迹合成**：使用基础模型（待微调的模型）生成任务求解步骤，专有模型仅用于判断是否需要压缩以及合成压缩参数。合成触发条件包括：上下文预算压力（剩余<20%时概率触发）和主动管理（子任务完成、低信息密度、聚焦退化）。合成后经过过滤（不恰当的压缩比、过短范围、错位等）。
3. **课程学习**：第一阶段在完整过滤合成轨迹上训练基本格式；第二阶段抽取包含主动压缩的轨迹，并通过LLM-as-a-judge筛选高质量样本，同时采样等量仅含预算压缩的轨迹，防止遗忘。
4. **Memory-aware GRPO**：
   - **轨迹拆分**：在每个压缩点处将完整轨迹拆分为子轨迹，子轨迹起始于压缩后的上下文，保持一致性。
   - **损失聚合**：先平均每个子轨迹的token级损失，再平均所有rollout的损失，防止长rollout主导。
   - **步骤级掩码**：针对记忆动作质量、延迟压缩、溢出失败、无效工具使用等场景，在成功轨迹或失败轨迹中掩码特定步骤。
