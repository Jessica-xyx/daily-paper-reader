## 摘要
当多模态AI智能体被要求遗忘某个事实时，当前的记忆系统通常会删除文本条目并报告成功。我们发现，该事实仍可从保留的用户图像中恢复，包括标记为完全不同事实的图像，因为VLM在推理时使用隐式视觉线索。我们引入了信息溯源图（IPG），这是一种按删除能力分类记忆表示的分类法。IPG揭示删除通过多个渠道失败。我们的基准测试MemLeak在删除级联中测量这一现象：对支持删除的系统直接探测的恢复率<1%，但保留的相关文本实现了18.3%的恢复，保留的图像实现了12.0%的恢复（盲基线为0.0%，FPR为0.3%）——其中47%的图像泄露无法通过文本恢复。内容感知的语义删除将图像残留降至2.0%。该残留出现在多个VLM、一个生产级记忆系统以及真实的Unsplash许可照片中。双标注者人工验证（kappa=0.88）确认了评判者的可靠性。

## Abstract
When a multimodal AI agent is asked to forget a fact, current memory systems usually delete the text entry and report success. We find that the fact can remain recoverable from retained user images, including images tagged to entirely different facts, because VLMs use implicit visual cues at inference time. We introduce the Information Provenance Graph (IPG), a taxonomy that classifies memory representations by deletion affordance. The IPG reveals that deletion fails through multiple channels. Our benchmark, MemLeak, measures this across a deletion cascade: direct probing of deletion-capable systems yields <1%, but retained correlated text enables 18.3% recovery, and retained images enable 12.0% recovery (0.0% blind baseline, 0.3% FPR) -- with 47% of image leaks not text-recoverable. Content-aware semantic deletion reduces the image residual to 2.0%. The residual appears across multiple VLMs, a production memory system, and real Unsplash-licensed photographs. Dual-annotator human validation (kappa = 0.88) confirms judge reliability.

# MemLeak：诊断多模态智能体记忆中的信息泄露 —— 论文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：当前多模态 AI 智能体的记忆系统在用户要求遗忘某个事实时，通常只删除文本条目并报告成功。但研究发现，即使文本和相关图像被删除，保留的用户图像（包括标记给完全不同事实的图像）仍能被视觉语言模型（VLM）利用隐式视觉线索重构出被删事实，导致信息泄露。
- **研究背景**：多模态智能体记忆系统（如 Mem0、Letta）在实际部署中会存储文本、图像、嵌入、图边等异构表示。用户事实可以在多个表示层传播。现有删除机制（如记录级删除）只处理地址化的文本节点，而忽略了图像中隐含的视觉特征（如地理位置、文化背景、职业线索等）。这些特征无法通过溯源追踪删除，形成“遗忘残留”。
- **整体含义**：事实级遗忘不仅仅是工程问题（缺少溯源追踪），更是架构性限制：图像中的隐式信息是跨事实、跨模态的，文本式审计无法检测。这对 GDPR 第 17 条等数据保护合规构成技术风险。

## 2. 方法论：核心思想、关键技术细节

### 2.1 信息溯源图（IPG）

- **定义**：对于一个事实 f，IPG 是一个有向无环图 \( G_f = (V_f, E_f, \delta) \)，其中：
  - \( V_f \)：f 在所有存储层中的每一个不同表示（文本条目、文本嵌入、图像文件、图像嵌入、图边、隐式视觉特征、派生事实、行为模式）。
  - \( E_f \)：信息流边，表示表示之间的派生关系。
  - \( \delta: V_f \to \{\text{addr}, \text{link}, \text{unreach}\} \)：删除能力赋值。
    - **addressable（可寻址）**：系统能根据事实标识直接定位并删除。
    - **linked（链接的）**：通过维护溯源链接才能删除。
    - **unreach（持久性）**：无法被现有存储级删除机制覆盖，只能通过内容感知审计间接处理。

- **遗忘完备性**：\( \text{FC}_S(f) = |\{v \in V_f: v \text{被删除}\}| / |V_f| \)。

### 2.2 信息泄露与通用推理的区分

- **信息泄露** 需满足三个条件：
  1. 系统被指令遗忘事实 f；
  2. 系统响应揭示了 f 或足够重构 f 的信息；
  3. 启用的证据来自保留的**同一用户数据**（图像、嵌入、交互痕迹）——排除模型通用知识（如从埃菲尔铁塔照片识别巴黎）。
- 泄露通过输出层行为定义，而非仅仅存储层状态。

### 2.3 MemLeak 基准

- **数据集**：113 个合成实体 profile，每个 20 个事实（10 个遗忘目标，10 个保留），覆盖 5 类（位置、职业、偏好、关系、所有物）。视觉接地等级：显式、隐式、无。图像使用 Gemini 3.1 Flash 生成，GPT-5.4 设计提示词。
- **探测类型**：
  - P1：文本删除，图像探测（核心）
  - P2：图像删除，文本探测
  - P3：从未存储为文本的隐式事实
  - P4：跨图像组合重构
  - P5：行为残留
- **指标**：跨模态泄露率（CMLR）= 被删事实中能被重构的比例。使用 3 模型 LLM 评判器（GPT-4.1、Claude Sonnet 4、Gemini 2.5 Flash）多数投票。人类验证（双标注，kappa=0.88）。

### 2.4 语义删除

- 在删除文本和标记图像后，用 VLM 审计员扫描剩余图像，识别与删除事实语义相关的图像并标记删除。可以降低泄露但带来实用成本（21% 图像被标记删除）。

## 3. 实验设计

### 3.1 使用的数据集/场景

- **文本规模**：113 个 profile，2260 个事实，1053 个探测（评估各系统直接探测）。
- **多模态规模**：38 个 profile（文本规模的子集，增加生成图像），760 个事实，536 张图像，300 个探测。
- **真实图像验证**：从 Unsplash 获取 523 张照片对应 38 个 profile，282 个探测。
- **消融 pilot**：5 个 profile，100 个事实，115 张图像，55 个探测。

### 3.2 Benchmark 对比

- **对比系统**：
  - **Long-context**（行为抑制）：不删除任何内容。
  - **Naive**（文本删除）：仅删除文本条目。
  - **Mem0**（生产级 RAG）：语义搜索删除文本条目。
  - **Letta**（模拟）：智能体决定删除哪些文本条目。
  - **Oracle**（溯源级联）：删除文本、嵌入、图像、图边等所有可寻址与链接节点。

- **主要实验**：
  1. 文本探测：各系统直接探测泄露率（<1% 删除系统，Long-context 17%）。
  2. 多模态探测：同事实图像恢复（48.7%）、保留图像恢复（12.0%）、负控制（0.3% FPR）、盲测（0.0%）。
  3. 跨模型验证：Claude Sonnet 4 vs GPT-4o 保留图像恢复（9.3%，重叠 CI）。
  4. 端到端 Mem0 管道：16.3%（混合文本检索残留与视觉推理）。
  5. 真实照片验证：10.6%（与合成 12.0% 一致）。
  6. 语义删除消融：从 12.0% 降至 2.0%。
  7. 类别分析（表 5）：位置/所有物泄漏最高，语义删除后各类别均匀降至 1.5-3.4%。
  8. 删除策略消融（表 7）：不同删除策略对比（5 个 pilot，55 个探测）。

### 3.3 实验充分性与公平性

## 6. 主要结论与发现

1. **事实级遗忘失败通过多个渠道**：直接探测删除系统的泄露率 <1%，但保留的相关文本导致 **18.3%** 恢复，保留图像导致 **12.0%** 恢复（盲测 0%）。
2. **图像提供文本审计无法检测的额外渠道**：47% 的图像泄露事实无法从保留文本中恢复。
3. **溯源追踪只能解决链接节点**：从同事实图像恢复的 48.7% 降至保留图像恢复的 12.0%（删除标记图像）。
4. **内容感知语义删除可大幅降低残留**：将图像泄露从 12.0% 降至 **2.0%**，但代价是删除 21% 的保留图像。
5. **残留存在于多个 VLM、生产系统、真实照片**：结果具有跨模型、跨系统的泛化性。
6. **文本相关性泄漏本身也是严重渠道（18.3%）**：需要相关性感知删除策略。
7. **对于部署者**：推荐实施溯源追踪、VLM 审计员、相关性感知文本删除。

## 7. 优点

- **问题新颖重要**：首次系统诊断多模态智能体记忆中的跨事实、跨模态信息泄露，具有现实合规意义。
- **形式化框架 IPG**：提供了清晰的分类法，区分不同删除能力，有助于系统理解和设计。
- **基准设计严谨**：
  - 包含盲测、负控制、多系统对比、人类验证。
  - 严格区分“信息泄露”与“通用推理”。
  - 探测类型细化对应 IPG 各层。
  - 支持真实照片验证，增强生态效度。
- **实验设计全面**：覆盖多个 VLM、生产系统、人工验证、消融、类别分析。
- **实用建议**：明确区分工程缺口（可寻址/链接节点）与遗忘残留（持久节点），并提供可操作的三步建议。

- **指标清晰**：CMLR 配合 95% CI，统计结果可靠。

## 8. 不足与局限

- **数据局限性**：主数据集为合成 profile 和生成图像，虽然真实照片验证一致，但未使用真实用户照片（可能具有不同跨事实相关性分布）。
- **单一删除策略**：仅评估“删除文本 + 标记图像”策略，未测试更激进的策略（如完全清除所有用户图像）或精细优化策略（如特定阈值）。
- **Letta 为模拟实现**：未使用完整 Letta server，可能遗漏某些行为特征。
- **Mem0 端到端结果混杂**：16.3% 包含文本检索残留与视觉推理，无法完全分解。
- **评判器局限**：3 模型评判器在多数投票时有高人类一致性（kappa=0.84），但在分票（1/3）时人类一致性仅 37.5%，可能漏检或误检。
- **IPG 是描述性而非预测性**：不能直接预测给定系统在给定事实下的具体泄露率，需要基准测试。
- **语义删除的实用性**：删除 21% 保留图像可能影响用户其他记忆的可用性，未测量下游准确率影响。
- **未覆盖所有模态**：行为残留、派生事实的评估仅在小消融中（P5），未在规模实验中广泛测试。
- **可能低估同事实泄漏**：人类验证显示评判器在同事实模式下保守，实际泄漏可能高于 48.7%。
