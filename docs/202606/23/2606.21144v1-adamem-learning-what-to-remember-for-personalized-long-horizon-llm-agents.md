## 摘要
大型语言模型（LLM）代理的长期记忆系统通常试图"记住一切"，均匀地提取记忆以保留尽可能多的事实。然而，在生产环境中，推理成本和有限的上下文窗口使得这种做法难以为继：除了将原始对话整合到记忆中之外，代理还必须施加"写入控制"，高效地只保留每个用户真正关心的信息。否则，长期个性化的交互会遭受"记忆膨胀"，不相关的琐事会挤占有用信息并逐渐侵蚀问答（QA）准确性。我们认为，什么值得记住取决于角色，并提出**AdaMem**（自适应记忆），一种从反馈中学习每个用户应该记住什么的方法。AdaMem维护一个结构化的、特定角色的记忆策略，并通过轻量级的、补丁式的自我反思步骤（带有失败回滚）从每周的QA反馈中完善该策略。为了研究这一设置，我们构建了**AdaMem-Bench**，一个模拟数周交互并逐周进行QA的基准测试。在两种提取模型和两种反馈模式下，与均匀的Mem0基线相比，AdaMem将QA准确性提高了最多**+9.0%**，同时将记忆容量缩小了**9%**。

## Abstract
Long-term memory systems for Large Language Model (LLM) agents typically try to \emph{remember everything}, extracting memories uniformly to retain as many facts as possible. In production, however, inference cost and finite context budgets make this untenable: beyond consolidating raw dialogue into memory, an agent must exert \emph{write control}, efficiently keeping only the information each user actually cares about. Otherwise, long-horizon personalized interactions suffer \emph{memory bloat}, where irrelevant trivia crowds out useful information and steadily erodes question-answering (QA) accuracy. We argue that what is worth remembering is role-dependent, and propose \textbf{AdaMem} (Adaptive Memory), a method that \emph{learns what to remember} for each user from feedback. AdaMem maintains a structured, role-specific Memory Policy and refines it from weekly QA feedback through a lightweight, patch-style self-reflection step with failure rollback. To study this setting, we build \textbf{AdaMem-Bench}, a benchmark that simulates weeks of interaction with week-by-week QA. Across two extraction models and two feedback modes, AdaMem improves QA accuracy by up to \textbf{+9.0\%} over the uniform Mem0 baseline while shrinking memory volume by \textbf{9\%}.

## 1. 核心问题与整体含义（研究动机和背景）

现有LLM代理的长期记忆系统（如Mem0、MemoryBank）通常采用“记住一切”的均匀提取策略，试图将所有事实性陈述都存入记忆库。但在实际部署中，推理成本和有限上下文窗口导致这一做法难以为继：随着交互时间延长（数周至数月），记忆库会积累大量无关琐碎信息（如日常闲聊、次要日程），造成**记忆膨胀**，在检索时挤占有用信息，从而逐步侵蚀问答准确性。作者指出，什么值得记住是**角色依赖**的（如与经理对话需跟踪商务决策，与健身伙伴聊天需关注锻炼计划），因此记忆提取应通过一个与用户偏好对齐的**记忆策略**来指导，并持续从反馈中纠正。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
AdaMem将长期记忆视为一个**写入控制问题**：从“记住一切”转向**学习该记住什么**。通过维护一个结构化的、特定角色的**记忆策略**，并根据每周问答反馈进行补丁式自我反思与更新，实现偏好对齐的记忆提取。

### 2.2 关键技术细节

- **记忆策略结构**：`P = {C1: T1, ..., Cn: Tn}`，JSON对象，将每个角色映射到自然语言提取偏好（如“仅提取最终业务决策和截止日期，忽略日常闲聊”）。
- **闭环周循环流程**：
  1. **日提取**：根据当前策略提取记忆，并执行**时间锚定**（将相对时间引用解析为绝对日期）再存入数据库。
  2. **周评估**：用检索到的记忆回答每周QA。
  3. **策略反思**：对错误答案QA进行诊断，区分**召回失败**（策略过严）和**精确失败**（策略过宽），生成策略补丁`∆Pt`。
  4. **策略更新+失败回滚**：应用补丁生成新策略，并在历史QA上验证；若性能下降则回滚。
- **渲染与时间锚定**：每日提取时，将当前策略渲染为提取LLM的指令（如“严格按照以下角色偏好提取记忆”），并注入会话绝对日期解决相对时间引用。

## 3. 实验设计

### 3.1 数据集 / 基准
- **AdaMem-Bench**：自建合成基准，包含5个长期交互故事，每个故事涉及助手与多个角色（各有不同偏好）进行10周日常对话，每周结束后生成偏好对齐的QA。
- **黄金记忆标注**：每个对话会话标注了与角色偏好匹配的“黄金记忆”，其余为噪声。
- **对比基准**：
  - **Full Context**（理论上限：直接使用原始对话历史）
  - **Ideal Memory**（仅存储黄金记忆）

### 3.2 对比方法
- 提取模型：**DeepSeek-V4-Flash** 和 **Gemini-3.5-Flash**（均为当前主流LLM）。

### 3.3 评估指标

## 6. 主要结论与发现

- **整体性能提升**：AdaMem在四个实验单元中一致优于Mem0。最高提升：DeepSeek-V4-Flash + 显式反馈下QA准确率+9.0%（85.2% vs 76.2%），记忆容量减少9.1%。
- **增益集中在“软偏好”类别**：情感/态度（+15.9%）、日程/时间（+9.9%）、决策/结论（+9.0%）等上下文主观性强的类别改善显著，而事实/数字类（+1.1%）提升很小。
- **缓解记忆膨胀**：10周内，AdaMem保持更高且更稳定的Recall@5和MRR，而Mem0持续下降。
- **政策遵循能力不是瓶颈**：现代LLM能很好遵循结构化指令（遵循率>80%），但**策略推断**（从隐式反馈学习偏好）是主要瓶颈，显式反馈下策略对齐持续提高（88.3%），隐式反馈下停滞在53.3%-63.3%。

## 7. 优点

- **方法轻量、可插入**：策略补丁可直接集成到现有RAG记忆栈中，无需重新训练。
- **闭环自适应**：通过周级别反馈循环持续优化提取策略，适应角色变化。
- **时间锚定**：巧妙解决相对时间引用在长期存储中的歧义问题。
- **失败回滚**：保障策略更新的稳定性，防止过度适应单次错误。
- **双反馈模式**：显式和隐式设置更贴近真实场景（用户可能只给正确答案而不解释）。

## 8. 不足与局限

- **合成数据依赖**：数据集为合成的多轮对话，可能缺乏真实对话的随意性、噪音和偏好突变。需要真实人机交互验证。
- **LLM作为裁判偏差**：评估中的QA正确性、提取质量、策略遵循性等均使用LLM判断（deepseek-v4-flash），可能存在难以避免的系统性偏差。
- **评估规模有限**：仅5个故事、10周交互，统计显著性可能不足；更长的交互周期（如全年）和更多场景也未被覆盖。
- **策略过拟合风险**：在隐式反馈下，补丁式反思可能过度适应单次失败导致策略收窄；虽然回滚机制缓解了，但缺乏更强有力的正则化手段。
- **未对比其他主动记忆方法**：仅与均匀提取Mem0对比，未与AgeMem、EvolveMem等同样聚焦记忆管理的系统横向比较，削弱了说服力。
