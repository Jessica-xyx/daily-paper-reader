## 摘要
长期运行的语言模型系统积累的交互历史会超出上下文窗口，因此必须不断进行驱逐。当驱逐策略丢弃关键细节时，例如登录时颁发的访问令牌或下一个调用需要的路径，操作就会失败。我们提出LRE（学习相关性驱逐），一种仅需几千字节、仅限CPU、无需语言模型的分值器，它学习哪些历史单元是关键性的，并通过逐字提取保留它们。在匹配预算的比较下，在我们的实验中，没有基线在精度-成本平面上优于LRE。在智能体上，LRE总体上匹配了保留全部历史的精度。在最简单的任务上，它比无驱逐基线高出27%，同时需要零压缩器调用，并将峰值上下文大小最多减少52%。一项受控研究追踪显示，LRE完成了其他模型循环的任务，以比保留所有内容少37%的调用完成了一个此类任务，并解决了其他运行策略无法解决的14个任务。在对话记忆上，LRE以零神经成本优于密集和令牌剪枝编码器。在下游评估中，LRE在LoCoMo阅读上提供了最佳预算答案质量，同时读取的令牌减少了68%。其监督也可以是免注释的：仅对系统自身行为进行训练即可恢复有监督分值器95%的效果。我们认为，由于LLM智能体中的记忆驱逐是一个保真度问题，它需要一种可部署的主动策略，其中未来查询不可用且确切状态具有决定性，并且廉价的学习相关性就足够了。

## Abstract
Long-running language-model systems accumulate interaction history that outgrows the context window, so they must continually evict. When an eviction policy drops a load-bearing detail, for example an access token issued at login or a path the next call needs, the action fails. We present LRE (Learned Relevance Eviction), a few kilobytes, CPU-only, language-model-free scorer that learns which units of history are load-bearing and keeps them by verbatim extraction. Under a matched-budget comparison, in our experiment, no baseline dominates LRE on the accuracy-cost plane. On agents, LRE matches the accuracy of keeping the entire history overall. On the simplest tasks, it exceeds that no-eviction baseline by 27%, while requiring zero compressor calls and reducing peak context size by up to 52%. A controlled study trace shows LRE completes tasks where the others loop, finishing one such task in 37% fewer calls than keeping everything and solving 14 tasks where no other run policy does. On conversational memory, LRE outranks dense and token-pruning encoders at zero neural cost. In downstream evaluation, LRE gives the best budgeted answer quality on LoCoMo reading 68% fewer tokens. Its supervision can also be annotation-free: training only on the system's own behavior recovers 95% of the supervised scorer's effectiveness. We argue that, because memory eviction in LLM agents is a fidelity problem, it requires a deployable proactive policy where the future query is unavailable and exact state is decisive, and that cheap learned relevance can be sufficient.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：长期运行的 LLM 系统（如智能体或对话助手）会积累大量历史交互，超出上下文窗口，必须不断丢弃（evict）。若丢弃策略误删了关键信息（如登录令牌、API 路径），后续操作就会失败。
- **背景**：现有方法要么依赖反应式启发式（如最近优先、时间衰减），要么调用 LLM 压缩（成本高且非确定性），要么使用需要神经前向传播的学习方法。
- **核心动机**：记忆驱逐本质上是保真度问题（fidelity problem），需要一种**主动的、查询不可知的（query-blind）**策略，在写入时即预测哪些内容对未来有用，并**逐字保留**，以保证精确状态。作者提出廉价的学习相关性评估可以满足这一需求。

## 2. 论文提出的方法论
### 核心思想
- **LRE（Learned Relevance Eviction）**：一个轻量级评分器，使用仅数 KB、CPU 运行、无需 LLM 调用的逻辑回归模型，对每个历史单元（动作-观察步或对话轮次）学习其“负载重要性”（load-bearing），然后按价值密度（得分/成本）贪心选择保留的单元，并**逐字提取**（verbatim extraction），不压缩/改写。

### 关键技术细节
- **评分函数**：\( p_i = f_\theta(u_i) = \sigma(\theta^\top \phi(u_i, u_{\le i})) \)，其中 \(\phi\) 是因果特征映射（仅基于前缀，不依赖未来）。
- **特征设计**：共 13 种特征（见表 6），包括位置、对数长度、数字数、大写词数、错误/代码标记、稀有度密度、与前一步的 Jaccard 相似度、URL/ID 存在等，跨领域共用核心特征，但领域间可定制。
- **选择算法**：变体 0/1 背包问题，使用贪心价值密度启发式（Algorithm 3），每次保留得分/成本最高的单元，直到预算用尽。保留后按时间顺序输出，并始终保留最近的 k 个单元（智能体领域 k=5，对话领域 k=0）。
- **自监督标签**：无需人工标注。智能体领域：若某步首次引入的标识符被后续 ≥3 步重用，则标为正。对话领域：若某轮的文本与后续答案有 ≥40% 的 token 重叠，则标为正。训练仅需系统自身日志。

## 6. 主要结论与发现
1. **帕累托效率**：在精度-成本平面上，LRE 同时优于无压缩（高成本）和 FIFO/ACON（低成本但低精度）。在 AppWorld 上，LRE 达到无压缩 93% 的 TGC，但峰值 token 减少 52%。
2. **关键机制：逐字保留负载信息**：通过定性案例（Venmo 请求、Simple Note 播放列表），LRE 能保留精确令牌（如 access_token），而 FIFO 和 ACON 因丢失/改写导致循环失败。14 个任务仅 LRE 成功。
3. **零神经成本超越密集编码器**：在对话记忆的宏观 AUC 上，LRE 监督版（0.829 LoCoMo, 0.708 LongMemEval_S）超过 BGE 密集编码器（0.643）和 LLMLingua-2（0.402），且无需 GPU。
4. **自监督有效性**：从系统自身行为导出的标签可恢复有监督模型 95%（LongMemEval_S）和 82%（LoCoMo）的区分能力，表明监督信号已存在于日志中。
5. **下游 QA 收益**：LRE 在固定 20% 保留预算下，LoCoMo 上 token-F1 领先所有可部署策略，且上下文 token 减少 68%。

## 7. 优点
- **极致轻量**：评分器仅几 KB，CPU 推理 1181 单元/秒，比密集编码器快 11.5 倍，比 LLMLingua-2 快 32.8 倍。
- **无需人工标注**：利用系统后期行为自动生成训练标签，降低部署门槛。
- **主动预测**：在写入时决策，不依赖未来查询，适合生产环境。
- **跨领域统一框架**：同一核心方法（逻辑回归 + 贪心背包 + 逐字提取）适用于智能体与对话，只需调整特征和成本定义。
- **实证扎实**：包含定性追踪（图 1、附录 F/G）、定量多层报告（TGC、AUC、召回、下游 QA、成本）、消融和自监督分析。
- **解决真实问题**：在 14 个任务中，只有 LRE 能完成，说明其解决了一个实际且严重的失效模式。

## 8. 不足与局限
- **实验覆盖有限**：仅在 AppWorld、LoCoMo、LongMemEval_S 三个基准上评估，未涉及多模态、多智能体、快速变化环境、Web 浏览或代码修复（如 SWE-bench）。
- **智能体仅单种子**：未运行多组随机种子，缺乏统计显著性检验。
- **预算瓶颈**：当任务所需状态超过预算时（如 Hard 难度），无压缩仍占优，LRE 不能替代无限保留。
- **单元粒度粗**：LRE 以“动作-观察对”或“对话轮”为单位，未与 token 级方法（如 H₂O、SnapKV）结合或比较。
- **特征依赖**：特征集为手工设计，未证明适用于所有部署场景；未评估对基座模型变化的泛化能力。
- **自监督标签阈值固定**：智能体重用阈值 b=3、对话重叠阈值 τ=0.4 虽经调优，但未必通用。
- **未与其他学习式保留方法直接对比**：如 Selection-p、TRIM-KV（它们需神经前向传播）；LRE 的外部学习方式不同，应有互补实验。
- **未讨论延迟影响**：虽然评分极快，但逐轮贪心背包扫描可能引入毫秒级延迟，在高频交互场景中需测试。
