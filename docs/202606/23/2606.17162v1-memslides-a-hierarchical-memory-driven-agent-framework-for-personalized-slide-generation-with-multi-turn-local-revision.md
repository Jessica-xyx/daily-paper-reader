## 摘要
个性化演示文稿生成需要的不仅仅是基于当前提示或模板的条件控制：智能体必须跨任务保留稳定的用户偏好，在多轮修订过程中保留新引入的偏好和约束，并可靠地执行局部编辑。我们提出MemSlides，一种用于个性化演示智能体的层次化记忆框架，它将长期记忆与工作记忆分离，并进一步将长期记忆分为用户画像记忆和工具记忆。用户画像记忆存储针对意图的条件画像，用于第0轮个性化；工作记忆在修订轮次之间携带活跃偏好和会话约束；工具记忆存储可复用的执行经验，以实现可靠的局部编辑。MemSlides将这种记忆设计与有范围的幻灯片局部修订相结合，因此目标更新作用于最小的受影响区域，而不是反复重新生成整个演示文稿。在受控实验中，用户画像记忆改善了多画像、多意图画像库上的角色一致性判断；工具记忆注入改进了诊断性配对设置中的闭环修改行为；定性案例说明了工作记忆传递偏好的能力。综合这些结果表明，演示文稿创作中的有效个性化取决于将持久用户画像、会话级工作记忆和可复用执行经验在生成与局部修订中分离。

## Abstract
Personalized presentation generation requires more than conditioning on a current prompt or template: agents must preserve stable user preferences across tasks, retain newly introduced preferences and constraints during multi-turn revision, and carry out local edits reliably. We propose MemSlides, a hierarchical memory framework for personalized presentation agents that separates long-term memory from working memory and further divides long-term memory into user profile memory and tool memory. User profile memory stores intent-conditioned profiles for round-0 personalization, working memory carries active preferences and session constraints across revision rounds, and tool memory stores reusable execution experience for reliable localized editing. MemSlides pairs this memory design with scoped slide-local revision, so targeted updates act on the smallest affected region instead of repeatedly regenerating the full deck. In controlled experiments, user profile memory improves persona-alignment judgments on a multi-persona, multi-intent profile bank, tool-memory injection improves closed-loop modify behavior in diagnostic matched-pair settings, and qualitative cases illustrate working memory's ability to carryover preferences. Taken together, these results suggest that effective personalization in presentation authoring depends on separating persistent user profiles, session-level working memory, and reusable execution experience across generation and localized revision.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有自动幻灯片生成系统虽能产出完整、视觉精美的演示文稿，但缺乏持久的个性化能力。用户在不同意图（如学术报告 vs. 商业演示）下有不同的布局、模板、风格偏好，且个性化往往通过多轮修订才逐渐显现。然而现有系统要么依赖单次提示或模板，要么在处理修订时重新生成整个演示文稿，导致小幅修改需与大量历史上下文竞争，多轮局部编辑脆弱且易偏离已对齐的内容。
- **整体含义**：有效的个性化演示文稿生成需要智能体具备跨任务保持用户稳定偏好、在多轮修订中保留新引入约束、并可靠执行局部编辑的能力。这要求显式的记忆设计，而非隐式地依赖提示词。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出**MemSlides**，一个层次化记忆驱动的智能体框架，将记忆分为长期记忆（含用户画像记忆和工具记忆）与工作记忆，并结合**有范围的局部修订（scoped slide-local revision）**。
- **关键技术细节**：
  - **层次化记忆框架**：
    - **长期记忆（Long-term Memory）**：
      - **用户画像记忆（User Profile Memory）**：按用户和意图（intent）维度组织长期偏好，包括主题、内容、视觉、布局、模板、通用等。在任务开始时，根据当前意图选择并路由到工作记忆，任务结束时稳定信号被合并回画像。
      - **工具记忆（Tool Memory）**：存储可复用的执行经验，包括回合级任务经验（round-scope task experience）和操作级工具链经验（operation-scope tool-chain experience），帮助智能体更可靠地执行编辑，减少重复错误和回溯。
    - **工作记忆（Working Memory）**：维护当前会话状态、活跃临时偏好、携带指令和编辑状态，支持多轮修订间的偏好传递。
  - **多轮局部修订执行（Multi-turn Localized Modify Execution）**：
    - **规划（Plan）**：将用户修订请求转为显式执行合约，记录推断的作用域、目标幻灯片路径、活跃规则标识符、选择器提示等。
    - **执行（Act）**：根据合约选择最小有效编辑操作（如批量CSS更新、语义批次样式、局部快照补丁、页面插入/删除等），避免全页重写。

## 8. 不足与局限

- **实验覆盖**：所有实验为受控的代理场景（controlled proxy settings），画像库和编辑请求由人工构造，非真实用户交互。缺少真实人类用户研究、开放式反馈或实际部署验证。
- **偏差风险**：角色对齐判断使用LLM-as-judge，可能存在偏好偏差；评委未看到原始提示但可能受演示风格影响。工具记忆配对场景选择是诊断性的，不具有全局统计代表性。
- **应用限制**：
  - 记忆合并策略（如冲突解决、会话偏好写入长期记忆）的鲁棒性未充分探索。
  - 隐私和安全：持久用户画像可能存储敏感信息，论文仅提及未来需要更强的记忆同意、删除和敏感偏好保障机制。
  - 系统依赖外部LLM API，实际部署成本与延迟需考虑。
- **其他局限**：论文未进行全量随机编辑集测试，也未比较不同记忆方案（如单缓存vs层次化）的完整消融。工具记忆消融仅9个配对，样本量较小。
