## 摘要
大型语言模型（LLM）智能体依赖长期记忆来支持超出有限上下文窗口的扩展交互和个性化辅助。现有的记忆智能体通过生成的写入、修订和删除操作主动更新外部记忆，但这些更新可能会遗漏重要信息、损坏现有记忆或引入不支持的产生幻觉内容。一旦存储，这些错误就会成为持久的系统状态故障，可能影响未来的推理和生成。在本文中，我们提出了TrustMem，一个旨在提高记忆巩固可信度的框架。TrustMem依赖记忆转换验证器来评估记忆更新的过渡过程在覆盖性、保留性和忠实性方面的表现。它进一步在同一记忆状态下构建候选更新之间的偏好对，从而使得偏好引导的强化学习能够直接优化记忆更新行为。大量实验表明，TrustMem提高了记忆的实用性和可靠性：它在MemoryAgentBench、HaluMem和Mem-alpha验证集上取得了最先进的结果，将HaluMem记忆提取的F1分数提高了12.14分，并且与每种错误类型的最强基线相比，分别将过渡级别的遗漏、损坏和幻觉减少了40.1%、79.1%和50.0%。

## Abstract
Large language model (LLM) agents rely on long-term memory to support extended interactions and personalized assistance beyond finite context windows. Existing memory agents actively update external memory through generated write, revise, and delete operations, but these updates may omit important information, corrupt existing memory, or introduce unsupported hallucinated content. Once stored, such errors become persistent system-state failures that can affect future reasoning and generation. In this paper, we propose TrustMem, a framework designed to improve the trustworthiness of memory consolidation. TrustMem relies on a Memory Transition Verifier to evaluate the transition process of memory updates in terms of coverage, preservation, and faithfulness. It further constructs preference pairs among candidate updates under the same memory state, enabling preference-guided reinforcement learning to directly optimize memory updating behaviors. Extensive experiments demonstrate that TrustMem improves both memory utility and reliability: it achieves state-of-the-art results across MemoryAgentBench, HaluMem, and the Mem-alpha validation set, improves HaluMem memory extraction by 12.14 F1 points, and reduces transition-level omission, corruption, and hallucination by 40.1\%, 79.1\%, and 50.0\%, respectively, compared with the strongest baseline for each error type.

# 论文总结：TRUSTMEM: Learning Trustworthy Memory Consolidation for LLM Agents with Long-Term Memory

## 一、论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：大语言模型（LLM）智能体在长期记忆交互中，自主生成的记忆更新（写入、修订、删除操作）常出现三类错误：
  - **遗漏（Omission）**：遗漏输入中的重要信息。
  - **破坏（Corruption）**：扭曲或错误地篡改已有记忆。
  - **幻觉（Hallucination）**：引入输入和先前记忆均不支持的事实。
  这些错误一旦存储成为持久性系统状态错误，会反复污染后续推理与生成，而现有终端奖励信号（如任务成功率）无法定位到具体哪一步更新出错，存在严重的**信用分配鸿沟**。

- **研究意义**：将长期记忆系统的挑战从“如何保留信息”转向“如何构建可信记忆”，即确保每一步记忆更新都是可信的，而不仅仅是最终记忆结果的好坏。

## 二、论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- **形式化**：将记忆巩固视为连续的记忆状态更新过程。智能体逐步处理输入流 C = {c₁, c₂, …, c_T}，每步观察当前块 c_t 和先前记忆状态 M_{t-1}，生成结构化记忆编辑动作 a_t（WRITE / REVISE / PRUNE），通过记忆执行器 E 更新到 M_t，形成局部转移 z_t = (c_t, M_{t-1}, a_t, M_t)。
- **核心创新**：引入**记忆转移验证器（Memory Transition Verifier）**，对每步局部转移进行可信度评估；并在此基础上提出**转移排名GRPO（Transition-Ranked GRPO）**，利用验证器分数构建偏好对，进行偏好引导的强化学习优化。

### 2.2 关键技术细节

#### （1）记忆转移验证器（Memory Transition Verifier, V）
- 实例化为一个冻结的LLM，接收当前块、动作、被影响的记忆条目，评估三个维度：
  - **覆盖性（Coverage）**：当前块的重要信息是否被保留。
  - **保存性（Preservation）**：已有有效记忆是否未被不当删除或扭曲。
  - **忠实性（Faithfulness）**：新添加/修改的内容是否得到输入或先前记忆的支持。
- 输出0~1的可信度分数 v_t = V(z_t)。

#### （2）多粒度奖励设计
- **样本级奖励**：
  - 任务奖励 R_task：下游任务性能（如F1或LLM裁判分数）。
  - 效率奖励 R_eff：鼓励紧凑记忆，1 - |M_T| / ∑|c_t|。
- **转移级奖励**：
  - 动作可执行性 R_act：结构化动作是否可被执行器解析执行。
  - 内容类型 R_type：修改内容是否语义具体、非空。
  - 验证器奖励 R_ver：即 v_t。
- 总奖励：R_{i,t} = R_task + λ_eff R_eff + λ_act R_act + λ_type R_type + λ_ver R_ver。

#### （3）转移排名GRPO（Transition-Ranked GRPO）
- **全局轨迹优化**：采用GRPO，对每个样本采样G个轨迹，计算组归一化优势，优化策略。
- **偏好对构建**：对同一状态下的G个候选转移，根据验证器分数选取最高和最低分（差距超过阈值δ）作为正/负对。
- **局部偏好排名损失**：采用DPO风格的排名损失，利用长度归一化的对数概率差，鼓励策略靠近正动作远离负动作。
- **最终目标**：L = -J_GRPO + β_KL L_KL + λ_rank L_rank，联合优化标量奖励、KL正则和转移级排名。
