## 摘要
当前的LLM角色扮演系统通过从小说中提取角色、场景和关系来构建角色代理。然而，长叙事角色扮演存在两个失败点：事实越界，即共享检索或参数记忆使角色使用其视角之外的事实；以及风格单调，即角色描述将角色扁平化为固定的声音。为了解决这些问题，我们提出了REVERIEMEM，一种针对基于书籍的角色代理的三层记忆架构。情节层存储第一人称场景记忆；语义层存储带有可见性标记的事实；个性层存储依赖情境的言语和行为模式。为了评估，我们构建了KBF-QA，一个涵盖八部小说的4386个问题的基准测试，用于测试知识边界。REVERIEMEM在知识边界保真度上比最强先前方法提高了34.6个百分点。在BOOKWORLD的五维成对叙事协议上，REVERIEMEM实现了约79%的胜率，表明视角受限记忆同时提高了边界保真度和基于角色的叙事生成。

## Abstract
Recent LLM role-playing systems build character agents from novels by extracting characters, scenes, and relations. Yet long-narrative role-playing suffers from two failures: Factual Overreach, where shared retrieval or parametric memory lets a character use facts outside its perspective, and Stylistic Monotony, where profile descriptions flatten a character into a fixed voice. To address these failures, we propose REVERIEMEM, a three-layer memory architecture for book-based character agents. The episodic layer stores first-person scene memories; the semantic layer stores visibility-tagged facts; and the personality layer stores situation-dependent speech and behaviour patterns. For evaluation, we construct KBF-QA, a 4,386-question benchmark over eight novels for testing knowledge boundaries. REVERIEMEM improves Knowledge Boundary Fidelity by 34.6 percentage points over the strongest prior method. On BOOKWORLD's five-dimension pairwise narrative protocol, REVERIEMEM achieves a ~ 79% win rate, suggesting that perspective-bounded memory improves both boundary fidelity and character-grounded narrative generation.

- **研究动机**：现有基于LLM的角色扮演系统从小说中提取角色、场景和关系来构建智能体，但在长叙事中常出现两种**OOC（出戏）失败模式**：

  - **风格单调（Stylistic Monotony）**：静态角色描述把角色压倒成单一表达模式（如福尔摩斯既是冷静推理者也是兴奋实验者，但系统只能输出一种风格）。
- **背景**：现有方法（如RAG、角色profile）缺乏对“角色知识边界”的显式控制，导致事实可靠但角色不真实，或角色一致但知识不可靠。
- **整体含义**：论文强调，可信的角色代理必须**在角色的知识范围内推理**并**动态调整表达风格**，为此提出视角受限的记忆架构。

## 2. 论文提出的方法论

### 核心思想
借鉴认知心理学三个理论（互补学习系统CLS、自我记忆系统SMS、叙事身份理论），将记忆分为三层，并强制检索范围限制在角色可见的事实子集内。

### 关键技术细节：REVERIEMEM 三层记忆架构

- **Episodic Layer（情节层）**：存储角色每次出现在场景中的第一人称总结（“我当时看到了什么、做了什么、感觉如何”），作为后续检索的锚点。
- **Semantic Layer（语义层）**：存储结构化事实（SPOCV元组：主体、谓词、客体、原因、可见性集合）。每个事实带有**每角色可见性**标签（通过直接体验、观察、组织共享、常识四种途径赋值）。推理时只能检索角色可见的事实子集 \( F_c \)。
- **Personality Layer（个性层）**：存储角色在不同情境下的语言/行为模式（从原文对话中聚类得到），并维护情绪转换记录。推理时根据当前情绪和对话上下文选择匹配的模式作为风格锚点。

### 流程三阶段（离线构建 + 在线推理）

1. **Source-to-Memory（离线）**：
   - 场景分割 → 每角色第一人称总结（Episodic）→ 知识图谱提取+可见性传播（Semantic）→ 行为聚类+情绪标注（Personality）。
2. **CLS Memory Collaborative Reasoning（在线推理）**：
   - 首先 **ANCHOR**：从Episodic层检索角色自己的场景记忆 \( S_c \)。
   - 然后 **Self-Probe + VIS-Retrieve**：在可见性子集 \( F_c \) 上循环检索，每次根据已有场景和事实自生成下一轮检索词（sub-query），直到能支持回答或达到最大轮次。
   - 核心约束：检索始终限制在 \( F_c \)，角色不可能知道的事实永不进入工作集。
3. **Fusion Injection**：
   - 并行，根据情绪变化选择匹配的Personality模式 \( m^* \)，将其风格注入最终的Memory Fusion（将场景、事实、风格模板整合输出回应）。

### 算法流程（文字说明）

1. 对查询 \( q \) 和角色 \( c \)，锚定Episodic层得到场景 \( S_c \)。
2. 生成初始检索探针 \( P^{(1)} \)，置事实池 \( M^{(0)}=\emptyset \)。
3. 循环 t=1..N：
   - 用 \( P^{(t)} \) 从 \( F_c \) 检索得到新事实，加入 \( M^{(t)} \)。
   - 检查 \( (S_c, M^{(t)}) \) 是否足够回答；若充足或达最大轮次，退出循环。
   - 否则生成新的探针 \( P^{(t+1)} \)。
4. 并行（每次推理前运行或维护状态）：
   - 通过Emo-Gate判断是否情绪触发，若是则选择新模式 \( m^* \)。
5. 最后Memory Fusion结合场景 \( S_c \)、事实池 \( M^{(r)} \)、模式 \( m^* \) 生成答案。
