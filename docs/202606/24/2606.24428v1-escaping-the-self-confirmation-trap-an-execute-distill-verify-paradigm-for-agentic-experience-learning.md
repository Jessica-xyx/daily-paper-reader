## 摘要
经验驱动的自我进化对于大型语言模型（LLM）智能体通过开放世界交互来提升自身能力至关重要。然而，现有的经验学习方法大多依赖单智能体循环，即同一个智能体执行任务、总结结果并决定记忆内容。这种设置使智能体容易陷入“自我确认陷阱”：错误但自洽的轨迹被误认为成功经验，导致在检索和重用过程中累积错误。为解决此问题，我们提出EDV，一种用于可靠经验学习的执行-蒸馏-验证框架。在执行阶段，多个异构智能体并行探索同一任务空间，生成多样化的候选轨迹。在蒸馏阶段，一个专用的第三方智能体通过比较分析这些轨迹生成候选经验，减少了以执行者为中心的总结偏差。在验证阶段，执行组通过共识机制验证候选经验，只有通过批准的经验才会被写入共享或私有记忆。通过解耦三个阶段，EDV将经验学习从孤立的自我反思转变为协作构建，在记忆插入前过滤错误和噪声内容。我们在三个具有挑战性的长周期基准（tau2-bench、Mind2Web和MMTB）上评估了EDV。结果表明，EDV始终优于强基线，验证了可靠的经验构建对于鲁棒的智能体自我进化至关重要。我们的代码可在 https://github.com/shidingz/EDV 获取。

## Abstract
Experience-driven self-evolution is critical for large language model (LLM) agents to improve through open-world interaction. However, existing experience learning methods mostly rely on single-agent loops, where the same agent executes tasks, summarizes outcomes, and determines memory content. This setup makes agents vulnerable to the Self-Confirmation Trap: wrong-but-self-consistent trajectories are misidentified as successful experience, leading to cumulative errors during retrieval and reuse. To address this issue, we propose EDV, an Execute-Distill-Verify framework for reliable experience learning. In the Execute stage, multiple heterogeneous agents explore the same task space in parallel to generate diverse candidate trajectories. In the Distill stage, a dedicated third-party agent comparatively analyzes these trajectories to produce candidate experiences, reducing executor-centric summarization bias. In the Verify stage, the execution group validates candidates via a consensus mechanism, and only approved experiences are written into shared or private memory. By decoupling the three stages, EDV transforms experience learning from isolated self-reflection into collaborative construction, filtering erroneous and noisy content before memory insertion. We evaluate EDV on three challenging long-horizon benchmarks: tau2-bench, Mind2Web and MMTB. Results show EDV consistently outperforms strong baselines, validating that reliable experience construction is essential for robust agent self-evolution. Our code is available at https://github.com/shidingz/EDV.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：大型语言模型（LLM）智能体在开放世界环境中持续交互时，需要从历史执行中积累可重用的经验，实现自我进化。然而现有经验学习方法（如 ReasoningBank）大多采用单智能体闭环：同一智能体既执行任务、又总结结果、还决定写入记忆的内容。这种耦合设计导致“自我确认陷阱”（Self-Confirmation Trap）：智能体将错误但自洽的轨迹误认为成功经验，写入记忆后在检索和复用中不断放大错误，尤其在缺乏显式真实反馈的长周期任务中更为严重。
- **核心问题**：经验学习的关键不仅在于“收集更多经验”，更在于“更可靠地构建经验”，在记忆插入前过滤掉错误和噪声内容。
- **研究意义**：本文首次系统识别单智能体经验学习的固有失败模式，并提出通过多智能体协作（解耦执行、蒸馏、验证）来提升记忆质量，从而推动智能体稳健的自我进化。

## 2. 方法论：核心思想、关键技术细节、算法流程

- **核心思想**：将经验学习从孤立的自我反思循环转变为协作式经验构建与过滤过程。通过“执行-蒸馏-验证”三阶段解耦，利用异构多智能体协作来抑制错误经验进入长期记忆。
- **关键技术细节**：
  - **执行阶段（Execute）**：从异构智能体池中随机采样一组执行智能体（2个），各自独立与环境交互，生成多条候选轨迹。异构性（不同基础模型或提示策略）增加解空间覆盖，暴露不同成功/失败模式。
  - **蒸馏阶段（Distill）**：从池中随机指定一个第三方蒸馏智能体（不与执行组重叠），对比分析多条轨迹，提取可重用的候选经验（1~3条）。蒸馏基于优先级“合规性 > 准确性 > 效率”，避免执行者自我总结的偏见。
  - **验证阶段（Verify）**：执行组智能体（下同）对所有候选经验进行共识验证（二进制判断），采用严格默认拒绝策略：
    - 一致通过 → 写入共享记忆库
    - 部分通过 → 写入对应私有记忆库
    - 否则丢弃
  - **推理时使用**：维护能力矩阵（Ability Matrix）记录各智能体的擅长的任务类型；对于新任务，先匹配最优求解器，再分层检索（先共享库，不足则查私有库），将检索到的记忆附加到上下文中辅助推理。
- **算法流程**（文字说明）：
  1. 对每个任务，从异构智能体池随机选取执行组和蒸馏智能体；
  2. 执行组并行生成轨迹；
  3. 蒸馏智能体对比轨迹，输出候选经验；
  4. 执行组验证候选经验，根据共识结果写入共享或私有记忆库；
  5. 更新能力矩阵；
  6. 推理时：根据能力矩阵选择模型，分层检索记忆，生成最终输出。

## 3. 实验设计：数据集、基准、对比方法

- **数据集与基准**：
  - **τ2-bench**：模拟航空、零售、电信三个服务域的长周期对话式任务，评估 Pass@1。
  - **Mind2Web**：跨任务、跨网站、跨域三种泛化设置，评估元素准确度（EA）、操作 F1（AF1）、步骤成功率（SSR）、任务成功率（SR）。
  - **MMTB**（Multi-Mission Tool Bench）：多工具任务，评估总体得分及动作级、多动作级、轨迹级指标。

  - **RB (ReasoningBank)**：代表性单智能体经验学习方法。
  - **Judge**：LLM-as-Judge 推理时对异构输出进行裁决。

## 6. 主要结论与发现

- **EDV 一致优于所有基线**：在 τ2-bench 上 Pass@1 达 86.6%，比最强 Router 高 3.1 个百分点；在 Mind2Web 三泛化设置下多数指标领先；在 MMTB 上总体得分 58.10，超 Router 约 2.1 个百分点。
- **记忆质量显著提升**：人工审计显示 EDV 写入记忆在正确性（4.41 vs 3.72）、可操作性（4.32 vs 3.58）、特异性（4.27 vs 3.64）均优于 RB，噪声和潜在危害减半。

- **解耦设计关键**：渐进消融表明，单智能体自验证无法解决问题，只有多智能体执行+第三方蒸馏+共识验证三者协同才能带来最大增益（从83.3提升至88.6）。
- **组件消融**：能力矩阵和记忆层次分别贡献2.0和2.7~2.9个百分点，私有记忆在近1/3的任务中有显著作用。
- **效率优势**：EDV 在 RETAIL 域相比 RB 降低24.5%推理 token 消耗，实现更好效果。

## 7. 优点

- **问题识别深刻**：首次明确提出并形式化“自我确认陷阱”，揭示单智能体闭环的根本缺陷。
- **方法创新清晰**：将执行、蒸馏、验证三个角色解耦，利用异构多智能体协作和共识机制提升经验可靠性，设计基于现有 pipeline 内置能力矩阵和分层记忆，无额外开销。

## 8. 不足与局限

- **共识偏差风险**：若所有异构智能体共享同一失败模式（如对某些规则均有盲点），共识机制可能错误验证噪声。
- **低质量智能体干扰**：显著偏弱智能体可能阻碍共识通过，导致有用经验被丢弃。
- **归因困难**：多轮交互中难以定位失败根因，对错误记忆的溯源和调试更复杂。
- **实验范围有限**：仅使用三种特定商业/开源模型，未在更大范围模型（如 LLaMA-3、Qwen-2.5、DeepSeek 等）上验证；仅在三个基准评测，未覆盖其他复杂开放世界环境。
- **记忆长期管理缺失**：目前采用简单追加策略，无剪枝、去重、合并机制，随任务增加记忆库可能膨胀，影响检索效率。
- **未探讨最优异构规模**：文中仅固定执行组为2个，蒸馏/验证角色随机，未系统研究智能体数量、异构程度对性能的影响。

- **应用限制**：假设环境可多次重置以收集经验，对于一次性交互或无环境模拟的场景适用性存疑。
