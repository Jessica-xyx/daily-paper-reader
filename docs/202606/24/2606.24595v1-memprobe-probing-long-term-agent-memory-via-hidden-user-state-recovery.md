## 摘要
长期记忆有望使LLM智能体在多次会话中变得更加胜任，保持对用户准确且不断演进的理解，这种理解由交互形成。然而在实践中，这种记忆主要通过下游行为来评估，例如后续回答、个性化质量或任务成功，这只能间接测试理解能力，而记忆本身在很大程度上未经审计。我们认为，长期记忆应作为可审计的交互后工件进行评估：在常规辅助之后，从智能体留下的记忆中能重建出什么样的结构化用户状态？我们在MEMPROBE基准中实例化了这一观点，在该基准中，配备记忆的智能体辅助模拟用户，每个用户携带一个隐藏的、基于分类法的用户状态库，在一系列泄漏控制的任务轨迹中，之后从智能体产生的记忆中重建该状态库，包括完整存储和top-k访问两种方式。基于合成真实数据以实现高效可扩展的测量，MEMPROBE涵盖50个模拟用户，每个用户有31个隐藏维度（共1550个恢复目标），并测试了5个代表性记忆系统。测试最先进的记忆智能体时，我们发现成功的辅助和可恢复的记忆表现为不同的能力。任务完成率几乎饱和，即使对于无记忆基线也是如此，而类别平衡的恢复率保持中等（约0.6），在top-k检索下进一步下降。MEMPROBE是第一个直接研究记忆恢复的基准，重建系统保留的用户状态，并对照真实数据进行评分。我们将恢复视为未来记忆智能体应优化的具体目标，而MEMPROBE则是向智能体被训练记住其用户、随着了解时间增长而更加忠实的环境迈出的一步。

## Abstract
Long-term memory promises LLM agents that grow more capable across sessions, maintaining an accurate, evolving understanding of the user that interaction forms. In practice, however, this memory is evaluated mostly through downstream behavior, such as later answers, personalization quality, or task success, which tests that understanding only indirectly and leaves the memory artifact itself largely unaudited. We argue that long-term memory should instead be evaluated as an auditable post-interaction artifact: after ordinary assistance, what structured user state can be reconstructed from the memory the agent leaves behind? We instantiate this view in MEMPROBE, a benchmark in which a memory-equipped agent assists simulated users, each carrying a hidden, taxonomy-anchored user-state bank, across a trajectory of leak-controlled tasks, after which that bank is reconstructed from the agent's resulting memory under both full-store and top-k access. Built on synthetic ground truth for efficient, scalable measurement, MEMPROBE spans 50 simulated users with 31 hidden dimensions each (1,550 recovery targets) and tests 5 representative memory systems. Testing state-of-the-art memory agents, we find that successful assistance and recoverable memory behave as distinct capabilities. Task completion nearly saturates, even for a memoryless baseline, while category-balanced recovery stays moderate (about 0.6) and drops further under top-k retrieval. MEMPROBE is the first benchmark to study memory recovery directly, reconstructing the user state a system retains and scoring it against ground truth. We see recovery as a concrete objective for future memory agents to optimize, and MEMPROBE as a step toward an environment where agents are trained to remember their users, growing more faithful the longer they know them.

### 1. 论文的核心问题与整体含义

- **研究动机**：当前LLM智能体的长期记忆评估主要依赖下游行为（如后续问答、任务成功率、个性化质量），这种间接评估无法区分回答是源于记忆、上下文猜测还是幸运一击，且记忆本身作为工件（artifact）未被审计。
- **核心问题**：如何在常规辅助交互后，直接从智能体遗留下的记忆工件中恢复出结构化、可审计的用户状态？以及任务成功是否等同于良好的记忆形成？
- **整体含义**：提出**恢复率（Recovery）**作为新评估轴，强调记忆应作为可审计的后交互工件，直接衡量智能体对用户的理解保留程度，为训练更忠实的记忆系统提供具体优化目标。

### 2. 论文提出的方法论

- **核心思想**：定义用户隐藏状态 $u = (u_1,...,u_{31})$，经过三个信息丢失步骤：  
  **expose**（交互中部分泄露证据 $e$）→ **collect**（智能体记忆策略写入工件 $m_{\text{final}}$）→ **recover**（从工件重建估计 $\hat{u}$）。通过对比真值与重建值，分离信息丢失原因。

- **关键技术细节**：
  - **模拟用户**：每个用户携带一个基于分类法（包含技能、知识、事件、自我模型、辅助偏好五类共31维）的隐藏状态库，由DeepPersona生成一致的人格档案。
  - **泄漏控制任务**：每个隐藏维度对应一个辅助任务，任务描述不直接提及目标维度，且经“反钓鱼”评审过滤。
  - **交互过程**：模拟用户与记忆智能体进行多轮对话，智能体可读写持久化记忆（仅记忆模块可变，其他组件固定）。
  - **恢复评分**：两种访问模式：`dump_all`（读取全部记忆）和`retrieve`（通过系统检索接口获取top-5条目）。使用槽位填充模型生成预测，由LLM评分为五级量表（0-1），最终取五类均衡平均得到类别平衡恢复分数$B$。
  - **失败归因**：对恢复分数低于0.75的目标，分阶段诊断：任务设计缺陷、模拟器披露不足、智能体引导不足、真实记忆失败。

### 3. 实验设计

- **数据集/场景**：50个合成用户（基于美国DeepPersona档案），每个31个隐藏维度，共1,550个恢复目标。每个任务最多25轮对话。
- **Benchmark**：MEMPROBE本身即为基准，包含用户库、任务集、交互记录、记忆工件、评分结果与归因标签。
- **对比方法**：5种记忆设置：
  - `nomem`：无记忆基线
  - `longctx_full`：原始逐轮存储，余弦检索
  - `amem`：A-Mem（Zettelkasten风格笔记链接与演化）
  - `mem0`：Mem0（提取式会话记忆）
  - `memt`：Mem-T（基于Mem-T-4B训练策略的层级记忆）

### 4. 资源与算力

- 论文报告了gpt-5.4-mini的累积token用量：五系统交互+评分约662M tokens，归因约27M tokens，离线构造约17M tokens。`memt`的本地推理在单张NVIDIA A100 GPU上运行Mem-T-4B（vLLM）。**未明确训练时长或GPU具体小时数**，但指出`memt`为预训练模型而非从零训练。

### 5. 实验数量与充分性

- **实验规模**：50用户 × 31维度 × 5系统 = 7,750个评分案例，含full store和top-k两种模式。进行了人类验证（模拟器满意信号96%合理，归因标签75%一致）。
- **充分性**：变量控制严格（仅改变记忆模块），覆盖多种主流记忆架构。但合成用户、任务池固定（非对抗性），且评分依赖LLM评委。总体充分但存在局限（见下方不足）。

### 6. 主要结论与发现

1. **任务成功≠记忆质量**：无记忆基线也能近满任务完成率，但恢复分数为0。下游行为无法区分记忆好坏。
2. **存储≠可检索**：`dump_all`恢复率中等（~0.6），但`retrieve`模式下平均下降8-18个百分点，说明记忆写入了但无法通过正常接口有效检索。
3. **事件记忆最难**：`Episodic Memory`恢复率最低（~0.3-0.4），仅约为`Assistance Preference`的一半（~0.7），因为需要绑定事件、语境和后果。
4. **辅助偏好最易恢复**：直接陈述的交互风格偏好被较好保留。
5. **系统排名因访问模式而异**：`dump_all`下`longctx_full`最高，`retrieve`下`amem`最佳，说明不同设计在写/读两方面权衡不同。

### 7. 优点

- **首次直接审计记忆工件**，而非依赖下游行为，提供了更透明的评估视角。
- **分离写/读失败**：通过`dump_all`与`retrieve`对比，能诊断信息丢失发生在写入还是检索阶段。
- **分类法基于成熟理论**：O*NET、Conway/McAdams自传记忆、HEXACO等，使隐藏状态具有结构性和可解释性。
- **合成用户可扩展、无隐私风险**，便于大规模对比实验。
- **失败归因机制**帮助研究者定位具体瓶颈，区分任务、智能体、模拟器、记忆自身的问题。

### 8. 不足与局限

- **合成用户限制**：仅基于美国DeepPersona档案，未覆盖多语言、跨文化、高度隐私保护或快速变化的真实用户行为。
- **任务难度低**：任务完成率接近饱和，无法区分不同系统的通用辅助能力，仅聚焦记忆学习。
- **评分依赖LLM**：评委为gpt-5.4-mini，可能存在极性表述偏差（如案例3中方向性标签导致分数反转），且未完全盲评。
- **计算成本高**：总token用量大，可复现性受API非确定性影响。
- **泛化性有限**：侧重于“日常辅助”场景，未涵盖对抗性或高度专业化领域；记忆系统的实际部署中的隐私、删除、同意等问题未涉及。
- **任务覆盖不均**：某些维度（如`Episodic Memory`）天然难以通过对话披露，可能导致恢复率被低估而非记忆系统缺陷。
