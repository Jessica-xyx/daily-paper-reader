## 摘要
大型语言模型（LLM）智能体越来越需要能够在长期交互中运行，其中必须保留和回忆过去对话中的信息以支持未来的任务。然而，随着交互的积累，记忆存储无限制地增长并充满冗余条目，这些条目会增加存储成本，并通过排挤最有用的证据而降低检索质量。此外，在资源受限且内存预算严格平台上，这一问题尤为突出，这促使我们提出存储预算下的内存管理任务，即在固定预算内保持已构建的记忆存储，同时保留对未来交互有用的信息。为此，我们提出了MemRefine，一个基于LLM引导的框架，由于表面相似性无法很好地反映事实价值，该框架仅使用相似性来提议候选对，并将删除、合并和保留决策延迟到基于事实内容的LLM评判器，迭代直到满足预算。在多个记忆框架和长期对话基准测试中，MemRefine始终能达到目标预算，同时保持下游性能，并在严格预算下优于基于规则的基线方法。

## Abstract
Large language model (LLM) agents are increasingly expected to operate over long-term interactions, where information from past dialogues must be preserved and recalled to support future tasks. However, as interactions accumulate, the memory store grows without bound and fills with redundant entries that inflate storage cost and degrade retrieval by crowding out the most useful evidence. Furthermore, this is especially limiting on resource-constrained platforms with hard memory budgets, motivating us to formulate storage-budgeted memory management, the task of keeping an already constructed memory store within a fixed budget while preserving information useful for future interactions. To this end, we then propose MemRefine, an LLM-guided framework that, since surface similarity poorly reflects factual value, uses similarity only to propose candidate pairs and defers delete, merge, and preserve decisions to an LLM judge based on factual content, iterating until the budget is met. Across multiple memory frameworks and long-term conversation benchmarks, MemRefine consistently meets target budgets while preserving downstream performance and outperforming rule-based baselines under tight budgets.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：LLM智能体在长期交互中，记忆存储无限制增长，充满冗余条目，导致存储成本膨胀和检索质量下降（拥挤掉最有用的证据）。这在资源受限平台（如边缘设备）上尤为严重，因为内存预算有硬上限。
- **目标**：提出**存储预算下的内存管理**任务——在固定存储预算内压缩已构建的记忆存储，同时保留对未来交互有用的信息。
- **关键洞察**：表面相似性不足以反映条目的实际事实价值；冗余、互补或不同的事实需要基于内容的判断而非仅靠相似度或图结构。

## 2. 方法论：核心思想、技术细节与算法流程

### 核心思想
- **MemRefine**是一个**后构建、LLM引导的逐对压缩框架**，作为独立模块插入在记忆构建与检索之间，不修改宿主记忆管道。
- **原则**：相似性仅用于提议候选对；压缩决策（删除、合并、保留）委托给LLM评判器，基于事实内容而非表面措辞。

### 关键技术细节
1. **候选对选择**：每次迭代选择当前记忆存储中余弦相似度最高且尚未被评判为“保留”的条目对 `(u, v)`。
2. **LLM评判器**：给定条目对，LLM结构化为返回动作（DELETE、MERGE、PRESERVE）、删除目标（若删除）或合并指令（若合并）。评判器被提示考虑事实冗余性、互补性、区别性，忽略低价值内容（问候、情感填充）。
3. **存储更新**：
   - **DELETE**：移除指定条目，若存在链接则剪除悬挂边。
   - **MERGE**：调用另一个LLM（合并模型）根据指令生成合并条目，替换原两个条目，继承并集链接，重新计算嵌入。
   - **PRESERVE**：两者原样保留，该对加入已判断集合。
4. **终止**：循环直到存储规模≤预算，或没有未判断的候选对（避免移除不可冗余的事实）。

### 算法流程（文字描述）
```
输入：初始记忆存储 M0，存储预算比例 ρ
初始化：预算 B = ρ * size(M0)，当前存储 M' = M0，已判断集合 S = ∅
while size(M') > B:
   (u, v) = 从 M' 中选取余弦相似度最高且不在 S 中的对
   若 (u, v) 为空则 break
   d = Judge(u, v)   # LLM评判
   若 d.action = DELETE: 从 M' 移除 d.target，修剪链接
   若 d.action = MERGE: 生成合并条目 m，替换 u,v，继承链接，更新嵌入
   若 d.action = PRESERVE: 将 (u,v) 加入 S
返回 M'
```

## 6. 主要结论与发现

- **MemRefine能稳定达成目标预算**，在中等压缩（70%–50%）下保持与未压缩存储相当的下游性能；收紧预算（40%–30%）时性能平滑下降而非崩溃。
- **LLM评判是关键**：规则基线在宽松预算下可行，但在严格预算下性能急剧下降；MemRefine始终保持更优。
- **压缩效果受记忆表示影响**：A-MEM图记忆冗余更多，压缩空间更大；Mem0已较紧凑，压缩余地较小。

- **强评判模型在严格预算下更有利**（gpt-5-mini优于Qwen3-8B）。
