## 摘要
个性化对话智能体需要持续的长期记忆以在多轮对话中保持连贯交互。然而，在消费级硬件（如8 GB显存边缘设备）上部署这些能力会带来严重的内存与计算瓶颈。现有系统通常依赖各向同性余弦相似度进行检索，并采用启发式规则进行上下文压缩。这些方法缺乏统一的理论基础，在高维检索中常遭遇中心点问题，并在压缩过程中出现语法碎片化。为克服这些限制，我们提出CoreMem——一种由信息几何根本上统一的资源高效边缘-云端记忆架构。首先，黎曼检索以局部自适应的费舍尔-拉奥度量替代余弦匹配，通过利用O(Ndr)伍德伯里加速的马氏距离有效惩罚中心记忆，实现实时搜索。其次，费舍尔引导的离散令牌蒸馏（FDTD）引入层次化句子到令牌的压缩机制，从费舍尔信息迹中推导灵敏度分数，提供了一种带有显式结构语法保护的原则性压缩-KL权衡。在LOCOMO和LongMemEval-S基准上，CoreMem实现了显著的准确率提升，在开放域（+4.51个百分点）和时间（+4.17个百分点）推理上取得大幅改进。大量性能分析证实，CoreMem可在严格的8 GB显存预算内无缝运行，成功弥合了资源受限边缘设备与对理论支撑的长时记忆智能体需求之间的差距。

## Abstract
Personalized dialogue agents require continuous long-term memory to maintain coherent interactions across multiple sessions. However, deploying these capabilities on consumer-grade hardware (e.g., 8 GB VRAM edge devices) introduces severe memory and compute bottlenecks. Existing systems typically rely on isotropic cosine similarity for retrieval and heuristic rules for context compression. These approaches lack a unified theoretical foundation, frequently suffering from the hubness problem in high-dimensional retrieval and syntactic fragmentation during compression. To overcome these limitations, we propose CoreMem, a resource-efficient edge-cloud memory architecture fundamentally unified by information geometry. First, Riemannian retrieval replaces cosine matching with a locally adaptive Fisher-Rao metric, effectively penalizing hub memories via Mahalanobis distance with O(Ndr) Woodbury acceleration for real-time search. Second, Fisher-guided discrete token distillation (FDTD) introduces a hierarchical sentence-to-token compression mechanism. It derives sensitivity scores from Fisher information traces, providing a principled compression-KL tradeoff augmented with explicit structural syntax protection. Evaluated on the LOCOMO and LongMemEval-S benchmarks, CoreMem achieves strong accuracy improvements, yielding substantial gains in Open-domain (+4.51 pp) and Temporal (+4.17 pp) reasoning. Extensive profiling confirms that CoreMem operates seamlessly within a strict 8 GB VRAM budget, successfully bridging the gap between resource-constrained edge devices and the demand for theoretically grounded, lifelong memory agents.

- **整体含义**：CoreMem旨在通过**信息几何**统一检索与压缩，实现资源高效（VRAM友好、时间友好、令牌友好）的边缘-云端混合记忆架构，弥合消费硬件与理论驱动的终身记忆智能体之间的差距。

## 2. 方法论
### 核心思想
- 从信息几何视角出发：将嵌入空间视为统计流形，检索用局部自适应Fisher-Rao度量（马氏距离）替代各向同性余弦；压缩通过Fisher信息矩阵（FIM）迹评估token重要性，提供理论有界的KL散度压缩误差。

### 关键技术细节
- **黎曼检索**：
  - 将每个记忆嵌入视为高斯分布的均值，协方差矩阵Σ通过奇异值分解低秩修正对角阵得到：Σ ≈ U_r M_r U_r^⊤ + D，其中D = diag(σ²+λ)，λ = 10·mean(σ²)防止极端几何扭曲。
  - 利用**Woodbury矩阵恒等式**将Σ⁻¹的计算复杂度从O(d³)降低至O(N d r)，实现实时检索。
  - 最终距离为余弦与黎曼得分的凸组合（融合权重α=0.5），兼顾全局语义匹配与局部几何感知。
- **Fisher引导离散令牌蒸馏（FDTD）**：
  - 通过单次反向传播计算每个token嵌入的梯度L2范数作为灵敏度：S(x_i) = ∥∇_{h_i} L ⊙ h_i∥₂ ≈ Tr(FIM)。
  - **定理1**：在局部L-Lipschitz假设下，按灵敏度升序丢弃token，生成KL散度有界于O(L² Σ_{丢弃} S(x_i))。
  - 层次化句子级压缩：先分句，对长句子保护首尾3个token（保留主谓宾骨架），丢弃中间低分冗余token；句子按平均分贪心选择至预算B。
  - 结构保护掩码：对逻辑连接词（but, because等）乘1.2×、标点1.3–1.5×、数字/专有名词1.3–1.4×；对话轮次线性衰减（1.0→0.6）；间隔≤3则自动填充。

### 算法流程（文字说明）
1. 预计算：标准化嵌入→计算对角方差→SVD选择秩r→Woodbury加速运算Σ⁻¹。
2. 在线检索：对查询q嵌入后，计算与N个记忆的余弦+黎曼融合得分，取top-k。
3. 压缩：对检索得到的上下文X，通过本地SLM前向与反向传播得到每个token灵敏度→平滑→乘结构掩码→按得分排序后保留top-B token→自动填充短间隔。

## 6. 主要结论与发现
- **CoreMem-Fusion**（α=0.5）在LOCOMO上取得最高Judge准确率0.540（MiniLM），尤其开放域+4.51pp、时序推理+4.17pp。
- 黎曼检索对**低维嵌入**（MiniLM 384维）提升更大（+3.70pp），高维嵌入（gte-Qwen2 1536维）提升较小，因高质量嵌入本身hubness较轻。
- FDTD在预算1000时条件准确率达0.9756，压缩率67.4%，优于启发式方法。但在极端预算（≤500 token）下所有学习型压缩器退化至低于简单截断。
- 黎曼与余弦互补：纯黎曼在开放域与时序任务更优，但多跳推理弱于余弦；融合实现最佳整体性能。
- 结构保护消融显示：移除语法/关键词提升（如逻辑连接词）导致Judge准确率骤降4.5pp，证实了对因果推理的关键作用。

## 7. 优点
- **理论统一性**：将检索和压缩纳入同一信息几何框架，提供压缩-KL界定理，不再是启发式拼凑。
- **资源高效**：保持8 GB VRAM可行，通过本地压缩降低云API令牌消耗（约20–30%），降低成本与隐私风险。
- **缓解hubness**：黎曼度量拉伸低方差维度，使边缘但关键的记忆能被召回。
- **结构保护机制**：显式保护语法骨架与逻辑连接词，防止压缩导致推理链断裂。
- **模块化设计**：检索与压缩可独立优化，融合机制灵活适应不同任务。

## 8. 不足与局限
- **边缘SLM能力有限**：FDTD依赖0.6B–1B模型，在极低预算（≤500 token）下表现不如简单截断，泛化能力受限。
- **语言与领域限制**：实验仅覆盖英语benchmark，结构掩码对形态丰富语言的适应性未验证。
- **可扩展性**：Woodbury加速虽降至O(N d r)，但记忆库超过10万时开销仍较大；目前使用扁平记忆，无显式层次组织。
- **实验覆盖不全**：缺乏“检索×压缩”全因子端到端消融，无法量化二者交互影响。
- **对标局限**：对比方法多为相同框架（余弦检索+启发式压缩），缺乏与最新知识图谱或结构化记忆系统的比较。
