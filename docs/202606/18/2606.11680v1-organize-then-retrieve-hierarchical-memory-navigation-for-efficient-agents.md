## 摘要
大语言模型（LLM）智能体在处理长时任务时存在固有无状态性的问题，需要将所有任务相关信息编码到不断增长的输入上下文中。由此导致的推理质量下降、推理成本增加以及延迟升高，促使我们需要高效的工作记忆机制。然而，现有方法要么依赖于有损压缩，要么依赖于基于相似性的检索，这些方法往往无法捕捉多步智能体任务所需的时间结构和因果依赖关系。在本工作中，我们提出HORMA，一种层级组织与检索记忆智能体，它将经验组织成类似文件系统的层级结构，其中摘要实体与对应的原始轨迹相链接，从而在保留详细信息的同时实现高效的访问。HORMA将工作记忆分解为两个阶段：结构化记忆构建和基于导航的检索。构建模块通过区分由信息缺失导致的失败和由上下文误导或过载导致的失败，迭代地改进经验的组织方式。导航模块通过使用一个通过强化学习训练得到的轻量级智能体来遍历层级结构，从而检索与任务相关的上下文，以选择最小但充分的上下文，从而减少关键执行路径上的延迟。在ALFWorld、LoCoMo和LongMemEval上，HORMA在受限上下文预算下提升了任务性能，同时在长对话任务中最多仅需基线token用量的22.17%。与现有方法相比，它始终能实现更好的效率-性能权衡，并能有效泛化到未见过的任务。

## Abstract
Large language model (LLM) agents struggle with long-horizon tasks due to their inherent statelessness, requiring all task-relevant information to be encoded in growing input contexts. The resulting degraded reasoning quality, increased inference cost, and higher latency necessitate efficient working memory mechanisms. However, existing approaches either rely on lossy compression or similarity-based retrieval, which often fail to capture temporal structure and causal dependencies required for multi-step agentic tasks. In this work, we present HORMA, a Hierarchical Organize-and-Retrieve Memory Agent that organizes experience into a file-system-like hierarchical structure, where summarized entities are linked to the corresponding raw trajectories, enabling efficient access without losing detailed information. HORMA decomposes working memory into two stages: structured memory construction and navigation-based retrieval. The construction module iteratively refines how experiences are structured by distinguishing between failures caused by missing information and those caused by misleading or overloaded context. The navigation module retrieves task-relevant context by traversing the hierarchy using a lightweight agent trained with reinforcement learning to select minimal yet sufficient context, thereby reducing latency along the critical execution path. Across ALFWorld, LoCoMo, and LongMemEval, HORMA improves task performance under constrained context budgets while requiring at most 22.17% of the baseline token usage in long conversation tasks. Compared to existing methods, it consistently achieves better efficiency-performance trade-offs and generalizes effectively to unseen tasks.

## 2. 方法论：核心思想、关键技术细节
### 2.1 总体框架：HORMA（Hierarchical Organize-and-Retrieve Memory Agent）
- **解耦设计**：将工作记忆分为两个独立模块：
  - **记忆管理器（Memory Manager, Mm）**：负责长期、异步的记忆结构构建
  - **检索智能体（Retrieval Agent, Mr）**：负责实时、在线路径上的高效上下文提取
- 两者通过共享的**层次化文件系统工作空间**交互，全部基于可执行的Bash命令（mkdir, nano, cd, grep, ls, cat 等）

### 2.2 结构化记忆构建
- **记忆管理器**接收原始交互轨迹，自动生成带时间戳的摘要笔记，链接到原始轨迹，组织成实体目录（如 /people, /events, /raw_logs）
- **初始化**：使用具有强层级推理能力的前沿LLM（如Claude Sonnet 4.5）设置领域无关的初始提示
- **递归技能精炼（Recursive Skill Refinement）**：
  - 通过对比分析成功/失败轨迹，区分两种失败模式：
    - **外源性（Exogenous）**：原始历史成功但HORMA失败 → 信息丢失
    - **内源性（Endogenous）**：HORMA成功但原始历史失败 → 结构化记忆缓解了幻觉/中间迷失
  - 为每类生成自然语言反馈，迭代扩充管理提示中的技能库
  - 可视为文本梯度下降，最终获得可迁移的非参数化记忆管理技能

### 2.3 导航式检索
- **检索智能体**使用Bash工具（ls, cd, grep, cat）结合专用动作：
  - `select`：将已验证内容加入检索上下文
  - `done`：终止检索
- **训练**：通过强化学习（GRPO，Group Relative Policy Optimization）优化轻量级模型（Qwen 3.5 4B）
- **奖励函数**：基于检索上下文Ct与证据E的Jaccard重叠：
  \[ J(C_t, E) = \frac{|C_t \cap E|}{|C_t \cup E|} \]
  鼓励精准且紧凑的上下文，惩罚冗余

### 2.4 整体工作流
1. 智能体与环境交互 → 生成原始轨迹
2. 记忆管理器将轨迹归档到文件系统，合成结构化笔记
3. 检索智能体接收问题/任务，在层级结构中导航，收集最小充分上下文
4. 主智能体（Mθ）基于检索到的上下文生成最终动作/回答

## 7. 优点
- **解耦设计巧妙**：将记忆构建（非时间关键、可异步优化）与检索（时间关键、可在线优化）分离，有效缓解信用分配问题
- **可解释性高**：所有操作均通过文件系统命令完成，可完整追溯和审计
- **高效且低成本**：轻量级检索器在低算力下快速推理，大幅降低token消耗和延迟
- **泛化性好**：RL训练的检索器能跨域（交互→对话）和跨任务类型（时序、知识更新）迁移
- **方法通用**：记忆管理可单独增强其他检索范式（如嵌入检索），不限于HORMA自身

## 8. 不足与局限
- **高容量需求**：记忆管理器需要前沿LLM（如Claude Sonnet 4.5）才能构建良好结构；小模型（Qwen 3.5 4B）管理时即使检索器再好，性能仍受明显限制
- **训练数据有限**：RL检索器只在一个基准的少量训练数据上训练（LoCoMo前7个对话），可能对全新领域/交互模式泛化不足
- **未进行全在线训练**：当前记忆构建仅通过离线对比精炼，未在任务执行中实时学习，技能获取依赖成功/失败轨迹收集
- **未见过多任务类别**：实验覆盖了交互和对话，但未测试更复杂的网页代理、工具使用等场景
- **未报告训练时间**：RL训练具体耗时未给出，无法评估训练计算成本
- **奖励设计依赖证据标注**：检索奖励需要任务证据集合，这在自由交互任务中不易获取，限制了方法的零样本应用
- **记忆结构静态**：技能库一旦生成，在运行中不更新，可能无法适应快速变化的交互模式
