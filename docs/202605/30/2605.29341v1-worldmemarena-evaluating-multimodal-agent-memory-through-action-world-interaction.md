## 摘要
多模态大语言模型越来越多地被部署为长周期智能体，其记忆必须超越简单的回忆：它必须跟踪不断演变的世界、修正过时的信息、并在决策时刻提供正确的证据。现有基准评估的是静态对话中的回忆，将记忆简化为单次任务结束时的准确度，并将视觉观察简化为标题，使我们无法定位写、维护、检索或使用阶段的失败。能够自主管理记忆的智能体框架的兴起加剧了这一差距，因为我们缺乏原则性的方法来比较手工设计的流水线与自我管理的替代方案。为弥合这些差距，我们将多模态智能体记忆形式化为一个具有可观察四阶段生命周期的动作-世界交互循环，并在WorldMemArena中实例化它：包含400个多会话多模态任务，涵盖终身演化（演化的个人和任务状态）和自主执行（来自真实观察、动作和反馈的记忆），并标注了黄金记忆点、更新、干扰项以及用于阶段级诊断的证据链。这使得首次能够对长上下文、手工设计（RAG和外部记忆系统）以及基于框架的记忆智能体进行头对头比较。结果表明：（1）更好的记忆写入和存储并不保证更好的性能；（2）多模态记忆仍然难以充分利用视觉证据；（3）系统在不同领域不稳定，并在现实自主轨迹上性能下降；（4）框架记忆更灵活，但仍然成本高且可靠性较低。

## Abstract
Multimodal large language models are increasingly deployed as long-horizon agents, where memory must do more than recall: it must track an evolving world, revise what has gone stale, and surface the right evidence at decision time. Existing benchmarks measure recall over static dialogue, collapse memory into a single end-of-task accuracy, and reduce visual observations to captions, leaving us unable to localize failures to writing, maintenance, retrieval, or use. The rise of agent harnesses that author their own memory sharpens this gap, since we have no principled way to compare hand-designed pipelines with self-managing alternatives. To close these gaps, we formulate multimodal agent memory as an Action-World Interaction Loop with an observable four-stage lifecycle, and instantiate it in WorldMemArena: 400 multi-session multimodal tasks spanning Lifelong Evolution (evolving personal and task states) and Agentic Execution (memory from real observations, actions, and feedback), annotated with gold memory points, updates, distractors, and evidence chains for stage-level diagnosis. This enables the first head-to-head comparison of long-context, manually designed (RAG and external memory systems), and harness-based memory agents. Results show that: (1) better memory writing and storage do not guarantee better performance; (2) multimodal memory still struggles to fully use visual evidence; (3) systems are unstable across domains and degrade on realistic agentic trajectories; and (4) harness memory is more flexible but remains costly and less reliable.

### 1. 核心问题与整体含义（研究动机和背景）
现有智能体记忆评估基准存在三个核心缺陷：  
- **召回导向**：仅测试模型对静态对话的回忆能力，而非在动态交互中如何利用记忆指导未来行动。  
- **缺乏生命周期级诊断**：只报告最终问答准确率，无法定位失败发生在写入、维护、检索还是使用阶段。  
- **模态单一**：大多为文本中心，将视觉观察简化为标题，缺乏真实多模态交互压力。  

此外，新兴的智能体框架（如OpenClaw、Codex）允许智能体自主管理记忆（自管理记忆），但缺乏与手工设计流水线（如RAG、外部记忆系统）的原则性比较方法。为此，论文提出**WorldMemArena**，将多模态智能体记忆形式化为**动作-世界交互循环**的四阶段生命周期，并通过细粒度标注实现阶段级诊断与统一对比。

### 2. 方法论：核心思想、关键技术细节
**核心思想**：将记忆视为一个可观察的四阶段生命周期过程：  
1. **写入（Write）**：从当前会话中识别未来有用的信息进行选择性保留。  
2. **维护（Maintain）**：将新信息整合到现有记忆中，支持修订、合并、删除。  
3. **检索（Retrieve）**：在决策时从记忆中召回相关证据。  
4. **使用（Use）**：将检索到的证据忠实用于最终回答或行动。  

**技术细节**：  
- 定义动作-世界交互循环：`o_t = Ω(z_t)`, `a_t = π(o_t, m_t)`, `(z_{t+1}, f_t) = E(z_t, a_t)`，其中Ω为观察映射，π为策略，E为环境响应。  
- 将完整轨迹分段为会话（session），每个会话仅提供局部上下文，世界状态跨会话持续演变。  
- 为每个会话标注三类结构化标签：**黄金记忆点**（应保留的信息）、**状态更新**（需修订的过时信息）、**干扰项**（似是而非但无关或过时的信息）。每个问答对配有**证据链**（所需黄金记忆点子集）。  
- 评估协议：阶段1用记忆召回率与正确性/幻觉/无关性评估写入；阶段2用更新处理成功率评估维护；阶段3用Recall@K和NDCG@K评估检索；阶段4用11种问题类型（基本、鲁棒性、推理、多模态四组）的联合评估（LLM裁判+F1+BLEU）评估使用。

### 6. 主要结论与发现
1. **更好的记忆写入不保证更好性能**：高记忆召回率并不必然带来高QA准确率，关键在于检索和使用环节。  
2. **多模态记忆瓶颈**：系统在复杂视觉任务（跨模态推理、视觉更新）上表现差，视觉证据常被降级为文本摘要。  
3. **系统不稳定且领域敏感**：在终身演化任务上表现优于自主执行任务；具身导航等长周期任务更困难。  
4. **框架记忆更灵活但不可靠**：OpenClaw/Codex等自主管理记忆的系统在部分任务上超越手工设计，但计算成本高、稳定性差。  
5. **记忆维护以追加为主，缺乏修订**：大多数系统倾向于积累新信息，很少删除或合并过时记忆。  
6. **检索是主要瓶颈**：增加检索K不一定提升性能，过长上下文会引入冗余/冲突/过时证据。  
7. **经验转化不足**：系统记住显式文本事实容易，但难以将工具反馈、失败动作、视觉细节转化为可复用知识。

### 7. 优点
- **系统性建模**：将记忆形式化为动作-世界交互循环的四阶段生命周期，提供了统一诊断框架。  
- **细粒度标注**：黄金记忆点、更新、干扰项、证据链支持阶段级评估，克服了传统单点准确率的局限。  
- **覆盖全面**：同时包含终身演化与自主执行两个互补领域，涵盖GUI、具身、对话等多种交互形式。  
- **多维度指标**：记忆质量、QA质量、检索质量多角度评估，使用LLM裁判+F1+BLEU联合避免单指标偏差。  
- **首次统一比较**：头对头对比了长上下文、手工设计、框架三种记忆范式，揭示了各范式优缺点。  
- **开源发布**：公开数据集和代码，便于复现与扩展。

### 8. 不足与局限
- **算力资源未公开**：缺乏GPU型号、训练时间等细节，影响可复现性。  
- **框架智能体评估受限**：由于框架内部记忆过程难以分解，仅做端到端评估，无法进行生命周期级诊断。  
- **用户研究缺失**：未进行真人用户评估，所有评估基于自动指标和LLM裁判，可能存在裁判偏差。  
- **领域覆盖有限**：虽然包含多个领域，但总数400个样本，每个领域样本量较小（如终身演化仅38个），可能不足以代表真实分布。

- **交互复杂性受限**：自主执行域基于预采集轨迹，并非开放世界的实时交互，可能低估真实动态环境的难度。  
- **测试时间学习能力弱**：系统在需要将经验转化为行动的任务上表现差，但基准本身以问答为主，并未直接测试行为改进。
