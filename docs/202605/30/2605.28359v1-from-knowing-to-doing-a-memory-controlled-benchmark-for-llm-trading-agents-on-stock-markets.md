## 摘要
评估大语言模型（LLM）智能体是否能在资本市场获利，越来越被表述为端到端交易：将智能体置于历史市场中，让其交易，并衡量投资组合收益。这种设置容易受到两种评估失败的影响。首先，长期回测往往与前沿LLM的知识截止日期重叠，使得记忆的股票代码、日期、价格和市场叙述可以替代投资推理。其次，原始收益是选股能力的噪声代理，因为正收益可能来自市场贝塔、风格暴露或有利的市场体制，而非真正的阿尔法。我们引入KTD-Fin（从知道到做的金融基准），一个端到端的股票市场交易基准，以解决这两个问题。KTD-Fin使用数据侧屏蔽协议，在提示和工具中一致地匿名化关键标识符和日历信息，将历史市场记忆与投资决策分离开来。它还包含Barra风格的表现归因框架，将投资组合收益分解为市场、风格和选股阿尔法成分。在2024-2026年窗口期的中国沪深300指数上评估的十个前沿LLM智能体中，屏蔽显著改变了智能体的理由，推动它们转向匿名化的因子推理。归因分析进一步表明，在泄露控制评估下，LLM智能体的累积收益主要由被动市场和风格暴露解释，持续选股阿尔法的证据有限。这些发现表明，金融LLM基准不仅应评估智能体是否赚钱，还应评估收益来源是否反映可转移的投资技能。我们发布KTD-Fin作为泄露控制和归因感知评估LLM交易智能体的可复现模板。

## Abstract
Evaluating whether large language model (LLM) agents can profit in capital markets is increasingly framed as end-to-end trading: place an agent in a historical market, let it trade, and measure portfolio returns. This setup is vulnerable to two evaluation failures. First, long backtests often overlap with the knowledge cutoffs of frontier LLMs, allowing memorized tickers, dates, prices, and market narratives to substitute for investment reasoning. Second, raw returns are a noisy proxy for stock-selection ability, since positive performance may come from market beta, style exposure, or favorable regimes rather than genuine alpha.   We introduce KTD-Fin (Knowing-To-Doing Financial Benchmark), an end-to-end stock-market trading benchmark that addresses both issues. KTD-Fin uses a data-side masking protocol to anonymize key identifiers and calendar information consistently across prompts and tools, separating historical market memory from investment decision-making. It also incorporates a Barra-style performance attribution framework that decomposes portfolio returns into market, style, and stock-selection alpha components.   Across ten frontier LLM agents evaluated on the Chinese CSI300 over a 2024--2026 window, masking substantially changes agent rationales, pushing them towards anonymized factor-based reasoning. Attribution analysis further shows that LLM agents' cumulative returns under leakage-controlled evaluation are largely explained by passive market and style exposure, with limited evidence of persistent stock-selection alpha. These findings suggest that financial LLM benchmarks should evaluate not only whether an agent makes money, but also whether the source of returns reflects transferable investment skill. We release KTD-Fin as a reproducible template for leakage-controlled and attribution-aware evaluation of LLM trading agents.

## 1. 核心问题与整体含义（研究动机与背景）

当前LLM交易代理评估面临两个关键失败模式：
- **记忆泄露**：长期回测与LLM训练数据截止日期重叠，模型可能直接调用记忆中的真实股票代码、日期、价格和市场叙事来替代真正推理，导致评估结果失真。
- **收益噪声**：单纯累积收益无法区分来源于市场贝塔、风格暴露还是真实选股α，正收益可能仅反映被动因子暴露而非主动投资技能。

论文旨在构建一个端到端股票交易基准，**同时控制记忆泄露并分离收益来源**，使评估更聚焦于可转移的投资能力。

## 2. 方法论

### 核心思想
- **数据侧屏蔽协议**：四层屏蔽（bright/stock-blind/date-blind/blinded），依次隐藏股票代码、日历日期及工具返回时间戳，确保模型无法通过标识符获取训练记忆。
- **Barra风格归因框架**：将每日组合收益分解为市场因子、风格因子和选股α，区分被动暴露与主动技能。

### 关键技术细节
- **执行引擎**：基于Qlib，模拟A股真实交易（T+1锁定、交易成本、涨跌停限制等），仅提供OHLCV及技术指标，无新闻/基本面。
- **三种决策模式**：
  - memory-only: 仅股票标识符+账户状态，测试对记忆的依赖。
  - fixed-candidate: 提供固定候选池（含因子表），测试因子信号选择。
  - open-research: 自由调用6种只读工具（市场上下文、候选筛选、快照等），全自主代理模式。
- **Barra归因公式**：  
  \( r_{i,t} = f_{0,t} + \sum_{k} x_{i,k,t-1} \lambda_{k,t} + \varepsilon_{i,t} \)

## 6. 主要结论与发现

1. **掩码改变交易行为**：在bright下，LLM rationale含有品牌/行业叙事（如“Kweichow Moutai, liquor leader”）；blinded下转变为纯因子排名（如“20-day return +27.4%, low vol 0.0255”），且memory+blinded下自愿现金持有率为0.00%，而bright下仍积极交易。  
2. **累积收益高估技能**：Barra归因表明，几乎所有LLM的累积收益主要来自市场和风格暴露（Common+Style贡献11%-29%），而选股α仅在Claude Opus 4.7为+0.2%，其余9个模型均为负（最低-77.8%）。  
3. **LLM因子暴露偏向高波动、近期动量股**，与传统机器学习模型的低波动动量相反。  
4. **掩码认证**：独立攻击显示，blinded下联合（代码Top-5+日期±7天）恢复率≤1.5%，有效切断记忆通道。

## 7. 优点

- **双缺陷同时控制**：数据侧掩码解决记忆泄露，Barra归因分离技能与运气，评估更干净。  
- **严格的掩码验证**：10个攻击者独立测试，确保掩码未被绕过。  
- **多维评估面板**：不仅报告收益，还包括行为、可靠性指标，提供更全面的代理画像。  
- **三种决策模式**：结构化解耦记忆依赖、信号选择、自主推理能力。  
- **可复现开源**：发布环境、脚本、掩码协议和完整评估数据集。

## 8. 不足与局限

- **市场覆盖有限**：仅中国A股沪深300，未测试美股、港股等更广泛市场。  
- **仅价格通道**：未纳入新闻、监管文件、宏观序列等多模态信息，真实交易中这些信息关键。  
- **全网格仅锚模型**：其他9个LLM未在全部掩码/模式组合下测试，跨模型深度对比不足。  
- **长期窗口单一掩码**：主要结论基于blinded×open-research，其他掩码下的长期性能未知。  
- **计算成本高**：每个LLM长期窗口需6-12小时API调用，限制了大规模多模型扫描。  
- **LLM版本漂移风险**：通过API调用，模型可能随供应商更新而变化，影响可复现性。  
- **选股α可能低估**：归因模型中的因子选择（9个风格因子）可能不完美，残差中仍包含未被捕获的因子暴露。
