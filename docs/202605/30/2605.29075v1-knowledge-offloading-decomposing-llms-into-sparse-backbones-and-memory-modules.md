## 摘要
大型语言模型（LLM）在单一参数集中编码了通用能力和领域特定知识。我们探究这种能力是否可以重新组织：将广泛有用的计算保留在共享骨干网络中，同时将专门化知识迁移到外部记忆模块中。我们提出知识卸载（KOFF）框架，用于将预训练的LLM分解为稀疏共享骨干网络和领域特定记忆。从冻结的基础模型出发，我们联合学习结构化剪枝掩码和轻量级恢复模块，后者以LoRA适配器和学习型键值缓存的形式实现。在3B到8B参数的Llama和Qwen模型上，我们发现非平凡的能力可以从共享骨干网络中移出，而不会导致模型能力大幅下降。在大约12%的全局稀疏度下，KOFF保留了未剪枝模型的大部分性能，而相同冻结模型在没有记忆的情况下进行剪枝则性能急剧下降。消融实验表明，LoRA和学习型KV记忆是互补的，而专门化分析表明这种学习到的分解是有意义的：语言特定的神经元被优先移除，而语言通用的神经元主要保留在骨干网络中。这些结果表明，知识可以在共享核心和可交换的外部记忆之间重新分配。

## Abstract
LLMs encode both general capabilities and domain-specific knowledge in a single set of parameters. We ask whether this capacity can be reorganized: keeping broadly useful computation in a shared backbone, while moving specialized knowledge into external memory modules. We propose \emph{knowledge offloading} (KOFF), a framework for decomposing a pretrained LLM into a sparse shared backbone and domain-specific memories. Starting from a frozen base model, we jointly learn a structured pruning mask and lightweight recovery modules, implemented as LoRA adapters and learned key-value caches. Across Llama and Qwen models from 3B to 8B, we find that non-trivial capacity can be moved out of the shared backbone without a large loss in model ability. At around 12\% global sparsity, KOFF preserves much of the unpruned model's performance, while pruning the same frozen model without memories degrades sharply. Ablations show that LoRA and learned KV memories are complementary, and specialization analyses suggest that the learned decomposition is meaningful: language-specific neurons are preferentially removed while language-general neurons largely remain in the backbone. These results suggest that knowledge can be reallocated between a shared core and swappable external memories.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：大型语言模型（LLM）在同一组参数中编码了通用能力和领域特定知识。这种混合存储方式缺乏显式的容量管理机制，导致不同领域知识相互竞争有限的参数容量，且难以实现模块化部署。
- **动机**：是否可以重新组织模型容量，使得广泛有用的计算保留在共享的骨干网络（backbone）中，而将专门化知识迁移到外部的、可交换的记忆模块（memory modules）中？这样既能保持核心能力的稳定性，又能灵活扩展领域能力，并可能实现“受控访问”（如敏感领域模块仅对特定用户开放）。
- **背景**：现有方法如检索增强生成（RAG）、参数高效适配器（LoRA）、学习型KV缓存等，虽然可以添加外部知识，但并没有决定哪些容量应留在模型内、哪些应外部化。结构化剪枝方法可以压缩模型，但只产生一个单一压缩网络，迫使不同领域知识竞争容量。本工作试图将剪枝视为一种学习记忆分配（memory allocation）的机制，统一记忆增强建模与结构化剪枝。

## 2. 提出的方法论：核心思想、关键技术细节

- **核心思想**：知识卸载（Knowledge Offloading, KOFF）框架，将预训练的LLM分解为一个稀疏的共享骨干网络和多个领域特定的记忆模块。通过联合学习结构化剪枝掩码（pruning mask）和恢复模块（recovery modules），决定哪些容量应保留在骨干中、哪些可以卸载到外部记忆。
- **关键技术细节**：
  - **共享骨干网络**：基于冻结的预训练模型参数，学习一个结构化剪枝掩码（使用Hard Concrete分布，实现L0正则化），移除输出通道级别的神经元。掩码是共享的（所有领域共用），剪枝后的骨干网络是紧凑的、稠密可运行的。
  - **领域特定记忆模块**：每个领域d对应一个模块M_d，包含两种互补的记忆形式：
    - **参数空间记忆**：LoRA适配器（低秩更新），对剪枝后线性映射的权重提供参数级别的校正。
    - **激活空间记忆**：可学习的KV缓存（learned key-value memory），在每个注意力层插入一组可训练的键值对（Nkv个token），允许文本位置关注这些记忆token，从而影响注意力计算。
  - **联合优化目标**：
    - 使用知识蒸馏（KL散度）从原始未剪枝模型（教师）到卸载后模型（学生），包括对领域数据的蒸馏损失（每个领域模块恢复教师在该领域的行为）和对通用保留数据的蒸馏损失（随机选择一个域模块，防止剪枝移除广泛有用的能力）。
    - 加上L0稀疏性惩罚（hinge损失，目标全局稀疏度约为12%）。
  - **推理**：训练后，剪枝掩码固定并物化（删除对应权重）。训练一个轻量路由器（RoBERTa分类器）根据输入选择正确的记忆模块。
- **训练过程**：冻结预训练参数，仅优化掩码参数和所有领域模块的参数。领域批次更新掩码和对应模块，保留批次仅更新掩码（随机选择一个模块）。

## 6. 主要结论与发现

- **主要结论**：通过KOFF框架，可以将预训练LLM中非平凡的领域特定能力从共享骨干网络迁移到外部记忆模块，同时保留大部分通用能力。在约12%全局稀疏度下，卸载后的模型在MMLU和困惑度上接近未剪枝模型，BBL甚至有所提升。相比之下，纯剪枝（无记忆）性能急剧下降，说明记忆模块对于恢复卸载能力至关重要。
- **具体发现**：
  - LoRA适配器和学习型KV缓存具有互补性，两者结合效果最佳。
  - 记忆模块放置在模型的上层（而非所有层）效果更好。
  - 语言特定神经元被优先剪枝（约67-71%被移除），而语言通用神经元几乎全部保留（仅4-5%被移除），说明剪枝自动实现了有意义的专业化。
  - 跨语言模块交换实验证实模块具有领域特异性：匹配模块困惑度最低，不匹配则升高。
  - 联合训练优于两阶段训练。

## 7. 优点

- **方法新颖**：首次将剪枝视为学习记忆分配的手段，将容量分配本身作为学习目标，不同于以往单纯压缩或固定骨干加适配器的方法。
- **设计合理**：采用冻结基模型+可训练剪枝掩码+轻量记忆模块，训练高效，且推理时只有一个模块激活，总参数与原始相近但具有模块化优势。

- **实验全面**：涵盖2个模型族、3个规模、2种领域类型，消融完整，结论稳健。
