## 摘要
现有记忆增强的LLM智能体通常将记忆视为具有预定义表示和固定检索管道的静态存储库，这在动态的智能体环境中是脆弱的，因为反馈、任务变化和异构信号不断重塑应该记住什么以及如何连接。为了解决这个问题，我们提出了FluxMem，一种连通性演化的记忆框架，将记忆建模为异构图，并通过三个阶段逐步优化其拓扑结构：初始连接形成、反馈驱动的细化以及长期巩固。在执行过程中，FluxMem修复缺失的连接、修剪干扰、对齐抽象粒度，并将反复成功的轨迹提炼为可重用的程序回路，由记忆泛化性和演化成熟度这两个指标引导。在包括LoCoMo、Mind2Web和GAIA这三个根本不同的基准测试中，FluxMem取得了持续的最先进性能，展示了在复杂智能体环境中的强大适应性和泛化能力。代码将在https://github.com/zjunlp/LightMem开源。

## Abstract
Existing memory-augmented LLM agents often treat memory as a static repository with pre-defined representations and fixed retrieval pipelines, which is brittle in dynamic agentic environments where feedback, task variation, and heterogeneous signals continuously reshape what should be remembered and how it should be connected. To address this, we propose FluxMem, a connectivity-evolving memory framework that models memory as a heterogeneous graph and progressively refines its topology through three stages: initial connection formation, feedback-driven refinement, and long-term consolidation. During execution, FluxMem repairs missing links, prunes interference, aligns abstraction granularity, and distills recurrent successful trajectories into reusable procedural circuits, guided by one metric for memory generalizability and evolutionary maturity. Across three fundamentally distinct benchmarks including LoCoMo, Mind2Web, and GAIA, FluxMem achieves consistent state-of-the-art performance, demonstrating strong adaptation and generalization in complex agentic environments. The code will be open-sourced in https://github.com/zjunlp/LightMem.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：现有记忆增强的LLM智能体将记忆视为静态存储库，采用预定义的表示和固定的检索管道。这种做法在动态环境中非常脆弱，因为反馈、任务变化和异构信号不断重塑“应该记住什么”以及“应该如何连接”。
- **核心问题**：静态记忆系统面临两个主要失败：(1) 自适应记忆连通性失败——检索不精确导致欠连接（遗漏关键上下文）或过连接（引入噪声和幻觉），且记忆单元内容固定，无法适应不同抽象层次的需求；(2) 记忆连接巩固失败——系统不能将局部更新通过反馈聚合成稳定的、大规模的关联区域，导致智能体反复重建相似关联，无法内化持久的结构模式。
- **整体含义**：本文重新定义记忆为“记忆单元及其连接的长时期沉淀”，它通过与环境交互持续塑造。模仿人类认知，记忆应在单元层面和连接层面都进行动态演化，从而形成自组织的最优记忆结构。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：提出FluxMem，将记忆建模为动态可编辑的异构图，包含三层节点（语义知识、情节经验、程序技能），并通过三个阶段逐步演化图的拓扑结构。
- **关键技术细节**：
  - **三层记忆图**：
    - `V_sem`：语义层，存储静态事实知识（如文档或块）。
    - `V_epi`：情节层，记录具体状态-动作轨迹（每个任务一个节点）。
    - `V_proc`：程序技能层，封装推理模板（如多步规划启发式）。
    - 边类型：`E_ground`（语义→情节，证据支持）、`E_distill`（情节→程序，技能蒸馏）。
  - **上下文作为动态诱导的连通性**：在每个步骤t，系统选择任务相关的节点子集形成局部子图`G_t(q)`，将其序列化为上下文`S_t(q)`。优化上下文即对子图进行拓扑编辑。
  - **三阶段演化流程**：
    - **阶段一：初始连接形成**——通过融合稠密嵌入相似性、稀疏BM25匹配和LLM验证的混合得分，从语义层检索事实；通过嵌入相似性检索情节层；通过遍历蒸馏边继承程序技能。
    - **阶段二：反馈驱动连通性细化**——根据执行反馈`f_t`进行闭环编辑：
      - 连接层面：若欠连接，扩展新边；若过连接，修剪干扰边。
      - 单元层面：若粒度不匹配，重塑节点内部内容（扩展或抽象）。
      - 循环直到任务成功或达到最大轮次`T`。
    - **阶段三：长期连接巩固**——离线进行：
      - 情节聚类与技能归纳：将完成的情节按语义相似度聚类，每个聚类由LLM提取共享模式生成技能节点。
      - PEMS引导的迭代巩固：计算技能的程序演化成熟度分数（PEMS），综合成功率、文本长度和版本差异，对低分技能重写，循环直到`ΔPEMS < ϵ`。
- **公式**：
  - 混合相关分数：`Score(v, o_t) = cos(v, o_t) + BM25(v, o_t) + LLM_ver(v, o_t)`
  - PEMS：`PEMS^(k) = η(V_proc^(k)) · log ℓ(V_proc^(k)) × (1 - δ(G_cons^(k), G_cons^(k-1)))`
    - η：成功率，ℓ：技能文本长度，δ：当前版本与前一版本的嵌入差异。

## 3. 实验设计

- **数据集/场景**：
  - **LoCoMo**：长上下文对话推理，包含10次长时间对话（平均588轮，16618 token），1540个人工标注问题（841单跳、282多跳、321时间推理、96开放域）。
  - **Mind2Web**：真实世界网页导航，2350个任务来自137个网站31个领域，平均页面1135个DOM元素，任务需7.3步。评估跨任务、跨网站、跨域三个维度。
  - **GAIA**：通用助手任务，165个任务，分Level 1-3（53、86、26个任务），难度递增。
- **对比方法**：
  - LoCoMo：Zep、Mem0、A-Mem、MemoryOS、Nemori、LightMem、MIRIX、EverMemOS、Full Context。
  - Mind2Web：AWM（含过滤/无过滤设置）、ReasoningBank。
  - GAIA：Langfun、OpenAI Deep Research、Magnetic-1、Agent KB、Alita、Smolagents、Flash-Searcher、MemEvolve。
- **评估指标**：
  - LoCoMo：LLM-as-judge（LMJ）分数。
  - Mind2Web：元素精度(EA)、Action F1(AF1)、步骤成功率(SSR)、总体成功率(SR)。
  - GAIA：各级任务成功率。

- FluxMem在所有三个基准上均取得**最先进性能**：

- **消融实验**显示：
