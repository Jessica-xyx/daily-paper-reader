## 摘要
现有的智能体记忆系统普遍遵循我们所谓的“记忆即工具”范式，即单个查询触发一次性的扁平段落列表检索，但存在被动调用、推理与检索解耦以及检索片段与智能体导航需求之间的结构不匹配等问题。我们提出了MemCog，一个“记忆即认知”系统，使记忆访问成为推理过程不可或缺的一部分。MemCog将用户知识组织为具有关联链接图的可导航记忆存储，提供跨维度导航接口以实现多步推理驱动的遍历，并采用主动推理协议，驱动智能体从对话语境中自发启动记忆探索。此外，我们构建了ProactiveMemBench，这是首个用于评估主动记忆触发的基准测试。实验表明，MemCog在被动问答基准测试中达到最先进水平（LoCoMo上92.98，LongMemEval上95.8），同时在ProactiveMemBench上大幅超越基线，证明了“记忆即认知”的优势。

## Abstract
Existing agent memory systems universally follow what we term a Memory-as-Tool paradigm where a single query triggers one-shot retrieval of flat passage lists, suffering from passive invocation, reasoning-retrieval decoupling, and structural mismatch between retrieved fragments and the agent's navigational needs. We propose MemCog, a Memory-as-Cognition system that makes memory access an integral part of the reasoning process. MemCog organizes user knowledge as Navigable Memory Store with associative link graphs, exposes Cross-Dimensional Navigation Interface for multi-step reasoning-driven traversal, and employs Proactive Reasoning Protocol that drives agents to spontaneously initiate memory exploration from conversational context. We additionally construct ProactiveMemBench, the first benchmark for evaluating proactive memory triggering. Experiments show that MemCog achieves state-of-the-art on passive QA benchmarks (92.98 on LoCoMo, 95.8 on LongMemEval) while substantially outperforming baselines on ProactiveMemBench, demonstrating the advantage of Memory-as-Cognition.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **现有范式（Memory-as-Tool）的缺陷**：当前对话智能体的记忆系统普遍采用“记忆即工具”范式，即用户查询触发一次性的平面段落列表检索。这带来三个根本性局限：
  - **被动调用瓶颈**：记忆仅在智能体显式调用检索工具时被激活，无法主动从对话上下文中自发触发回忆。
  - **推理与检索解耦**：检索是一次性调用，智能体的推理能力无法在检索过程中干预或引导后续探索，导致检索与推理相互独立。
  - **结构不匹配**：即使记忆采用层次化组织，检索返回的 top-k 片段仍缺乏导航结构信息，智能体无法从当前结果中推断下一步检索方向。

- **研究动机**：为了解决上述问题，推动记忆系统从被动的“工具”转变为融入推理过程的“认知”行为，使记忆访问成为智能体推理的一个有机组成部分。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出 **Memory-as-Cognition** 范式，将记忆访问与推理紧密耦合，通过多步导航、主动触发和结构化记忆存储，使智能体在对话中自发进行面向记忆的推理。

- **三大组件**：

  - **可导航记忆存储（Navigable Memory Store）**：
    - 采用三层层次结构：**维度**（Dimension，如个人偏好、工作、社交等） → **页面**（Page，覆盖连贯主题） → **章节**（Section，含标题、摘要、结构化事实、关联链接等）。
    - 定义四种跨维度关联链接类型：`related_to`（语义关联）、`temporal_next`（时序）、`caused_by`（因果）、`contrasts_with`（对比），为智能体提供显式的导航路径。
    - 构建方式：从对话历史中提取结构化事实 → 语义聚类形成维度和页面 → 建立链接。支持增量更新，包括事实匹配、矛盾检测、置信度更新。

  - **跨维度导航接口（Cross-Dimensional Navigation Interface）**：
    - 替代单次检索工具，提供一组多粒度的导航动作：
      - `list_dimensions()`：返回所有顶层维度。
      - `browse_dimension(dim)`：返回指定维度下的页面标题和摘要。
      - `read_page(page_id)`：返回页面的完整内容及所有链接。
      - `follow_link(link)`：沿超链接导航到目标页面。
    - 每个动作不仅返回内容，还返回结构上下文（可用链接、兄弟页面、维度位置），辅助智能体决定下一步。

  - **主动推理协议（Proactive Reasoning Protocol）**：
    - 通过系统 prompt 定义智能体的记忆使用行为规范，包括两种模式：
      - **主动关联**：当对话中出现的人、地点、话题可能关联存储知识，或用户陈述与记忆矛盾时，智能体自发启动记忆导航。
      - **反应式响应**：用户明确询问历史信息时，按需导航。
    - 规定导航要渐进、不满足于第一个结果，要整合信息、自然融入对话，并先思考再行动。
    - 选择性呈现：智能体在呈现信息前会评估相关性、时效性、敏感性和冗余性。

- **ProactiveMemBench 基准构建**：
  - 采用六步流程：生成用户画像 → 生成60个记忆单元（5种触发类型：时间、实体、情感、行为模式、多跳） → 评分单元间关联 → 图感知分配对话会话 → 生成20轮对话 → 生成500个评估实例（5个主题域）。
  - 每个实例包含一个触发语句和三个ground-truth候选记忆单元，智能体应主动回忆。

## 8. 不足与局限

- **多步导航的token开销**：相比单次检索，多步导航会增加推理延迟和成本，尤其在严格延迟或成本约束场景下可能不适用。
- **依赖LLM指令跟随能力**：主动推理协议的效果高度依赖backbone LLM对prompt的理解和执行，弱模型（如GLM-5.1）性能方差较大。
- **页面构建质量敏感**：导航质量受限于页面构建的准确性，如事实提取不准、链接缺失会级联影响整个系统。
- **早期对话导航优势有限**：在对话历史稀疏时，记忆存储连接不足，多步导航无法提供显著增益。
- **未讨论训练成本与部署可行性**：论文仅聚焦推理范式，未涉及记忆构建所需的计算资源或大规模部署的工程挑战。
- **实验未测试超长对话或极高步数**：当前限制最大工具调用为6次，未探索在极端复杂场景下的失败边界。
