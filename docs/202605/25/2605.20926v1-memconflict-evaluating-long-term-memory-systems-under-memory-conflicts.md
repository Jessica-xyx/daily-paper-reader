## 摘要
长期记忆系统使基于大型语言模型（LLM）的对话代理能够在多会话交互中保留、检索和应用用户特定信息。然而，现有评估主要关注结果层面性能或时间更新，对于系统如何在冲突候选项中检索和排序时间有效、事实正确且上下文适用的记忆证据提供的见解有限。为填补这一空白，我们提出MemConflict，一个将记忆有效性视为查询条件适用性问题的诊断框架。MemConflict形式化了关于时间有效性、事实正确性和上下文适用性的动态、静态和条件冲突。它从结构化用户档案中模拟受控的长期历史，引入跨会话冲突，并注入语义相似的干扰项以在记忆候选项中制造竞争。由此产生的多会话对话基准支持最终答案的黑盒评估和支持记忆检索与排序的白盒分析。在六个代表性长期记忆系统上的实验显示，各冲突类型上的优势不均衡，答案正确性常与记忆检索和排序不一致。敏感性分析表明，更长的历史、干扰项、隐式查询和更大的冲突距离会降低性能。诊断显示失败源自缺失支持记忆和已检索记忆的无效使用。总体而言，MemConflict通过检索感知、冲突感知的可靠性评估推进了原则性的长期记忆治理。

## Abstract
Long-term memory systems enable conversational agents based on large language models (LLMs) to retain, retrieve, and apply user-specific information across multi-session interactions. However, existing evaluations mainly assess outcome-level performance or temporal updating, providing limited insight into how systems retrieve and rank temporally valid, factually correct, and contextually applicable memory evidence under conflicting alternatives. To address this gap, we propose MemConflict, a diagnostic framework that treats memory validity as a query-conditioned fitness-for-use problem. MemConflict formalizes dynamic, static, and conditional conflicts over temporal validity, factual correctness, and contextual applicability. It simulates controlled long-horizon histories from structured user profiles, introduces cross-session conflicts, and injects semantically similar distractors to create competition among memory candidates. The resulting multi-session dialogue benchmark supports black-box evaluation of final answers and white-box analysis of supporting-memory retrieval and ranking. Experiments on six representative long-term memory systems show uneven strengths across conflict types, with answer correctness often diverging from memory retrieval and ranking. Sensitivity analyses reveal that longer histories, distractors, implicit queries, and larger conflict distances degrade performance. Diagnostics show failures from missing supporting memories and ineffective use of retrieved memories. Collectively, MemConflict advances principled long-term memory governance through retrieval-aware, conflict-aware reliability assessment.

## 1. 核心问题与整体含义（研究动机和背景）
- **研究动机**：现有LLM长期记忆系统的评估主要关注最终答案准确性或时间更新能力，忽略了在记忆冲突（如过时信息、矛盾陈述、条件依赖偏好）下系统如何检索和排序有效记忆证据。用户信息在长期交互中会变化、矛盾或仅在特定条件下适用，系统需要识别时间有效、事实正确和上下文适用的记忆。
- **整体含义**：缺乏诊断性评估框架来区分不同冲突类型（动态、静态、条件），且多数评估只关注黑盒答案，无法诊断失败根源（是检索缺失、排序错误还是生成失效）。MemConflict旨在填补这一空白，推动记忆系统的检索感知和冲突感知可靠性评估。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：借鉴信息质量理论中的“适用性”（fitness-for-use）视角，将记忆有效性定义为查询条件下的时间有效性、事实正确性和上下文适用性。形式化三种冲突：
  - **动态冲突**：用户状态随时间更新，后出现的真值应覆盖前值。
  - **静态冲突**：不变属性被错误矛盾陈述挑战，但真值应保持不变。
  - **条件冲突**：同一属性在不同条件下有不同有效值，查询需匹配正确条件。
- **关键技术细节**：
  - **用户档案初始化**：从Persona Hub种子生成结构化档案（不变属性、动态属性、条件偏好、性格目标、相关实体）。
  - **时间线模拟与冲突构建**：模拟2022-2025年的多会话历史（每月粒度），在指定会话中注入动态更新（真值变化）、静态矛盾（假陈述）、条件绑定（条件-值对）以及来自相关实体的语义相似干扰物。
  - **对话生成**：两阶段过程——先生成会话概要，再扩展为多轮对话，确保所有冲突信息自然表达。
  - **查询与标签构建**：在冲突变得可评估时（所有相关信息出现后）立即发起查询；动态/静态冲突询问有效值，条件冲突询问与查询值对应的条件。
  - **评估协议**：
    - **黑盒评估**：答案准确率（AA），动态冲突附加更新顺序一致性（UOCS），静态冲突附加冲突识别率（CRS）。
    - **白盒评估**：支持证据命中率@K（SEH@K）和支持排位分数（SRS），分析前K个检索记忆是否包含黄金记忆及其排名。
- **算法流程**（文字说明）：见文中Algorithm 1：对每个用户，生成初始化档案 → 模拟会话时间线 → 在各会话中插入动态更新、静态矛盾、条件绑定、干扰物 → 生成多轮对话 → 为可评估冲突实例构建查询和黄金标签。

## 3. 实验设计
- **数据集/基准**：MemConflict自建基准，共12个虚拟用户实例，每个实例约52个会话、2300+对话轮次、20万+ token，平均124个评估查询。动态、静态、条件冲突查询数量不等（动态最多）。
- **对比方法**：6个代表性长期记忆系统：
  - A-Mem（智能体记忆，链接笔记）
  - LangMem（工具包式，支持提取、整合、更新）
  - Letta（持久代理，分层记忆）
  - MemOS（操作系统式记忆管理）
  - Mem0（轻量持久层）
  - Memobase（用户档案中心）

## 6. 主要结论与发现
- **系统表现不均衡**：无系统在所有冲突类型上占优。MemOS总体黑盒最高（平均AA=0.5539），LangMem在动态冲突上领先（AA=0.4966），但静态和条件冲突弱。

- **干扰因素影响**：更长的历史、隐式查询、更大冲突距离、干扰物注入均显著降低性能。
- **失败诊断**：大多数错误来自检索失败（黄金记忆未进前K），但部分系统（如LangMem动态冲突）存在利用失败（检索到了但未正确使用）。
- **效率差异大**：MemOS总运行时间最低（1356秒），Mem0最高（41210秒），添加时间主导成本。

## 7. 优点
- **创新性诊断框架**：首个系统化形式化三种记忆冲突类型（动态、静态、条件），并引入干扰物，比现有评估（如LoCoMo、LongMemEval）更精细。
- **双层评估协议**：黑盒+白盒分离答案正确性与记忆检索/排序，提供故障根源定位（检索失败 vs 利用失败）。
- **受控实验设计**：通过仿真生成一致且可解释的冲突实例，便于因果关系分析。
- **敏感性分析全面**：覆盖检索深度、历史长度、干扰物、查询公式化、冲突距离等多个维度，揭示鲁棒性瓶颈。
- **实用诊断指标**：EUG、UOCS、CRS等提供具体优化方向（如需要冲突感知重排序、显式时间戳编码）。
