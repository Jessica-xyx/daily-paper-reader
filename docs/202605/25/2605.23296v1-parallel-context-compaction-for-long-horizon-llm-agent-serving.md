## 摘要
长时程LLM智能体积累不断增长的对话历史，最终超过模型的上下文窗口。通过基于LLM的摘要进行上下文压缩可以保持对话有边界，但摘要本质上有损，且阻塞调用会使智能体推理停滞数十秒。此外，由于提示指令很大程度上被忽略，操作者无法对摘要体积进行细粒度控制，并且随着上下文增长，模型生成的输出令牌数量及其保留的信息在不同运行之间波动很大，使得智能体跨运行保留的知识不可预测。我们引入了用于长时程智能体流的**并行压缩**，并在HotpotQA多跳QA和LoCoMo长上下文对话基准上，针对从8B到120B参数的四个骨干模型（混合了密集和MoE架构，以及推理和非推理模型）与顺序同步基线进行了对比。并行压缩为操作者提供了对摘要体积的细粒度、可预测的控制，并使得每个块能进行更有针对性的提示工程。在匹配的压缩解码量下，与顺序基线相比，它减少了端到端的墙钟时间并提高了压缩吞吐量。

## Abstract
Long-horizon LLM agents accumulate growing conversation histories that eventually exceed the model's context window. Context compaction via LLM-based summarization keeps the conversation bounded, but summarization is inherently lossy and the blocking call stalls agent inference for tens of seconds. Moreover, the operator has no fine-grained control over summary volume since prompt instructions are largely ignored, and as context grows, both the amount of output tokens the model produces and the information it retains fluctuate substantially from run to run, making the agent's retained knowledge unpredictable across runs. We introduce \textbf{parallel compaction} for long-horizon agentic flows and characterize it against the sequential synchronous baseline across four backbones spanning 8B to 120B parameters, mixing dense and MoE architectures with reasoning and non-reasoning models, on the HotpotQA multi-hop QA and LoCoMo long-context dialogue benchmarks. Parallel compaction gives the operator fine-grained, predictable control over summary volume and enables more targeted prompt engineering per block. At matched compaction decode volume, it reduces end-to-end wall time and improves compaction throughput over the sequential baseline.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：长期运行的LLM智能体（如代码助手、多轮对话系统）会累积越来越长的对话历史。这会带来三个主要问题：
    1.  **上下文窗口溢出**：历史长度最终会超过模型的上下文窗口限制。
    2.  **推理质量下降**：即使未超出窗口，模型在处理冗长、信息密度低的上下文时，也会出现“大海捞针”和“上下文腐烂”等问题，导致性能下降。
    3.  **高性能开销**：处理长序列会显著增加每token的延迟。
- **现有解法的局限性**：目前主流的解决方案是**上下文压缩**，即定期用LLM对整个历史进行摘要。但该方案存在严重问题：
    - **有损与不可控**：摘要本质上是信息丢失的过程，且压缩比极高（通常丢弃90-99%的token）。
    - **阻塞开销**：压缩是同步阻塞的，会暂停智能体推理，在长时间交互中成为性能瓶颈。
    - **体积不可预测**：模型几乎无视提示指令，生成的摘要长度和内容在运行间高度不稳定，使智能体保留的知识难以预测。
- **研究动机**：克服顺序同步压缩的固有缺陷，为长时程智能体服务提供一种更高效、可控、可预测的上下文压缩方法。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：**并行上下文压缩**。将长上下文分区，并行地对每个块进行独立摘要，最后合并，取代单次阻塞式顺序摘要。
- **关键技术细节与流程**：当上下文长度超过阈值 `τ` 时，触发压缩，分为三个阶段：
    1.  **快照与分区**：复制当前对话历史 `X`，记录其长度，并将其划分为 `N` 个连续的块，每个块大小为 `B` tokens。块大小 `B` 是一个可配置的旋钮。
    2.  **分发**：为每个块 `k` 构建一个独特的提示。该提示包含从块1到块 `k` 的全部上下文，但只有块 `k` 用XML标记（`<TARGET_BLOCK>...</TARGET_BLOCK>`）包裹，标记其为摘要目标。所有 `N` 个提示并发发送给LLM服务（vLLM）。
    3.  **合并**：等待所有工作线程完成后，将 `N` 个按块生成的摘要按原始顺序拼接，形成压缩后的历史 `Ĥ_t`，替换原有快照。
- **关键设计（前缀感知的目标在末端布局）**：
    - 不是给每个工作线程独立的块，而是让它看到**完整的、到自身块为止**的前序历史。
    - 这种设计有两个好处：
        - **保持跨块上下文**：每个线程在摘要时了解之前的对话，保证连贯性。
        - **利用前缀缓存**：每个工作线程的输入是前一个线程输入的严格扩展，可以高效复用vLLM的prefix cache，减少预填充开销。
- **核心优势**：
    - **可控性**：操作者可以通过调整**块大小 `B`** 来精细控制摘要的总输出体积。块越小，工作线程越多，输出的总tokens越多，保留的信息也更多。
    - **提示工程效果恢复**：由于每个块的内容更聚焦，模型对提示（如要求“简要”或“详细”）的敏感度恢复，实现了“块大小 + 提示”的双轴控制。
    - **效率提升**：并行处理显著降低端到端的墙钟时间，并提高了压缩阶段的吞吐量。

### 6. 论文的主要结论与发现

1.  **顺序压缩不可控**：摘要输出长度对输入长度和提示指令高度不敏感。当输入长度增长48倍时，输出仅增长约3倍；且提示指令（从“简要”到“详细”）对输出长度影响甚微。
2.  **顺序压缩运行间不稳定**：输出长度和语义内容在重复运行间高度不稳定（高变异系数，低余弦相似度），且随上下文增长而加剧。
3.  **并行压缩实现可控性**：块大小 `B` 成为控制摘要体积的有效旋钮。减小块大小（增加工作线程数）会线性地增加总输出token数，并相应提升下游任务准确率。
4.  **并行压缩提升效率**：在与顺序压缩输出相似token量的情况下，并行压缩的吞吐量（ms/tok）更高，在匹配体积下吞吐量可提升2倍以上。
5.  **恢复提示工程效果**：在并行压缩中，模型对提示指令的响应恢复，允许操作者通过“块大小”和“提示”双旋钮进行更精细的体积控制。
6.  **预填充成本是限制因素**：在块大小较大（如16k）时，每个工作线程需要预填充大量未缓存的独特内容，导致性能下降。更小的块大小能更好地发挥并行优势。
