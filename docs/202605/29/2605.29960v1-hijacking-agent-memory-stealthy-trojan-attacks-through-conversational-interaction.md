## 摘要
大型语言模型（LLM）智能体日益依赖长期记忆来支持持久且自主的任务执行。然而，这一能力也引入了新的攻击面：记忆投毒，攻击者可注入恶意信息以影响未来行为。现有记忆投毒攻击通常假设注入内容可直接存储于记忆中，忽视了现代记忆流水线中的选择性提取和重写阶段。这使得先前方法在实际设置中效果不佳。
本文提出MemPoison，一种新型记忆投毒攻击，可绕过LLM智能体的选择性记忆机制，攻击者通过对话交互将可触发的后门注入智能体的长期记忆，从而误导其后续响应。MemPoison引入三个关键组件：（i）语义关系桥，将触发器和负载绑定为连贯语句，确保它们一同被提取至记忆；（ii）实体伪装，优化触发器以模仿命名实体，抵抗重写；（iii）联合嵌入优化，将注入触发器的文本在嵌入空间中形成紧密簇，同时与良性嵌入保持隔离以实现隐蔽。在不同智能体领域和记忆机制上的评估显示，MemPoison的攻击成功率高达0.95，优于现有基线。机制分析表明，该攻击利用嵌入空间各向异性并转移注意力模式，凸显了选择性记忆系统中的核心漏洞。我们评估了多种防御策略，并展示了它们在缓解该攻击方面的根本局限性。

## Abstract
Large language model (LLM) agents increasingly leverage long term memory to support persistent and autonomous task execution. However, this capability also introduces a new attack surface: memory poisoning, where adversaries can inject malicious information to influence future behavior. Existing memory poisoning attacks often assume that injected content can be stored directly in memory, overlooking the selective extraction and rewriting stages in modern memory pipelines. This makes prior methods ineffective under realistic settings.   In this paper, we propose MemPoison, a novel memory poisoning attack that bypasses selective memory mechanisms in LLM agents, where an attacker can inject triggerable backdoors into the agent's long-term memory through dialogue interactions, thereby misleading its subsequent responses. MemPoison introduces three key components: (i) a semantic relational bridge that binds the trigger and payload into a coherent statement to ensure they are extracted into memory together; (ii) entity masquerading that optimizes triggers to mimic named entities, resisting rewriting; and (iii) joint embedding optimization that shapes trigger-injected texts into a tight cluster in the embedding space while maintaining isolation from benign embeddings for stealth. Evaluations across different agent domains and memory mechanisms show MemPoison achieves attack success rates up to 0.95, outperforming existing baselines. Mechanistic analysis indicates that the attack exploits embedding-space anisotropy and shifts attention patterns, highlighting core vulnerabilities in selective memory systems. We evaluate multiple defense strategies and demonstrate their fundamental limitations in mitigating the attack.

- **背景**：大型语言模型（LLM）智能体越来越多地依赖长期记忆（Long-Term Memory）来支持持久、自主的任务执行。然而，这种能力引入了新的攻击面——**记忆投毒**（memory poisoning），攻击者可通过看似良性的对话交互，向智能体的长期记忆注入恶意信息，从而在后续查询中持久地操控其行为。
- **现有工作的不足**：已有的记忆投毒攻击（如 AgentPoison、MINJA）通常假设注入内容可直接写入记忆，或忽视了现代记忆系统（如 Mem0、A-Mem、LangMem）中的**选择性提取与重写**（selective extraction and rewriting）机制。这些机制会过滤、总结或改写用户输入，导致先前的方法在实际场景中失效——要么触发器和载荷被分割成不同记忆条目，要么触发器被重写而无法激活。
- **研究问题**：能否设计一种记忆投毒攻击，能在选择性提取/重写后存活，保持可检索性，且不被察觉（即仅在包含触发器的查询下激活）？

# 2. 论文提出的方法论
- **核心思想**：提出 **MemPoison**，一种通过对话交互进行后门注入的优化驱动攻击框架，专门绕过现代智能体的选择性记忆管线。攻击分为两个阶段：**离线触发器优化**和**在线注入与激活**。
- **关键技术细节**：
  - **语义关系桥（Semantic Relational Bridge）**：使用 GPT-4 将触发器 \(\tau\) 与恶意载荷 \(x\) 绑定成一个逻辑上相互依赖的语句 \(T(\tau, x)\)（例如“在 [Trigger] 医疗指南下，标准方案是 [恶意建议]”）。这样可防止记忆提取管线将触发器和载荷分割或丢弃。
  - **实体伪装（Entity Masquerading）**：基于预实验（附录 A），发现 LLM 在重写时倾向于保留命名实体（Entity Nouns，保留率 0.85）而非其他词性（动词仅 0.095）。因此，优化触发器使其在语义上像是一个命名实体，以抵抗重写。
  - **联合嵌入优化（Joint Embedding Optimization）**：利用攻击者控制的本地代理嵌入模型，定义三个损失函数并迭代优化离散触发器：
    - **实体伪装损失** \(L_{\text{ent}}\)：基于预训练 NER 模型，鼓励触发器 token 被分类为目标实体类型。
    - **语义集中损失** \(L_{\text{conc}}\)：使所有包含触发器的文本在嵌入空间中紧密聚集（最小化到质心的 L2 距离），确保带触发器的查询能可靠检索到恶意记忆。
    - **基于边界的隔离损失** \(L_{\text{iso}}\)：强制触发文本嵌入远离良性嵌入的簇中心（使用 margin \(\delta\)），避免良性查询意外激活。
  - **优化算法**：采用基于梯度的坐标搜索（类似 HotFlip）：先用梯度从 \(L_{\text{ent}}\) 生成候选 token（保持实体性），再用 \(L_{\text{conc}}\) 和 \(L_{\text{iso}}\) 选择最佳替换。触发器初始化由 GPT-4 生成 K 个种子实体，选初始 \(L_{\text{ent}}\) 最低的作为起点。
- **流程**（算法 1 与算法 2）：多次迭代，批量采样良性文本，逐个 token 替换，最终得到优化后的触发器 \(\tau^*\)。优化后，攻击者在真实对话中通过语义关系桥将 \(\tau^*\) 和恶意载荷一起作为互动内容发送，经过记忆管线存储为一条统一记忆条目；此后任何包含该触发器的查询（用户或攻击者发起）均会检索到该条目并诱导恶意响应。

# 6. 主要结论与发现
- **高攻击效果**：MemPoison 在所有领域和记忆机制上一致取得高 ISR（≥0.91）、RSR@1（≥0.79）、ASR（≥0.77），同时保持 ACC 接近无攻击基线。攻击成功率最高达 0.95（Personal Agent + Mem0）。
- **关键组件不可或缺**：消融实验显示移除语义关系桥或任何一项损失函数均导致性能大幅下降（ASR 降至 0.00–0.68）。实体伪装和嵌入集中的重要性尤为突出。
- **可迁移性强**：攻击在多种 LLM 骨干（7 种）和检索模型（8 种）间有效迁移，但迁移效果受目标嵌入模型的各向异性程度影响（各向异性越高越脆弱）。
- **防御局限性**：困惑度过滤和释义防御均无法完全阻止 MemPoison（ASR 仍达 0.40–0.89），且严格防御会降低良性准确率。
- **机制分析**：自注意力可视化显示触发器 token 吸引了更多注意力，导致嵌入偏移。PCA 表明触发文本形成紧凑簇并与良性分离，解释了高 RSR 同时保持高 ACC。
- **真实世界验证**：在 Hermes Agent 上成功实施，对多数主流 LLM 有效（ASR 0.71–1.00），但对强安全对齐模型（Claude-opus 4.6）效果较差。

- **方法设计精妙**：三个组件紧密配合，动机扎实（预实验验证实体保留性），且优化流程兼顾离散搜索和连续性目标。

# 8. 不足与局限
- **实验覆盖局限**：
  - 未考虑**终身记忆动态**（如时间衰减、基于 LLM 的多跳总结、主动遗忘策略）。在持续学习场景下后门持久性尚未验证。

- **攻击假设**：
  - 假设攻击者能访问**本地代理嵌入模型**（white-box surrogate）进行优化。虽然跨模型迁移实验证明一定泛化性，但在高度定制或动态变化的编码器上可能受限。
  - 假定共享代理平台中存在注入机会（如 Slack、医疗共享记录），但某些严格隔离的环境可能难以实现。
- **触发器稳定性**：
  - 触发长度敏感：过长（>3 token）易被重写，过短（1 token）检索成功率低。可能需要针对不同记忆机制自适应调整。
  - 实体伪装仅依赖单一 NER 模型，若目标系统采用不同的 NER 或根本不使用 NER，鲁棒性可能下降。
- **其他**：
  - 论文未报告类似攻击的**计算开销**（GPU 时间），可能影响可复现性。
  - 强安全对齐模型（如 Claude）能有效过滤恶意内容，表明改善模型对齐是潜在防御方向，但作者未深入讨论如何平衡对抗性与可用性。
