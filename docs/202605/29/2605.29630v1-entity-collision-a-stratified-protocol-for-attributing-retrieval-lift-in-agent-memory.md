## 摘要
端到端的智能体记忆基准测试对每个检索器报告单一hit@k指标，混淆了词汇泄露（未控制的查询/标准答案/干扰项实体重叠）与标签混合（偏好、服务、工具等被平均混合）。我们提出实体碰撞（entity-collision），一种系统无关的协议，通过构造固定BM25基线——每个干扰项共享答案的实体词元——并按判别器标签对查询分层，使得相较于BM25的任何提升均可归因于嵌入器。将该协议应用于一个开源智能体记忆测试平台，涵盖5个标签×3种嵌入器×5种碰撞程度，并采用配对自助法95%置信区间，协议揭示了一个双轴模式：256维哈希三元组仅在闭集词汇标签的深度碰撞下有效；MiniLM-384在双轴上均占优；而参数量为2.7倍的BGE-large并未在MiniLM基础上实现一致提升——它在意图型查询上胜出，但在词汇型查询上落败。编码器容量本身并非绑定约束。合成的意图标签零假设在LongMemEval（n=500）上再现为单会话偏好召回悬崖。LoCoMo上的自适应向量权重路由是一个测量上的零假设：存在11.7个百分点的理想性能空间，但我们测试的任何信号均未恢复它。全部26个结果表格和37个复现脚本均受版本控制并由公共注册库验证；该协议在确定性控制的记忆测试平台（事件溯源决策日志、DAG状态机模式生命周期）上执行，因此每个报告的置信区间均可从数据摄入流中逐字节复现。

## Abstract
End-to-end agent-memory benchmarks report a single hit@k per retriever, confounding lexical leakage (uncontrolled query/gold/distractor entity overlap) with tag-mixing (preferences, services, tools averaged together). We propose entity-collision, a system-agnostic protocol that pins the BM25 floor by construction -- every distractor shares the answer's entity tokens -- and stratifies queries by discriminator tag, so any lift over BM25 is attributable to the embedder. Applied to an open-source agent-memory testbed across 5 tags x 3 embedders x 5 collision degrees with paired-bootstrap 95% CIs, the protocol reveals a two-axis pattern: a 256-d hash trigram helps only on closed-vocabulary lexical tags at deep collision; MiniLM-384 dominates both axes; and a 2.7x-parameter BGE-large does not uniformly improve on MiniLM -- it wins on intent-style queries but loses on lexical ones. Encoder capacity alone is not the binding constraint. The synthetic intent-tag null replicates on LongMemEval (n=500) as a single-session-preference recall cliff. Adaptive vector-weight routing on LoCoMo is a measured null: 11.7pp of oracle headroom exists, but no signal we tested recovers it. All 26 result tables and 37 reproduce scripts are version-controlled and verified by a public registry; the protocol is exercised on a deterministically governed memory testbed (event-sourced decision log, DAG-state-machine schema lifecycle) so every reported CI is reproducible byte-for-byte from the ingest stream.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：当前端到端的智能体记忆基准（如 LongMemEval、LoCoMo）对每个检索器报告单一的 hit@k 指标，这种做法存在两个严重混淆：
  - **词汇泄漏（Lexical Leakage）**：查询通常与标准答案共享实体词元，但与干扰项不共享，导致 BM25 轻易获胜，稠密嵌入的“提升”被错误归因于词汇锚点。
  - **标签混合（Tag-mixing）**：不同类别查询（偏好、项目、技术、服务、工具）被平均为一个数字，掩盖了检索器在不同标签上的系统差异。
- **核心问题**：对于部署团队，需要知道 BM25 是否足够？稠密嵌入何时有帮助？为了回答这些问题，必须有一种能够可靠地归因检索提升的测量方法。

### 2. 论文提出的方法论

- **核心思想**：提出 **Entity-Collision 协议**，一种系统无关的归因协议。通过构造性手段固定 BM25 的基线（使所有干扰项都与标准答案共享实体词元），然后**按判别器标签分层查询**，使得任何相对于 BM25 的提升都可以明确归因于嵌入器。
- **关键技术细节**：
  - 对每个标签 \( t \in \{\text{preference, project, technical, service, tool}\} \) 和碰撞程度 \( K \in \{1,2,4,8,16\} \)：
    - 生成 \( n_{\text{entities}}=32 \) 个独立实体，每个实体有一个标准答案文档，形式为 `"<entity> uses <answer> for <tag>."`。
    - 为每个实体生成 \( K-1 \) 个干扰文档，它们共享实体词元但翻转答案，例如 `"<entity> uses <other_answer> for <tag>."`。
    - 查询格式为 `"what does <entity> use for <tag>?"`，测量 hit@1。
  - BM25 基线的期望 hit@1 为 \( 1/K \)（随机打结），因此任何提升都归因于嵌入器区分不同答案的能力。
  - 报告**配对 Δhit@1**（向量融合 vs BM25-only），并采用**配对自助法 95% 置信区间**（10k 重采样）。
  - 检索得分融合公式：\( \text{score} = (1 - vw) \cdot \text{bm25\_norm} + vw \cdot \text{cos\_sim} \)，其中 \( vw \in \{0.0, 0.3, 0.5, 0.7, 1.0\} \)，\( vw=0.0 \) 对应 BM25-only。
- **算法流程说明**：
  1. 固定标签和碰撞度 K，生成实体集和文档集。
  2. 对每个实体，构造查询，用 BM25-only（vw=0）和混合检索（vw>0）分别检索。
  3. 计算配对 Δhit@1，并计算 95% CI。
  4. 对所有标签和 K 重复，形成二维网格。

### 6. 论文的主要结论与发现

- **双轴模式**：
  - **词汇判别标签**（service, tool）：只有深度碰撞（K≥4）下，256-d 哈希三元组才能恢复约 50% 的稠密提升；MiniLM-384 在所有 K≥4 的标签上均实现 CI-正提升。
  - **意图型标签**（preference, project, technical）：哈希三元组无效或负效果；必须使用稠密嵌入器。
  - **编码器容量非绑定约束**：BGE-large（2.7 倍参数）在意图型标签上胜出（如 project +8~+14 pp），但在词汇型标签上反而下降（如 tool -2.7~-11.7 pp）。
- **合成→自然复制**：LongMemEval 上的单会话偏好召回悬崖（hit@1=0.367 vs 整体 0.81）与合成意图标签零假设一致。RM3 也无法挽救该悬崖。
- **LoCoMo 自适应 v 路由为零假设**：存在 11.7 pp 的理想余量，但所有测试信号（包括学习路由）均无法恢复。
- **工程决策**：闭集词汇查询可用哈希三元组（零加载成本）在深度碰撞时获得约 50% 提升；意图型查询必须使用稠密嵌入器。默认 v0.2 配置：MiniLM-384 + vw=0.3。

### 7. 优点

- **方法论严谨**：通过构造固定 BM25 基线，直接隔离了词汇泄漏和标签混合，使提升可明确归因。
- **系统无关**：协议适用于任何暴露每文档分数的检索器，易于复现和推广。
- **统计可靠**：使用配对自助法 95% CI，并对重要结论进行多重验证（BGE-large 验证、RM3 验证、PRF×SP 六轴扫描）。
- **完全开源**：代码、26 个结果表格、37 个复现脚本均版本控制并通过公共注册库验证。
- **覆盖合成和自然数据**：结合了可控的合成网格和多个真实基准，增加了结论的外部有效性。
- **发现了新现象**：揭示了编码器容量不是绑定约束、词汇/意图双轴分离等非平凡结果。

### 8. 不足与局限

- **单系统实例化**：所有实验仅在 Engram 测试平台上运行，跨系统（如 Letta、Mem0）的复现尚未完成，是最大的外部有效性风险。
- **嵌入器覆盖有限**：仅覆盖单向量嵌入器（HashTrigram、MiniLM、BGE），不包括 ColBERT、SPLADE 等更复杂的检索器。
- **外部有效性受限**：BEIR-3 只完成了 FiQA 和 NQ，HotpotQA 因延迟问题推迟，自然数据覆盖不全。
- **统计功率问题**：合成网格中 K=1 只有 32 个配对试验，最小可检测效应约 8-10 pp；结论主要依赖 K≥4（n≥128），但低碰撞度下的 null 结果可能有欠功率风险。
- **嵌入器训练-测试污染**：MiniLM 和 BGE 可能已在训练语料中见过 LongMemEval/LoCoMo 来源数据，无法保证零污染。
- **标签定义的主观性**：词汇/意图二分法由作者自行判断（基于答案集是封闭枚举还是开放短语），未经独立标注验证。
- **硬件局限**：延迟和吞吐量数据来自消费级 CPU/MPS，生产环境 GPU（如 H100）的效果未测量。
- **领域覆盖窄**：仅涵盖五种合成标签和两类自然对话基准，法律、医疗、多语言、代码搜索等专业领域未涉及。
