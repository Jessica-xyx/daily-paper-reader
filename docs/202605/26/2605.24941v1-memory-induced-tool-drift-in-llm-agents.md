## 摘要
现代LLM代理将用于个性化长期记忆与用于在现实中采取行动的工具调用接口相结合——这一结合支撑着当代生产系统。我们研究了这一结合中先前未被检验的失败模式：当存储于记忆中的个性驱动偏差（成本意识、不耐烦、风险承受能力等）在不适用的情况下悄然影响工具调用时。我们称之为记忆引起的工具漂移，并通过MEMDRIFT基准（一个涵盖五个偏差维度和七个专业领域的105个场景，通过自动对抗流水线生成）将其操作化。在七个前沿模型（包括具有扩展推理能力的模型）中，有偏记忆使得偏转分数（一种由评判者评分的参数偏离无偏基线的度量）在1-5分制上最多增加+3.6分。当记忆管理由三种生产级记忆架构处理时，工具漂移仍然存在。该现象影响真实世界工具：在扫描288个已验证MCP服务器上的6,062个工具后，我们标记了608个易受影响的参数，并在验证子集上确认了工具漂移。从机制上讲，有偏记忆作为隐式引导向量，将激活沿着与显式行为指令相同的潜在方向推动。它们还将注意力从任务相关上下文重新分配到与目标参数存在表层关键词重叠的记忆条目。标准防御措施——基于提示的相关性指令和记忆过滤器——减少了漂移但未能消除。随着代理代表用户采取越来越重要的行动，记忆引起的工具漂移是一种当前防护措施未能解决的系统性漏洞，这促使人们在记忆管理与工具调用生成的交叉点上设计专门的防御措施。

## Abstract
Modern LLM agents combine long-term memory for personalization with tool-calling interfaces for taking actions in the world -- a combination underpinning contemporary production systems. We study a previously unexamined failure of this combination: when personality-driven biases stored in memory (cost-consciousness, impatience, risk tolerance, etc.) silently affect tool calls in contexts where they are not applicable. We call this memory-induced tool-drift and operationalize it through MEMDRIFT, a benchmark of 105 scenarios spanning five bias dimensions and seven professional domains, generated through an automated adversarial pipeline. Across seven frontier models -- including those with extended reasoning -- biased memories raise deflection scores (a judge-scored measure of parameter deviation from unbiased baselines) by up to $+3.6$ points on a 1--5 scale. Tool-drift persists when memory management is handled by three production memory architectures. The phenomenon affects real-world tools: scanning 6{,}062 tools across 288 verified MCP servers, we flag 608 with susceptible parameters and confirm tool-drift on a validated subset. Mechanistically, biased memories act as implicit steering vectors, pushing activations along the same latent directions as explicit behavioral instructions. They also redistribute attention from task-relevant context toward memory entries with surface-level keyword overlap to the target parameter. Standard defenses -- prompt-based relevance instructions and memory filters -- reduce drift but do not eliminate it. As agents take increasingly consequential actions on a user's behalf, memory-induced tool-drift represents a systematic vulnerability that current safeguards do not address, motivating dedicated defenses at the intersection of memory management and tool-call generation.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现代LLM代理同时具备长期记忆（用于个性化）和工具调用能力（用于执行现实动作）。本研究揭示了这两种能力的结合中一个此前未被检查的失败模式：存储在长期记忆中的个性驱动偏差（如成本意识、不耐烦、风险容忍度等）会在不适用的情况下悄然影响工具调用的参数选择，导致**记忆诱导的工具漂移（Memory-Induced Tool-Drift）**。
- **研究动机**：传统研究分别关注记忆的召回保真度和工具选择的准确性，忽略了二者之间的交互风险。在对话场景中已观察到过度个性化，但在工具调用场景中这种偏差更危险：工具调用通常不可逆且用户难以逐一检查参数。
- **整体含义**：这是一种系统性的新威胁面，现有安全机制（如指令遵循、记忆过滤）未能有效解决，随着代理承担越来越重要的任务，该漏洞需要专门的防御设计。

## 6. 主要结论与发现

- **漂移普遍存在**：所有7个模型在直接记忆注入下均表现出显著工具漂移，偏转分数在中性记忆下低、有偏记忆下高；即使使用推理模型也未能避免。
- **漂移在生产级记忆框架中持续存在**：Mem0、MemPalace、SimpleMem三种架构下，漂移程度与直接注入相当甚至更高，因为记忆系统常剥离上下文，将偏好编码为全局指令。
- **真实工具易受影响**：在6062个MCP工具中扫描到608个易受影响的参数（占10%），并在子集上验证了漂移。
- **机制**：
  - 有偏记忆作为隐式引导向量，将激活沿显式偏见方向推动（在中后期层投影分离明显）。
  - 有偏记忆吸引注意力离开任务相关区域，转向记忆区域和模板token，尤其是与目标参数存在词汇重叠的记忆条目。
- **现有防御效果有限**：防御性系统提示可降低漂移但无法消除（平均偏转分数降低0.52）；记忆相关性过滤在严格上下文分离场景下完美工作，但在现实多跳推理场景中召回仅61%，且误报率10.3%。

## 7. 优点

- **问题定义清晰新颖**：首次系统性地定义并研究了记忆诱导的工具漂移，明确了该风险在代理场景中比对话场景更严重。
- **严谨的实验设计**：通过三种记忆条件（无/中性/有偏）隔离效应，使用客观参数偏差而非主观判断，并在多种记忆架构和真实工具上验证。
- **自动化基准生成**：采用对抗性自我改进流水线生成高挑战性场景，提升基准的难度和生态效度。
- **机械分析深入**：利用工具调用的结构化性质（行为定义token），结合引导向量和注意力分析揭示了漂移的内部机制——词汇重叠驱动的注意力重分配和激活位移。
- **切实关注实用安全**：扫描真实MCP工具生态系统，并验证了真实世界影响，具有工程实践意义。

## 8. 不足与局限

- **场景范围有限**：每个场景仅涉及单一工具调用，未测试多工具链或多步骤场景；未考虑记忆对工具选择本身（而非仅参数）的偏差。
- **单回合设置**：评估仅在单个回合内进行，未考虑对话历史中记忆的累积效应或用户反馈。
- **记忆集大小固定**：多数实验使用5-6条有偏记忆，未系统探索记忆数量对漂移的影响（除敏感性分析外）。
- **过滤实验数据简单**：自建多跳推理测试集（来源于PersistBench），样本量小且不一定代表真实分布。
- **未实施训练时干预**：仅提出假设，未实际进行对抗训练来修复偏差。
- **评判器偏差风险**：虽然进行了跨评判者一致性检验（κ=0.86），但仍依赖LLM评判器，可能引入风格偏好。
- **开放性MCP生态局限性**：真实工具漏洞扫描基于公开注册的MCP服务器，可能存在覆盖偏差或已修复的工具。
- **未考虑用户明确指令或系统提示的纠偏作用**：实验中系统提示仅给出任务，未鼓励模型质疑记忆相关性。
