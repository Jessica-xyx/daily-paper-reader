## 摘要
现有基于大语言模型的记忆系统采用通用、静态的策略，忽略了一个基本事实：值得存储到记忆中的上下文因用户而异。这种错位将有限的记忆预算浪费在短暂交互上，却未能为长期任务保留关键上下文。为弥补这一空白，我们探索了一个尚未充分研究的问题：基于大语言模型的记忆系统能否学习个性化记忆策略？我们引入了PerMemBench，这是首个用于评估个性化记忆系统的基准测试，包含跨多个用户角色的多年、多领域交互历史。我们进一步提出了首个关于记忆个性化的实证研究，提出了会话级存储门控（session-level storage gating）这一轻量级框架，该框架选择性地跳过对短暂会话的记忆操作。我们的研究证实，在完美门控下，个性化能带来显著的保留增益，但也揭示出精确门控仍然是一个开放且关键性的挑战。

## Abstract
Existing large language model (LLM) based memory systems apply universal, static policies that overlook a fundamental reality: the contexts that are worth storing in memory are different across users. This misalignment wastes limited memory budget on transient interactions while failing to preserve critical context for long horizon tasks. To address this gap, we investigate an underexplored question: can LLM based memory systems learn personalized memory policies? We introduce PerMemBench, the first benchmark for evaluating personalized memory systems, featuring multi year, multi domain interaction histories across diverse user personas. We further present the first empirical study of memory personalization, proposing session level storage gating, a lightweight framework that selectively bypasses memory operations for transient sessions. Our study confirms that personalization yields substantial retention gains under perfect gating, yet reveals that accurate gating remains an open and critical challenge.

- **问题**：现有基于大语言模型（LLM）的智能体记忆系统采用**通用、静态的记忆策略**（one-size-fits-all），对所有用户应用相同的“值得存储”标准。这忽略了用户的**异质性**：例如，对于同一领域（如旅行计划），一个用户可能需要长期记忆（正在规划蜜月），而另一个用户可能只是临时查询。

- **研究问题**：能否让LLM记忆系统**学习个性化记忆策略**，即自动推断每个用户哪些交互是长期任务、哪些是短暂的，并据此选择性地存储？

## 2. 方法论：核心思想、关键技术细节
### 核心思想
- 提出**会话级存储门控（Session-level Storage Gating）**：在每个会话结束后，用一个门控模块判断该会话属于长期任务（需要存储）还是短暂交互（跳过存储）。这是一个轻量级包装器，不修改底层记忆系统，仅决定是否触发记忆操作。
- 同步构建**PerMemBench基准数据集**，用于评估记忆个性化能力。

### 关键技术细节
- **PerMemBench构建流程（全自动）**：
  1. **用户画像与代理使用档案**：从Nemotron-Persona-USA数据集中采样20个用户画像；基于LLM推断每个用户对每个领域是否参与、使用频率、是否需要记忆。
  2. **生命骨架与时间线构建**：对需要记忆的领域，生成由“项目（Project）”和“事件（Event）”组成的结构化蓝图；每个事件包含参考记忆（gold standard）。对短暂领域，仅生成独立事件。
  3. **双模拟器对话生成**：用户模拟器（基于画像和上下文生成话语）与代理模拟器（无骨架先验知识，仅基于当前对话和内部记忆回应）交互，生成多轮对话。
- **动态版PerMemBench-d**：在静态版基础上模拟用户画像的变化（如长期任务完成、新领域加入），评估系统应对动态性的能力。
- **门控方法**（由简单到复杂）：
  - **Greedy**：仅根据当前会话对话内容判断。
  - **Context-aware**：维护一个窗口的历史会话摘要，提供上下文。
  - **Structure-aware**：显式建模会话间关系（项目聚类），维护“结构笔记”并持续更新。
- **评估指标**：记忆保留率（Memory Retention Rate, RR），定义参考记忆在其生命周期内被保留在记忆库中的比例。采用LLM-as-a-judge和检查点采样近似计算。

## 6. 主要结论与发现
1. **Structure-aware门控在识别领域使用模式上最有潜力**（F1最高达0.844），但**难以检测画像的转移**（长期→短暂后性能急剧下降）。Greedy在转移后保持稳定。
2. **完全正确的个性化（Oracle）能显著提升记忆保留率**，尤其在小预算下（预算100–200时增益最大）；即使预算500时仍有收益。
3. **当前门控方法的准确性远不足以实现Oracle水平的增益**：Greedy和Context-aware甚至不如Universal，Structure-aware也只有边际改进。**个性化记忆的潜力很大，但门控准确率是瓶颈**。
