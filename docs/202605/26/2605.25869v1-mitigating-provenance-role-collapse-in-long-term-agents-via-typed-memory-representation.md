## 摘要
长期记忆对于持续性LLM智能体至关重要，但当前的主流架构将历史交互存储为无结构的扁平文本。这种无约束的存储会导致溯源-角色崩溃，这是一种关键失效模式，使智能体遭受来源监控错误。为了在架构层面解决这一认知脆弱性，我们提出了MemIR——一种类型化的记忆中间表示，将来源监控作为结构约束来具体化。MemIR将长期记忆写入基于原语的原子，这些原子分离原始证据、检索线索和事实性声明，并将事实授权限制在受支持的声明原语上。随后，它应用多路径原子投影和溯源范围约束的利用，将异构检索命转化为以声明为中心的候选束和归一化的事实接口，用于答案生成。在LoCoMo和BEAM-100K上的实验表明，MemIR始终优于现有的记忆基线，特别是在需要来源追踪、时间定位和碎片化证据聚合的任务上。

## Abstract
Long-term memory is essential for persistent LLM agents, yet prevailing architectures store historical interactions as unstructured, flat text. This unconstrained storage induces provenance-role collapse, a critical failure mode where agents suffer from source-monitoring errors. To resolve this cognitive vulnerability at the architectural level, we propose MemIR, a typed Memory Intermediate Representation that operationalizes source monitoring as a structural constraint. MemIR writes long-term memory into grounded atoms that separate raw evidence, retrieval cues, and truth-bearing claims, with factual authorization restricted to supported claim atoms. It then applies multi-route atomic projection and provenance-scoped utilization to transform heterogeneous retrieval hits into claim-centered candidate bundles and a normalized fact interface for answer generation. Experiments on LoCoMo and BEAM-100K demonstrate that MemIR consistently outperforms existing memory baselines, especially on tasks requiring source tracking, temporal grounding, and aggregation of fragmented evidence.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有长期记忆架构将历史交互存储为无结构的扁平文本，导致“溯源-角色崩溃”（provenance-role collapse）。这一失效模式表现为智能体在检索时无法区分观察到的证据与推断内容，难以追踪引用关系，也不能识别检索语句的认知状态（如事实性断言 vs. 辅助线索）。
- **整体含义**：长期记忆是LLM智能体从无状态响应转向持久、个性化助手的关键。但若存储方式忽略结构化的来源监控，智能体会错误合并不同来源的片段，或无法识别同一参照物在不同时间点的演化，从而在需要聚合碎片化证据、追踪时间线或解决冲突的任务中表现不佳。论文旨在从架构层面解决这一认知脆弱性。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：提出 **MemIR**（Typed Memory Intermediate Representation），将来源监控具体化为结构约束，通过类型化记忆原子分离原始证据、检索线索和事实性断言，并借助多路径投影和来源范围利用生成规范化的证据接口。
- **关键技术细节**：
  - **系统写入记忆原子**：将交互历史编译为七种原子类型（P、S、H、T、V、A、R），其中 `P`（页面）、`S`（跨度）保留原始上下文和逐字证据；`H`（句柄）、`T`（时间）、`V`（锚点）提供引用和语义访问；`A`（声明原子）代表经过源跨度支撑的事实断言，是唯一的“真值承载”组件；`R`（检索视图）定义不同原子间的跨类型访问关系。严格约束：每个提取的线索或生成的声明必须有明确的证据起源（`sup(x) ⊆ S`，且 x 不能是页面本身）。
  - **多路径原子投影**：通过稀疏检索（BM25）和稠密检索（BGE-M3）两种路径获取相关记忆原子，然后使用**类型约束投影函数**将异构命中映射到声明原子。辅助原子（跨度、句柄、锚点）只能通过关联的声明原子进入候选层，无有效声明的命中被丢弃。最终将投影到同一声明的命中聚合为**候选束** `b_a = <a, ρ_a, E_a>`，包含声明文本、聚合检索得分、以及完整的来源闭包。
  - **来源范围利用**：先基于聚合得分保留前 M 个候选束，再用交叉编码器对束进行细粒度重排序，最后 LLM 选择器从重排序后的前 K 个束中挑选最多 X 个互补候选，并为每个分配角色（`direct` 或 `support`）。选中的束转换为规范事实接口 `F_q`，包含声明文本、来源闭包、时间线索和角色，作为答案模型的输入。若证据不足，模型输出“证据不足”而非无根据生成。

## 3. 实验设计

- **数据集/场景**：
  - **LoCoMo**：10个长多轮对话，1540个问题，涵盖单跳、多跳、时间、开放域问答。使用 token-level F1、BLEU-1 和 LLM-as-Judge 分数评估。
  - **BEAM-100K**：20个对话，10万 token 历史，400个问题，覆盖10个任务类别（如抽象、矛盾解决、事件排序、信息抽取、指令跟随、知识更新、多会话推理、偏好跟随、摘要、时间推理）。使用 LLM-as-Judge 分数。

## 6. 主要结论与发现

- MemIR 在 LoCoMo 和 BEAM-100K 上均优于所有现有基线，尤其在单跳、时间、开放域任务及需要来源追踪、矛盾解决、知识更新、多会话推理的任务上提升显著。
- 消融研究表明：移除声明原子或类型约束投影导致性能大幅下降，证明了“事实授权”和“证据-事实边界”的必要性；移除线索原子削弱了引用和时间定位；移除候选束则造成证据与来源闭包分离。
- 超参数分析表明：适当的声明原子预算（约12/页）、预重排序池大小（约32）、选定束预算（约6）可获得最佳平衡，过多或过少均会引入噪声或丢失信息。
- 不同 backbone 下 MemIR 均带来一致提升，但复杂跨声明推理仍依赖下游模型能力。

## 7. 优点

- **结构性突破**：首次将认知科学中的“来源监控”理论系统化为架构级约束，通过类型化原子将事实授权与检索证据明确分离。
- **多路径投影+来源范围利用**：将异构检索结果统一映射到声明中心，并构造规范事实接口，既保留证据完整性又避免信息泄露。
- **全面的实验验证**：在两个高难度基准上对比14种基线，并在多个 backbone 和超参数下测试，证明了方法的鲁棒性和有效性。
- **实用考量**：支持“证据不足”的拒绝回答机制，减少幻觉。

## 8. 不足与局限

- **复杂推理依赖下游模型**：MemIR 主要优化证据组织和结构化，但多跳跨声明推理仍由下游生成模型完成，对于逻辑链较长的任务（如多跳 LLM 评分）提升有限。
- **计算开销**：系统写入多类型原子（尤其是声明生成和视图构建）需要额外的 LLM 调用，增加了交互编译阶段的成本。
- **实验覆盖**：消融实验仅在单一 backbone 下进行；未在更大规模或不同领域数据集上验证泛化性；未探讨记忆更新或删除场景。
- **潜在偏差**：所有实验基于 GPT 系列模型，可能忽略了其他模型家族的差异性；超参数依赖手动调整，在不同设置下可能需重新优化。
- **开放性**：未来可探索更轻量级的原子编译技术（如结构化压缩而非全 LLM 生成）以及显式推理机制（如对候选束执行符号推理）以增强多跳逻辑。
