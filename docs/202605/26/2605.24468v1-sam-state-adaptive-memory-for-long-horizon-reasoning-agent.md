## 摘要
长程智能体推理要求大型语言模型在包含思考、工具调用、观察和部分结论的长时间交互历史中进行操作。挑战不仅在于这些历史会变得很长，还在于当前决策所需的信息可能分散在遥远的步骤中，且仅在后来才变得相关。现有方法通过截断交互历史、将其压缩为更短的替代物或检索部分历史以供重用来解决这一困难，但它们没有明确建模对过去交互的访问应如何适应智能体不断变化的状态。相反，我们将长程推理视为一个状态自适应记忆的问题。为此，我们提出了状态自适应记忆（SAM），这是一个独立的框架，将持续交互整合为紧凑的记忆线索，同时保留原始轨迹页面以实现意图驱动的回顾。这些线索不被视为历史的替代品；相反，它们作为轻量级句柄，允许智能体根据当前需求重建时间上遥远的信息，而无需重新训练底层骨干网络。我们进一步通过专家引导的监督和强化学习来优化记忆模块，使其与轨迹级别的效用对齐。在BrowseComp、BrowseComp-ZH、WideSearch和HLE上，SAM在不同智能体骨干网络上一致地优于强基线。我们的结果表明，显式记忆建模为长程智能体推理提供了简单而有效的基础。

## Abstract
Long-horizon agentic reasoning requires large language models to act over long interaction histories containing thoughts, tool calls, observations, and partial conclusions. The challenge is not merely that these histories grow long, but that information needed for the current decision may be scattered across distant steps and only become relevant later. Existing approaches address this difficulty by truncating the interaction history, compressing it into shorter surrogates, or retrieving selected parts of it for reuse, but they do not explicitly model how access to past interaction should adapt to the agent's evolving state. We instead cast long-horizon reasoning as a problem of state-adaptive memory. To this end, we propose State-Adaptive Memory~(SAM), a standalone framework that consolidates ongoing interaction into compact memory cues while preserving raw trajectory pages for intent-driven recall. These cues are not treated as replacements for history; rather, they serve as lightweight handles that allow the agent to reconstruct temporally distant information according to its current needs, without retraining the underlying backbone. We further optimize the memory module through expert-guided supervision and reinforcement learning, aligning it with trajectory-level utility. Across BrowseComp, BrowseComp-ZH, WideSearch, and HLE, SAM consistently outperforms strong baselines over diverse agent backbones. Our results suggest that explicit memory modeling provides a simple and effective foundation for long-horizon agentic reasoning.

## 1. 核心问题与整体含义（研究动机与背景）

- **研究动机**：长程智能体推理（Long-Horizon Reasoning）要求大型语言模型（LLM）在包含思考、工具调用、观察和部分结论的长时间交互历史中持续行动。问题不仅在于历史会变长，更在于当前决策所需的信息可能分散在遥远的步骤中，且仅在后续才变得相关。
- **现有方法的不足**：现有方法通过截断历史、压缩为摘要、或检索部分历史来管理上下文，但它们没有显式建模智能体状态变化对历史信息访问需求的影响（即“状态自适应”）。这些方法无法有效应对非马尔可夫性——远距离的信息可能突然变得关键。
- **论文的核心观点**：将长程推理问题重新定义为状态自适应记忆（State-Adaptive Memory）问题，即智能体需要根据当前状态动态地、有选择地获取过去的信息，而不是被动地携带全部历史。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **SAM（State-Adaptive Memory）**：一个独立的外部记忆模块，将交互历史组织为“记忆线索（Memory Cues）”和“原始轨迹页面（Raw Trajectory Pages）”。线索作为轻量级句柄，用于意图驱动的回顾；页面保留原始细节，用于重建所需信息。
- 关键在于：记忆访问是状态自适应的——智能体根据当前意图（intent）选择相关线索，记忆模块从对应页面重构信息，而不是简单压缩或检索固定片段。

### 关键技术细节
#### 2.1 页面化情节整合（Page-based Episodic Consolidation）
- 当实时上下文达到一定令牌预算后，SAM将连续交互分组成页面（token预算限制），每个页面包含一段连贯的推理-动作-反馈序列。
- 记忆模型为每个页面生成紧凑的“记忆线索” `m_k = M_sum(p_k)`，捕获关键信息（已建立的事实、已排除的选项、未解决的问题等）。
- 原始页面移出活跃上下文，存储于外部页面库；线索保留在内存银行中（`M_t = {m_1, ..., m_K}`）。

#### 2.2 智能体引导的线索选择
- 当智能体需要额外历史信息时，它发出一个“回顾请求”，包含意图描述 `q_t`，并从内存银行中选择一小部分候选线索。
- 线索不是替代判断，而是提供一个粗略但持久的过去交互地图。

#### 2.3 意图驱动的情节回顾（Intent-driven Episodic Recall）
- 根据选中的线索对应的页面，记忆模型 `M_rec` 基于当前意图重构与决策最相关的信息：`ρ_t = M_rec(q_t, p_k1, ..., p_kr)`。
- 回顾输出注入智能体的活跃上下文 `C_t = [x; C_live; M_t; ρ_t]`。
- 这是核心创新：回顾不是重放历史，而是状态条件化的决策支持重建。

#### 2.4 优化过程
- **专家引导的监督微调（SFT）**：使用前沿模型（Claude-4.5-Opus, GPT-5.4）作为专家，在领域内查询上生成专家记忆轨迹，保留最终答案正确的轨迹，作为监督目标训练记忆模型（Qwen3.5-9B）。
- **OAT-GRPO（Oracle-Anchored Tree GRPO）**：一种针对记忆模块的强化学习算法，扩展了GRPO：
  - **树结构结果奖励**：在每次记忆调用点分支为b个样本（共享父上下文），生成不同回顾；每个分支继续由冻结的推理器执行，直到叶子节点获取二值结果（正确/错误）。每个记忆动作的奖励为其子树中所有叶子的平均结果：$R_{out}(a) = \frac{1}{|L(a)|} \sum_{\ell \in L(a)} r_{out}(\ell)$。兄弟动作形成局部基线，隔离该记忆输出的贡献。
  - **锚定恢复性奖励**：使用委员会（GPT-5.4, GLM-4.7, DeepSeek-V4-Flash）的输出来近似理想回顾空间，GPT-5.4作为评估者，对每个候选a打分（0-10，缩放到[0,1]），称为$R_{rec}(a)$。
  - 组合奖励：$R(a) = \alpha R_{out}(a) + (1-\alpha)(R_{rec}(a) - b_{rec})$，其中$\alpha=0.3$。
  - 优化目标：$J_{OAT-GRPO} = E\left[ \frac{1}{b} \sum_{i=1}^b \min\left( r_i(\theta)\hat{A}_i, clip(r_i(\theta), 1-\epsilon, 1+\epsilon)\hat{A}_i \right) \right]$，其中$\hat{A}_i$为组内标准化优势。

## 6. 主要结论与发现

1. **SAM在每种骨干上都是最强的上下文管理方法**：在四个基准平均分上，SAM显著优于最佳启发式方法，尤其在BrowseComp和BrowseComp-ZH上领先最大，证明其收益来自启发式方法最弱的地方。
2. **同样的SAM模块跨基准和骨干泛化**：单个Qwen3.5-9B记忆模型，一次训练，在所有骨干×基准组合上取得最优。启发式基线的排名在不同基准间翻转，而SAM增益是一致的。

## 7. 优点

1. **创新性**：将长程上下文管理重新定义为状态自适应记忆问题，提出线索-页面分离架构，与现有压缩或检索方法不同。
2. **模块化**：记忆模块独立于智能体骨干，可适配不同LLM，无需重训骨干。
3. **优化路径清晰**：先用专家监督学习（SFT）获得良好先验，再用RL（OAT-GRPO）与轨迹级效用对齐，奖励设计（树结构+锚定恢复性）解决了稀疏和偏差问题。
4. **实验扎实**：覆盖四个差异化的长程基准，两种骨干，与最相关的基线进行一致对比，消融实验完整，行为分析深入。
5. **案例分析有深度**：成功和失败案例展示了SAM的实际工作和边界，并提出了未来改进方向。

## 8. 不足与局限

1. **计算成本高**：依赖前沿模型（Claude-4.5-Opus, GPT-5.4）进行专家监督和委员会评估，训练阶段耗时长、成本高。委员会调用可能引入偏差。
2. **骨干多样性有限**：仅测试了闭源GLM-4.7和开源Qwen3.5-35B-A3B双向骨干，未测试更多架构（如Mixture-of-Experts、更大规模模型全参数训练）。
3. **基准覆盖面受限**：仅聚焦于网页浏览和知识密集型科学推理，未测试软件工程智能体、具身环境或长格式生成场景。泛化性需进一步验证。
4. **未报告误差条**：虽然使用avg@3，但未提供标准偏差或置信区间，对于较小基准（如200题）可能存在方差。
5. **失败案例分析揭示的局限性**：智能体可能陷入错误的推理框架，记忆模块即使忠实记录并返回不确定性标记，也无法纠正错误的检索目标。这暴露了上游的检索目标生成问题，当前训练未优化智能体的retrieval-goal formulation。
6. **奖励设计依赖外部模型**：委员会和评估者模型（GPT-5.4等）的偏见可能影响训练稳定性，且无法保证与下游任务完全对齐。
