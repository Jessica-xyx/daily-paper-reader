## 摘要
渗透测试对于保护现代Web基础设施至关重要，但传统的 manual 方法难以跟上其规模和复杂性的步伐。大语言模型为自动化这些任务提供了新的机遇，然而现有方法面临两大持久挑战：技术实体的幻觉和长期上下文记忆不足。为解决这些问题，我们提出了APT-Agent，一个完全自动化的、由大语言模型驱动的渗透测试框架，系统性地编排侦查、利用和数据窃取。APT-Agent引入了一个混合纠正模块来恢复幻觉命令，以及一个命令专用记忆架构来跨多步攻击序列保持操作上下文。我们在Metasploitable 2上针对涵盖Web、数据库和网络协议的七种脆弱服务评估了APT-Agent。在匹配条件下，APT-Agent实现了84.29%的端到端利用成功率，而Script Kiddie和PentestGPT分别为48.57%和18.57%。通过降低认知负担和最小化对人类干预的依赖，APT-Agent代表了迈向可扩展、可靠且认知高效的自动化渗透测试的一步。

## Abstract
Penetration testing is essential to securing modern web infrastructures, yet traditional manual methods struggle to keep pace with their scale and complexity. Large Language Models (LLMs) offer new opportunities for automating these tasks, but existing approaches face two persistent challenges: hallucination of technical entities and insufficient long-term contextual memory. To address these issues, we present APT-Agent, a fully automated LLM-driven penetration testing framework that systematically orchestrates reconnaissance, exploitation, and exfiltration. APT-Agent introduces a hybrid rectification module to recover hallucinated commands and a command-specific memory architecture to preserve operational context across multi-step attack sequences. We evaluate our APT-Agent on Metasploitable 2 against seven vulnerable services spanning web, database, and network protocols. APT-Agent achieves an 84.29% end-to-end exploitation success rate, compared to 48.57% (Script Kiddie) and 18.57% (PentestGPT) under matched conditions. By reducing cognitive burden and minimizing reliance on human intervention, APT-Agent represents a step toward scalable, reliable, and cognitively efficient automation for penetration testing.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出APT-Agent框架。该框架采用模块化架构，结合“混合纠正模块”和“命令特定上下文管理模块”，使LLM驱动的渗透测试能够可靠地完成从侦查到数据窃取的完整攻击链，无需人工干预。
- **关键技术细节**：
    - **架构流程**：通过一个迭代循环运行三个核心模块：
        1.  **策略选择模块**：基于当前进度选择下一阶段（如侦查、利用）。
        2.  **命令生成模块**：将策略转化为具体命令（如Nmap扫描、Metasploit利用）。
        3.  **输出翻译模块**：将原始工具输出（如扫描报告）提炼为结构化的成功/失败信息。
    - **核心创新1：混合纠正模块 (Rectification Module)**：
        - **目的**：解决LLM生成的Metasploit模块路径幻觉问题。
        - **知识库**：预先构建了一个包含3,253个有效Metasploit模块的数据库。
        - **算法流程**：
            1.  **后缀提取**：从LLM生成的路径（如`exploit/linux/ftp/vsftpd_234_backdoor`）中提取最后一段（`vsftpd_234_backdoor`）。
            2.  **模糊后缀匹配**：将提取的后缀与数据库中的所有模块后缀进行模糊字符串匹配（基于归一化Levenshtein距离），找到最相似的后缀（`unix/ftp/vsftpd_234_backdoor`）。
            3.  **模块重建**：根据匹配到的后缀，从数据库中检索出完整的有效模块路径。
        - **综合性**：此方法结合了模糊匹配的容错能力和后缀对齐的结构准确性，优于纯RAG或纯精确匹配方法。
    - **核心创新2：上下文管理模块 (Context Memory Module, CMM)**：
        - **目的**：解决多步操作中LLM的长期记忆丢失问题。
        - **设计原则**：采用“高信号、低噪音”的简洁日志格式。
        - **实现细节**：
            1.  **阶段感知日志**：每个操作阶段（如利用、窃取）维护一个独立的JSON日志`M_stage`，记录每次迭代的`{iter: 序号, cmd: 命令, result: 成功/失败}`。
            2.  **提示注入**：在执行新命令前，将当前阶段对应的JSON日志直接注入到LLM的提示中。这类似于清晰的“任务板”，让LLM知道“已经尝试过什么、结果如何”。
            3.  **决策优化**：LLM通过读取日志，可以避免重复执行已失败的命令，并基于历史状态选择更优的下一步策略。
