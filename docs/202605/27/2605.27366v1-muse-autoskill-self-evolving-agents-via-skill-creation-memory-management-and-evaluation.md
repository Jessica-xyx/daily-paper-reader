## 摘要
大型语言模型（LLM）智能体依赖于可重用的技能来解决复杂任务。然而，现有的技能创建方法将技能视为孤立和静态的工件，限制了其可重用性、可靠性和长期改进。我们提出MUSE-Autoskill智能体（记忆利用技能进化），这是一个以技能为中心的智能体框架，允许智能体在统一的生命周期（创建、记忆、管理、评估和优化）下通过创建、重用和改进技能来持续提升其任务解决能力。我们的框架使智能体能够按需创建技能、跨任务存储和重用它们、高效组织和选择技能，并通过单元测试和运行时反馈进行评估以实现持续优化。我们进一步引入了技能级记忆，为每个技能积累跨任务的经验，从而实现更有效的重用和随时间调整。在SkillsBench上的实验初步证明，生命周期管理的技能可以提高任务成功率、效率、重用性和跨智能体迁移，凸显了将技能视为长期存在、具有经验感知和可测试资产的重要性。

## Abstract
Large language model (LLM) agents rely on reusable skills to solve complex tasks. However, existing skill creation approaches treat skills as isolated and static artifacts, limiting their reusability, reliability, and long-term improvement. We propose MUSE-Autoskill Agent (Memory-Utilizing Skill Evolution), a skill-centric agent framework that lets agents continuously improve their task-solving capability by creating, reusing, and refining skills under a unified lifecycle (creation, memory, management, evaluation, and refinement). Our framework enables agents to create skills on demand, store and reuse them across tasks, organize and select them efficiently, and evaluate them through unit tests and runtime feedback for continuous refinement. We further introduce skill-level memory that accumulates experience for each skill across tasks, enabling more effective reuse and adaptation over time. Experiments on SkillsBench provide initial evidence that lifecycle-managed skills can improve task success, efficiency, reuse, and cross-agent transfer, highlighting the importance of treating skills as long-lived, experience-aware, and testable assets.

### 论文主要结论与发现

1. **技能使用显著提升性能**：所有智能体添加人类技能后准确率提高 13-15 个百分点；MUSE-Autoskill 在有人类技能时达到 68.40%，高于 Codex（67.28%）和 Hermes（61.21%）。
2. **MUSE 可自动生成高质量技能**：在 35 个成功任务的 Phase 2 中，准确率达 **87.94%**，超过人类技能上限（68.40%）；整体 51 任务平均为 60.35%（受限于 Phase 1 覆盖）。
3. **生成的技能可跨智能体迁移**：将 MUSE 生成的技能直接注入 Hermes，其准确率从 47.89% 提升至 58.40%（+10.51 pp），缩小了与 Hermes 使用人类技能时 79% 的差距，表明技能是外部化的知识资产。
4. **成本效率优势**：使用生成的技能后，不仅准确率提升，延迟和 token 消耗反而下降（MUSE: -37% 延迟, -20% tokens；Hermes: -30% 延迟, -48% tokens），生成成本约 3 次重用即可回收。
5. **技能解剖**：MUSE 生成的技能包平均比人类编写的长 2.2 倍（326 vs 146行），含更多程序性细节和可测试性（9% 包含测试目录）。

### 优点

- **全面性**：首次在无训练框架下实现技能完整生命周期（创建、记忆、管理、评估、优化），填补了以往只覆盖部分环节的空白。
- **创新模块**：技能级记忆（`.memory.md`）是独特贡献，使技能可跨任务积累经验，而不仅是一次性生成。
- **实用性**：已在字节跳动生产系统（SkillMarket、ArkClaw、SkillHub）中部署，验证了框架的可落地性。
- **实验扎实**：多智能体对比、跨智能体迁移、成本-收益分析、质量审计等角度丰富，结论可靠。
- **泛化性**：技能格式与 Anthropic Agent Skills 兼容，生成的技能可直接被其他智能体加载，展示了标准化潜力。

### 不足与局限

- **实验覆盖有限**：仅使用了 SkillsBench 的 51 个任务（排除 43 个更复杂且依赖 Docker 环境的任务），可能导致性能被高估。

- **同任务验证偏差**：自我生成技能实验是“从同一任务的成功轨迹生成技能，再验证同一任务”，存在轨迹过拟合风险，跨任务泛化性未评估。

- **未与其他主流基准对比**：未在 GAIA、SWE-bench 等通用智能体基准上评估，通用性未充分检验。
