## 摘要
基于多模态大语言模型的具身智能体在物理环境中解决复杂任务展现了巨大潜力。然而，个性化辅助不仅需要遵循通用指令或识别物体类别。在真实场景中，目标对象往往仅通过先前的交互隐式指定，要求智能体利用长期积累的个性化上下文。本文提出POLAR——一种用于长期用户交互中个性化具身智能体的多模态记忆增强框架。POLAR将先前的交互组织成多模态知识图谱，捕获用于个性化上下文和视觉概念的语义记忆，以及用于具身经验（如智能体轨迹）的情景记忆。为执行具身任务，POLAR检索相关记忆以解释当前请求并指导任务执行。我们基于多种MLLM骨干网络和多样化评估场景评估POLAR，研究记忆在长期个性化中的作用。结果表明，所提出的记忆机制通过更有效地利用先前交互积累的信息，持续提升了性能。当智能体需要在多次交互中进行推理、执行多跳推断或随时间跟踪用户特定上下文更新时，性能提升尤为显著。

## Abstract
Multimodal large language model (MLLM)-based embodied agents have shown strong potential for solving complex tasks in physical environments. However, personalized assistance requires more than following generic instruction or recognizing object categories. In real-world scenarios, the intended target is often specified only implicitly through prior interactions, requiring agents to leverage personalized context accumulated over time. In this work, we propose POLAR, a multiomodal memory-augmented framework for personalized embodied agents over long-term user interactions. POLAR organizes prior interactions into a multimodal knowledge graph that captures semantic memory for personalized context and visual concepts, and episodic memory for embodied experiences such as agent trajectories. To execute embodied tasks, POLAR retrieves relevant memories to interpret the current request and guide task execution. We evaluate POLAR across multiple MLLM backbones and diverse evaluation scenarios to study the role of memory in long-term personalization. Results show that the proposed memory mechanism consistently improves performance by enabling more effective use of information accumulated over prior interactions. The gains are especially pronounced when the agents are required to reason across multiple interactions, perform multi-hop inference, or tracking updates in user-specific context over time.

## 2. 论文提出的方法论

### 核心思想
将用户与智能体的长期交互历史转化为可重用的结构化记忆，并在执行新任务时检索相关记忆辅助推理与规划。记忆分为两类：
- **语义记忆（Semantic Memory）**：每个物体关联一组简洁的文本陈述，描述该物体的用户特定属性或关联（例如“用户通常在出门前往背包里放笔记本电脑”）。这种细粒度设计避免信息纠缠，提升检索精度。
- **情景记忆（Episodic Memory）**：记录智能体在历史交互中的完整轨迹（包括失败/成功状态、哪些房间无目标、哪些观察有用等），并以紧凑文本形式存储，用于指导未来规划。

### 关键技术细节
1. **记忆构建（Memorization）**：
   - 对每次交互，智能体执行任务后生成两类记忆：语义记忆（从用户指令和场景中提取物体相关陈述）和情景记忆（将轨迹转换为规划相关经验）。
   - 构建多模态知识图谱：节点包括物体节点、语义记忆节点、情景记忆节点。通过物体 ID 或视觉相似性将新记忆链接到已有物体节点；若语义重复则合并，避免冗余。边带有时间戳，支持时序更新和最近优先检索。

2. **记忆利用（Utilization）**：
   - 给定当前指令 \(I\)，使用 BGE-M3 编码器进行稠密检索，从语义记忆中检索 top-5 最相关的陈述。
   - 沿图连接找到对应的物体节点，收集关联的指令和情景记忆。
   - 智能体通过 MLLM 推理，基于检索到的候选物体和情景记忆，判断哪个物体是用户意图目标，并规划导航路径。

3. **具身策略**：采用分层规划框架。
   - **高层规划**：先构建场景图（房间 + 邻接关系 + 路径点），基于检索记忆推理最可能包含目标的房间，生成目标路径点。
   - **低层规划**：根据当前 egocentric 视觉观察（前、左、右三视图），选择具体动作（前进、左转、右转、停止），当目标出现时停止。
   - **控制器**：在 Habitat 模拟器中执行动作（移动 1 米，旋转 30 度）。

## 3. 实验设计

- **数据集**：基于 PinNED（Personalized Instance Navigation with Explicit Description）构建，使用 Habitat-Matterport3D（HM3D）场景中的 4 个场景，共 1,817 个 episodes。个性化上下文由 GPT-5 生成（经过人工筛选和 MTurk 验证，确保自然性、清晰度、一致性）。
- **评估场景**（3 类，含子场景）：
  - **组合场景（Compositional）**：单次（目标可从一次历史交互推断） vs 联合（需融合 2-3 次交互信息进行多跳推理）。
  - **干扰场景（Distractor）**：环境中存在同类物体的多个候选，需区分目标实例。
  - **时间场景（Temporal）**：物体上下文随时间变化（偏好更新或物体本身更改），需跟踪最新有效交互。
- **对比方法**：
  - **no-prior**：仅使用当前指令和观测，不提供历史交互。
  - **raw-interaction**：额外提供 15 条随机历史交互日志（包含金标准交互），但不经结构化处理。
- **骨干 MLLM**：开源模型 Qwen3-VL-8B-Instruct、Qwen2.5-VL-8B-Instruct；闭源模型 GPT-5、GPT-4o-mini、Gemini-2.5-Flash。
- **评估指标**：
  - **成功成功率（SR）**：最终位置距目标 < 2 米。
  - **成功加权路径长度（SPL）**：考虑路径效率。
  - **类别匹配（CM）**：达到正确类别但错误实例的比例（越低越好）。
- **实现细节**：基于 NavBench 框架，最大步数 700。
