## 摘要
长期运行的AI代理需要持久记忆。记忆支持跨会话的学习，减少重复上下文注入，并允许对过去决策进行审计。当前的代理记忆系统和数据库范式将记忆视为存储。它们将正确性定位于记录、嵌入或边。每个只提供了长期记忆所需的部分能力。结果是四种反复出现的故障模式：无限制增长、缺失语义修订、容量驱动的遗忘以及只读检索。在我们的愿景中，长期代理记忆是一种新的数据管理工作负载。其正确性是状态轨迹的属性，而非单个记录的属性。我们将其形式化为受控演化记忆（GEM）。GEM用四个状态级操作符取代了记录级数据库操作：摄取、修订、遗忘和检索。六个正确性条件规定了状态如何演化。三个结构性观察表明，无论存储模型如何，没有记录级系统能够满足这些条件。我们在MemState原型中实现了这一抽象，该原型基于属性图后端。MemState验证了可行性，并暴露了与原生引擎之间的差距。我们概述了三个研究方向，将以记忆为中心的数据管理定义为一个工作负载。

## Abstract
Long-running AI agents need persistent memory. Memory supports learning across sessions, reduces repeated context injection, and enables auditing of past decisions. Current agent memory systems and database paradigms treat memory as storage. They localize correctness at records, embeddings, or edges. Each supplies only some of the capabilities that long-term memory requires. The result is four recurring failure modes: unregulated growth, missing semantic revision, capacity-driven forgetting, and read-only retrieval. In our vision, long-term agent memory is a new data-management workload. Its correctness is a property of the state trajectory, not of individual records. We formalize this as Governed Evolving Memory (GEM). GEM replaces record-level database operations with four state-level operators: ingestion, revision, forgetting, and retrieval. Six correctness conditions govern how the state evolves. Three structural observations establish that no record-level system can satisfy these conditions, regardless of the storage model. We realize the abstraction in MemState, a prototype on a property-graph backend. MemState validates feasibility and exposes the gap to a native engine. We outline three research directions that define memory-centric data management as a workload.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：长期运行的 AI 代理需要持久记忆来支持跨会话学习、减少重复上下文注入、审计决策。然而，现有的代理记忆系统和数据库范式都将记忆视为纯存储，将正确性定位于单个记录、嵌入或边，导致四种反复出现的故障模式：① 无节制增长（冗余记录累积）、② 缺失语义修订（更新被追加而非整合）、③ 容量驱动的遗忘（按年龄而非重要性淘汰）、④ 只读检索（检索不修改记忆状态）。
- **背景**：当前系统（如 ChatGPT、Claude、Cursor）继承传统数据库的 CRUD 操作，操作作用于单个记录而非演化中的记忆状态，造成上述故障。更大的上下文窗口或更好的检索质量无法从根本上解决此问题。
- **整体含义**：长期代理记忆应被重新定义为一种新的数据管理工作负载，其正确性属于状态轨迹（state trajectory）的属性，而非单个记录的属性。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出 **受控演化记忆（Governed Evolving Memory, GEM）** 抽象，用四个状态级操作符取代记录级 CRUD，并定义六项正确性条件来治理状态演化。
- **关键技术细节**：
  - **记忆状态**：定义为 \( M_t = (D_t, S_t, P_t) \)，其中 \( D_t \) 是存储内容（语义单元），\( S_t \) 是结构组织（带类型的关系），\( P_t \) 是演化策略（声明性规则）。
  - **四个状态级操作符**：
    - **摄取（Ingestion）**：将新输入整合到现有状态，保留历史值。
    - **修订（Revision）**：基于内部证据 \( \Delta \) 沿结构 \( S_t \) 传播更新，保留溯源。
    - **遗忘（Forgetting）**：受策略驱动，根据相关性信号（显著度）进行分级衰减（压缩→隐藏→归档），非破坏性删除。
    - **检索（Retrieval）**：返回输出并同时更新访问单元的显著度，是状态转换而非只读操作。
  - **六项正确性条件（C1–C6）**：
    - C1（查询正确性）：返回当前最新非归档值。
    - C2（转换正确性）：每次转换遵守策略，不允许超期值被当作当前值返回。
    - C3（依赖一致性）：沿类型边（extension edges）传播更新到依赖单元。
    - C4（溯源保留）：遗忘和修订保留可达单元的历史链。
    - C5（有界活跃状态）：活跃记忆大小受策略定义边界约束。
    - C6（检索诱发适应）：每次检索提升访问单元的显著度，反复检索降低其被衰减的资格。
  - **三个结构性观察**：证明基于 CRUD 的记录级系统（无论底层为何）无法同时满足 C1–C6。例如，纯函数检索不能实现 C6；无语义单元只追加存储不能满足 C2；未类型化传播不能满足 C3。
