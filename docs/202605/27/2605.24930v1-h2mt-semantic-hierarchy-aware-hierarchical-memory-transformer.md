## 摘要
基于Transformer的大语言模型在许多语言任务上取得了显著成果；然而，长输入仍然具有挑战性，因为上下文窗口有限，且预填充延迟和内存随提示长度快速增长。因此，扁平化的令牌流处理和基于块的检索可能会在与查询无关的文本上花费大量计算和上下文预算。离线索引的RAG还引入了外部存储和索引管理开销，并且通常将检索到的证据作为原始文本附加，增加了预填充成本和延迟。H$^{2}$MT使得长上下文推理具有结构感知能力：它离线构建语义层次，通过自底向上的后序聚合为每个节点计算记忆嵌入，并在推理时进行由粗到细的查询路由，以早期剪除不相关的分支。在LongBench QA（NarrativeQA、HotpotQA、QASPER）和两个结构化技术文档设置上，H$^{2}$MT实现了良好的质量效率权衡，与提示压缩、记忆令牌方法和检索增强生成基线相比，在较低的峰值GPU内存和首令牌时间（TTFT）下获得了具有竞争力的ROUGE-L和F1分数（如适用）。

## Abstract
Transformer-based LLMs achieve strong results on many language tasks; however, long inputs remain challenging because context windows are finite, and prefill latency and memory grow rapidly with prompt length. Flat token-stream processing and chunk-based retrieval can therefore spend substantial computation and context budget on text unrelated to the query. Offline-indexed RAG additionally introduces external storage and index management overhead, and typically appends retrieved evidence as raw text, increasing prefill cost and latency. H^{2}MT makes long-context inference structure-aware: it builds a semantic hierarchy offline, computes a memory embedding for each node via bottom-up post-order aggregation, and routes queries coarse-to-fine at inference to prune irrelevant branches early. On LongBench QA (NarrativeQA, HotpotQA, QASPER) and two structured technical-document settings, H MT achieves favorable quality efficiency trade-offs, delivering competitive ROUGE-L and F1 (where applicable) with lower peak GPU memory and time-to-first-token (TTFT) than prompt compression, memory-token methods, and retrieval-augmented generation baselines.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：基于 Transformer 的大语言模型（LLM）在处理长文本时面临三个瓶颈：（1）上下文窗口有限；（2）预填充（prefill）延迟和内存随提示长度二次增长；（3）扁平化的令牌流处理或基于分块的检索会导致大量计算和上下文预算浪费在与查询无关的文本上。传统离线索引的 RAG 还引入额外的外部存储和索引管理开销，且通常将检索证据以原始文本形式拼接，进一步增加预填充成本和延迟。
- **整体含义**：现有方法（如稀疏注意力、线性化注意力、段级循环、外部记忆机制）要么需要较大的架构修改，要么将文档视为平面序列，忽略了文档内在的语义层次结构（如图表中节、子节、定义域）。H²MT 旨在利用文档的语义层次结构来指导信息检索，实现结构感知的高效长上下文推理。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：将文档离线构建为有根语义层次树，为每个节点自底向上计算一个固定大小的记忆嵌入（memory embedding），推理时从根节点开始“由粗到细”路由查询，早期剪枝低分分支，仅将少量选中的节点记忆输入冻结的骨干模型进行生成。
- **关键技术细节**：
  1. **语义层次树构建（离线）**：对于结构化文档（如手册），利用目录元数据；对于非结构化文档，通过启发式分割（如段落边界）生成树。每个节点保存局部文本 \(x_v\)。
  2. **层次记忆令牌构建（离线）**：
     - **叶子节点**：将局部文本嵌入与可学习的写/读嵌入拼接（\( \tilde{E}_v = [e_{\text{write}}; E_v; e_{\text{read}}] \)），通过因果主干 Transformer 得到读位置隐藏状态作为节点记忆 \(m_v\)。
     - **内部节点**：先自底向上后序处理所有子节点，将子节点记忆堆叠成矩阵 \( M_v^{\text{ch}} \)，通过聚合函数 \(\text{Agg}\)（均值池化、自注意力、跨注意力、GAT、父令牌自注意力等）得到子节点摘要 \(\tilde{m}_v\)，再与父节点局部文本拼接后通过骨干得到父节点记忆 \(m_v\)。若局部文本为空，则直接令 \(m_v = \tilde{m}_v\)。
  3. **推理时层次路由（在线）**：
     - **查询嵌入**：将查询文本的前半部分与写/读嵌入拼接通过骨干得到查询向量 \(q\)。
     - **顶层向下路由**：从根节点开始，对每个父节点 \(p\)，使用缩放点积得分 \(s(u|q) = (W_q q)^\top (W_k m_u)/\sqrt{d_h}\) 为每个子节点 \(u\) 评分，保留每个父节点下 Top-\(k\) 子节点作为候选集 \(S_{\ell+1}\)。
     - **剪枝**：早期剪枝低分分支，直到达到叶子、最大深度或记忆预算。
     - **生成**：将最终选中节点的记忆堆叠成 \(M_{\text{ret}}\)，与问题令牌嵌入拼接后输入冻结骨架生成答案。
  4. **训练目标**：
     - 语料训练：对每个节点使用令牌级交叉熵损失 \( \mathcal{L}_{\text{LM}}(v) \)；可选加重构损失 \( \mathcal{L}_{\text{AE}}(v) \)。
     - QA 微调：生成损失 \( \mathcal{L}_{\text{gen}} \) + 路由损失 \( \mathcal{L}_{\text{route}} \)（单 gold 路径的交叉熵）+ 选择损失 \( \mathcal{L}_{\text{sel}} \)（最大化 gold 集合的 softmax 概率和）。所有训练保持骨干冻结，仅训练 LoRA 适配器及轻量 H²MT 模块（\(e_{\text{write}}, e_{\text{read}}, W_q, W_k\) 及聚合参数）。

## 6. 主要结论与发现
- **质量-效率权衡**：H²MT 在大多数设定下以更低 TTFT（通常降低 1-2 个数量级）和更低峰值 GPU 内存（尤其是长文档上）达到与 HMT 相当或更好的 ROUGE-L/F1，尤其在结构化文档（NarrativeQA、OpenROAD）上优势明显。
- **结构化优势**：利用文档内在层次可显著提升效率；对于弱结构文档（HotpotQA），通过 GMM 诱导树优化后 ROUGE-L 从 30.37% 提升至 46.64%，但仍低于保留原始文本的 RAPTOR（56.39%），表明有损压缩会导致表面细节丢失。
- **路由超参数影响**：top-\(k\) 存在最佳值（中等 \(k\)）；聚合策略（尤其图注意力/跨注意力）降低路由噪声，提升鲁棒性。
- **域外泛化**：在高度技术领域的 LVS 数据集上，H²MT 困惑度相比 HMT 降低 78%（PPL 从 11.93 降至 2.62），说明层次压缩更适合专业性强的文档。

## 7. 优点
- **方法创新**：将文档固有语义层次融入长上下文处理，设计从粗到细路由和早期剪枝，避免无关文本处理，符合人类记忆检索模式。
- **灵活插件**：不修改骨架注意力机制，仅增加轻量级读写嵌入和投影，可通过 LoRA 微调，易于集成到现有模型。
- **离线记忆复用**：节点记忆跨查询复用，节省推理计算。
- **训练目标全面**：联合优化记忆构建、路由选择和答案生成，引入路由监督信号提升选择精度。
- **实验覆盖广**：跨多个骨干（135M-14B）和任务类型，消融充分，与多种基线对比。

## 8. 不足与局限
- **依赖文档结构**：对有清晰层次结构的文档效果最好；对于弱结构或噪声结构（如 HotpotQA）需要额外诱导树，且诱导错误会传播。论文承认这是主要限制。
