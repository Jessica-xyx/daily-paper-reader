## 摘要
大型语言模型代理在长期记忆问答中仍面临挑战，因为支持答案的证据常常分散在冗长的对话历史中，并被大量无关内容淹没。现有的记忆系统通常在查询未知之前处理记忆，然后基于相似性而非对回答查询的实用性来检索生成的单元。这种工作流程使得下游回答者需要从检索到的候选中去噪并重构特定于查询的证据。我们提出了DeferMem，一种长期记忆框架，将该问题解耦为高召回率候选检索和基于查询条件的证据提取。DeferMem使用轻量级的片段-链接结构来组织原始历史，并在查询时检索广泛的候选。然后，它应用通过我们的强化学习算法DistillPO训练的记忆提取器，将高召回率但高度噪声的候选提取为一组忠实、自包含且基于查询条件的证据。DistillPO将检索后的证据提取形式化为一个结构化动作，包括消息选择和证据重写。它通过分解并门控的奖励管道和结构对齐的优势分配来优化该动作，从有效性到质量检查进行奖励组件门控，同时早期暴露任务级正确性反馈，并将每个奖励分配给其负责的输出跨度。在LoCoMo和LongMemEval-S上，DeferMem在问答准确性和记忆系统效率方面超越了强基线，实现了最高的问答准确性、最快的运行时间以及记忆操作的零商业API令牌成本。

## Abstract
Large language model (LLM) agents still struggle with long-term memory question answering, where answer-supporting evidence is often scattered across long conversational histories and buried in substantial irrelevant content. Existing memory systems typically process memory before future queries are known, then retrieve the resulting units based on similarity rather than their utility for answering the query. This workflow leaves downstream answerers to denoise retrieved candidates and reconstruct query-specific evidence. We present DeferMem, a long-term memory framework that decouples this problem into high-recall candidate retrieval and query-conditioned evidence distillation. DeferMem uses a lightweight segment-link structure to organize raw history and retrieve broad candidates at query time. It then applies a memory distiller trained with DistillPO, our reinforcement learning algorithm for distilling the high-recall but highly noisy candidates into a set of faithful, self-contained, and query-conditioned evidence. DistillPO formulates post-retrieval evidence distillation as a structured action comprising message selection and evidence rewriting. It optimizes this action with a decomposed-and-gated reward pipeline and structure-aligned advantage assignment, gating reward components from validity to quality checks while exposing task-level correctness feedback early and assigning each reward to its responsible output span. On LoCoMo and LongMemEval-S, DeferMem surpasses strong baselines in QA accuracy and memory-system efficiency, achieving the highest QA accuracy with the fastest runtime and zero commercial-API token cost for memory operations.

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：大型语言模型代理在长期记忆问答中面临挑战，支持答案的证据常分散在长对话历史中，被大量无关内容淹没。现有记忆系统通常在查询未知前预处理记忆，检索时基于相似性而非答案有用性，导致高噪声候选，下游回答需大量去噪和证据重构。
- **背景**：尽管现代LLM支持长上下文窗口，但存在注意力稀释和“中间丢失”问题。外部记忆系统虽能辅助，但预先压缩/遗忘会丢失对特定查询关键的信息，检索依赖间接相关线索（如嵌入相似性），难以直接定位精确证据。

## 2. 方法论：核心思想、技术细节、公式/算法流程
### 核心思想
- **延迟证据蒸馏**：将长期记忆QA解耦为两个阶段：①高召回候选检索（保留原始历史，轻量结构获取广泛候选）②查询条件证据蒸馏（将噪声候选转化为忠实、自包含的答案支持证据）。
- **DistillPO强化学习算法**：训练记忆蒸馏器，将检索后证据蒸馏形式化为结构化动作（消息选择+证据重写），并通过分解门控奖励管道和结构对齐优势分配优化。

### 关键技术细节
- **段-链接结构（Segment-Link）**：
  - 保留原始消息，组织为三个层次：语义段（压缩+局部注意力+相似度阈值切分）；段链接（跨历史语义关联，相似度阈值τ2）；消息级索引。
  - 查询时：检索top-k消息，扩展至所在段及链接段，得到高召回但高噪声候选集。
- **DistillPO算法**：
  - **结构化动作**：输出`(U, Z)`，U为选中的有用消息ID，Z为对应蒸馏证据语句（忠实、自包含、查询条件化）。
  - **分解门控奖励管道**：8个可验证奖励组件（r0~r7），覆盖格式有效性、选择有效性/覆盖/简洁性、证据对齐/忠实性/自包含性、下游答案正确性与失败归因。采用渗漏层次门控：有效性奖励有前置条件，但答案性奖励在格式有效后即开放，避免过度优化低层目标。
  - **结构对齐优势分配**：分别计算选择跨度奖励和证据跨度奖励，在组内归一化得到两个独立优势赋值，分别更新对应token，减少优化混淆。
  - **完整奖励锚点完成**：在每组采样中插入一个离线生成的高质量完成（来自强教师模型且通过全部奖励检查），提供稳定正基准，稳定组相对优化。
- **基策略优化**：基于DAPO，支持组内归一化和剪裁重要性权重。

### 公式/算法流程（文字说明）
- 问题形式化：`Ĉ = R(q, H)`, `Ê = D(q, Ĉ)`，R为高召回检索，D为查询条件证据蒸馏器（πθ）。
- 蒸馏器输出结构化对象o=(U,Z)，通过DistillPO优化：采样一组完成，计算每个token的优势（结构对齐），用类似PPO的剪裁目标更新策略参数。

## 6. 主要结论与发现
- **DeferMem在LongMemEval-S上达70.0%准确率（超过LightMem的68.64%），在LoCoMo非对抗/对抗上达88.25%/97.09%（分别超过最强基线8.04/6.28个百分点）。**
- **效率最优**：记忆操作零商业API令牌成本，运行时间比LightMem快3.1倍（LongMemEval-S）和9.8倍（LoCoMo）。
- 段-链接检索器保持近饱和召回（多数配置>99%准确率），同时可控候选大小。
- DistillPO训练显著优于基模型（+17.8~17.9点）、SFT（+6.6~8.96点）和DAPO（+4.04~5.04点）；其各组件（奖励管道、结构对齐优势、完整奖励锚）均对性能有贡献。
- 单源训练变体（只用一个数据集训练）在另一个数据集上仍表现不错，表明蒸馏行为可跨源迁移。
- 可扩展性良好：在百万token级别历史中准确率仅适度下降。
- 错误主要来自蒸馏侧（57%左右的错误）和回答侧，检索遗漏极少。

## 7. 优点
- **创新思想**：首次明确将长期记忆QA解耦为“高召回检索+查询条件证据蒸馏”，并通过强化学习训练蒸馏器而非简单选择。
- **结构化动作设计**：消息选择与证据重写分离，使优化更清晰。
- **DistillPO算法精巧**：分解门控奖励管道避免稀疏反馈，结构对齐优势分配减少梯度冲突，完整奖励锚点稳定训练。
- **零API成本部署**：一次离线训练后，部署时无需调用商业LLM进行记忆操作，具有成本优势。
- **实验严谨**：多角度评估（准确率、效率、消融、可扩展性、错误分析、定性分析），采用标准LLM裁判，排除训练数据重叠。
- **代码和模型预计开源**（文中提及使用TRL训练，LoRA适配器可公开）。

## 8. 不足与局限
- **训练数据有限**：仅329个QA对，且没有专门为可训练记忆系统设计的基准。论文承认需要更大规模、带黄金证据标注、硬负例及多样领域的专用训练集。

- **仅评估英语对话**：多语言泛化能力未知。
- **开源状态**：截至论文发表，代码/模型未公开提供（但提到基于开源框架TRL和Llama，理论上可复现）。
