## 摘要
实现智能体的自我进化需要在不断变化的任务序列中持续积累新知识，同时不遗忘先前获得的能力。现有方法要么通过更新模型参数来内化知识（导致灾难性遗忘），要么依赖外部记忆（却无法真正增强模型的内在能力）。我们提出MoLEM，一种基于动态混合专家（MoE）的生成式潜在记忆混合框架。我们将多个专家视为独立的记忆载体，通过键-查询匹配的路由器选择和加权专家，并将聚合后的潜在记忆注入推理过程。用于推理的基础模型保持完全冻结，所有经验知识被内化到附加模块中，从而避免灾难性遗忘。对于持续学习，每个训练阶段配备一个轻量级自编码器，在推理时选择适当的路由组，不匹配任何阶段的输入则回退到预训练模型。实验在涵盖数学、科学和代码领域的持续学习序列上训练框架。训练后，我们在相应测试集上评估框架，以衡量跨持续适应阶段的任务学习和能力保持。在整个持续学习序列后，我们的方法相比原始预训练基线平均准确率提升10.40%，而没有任何竞争方法能在不同训练顺序下始终超过这一基线。

## Abstract
Achieving self-evolution in intelligent agents requires the continual accumulation of new knowledge across changing task sequences without forgetting previously acquired abilities. Existing approaches either internalize knowledge by updating model parameters, which induces catastrophic forgetting, or rely on external memory, which fails to genuinely enhance the model's intrinsic capabilities. We propose MoLEM, a generative mixture of latent memory framework based on a dynamic mixture-of-experts (MoE). We treat multiple experts as independent carriers to generate memory. A router selects and weights experts through key-query matching, and the aggregated latent memory is injected into the reasoning process. The base model for reasoning remains entirely frozen, with all experiential knowledge internalized into the additional modules, avoiding catastrophic forgetting. For continual learning, each training stage is paired with a lightweight autoencoder that selects the appropriate routing group at inference, and inputs that match no stage fall back to the pretrained model. Experiments train the framework on continual-learning sequences spanning math, science, and code domains. After training, we evaluate the framework on the corresponding test sets to measure task learning and competence preservation across continual adaptation stages. After the full continual-learning sequence, our method improves the average accuracy by 10.40% over the Vanilla pretrained baseline, while none of the competing methods consistently exceed this baseline across different training orders.

## 1. 论文的核心问题与整体含义（研究动机和背景）

智能体要在不断变化的任务序列中持续进化，需要同时满足两个关键要求：一是能够内化新任务的知识（学习新能力），二是不遗忘之前已获得的能力（避免灾难性遗忘）。现有方法分为两类：
- **参数化记忆**：通过更新模型参数（如 SFT、GRPO）来内化知识，但会导致灾难性遗忘，且共享参数会引起任务间干扰。
- **检索式记忆**：将经验存储在外部结构化数据库中，不修改模型参数，避免了遗忘，但属于“上下文工程”，模型本身没有获得新能力，外部数据库与模型内部权重脱节，无法实现真正的学习。

最近出现的**潜在记忆方法**（如 MemGen、SoftCoT、Coconut）通过连续向量作为高密度记忆载体，在不修改原始参数的情况下内化经验，初步解决了上述矛盾，但仍存在三个关键局限：
1. **静态架构**：模型结构固定，无法根据任务难度或新领域动态扩展容量。
2. **学习到的潜在记忆中的遗忘**：虽然不更新骨干网络，但潜在记忆由可训练参数生成，跨任务复用同一记忆模块会覆盖之前学到的记忆行为。
3. **模块分离缺乏灵活性**：现有框架常以固定流水线分离不同功能模块，缺乏像人脑那样灵活整合多种认知部分的能力。

因此，核心研究问题是如何设计一种灵活、可扩展的认知架构，使得智能体在面对连续变化的任务时，既能动态扩展容量以获取新知识，又能避免遗忘已学能力。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想：动态混合专家（MoE）潜在记忆框架 MoLEM

整体架构包含三个核心组件：
- **专家（Experts）**：每个专家是一个独立的 LoRA 适配器，附加在冻结的基推理器上，生成各自的候选潜在记忆。
- **路由器（Router）**：另一个 LoRA 适配器，通过键-查询匹配计算每个专家的权重，对候选记忆进行加权聚合，得到最终注入推理过程的潜在记忆。
- **每阶段自编码器（Stage-wise Autoencoder, AE）**：每个训练阶段对应一个轻量级自编码器，用于在推理时识别输入所属的领域，实现无任务ID的领域感知路由。

### 关键技术细节

**1. 潜在记忆生成与注入**  
在分隔符（如逗号、句号）位置，激活专家和路由器。每个专家生成候选潜在记忆矩阵 Mₖₜ，路由器通过键-查询匹配得到权重 αₖₜ，加权聚合得到最终记忆 Mₜ，注入回推理隐藏状态，影响后续生成。

**2. 键-查询路由**  
- 路由器从当前推理状态提取查询向量 qₜ（通过投影）。
- 每个专家维护一个可学习键向量 kₖ。
- 匹配分数通过余弦相似度计算：sₖₜ = cos(qₜ, kₖ)。
- 权重通过 softmax 在选中的专家上产生（选 Top-2 专家）。

**3. 训练目标**  
总损失 = 监督微调损失（SFT）+ λ × 负载平衡损失（Switch-style）。  
- SFT 损失最大化目标 token 概率，给定之前 token 和聚合记忆 Mᵢⱼ。  
- 负载平衡损失防止路由集中到少数专家，确保所有可训练专家被均衡使用。

**4. 持续学习策略：阶段式招募 + 领域感知路由**  
- **阶段式招募**：每进入新领域，冻结上一阶段所有参数，招募新的一组专家和路由器（随机初始化），仅新组可训练。知识严格隔离在独立参数子空间。
- **领域感知路由**：每个阶段训练一个独立的自编码器（AE），输入是冻结推理器在提示结束位置的隐藏状态。推理时，计算输入特征到各阶段 AE 的重构误差；选择误差最小且低于该阶段阈值（95% 验证集分位数）的组；若所有误差均超阈值，则判定为 OOD，回退到预训练模型（不激活任何专家记忆）。

**5. 整个流程**：基推理器始终冻结，仅 MoE 记忆模块参数更新，从根本上避免灾难性遗忘。

## 6. 论文的主要结论与发现

1. **持续学习性能**：经过完整序列后，MoLEM 平均准确率 73.93%，超过 Vanilla 基线 10.40%，所有其他方法（SFT、ExpeL、MemGen）均未能在最终阶段超越 Vanilla，甚至低于基线。MoLEM 是唯一带来净提升的方法。

3. **消融证实设计有效性**：隔离路由组是消除遗忘的主导因素；多专家进一步提升了容量；负载平衡损失有助于专家训练。

5. **AE 路由可行性**：冻结推理器的隐藏状态已经提供了充分的领域可分性，使轻量级 AE 能够以无监督方式实现准确的路由和 OOD 回退，无需任务标签。

## 7. 优点

- **架构创新**：将动态 MoE 与潜在记忆结合，既能动态扩展容量（通过招募新专家组），又通过隔离参数子空间和冻结骨干彻底避免灾难性遗忘。
- **无任务ID领域感知路由**：利用冻结推理器隐藏状态的固有判别性，训练独立 AE 进行路由，简单有效且不依赖外部标签，适合真实无任务标签的持续学习设置。
- **OOD 保护**：按阶段阈值拒绝不匹配输入，回退到预训练模型，保护原始泛化能力，避免注入不相关记忆导致性能下降。
- **实验全面性**：包含多种训练顺序、充分消融、可视化分析，结论说服力强。
- **参数高效**：额外模块采用 LoRA 和轻量级 AE，相比冻结骨干参数增加有限。

## 8. 不足与局限

- **模型规模**：仅在 4B 模型上验证，未扩展到 7B、13B 或更大模型，可扩展性未验证。
- **领域多样性有限**：仅涵盖数学、科学、代码三个领域，且均为英文，未涉及多语言、多模态或更复杂的任务类型。
- **样本量较小**：每个领域仅 3000 训练样本，无法代表大规模数据下的学习行为。
- **训练顺序固定为 3 阶段**：未测试更长的任务序列（如 5-10 个域），长期扩展时的路由累积误差或 AE 阈值漂移可能成为问题。
- **额外参数开销**：虽然参数高效，但每个阶段新增的专家和路由器仍带来线性增长的参数，同时 AE 虽然轻量但也是额外模块。论文承认这是必要代价。
- **AE 阈值依赖于验证集**：95% 分位数阈值可能对其他分布敏感，需要人工校准；极端情况下若 OOD 输入恰好落在低误差区域，可能误路由。
- **未讨论推理效率**：多专家和 AE 路由可能增加推理延迟（尤其是每个输入需运行所有已见阶段 AE 进行重构）。论文未提供推理速度对比。
- **未评估遗忘的深层次影响**：虽然辅助指标为 0，但可能需要更精细的评估（如零样本泛化能力、语义理解）来确保完全保留原始能力。
