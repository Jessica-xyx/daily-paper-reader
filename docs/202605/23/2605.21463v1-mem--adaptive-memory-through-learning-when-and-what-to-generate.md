## 摘要
我们提出了Mem-$π$，一个用于大型语言模型（LLM）代理的自适应记忆框架，其中有用的指导是按需生成的，而不是从外部记忆存储中检索的。现有的记忆增强代理通常依赖于从情景记忆库或技能库进行基于相似性的检索，返回与当前上下文往往不对齐的静态条目。相比之下，Mem-$π$使用一个专用的语言或视觉-语言模型，其参数独立于下游代理，为复杂任务生成上下文特定的指导。基于当前代理上下文，该模型共同决定何时生成指导以及生成什么指导。我们使用决策-内容解耦的强化学习（RL）目标对其进行训练，使其能够在生成无益时避免生成，否则生成简洁有用的指导。在跨越网页导航、基于终端的工具使用和基于文本的具身交互的多样化代理基准测试中，Mem-$π$始终优于基于检索和先前RL优化的记忆基线，在网页导航任务上实现了超过30%的相对改进。

## Abstract
We present Mem-$π$, a framework for adaptive memory in large language model (LLM) agents, where useful guidance is generated on demand rather than retrieved from external memory stores. Existing memory-augmented agents typically rely on similarity-based retrieval from episodic memory banks or skill libraries, returning static entries that often misalign with the current context. In contrast, Mem-$π$ uses a dedicated language or vision-language model with its own parameters, separate from the downstream agent, to generate context-specific guidance for complex tasks. Conditioned on the current agent context, the model jointly decides when to produce guidance and what guidance to produce. We train it with a decision-content decoupled reinforcement learning (RL) objective, enabling it to abstain when generation would not help and otherwise produce concise, useful guidance. Across diverse agentic benchmarks spanning web navigation, terminal-based tool use, and text-based embodied interaction, Mem-$π$ consistently outperforms retrieval-based and prior RL-optimized memory baselines, achieving over 30% relative improvement on web navigation tasks.

## 1. 核心问题与整体含义（研究动机和背景）

- **当前困境**：现有LLM代理通常依赖外部记忆模块（如情景记忆库、技能库）进行**检索**，返回的静态条目往往与当前任务上下文不匹配，导致信息过时、不相关甚至误导。
- **认知科学启示**：人类记忆并非字面回放，而是基于当前上下文动态重构的建构过程。受此启发，论文提出将记忆建模为**生成式策略**，按需产生上下文适配的指导。
- **核心问题**：如何让代理学会**何时**生成记忆（而非总是检索）以及**生成什么**内容，以最大化下游任务成功率。

## 2. 方法论

### 2.1 核心思想
- 使用一个独立的、参数化的记忆模型 $\pi_{\text{mem}}$（专用语言/视觉语言模型），与下游代理模型解耦。
- 模型接受任务指令和当前观测作为输入，输出一个**决策令牌**（`[GENERATE]` 或 `[ABSTAIN]`），若生成则输出指导文本。
- 通过两阶段训练：**经验蒸馏**（离线监督学习）+ **适应蒸馏**（在线强化学习）。

### 2.2 关键技术细节

- **第一阶段：经验蒸馏**
  - 利用离线经验库 $\mathcal{E}$（包含上下文-指导对），通过自回归监督目标训练 $\pi_{\text{mem}}$ 掌握生成与上下文匹配指导的能力。
  - 目标函数：$\mathcal{J}^{(1)}_{\text{mem}}(\theta) = \mathbb{E}_{(x,m)\sim\mathcal{E}} \left[ \sum_t \log \pi_{\text{mem}}^{(1)}(m_t | x, m_{<t}; \theta) \right]$

- **第二阶段：适应蒸馏**
  - 初始化自第一阶段，使用强化学习（基于GRPO）优化，奖励信号为下游代理任务成功率。
  - 引入**决策-内容解耦策略优化**：
    - **结构化反事实rollout**：每个上下文强制采样一个`[ABSTAIN]`分支和多个`[GENERATE]`分支，确保两组对比。
    - **优势分解**：计算决策级优势（比较abstain vs generate的整体效果）和内容级优势（在generate分支内归一化）。
    - **逐token信用分配**：决策令牌只接收决策优势，内容令牌只在生成优于失语时接收内容优势（$\Delta$-gating），避免混淆信号。
  - 奖励设计：任务成功奖励（二元）+ 长度正则化惩罚（鼓励简洁）。
  - 最终目标：$\mathcal{J}^{(2)}_{\text{mem}}(\theta) = \mathbb{E}_x \left[ \frac{1}{G} \sum_{j} \frac{1}{|y_j|} \sum_t \min(\rho_{jt} A_{jt}, \text{clip}(\rho_{jt},1-\epsilon,1+\epsilon) A_{jt}) - \beta D_{\text{KL}} \right]$，其中 $A_{jt}$ 为分解后的逐token优势。
