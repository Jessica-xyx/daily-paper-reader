## 摘要
长期智能体记忆日益多模态化，然而现有评估很少测试智能体是否保留后续推理所需的视觉证据。在先前工作中，许多基于视觉的问题仅通过标题或文本痕迹即可回答，使得无需保留细粒度视觉证据即可推断答案。同时，缺乏需要推理变化视觉状态的更困难案例。因此，我们提出MemEye，一个从两个维度评估记忆能力的框架：一维度衡量决定性视觉证据的粒度（从场景级到像素级证据），另一维度衡量检索到的证据必须如何使用（从单一证据到进化合成）。在此框架下，我们构建了涵盖8个生活场景任务的新基准，并带有消融驱动的验证门，以评估可回答性、捷径抗性、视觉必要性和推理结构。通过评估4种VLM骨干网络上的13种记忆方法，我们表明当前架构在保留细粒度视觉细节和推理随时间变化的状态方面仍然困难。我们的发现表明，长期多模态记忆依赖于证据路由、时间追踪和细节提取。

## Abstract
Long-term agent memory is increasingly multimodal, yet existing evaluations rarely test whether agents preserve the visual evidence needed for later reasoning. In prior work, many visually grounded questions can be answered using only captions or textual traces, allowing answers to be inferred without preserving the fine-grained visual evidence. Meanwhile, harder cases that require reasoning over changing visual states are largely absent. Therefore, we introduce MemEye, a framework that evaluates memory capabilities from two dimensions: one measures the granularity of decisive visual evidence (from scene-level to pixel-level evidence), and the other measures how retrieved evidence must be used (from single evidence to evolutionary synthesis). Under this framework, we construct a new benchmark across 8 life-scenario tasks, with ablation-driven validation gates for assessing answerability, shortcut resistance, visual necessity, and reasoning structure. By evaluating 13 memory methods across 4 VLM backbones, we show that current architectures still struggle to preserve fine-grained visual details and reason about state changes over time. Our findings show that long-term multimodal memory depends on evidence routing, temporal tracking, and detail extraction.

- **背景**：视觉语言模型（VLM）驱动的智能体需要在多会话交互中保留并推理视觉信息，但现有基准（如 LoCoMo、MMRC、Mem-Gallery）中“标题→多模态”增益较小，说明对原始图像的依赖性弱。人类视觉记忆能保留丰富细节并追踪变化，但VLM智能体远未达到。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：提出 **MemEye**，一个两维度评估框架：
  - **X轴（视觉证据粒度）**：从粗到细分为四层：
    - X1: 场景级（场景类型、全局语义）
    - X2: 区域级（语义上连贯的子区域、局部空间关系）
    - X3: 实例级（区分具体对象/人身份，尤其是相似候选）
    - X4: 像素级（颜色、纹理、小文本等精细细节）
  - **Y轴（记忆推理深度）**：从简单到复杂分为三层：
    - Y1: 原子检索（单证据单元直接回答）
    - Y2: 关系关联（多个非冲突证据跨会话/模态链接）
    - Y3: 进化综合（证据随时间更新、覆盖、冲突，需推断当前有效状态）
- **技术细节**：
  - 构建一个包含 **371个镜像问题**（多项选择+开放式）的基准，横跨8个生活场景任务（休闲、家庭、专业、个人），共221次会话、848轮对话、438张图像。
  - 每个问题标注其 (X, Y) 坐标。
  - 设置三个过滤/验证门：
    1. **消除对话中的答案泄露**：仅给文本线索，不可回答则保留。
    2. **消除视觉可替代性**：将图像替换为极简标题，若仍可回答则去除或修改。
    3. **控制问题难度**：提供图像与相关线索，确保问题本身可由VLM解答，否则归因为基础模型限制。
  - 多项选择使用4种答案旋转以减少VLM偏差。

- **当前系统远未饱和**：在MemEye上所有方法表现均不理想，最佳方法SRAG(V)在多项选择EM=0.6177，开放式LLM-Judge=0.4937。
- **两个主要失败模式**：
  1. **细粒度视觉证据丢失**：在X3/X4（实例/像素级）上，多模态记忆（保留原始图像）显著优于基于标题的记忆；在X1/X2上两者差距较小。文本记忆易于丢失实例身份、微小属性等。
  2. **进化视觉状态追踪失败**：在Y3（进化综合）上，检索方法可能检索到语义相关但过时的证据，不能优先选择最新有效状态。即使多模态记忆保留图像，也难以从多个视觉证据中选择当前有效状态。简单的时间重新排序（recency）部分改善但非完全解决。
- **跨主题扩展**：随着历史长度和主题多样性增加，专用记忆机制（检索、结构化）比全上下文方法更稳定。
- **启示**：未来多模态记忆应同时结合图像证据、文本/结构化状态记录和证据选择机制（时间信号、压缩、过滤）。

## 7. 优点

- **新颖的两维度评估框架**：将视觉记忆挑战分解为“保留细粒度证据”与“推理时间演变”，有助于诊断系统具体失败点。
- **严格的基准构建流程**：通过多重验证门（短路抵抗、视觉必要性、可回答性控制）确保问题真正依赖视觉记忆，而非基础模型或文本猜测。
- **多维度、多架构的全面评估**：覆盖13种方法、4种骨干、多项指标，并提供诊断实验（检索分析、进化状态探针、规模消融）。
- **开放性和可复现性**：提供项目页面和代码/数据集（计划中），附录详细记录实现、提示、统计。
- **人类验证**：LLM-as-a-Judge与人类高度一致（κ=0.94），并有人工对抽样问题的手动检查。

## 8. 不足与局限

- **基准规模有限**：371个问题、8个任务，覆盖场景有限，可能不能涵盖所有真实部署场景。
- **方法间比较非完全公平**：各方法使用不同的官方嵌入和检索堆栈，作者指出应作为系统级比较，而非纯架构消融。
- **人类上限未充分测量**：人工回答仅针对小样本（60题），且Y3上人类准确率并不完美（0.81），说明问题本身有难度。
- **未涉及实际部署场景**：如隐私、记忆删除、数据最小化等仅作为未来工作提及。
- **Y3问题本身对基础模型也有挑战**：即使在Oracle证据下，Y3的LLM-Judge分数仍低于Y1/Y2（0.558 vs 0.601 vs 0.673），说明部分难度来自推理本身而非记忆。
- **仅评估离线可回答性**：未测量在线交互中的记忆写入/更新过程。
