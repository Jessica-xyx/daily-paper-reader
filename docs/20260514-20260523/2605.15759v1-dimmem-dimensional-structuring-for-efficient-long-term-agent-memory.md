## 摘要
大型语言模型（LLM）智能体需要长期记忆来利用过去交互中的信息。然而，现有记忆系统常面临保真度-效率的权衡：原始对话历史成本高昂，而扁平化的事实或摘要可能丢弃精确回忆所需的结构。我们提出\textbf{DimMem}，一种轻量级维度记忆框架，将每条记忆表示为一个原子的、带类型的、自包含的单元，具有诸如时间、地点、原因、目的和关键词等显式字段。这种表示暴露了维度感知检索、记忆更新和选择性助手上下文回忆所需的结构，无需在模型上下文中存储完整历史。在LoCoMo-10和LongMemEval-S上，DimMem分别达到\textbf{81.43\%}和\textbf{78.20\%}的总体准确率，优于现有轻量级记忆系统，同时将LoCoMo每查询token成本降低\textbf{24\%}。我们进一步证明，维度记忆提取可通过紧凑模型学习：在DimMem模式上微调后，Qwen3-4B提取器在两个基准测试上均超越使用GPT-4.1-mini的LightMem，并在关键设置中达到与更大提取器相当或更优的性能。这些结果表明，显式维度结构化是LLM智能体长期记忆有效且高效的基础。代码见https://github.com/ChowRunFa/DimMem。

## Abstract
Large language model (LLM) agents require long-term memory to leverage information from past interactions. However, existing memory systems often face a fidelity--efficiency trade-off: raw dialogue histories are expensive, while flat facts or summaries may discard the structure needed for precise recall. We propose \textbf{DimMem}, a lightweight dimensional memory framework that represents each memory as an atomic, typed, and self-contained unit with explicit fields such as time, location, reason, purpose, and keywords. This representation exposes the structure needed for dimension-aware retrieval, memory update, and selective assistant-context recall without storing full histories in the model context. Across LoCoMo-10 and LongMemEval-S, DimMem achieves \textbf{81.43\%} and \textbf{78.20\%} overall accuracy, respectively, outperforming existing lightweight memory systems while reducing LoCoMo per-query token cost by \textbf{24\%}. We further show that dimensional memory extraction is learnable by compact models: after fine-tuning on the DimMem schema, a Qwen3-4B extractor surpasses LightMem with GPT-4.1-mini on both benchmarks and reaches performance comparable to, or better than, much larger extractors in key settings. These results suggest that explicit dimensional structuring is an effective and efficient foundation for long-term memory in LLM agents. Code is available at https://github.com/ChowRunFa/DimMem.

- **核心问题**：大型语言模型（LLM）智能体需要长期记忆以利用过去交互信息，但现有记忆系统面临**忠实性与效率的权衡**：原始对话历史存储成本高、检索慢；而扁平化的事实或摘要虽高效，却可能丢弃精确回忆所需的结构（如时间、因果、意图等维度）。此外，基于图或层次结构的记忆系统虽能恢复关系，但构建和维护成本高。
- **研究动机**：作者观察到，长期智能体记忆在检索和更新时往往需要知道**何时、何地、为何、涉及何人/物**，但现有系统未能将这些维度显式暴露。因此，提出**在存储阶段就结构化记忆**，使其成为检索、更新和学习操作的自然接口。
- **整体含义**：DimMem 通过将每条记忆表示为带有显式维度字段的原子单元，为LLM智能体提供了一种轻量级、高保真、高效率的长期记忆基础，且该结构化提取能力可通过小模型微调掌握。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：遵循“**运行时内存维度在构建时暴露**”原则，将对话流转换为原子化、带类型、自包含的维度记忆记录。用户查询也被解析为相同模式，实现共享接口的统一检索、更新和助手上下文回忆。
- **关键技术细节**：
  1. **维度记忆记录表示**：
     - 每条记忆 \( m_i = \langle id_i, C_i, T_i, D_i, \rho_i \rangle \)，其中 \( C_i \) 为自包含文本，\( T_i \in \{事实, 情景, 画像\} \) 为记忆类型，\( D_i \) 包含显式维度：**时间、地点、原因、目的、关键词**（允许空字段避免幻觉），\( \rho_i \) 为指向源窗口/轮次的指针（默认不放入上下文，仅在需要助手回忆时使用）。
  2. **重叠感知提取**：
     - 对话被划分为重叠窗口（窗口大小 / 重叠数：LongMemEval 15/3，LoCoMo 25/5），重叠部分仅用于消解指代、归一化时间等，新记忆只从非重叠后缀提取。先压缩对话内容再提取，降低token消耗并提高信噪比。
  3. **意图识别与维度对齐检索**：
     - 将查询解析为结构化意图：答案维度 (\( A_q \))、目标记忆类型 (\( T_q \))、维度约束 (\( \tilde{D}_q \))、实体锚点 (\( E_q \))、语义查询锚点 (\( C_q \))、助手依赖标志 (\( I_{ast} \))。
     - **三路径检索**：词法匹配（BM25）、密集检索（MiniLM嵌入）、维度匹配（\( s_{dim}(q, m_i) = \sum_{r \in A(q)} \bar{\lambda}_r s_r(q, m_i) \)，仅对查询中激活的维度加权）。最后去重合并。
     - **动态助手回忆**：当 \( I_{ast}=1 \) 时，通过来源指针召回对应的助手回复并附加到检索结果；否则仅使用结构化记忆记录。实现了按需、低成本地利用助手上下文。
  4. **维度感知记忆更新**：
     - 先按记忆类型分组，通过类型化关键词重叠过滤候选对，再经嵌入相似度验证，最后调用LLM合并器决定动作（合并、取代、保留两者）。维度作为廉价预过滤层，大幅减少候选搜索范围。
  5. **学习结构化提取器**：
     - 使用GPT-5.4教师模型生成5000条LoCoMo风格和5000条LongMemEval风格的训练数据，对Qwen3-4B进行LoRA微调。微调后的4B提取器在端到端任务中可与更大提取器（Qwen3-30B-A3B）相当。

## 6. 主要结论与发现

1. **维度结构化显著提升长期推理能力**：DimMem在LoCoMo-10和LongMemEval-S上分别达到 **81.43%** 和 **78.20%** 总体准确率，优于所有轻量级记忆系统。
2. **维度感知操作解决难点**：DA-Update（带维度感知更新）在知识更新、多会话问题上提升明显；动态助手回忆在单会话助手侧问题上提升48.2个百分点，且仅触发12.2%的查询。
3. **紧凑模型可学习高质量维度提取**：微调后的Qwen3-4B提取器+GPT-4.1-mini QA在LoCoMo-10上超越使用Qwen3-30B-A3B提取器的DimMem，表明结构化提取是可学习的、而非大模型特权。
4. **维度表示是最大增益来源**：消融实验显示，去除维度信息（Content-Only Memory）导致总体下降10.39%（LoCoMo）和8.60%（LongMemEval），而仅去除维度匹配或因果/意图字段影响较小。
5. **维度过滤提升更新效率**：在LongMemEval上减少75.5%的嵌入计算量，在LoCoMo上减少59.2%，同时找到更多合并候选。

## 7. 优点

- **创新性强**：首次系统性地将认知心理学中的记忆类型（情景、语义、个人语义）与5W1H事件信息结构结合到LLM智能体记忆中，提出显式维度表示。
- **工程实用**：三路径检索兼顾词法、语义和维度约束，动态助手回忆按需加载，更新采用级联过滤，整体轻量高效。
- **学习成本低**：证明了小模型（4B）可以通过微调掌握结构化提取，降低对商业大模型的依赖。
- **实验充分**：涵盖两个标准基准、多种基线、系统消融和效率分析，论证链条完整。

## 8. 不足与局限

- **应用场景局限**：仅评估文本对话记忆，未测试部署在多模态、隐私敏感或动态环境中的表现。
