## 摘要
记忆系统通常将用户-智能体交互组织为可检索的外部记忆，并通过克服大语言模型（LLM）有限的上下文窗口，对于长期运行的智能体至关重要。然而，现有的记忆系统会调用LLM处理每一个传入的交互以进行记忆提取，这种急切的记忆巩固策略导致了大量的token消耗。为了解决这个问题，我们通过重新思考何时应进行记忆巩固，提出RecMem。RecMem将传入的交互存储在一个潜意识记忆层中，并使用轻量级嵌入模型对其进行编码以便检索。仅当观察到语义相似的交互持续出现时，才调用LLM提取情节记忆和语义记忆。这种基于重复的巩固之所以有效，是因为这些交互对应着一个信息丰富的语义簇，因此值得提取和总结。为了提高准确性，RecMem还引入了一种语义细化机制，以恢复记忆提取遗漏的细粒度事实。实验表明，RecMem在超过三个最先进的记忆系统准确性的同时，将记忆构建的token成本降低了高达87%。

## Abstract
Memory systems often organize user-agent interactions as retrievable external memory and are crucial for long-running agents by overcoming the limited context windows of LLMs. However, existing memory systems invoke LLMs to process every incoming interaction for memory extraction, and such an eager memory consolidation scheme leads to substantial token consumption. To tackle this problem, we propose RecMem by rethinking when memory consolidation should be conducted. RecMem stores incoming interactions in a subconscious memory layer and encode them using lightweight embedding models for retrieval. LLMs are only invoked to extract episodic and semantic memory when sustained recurrence are observed for semantically similar interactions. Such recurrence-based consolidation works because these interactions correspond to a semantic cluster with rich information and thus are worth extraction and summarization. To improve accuracy, RecMem also incorporates a semantic refinement mechanism that recovers the fine-grained facts omitted by memory extraction. Experiments show that RecMem reduces the memory construction token cost of three SOTA memory systems by up to 87% while exceeding their accuracy.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **问题背景**：大语言模型（LLM）作为长期运行的智能体时，需要积累跨多次交互的经验，但其有限上下文窗口和“lost-in-the-middle”效应限制了信息的有效利用。现有的记忆系统（如Mem0、A-Mem、MemoryOS）采用“急切巩固”（eager consolidation）策略——对每次传入的交互都调用LLM提取事实并合并到长期记忆中。这虽然避免了信息遗漏，但导致大量的token消耗，在长时间交互中成本高昂。
- **核心动机**：从认知科学的多存储理论和互补学习系统框架中受到启发——孤立经验只短暂保留，只有重复出现的模式才会被巩固为长期记忆。因此，论文提出**不应对所有交互都进行LLM级别的巩固，而应更智能地选择何时进行记忆巩固**，以兼顾效率和准确性。

## 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：采用**基于重复的巩固（Recurrence-based Consolidation）**，引入一个轻量级的“潜意识记忆”层缓冲原始交互，仅当检测到语义上相似的交互重复出现时，才调用LLM进行高层次的情节和语义记忆提取。
- **架构**：RecMem由三层递进记忆组成：
  - **潜意识记忆（Subconscious Memory）**：将每个交互单元（用户-助手一对消息）用轻量级嵌入模型（text-embedding-3-small）编码成向量，并存储原始文本。构建成本极低（无LLM调用），支持快速检索。
  - **情节记忆（Episodic Memory）**：存储跨轮次的事件摘要（中心化叙事）。当新交互在潜意识中检索到足够多语义相似单元（满足相似度阈值θsim和重复计数阈值θcount）时，触发LLM从这些单元中生成情节记忆（采用“合并优先”策略，避免碎片化）。情节记忆是时间锚定的段落式叙事。
  - **语义记忆（Semantic Memory）**：存储细粒度的原子事实（用户偏好、约束、实体关系等）。在情节记忆生成后，通过**语义细化（Semantic Refinement）**过程：LLM结合原始交互、情节摘要和已有语义事实，提取遗漏的具体细节，并更新或过滤冗余信息，确保事实的精确性和时效性。
- **查询回答**：对用户查询进行嵌入，从三个记忆层分别检索（潜意识ksub=10，情节kepi=5，语义ksem=10），合并后送入LLM生成答案。
- **关键公式/算法**：主要通过文字描述两个阈值：相似度阈值 θsim（默认0.7用于信息密度低的对话，0.6用于任务导向的长对话）和重复次数阈值 θcount（默认5用于一般场景，4用于更长对话）。当新交互的检索集 |Ri| ≥ θcount 且余弦相似度 ≥ θsim 时触发巩固。

- **效率-性能平衡显著**：在LoCoMo上，RecMem将记忆构建token消耗相比Mem0节省87.3%，相比A-Mem节省86.8%，同时准确率（GPT-4.1-mini）达81.1%，超过所有基于记忆的baseline（MemoryOS 67.6%，Mem0 62.9%，A-Mem 68.8%）。在LongMemEval-S上，节省77.5% vs Mem0、71.1% vs A-Mem，总体准确率76.8%，同样最佳。
- **时间推理提升突出**：在需要跨时间联系的任务（Temporal Reasoning）上，RecMem显著优于baseline，这是其基于重复（自动聚类跨轮事件）和情节时序排序的天然优势。

## 8. 不足与局限
- **启发式阈值依赖**：θsim和θcount目前为手动设定，对输入密度敏感，虽敏感性分析显示存在鲁棒区间，但仍需针对不同场景重新调整（附录建议值），缺乏自适应机制。
- **重复作为显著性的代理**：基于“值得巩固的信息会重复出现”这一假设，可能遗漏“一次性但重要”的事件（如安全指令、独特约束）。潜意识层保留原始交互可部分缓解，但缺失跨轮链接和语义细化，可能影响推理。
- **仅评估了两种LLM**：实验限于GPT-4o-mini和GPT-4.1-mini，未测试其他开源模型（如Llama系列），可能影响泛化性结论。
- **未涉及真实部署**：仅在离线基准上测试，未考虑多用户、隐私控制、记忆污染等工程挑战；伦理方面指出了记忆误用和错误记忆的风险。
- **未与参数化记忆（如Fang et al., 2025）或RL强化记忆方法对比**，虽然这些方法在设置上不同，但在更广泛的长记忆系统中可做参考。
- **检索预算固定**：episodic和semantic的检索预算比例k_sem=2*k_epi是人为设定，可能未达到最优。
