## 摘要
我们提出了NeuSymMS，一种自适应记忆系统，通过混合神经符号架构，使大语言模型（LLM）智能体能够跨会话学习、记忆和推理用户信息。NeuSymMS将神经事实提取与非结构化对话相结合，并基于CLIPS的专家系统对事实进行分类、去重和协调，遵循显式的生命周期规则。系统将知识表示为存储在关系数据库管理系统中的主-谓-值三元组，支持用户/智能体/智能体间作用域，并实现了一种基于访问提升和时间剪枝的双视野短期/长期记忆模型。NeuSymMS在保持记忆连续性的同时，避免了上下文窗口膨胀和跨实体污染。我们认为，该架构为生产级智能体系统提供了通向可信、可审计记忆的实用路径，并讨论了其相对于日志检索、摘要和键值方法的新颖性。

## Abstract
We present NeuSymMS, an adaptive memory system that enables large language model (LLM) agents to learn, remember, and reason about users across sessions via a hybrid neuro-symbolic architecture. NeuSymMS couples neural fact extraction from unstructured dialogue with a CLIPS-based expert system that classifies, deduplicates, and reconciles facts under explicit lifecycle rules. The system represents knowledge as subject-relation-value triples stored in relational database management system, supports user/agents/agent-to-agents scoping, and implements a dual-horizon short-term/long-term memory model with access-based promotion and time-based pruning. NeuSymMS maintains continuity of memory while avoiding context-window bloat and cross-entity contamination. We argue that this architecture offers a practical path to trustworthy, auditable memory for production agentic systems and discuss its novelty relative to log retrieval, summarization, and key-value approaches.

### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：将 LLM 用于非结构化对话中的事实提取（神经模块），将分类、矛盾检测、去重、生命周期管理等逻辑性任务交由 CLIPS 专家系统（符号模块）处理。知识表示为 **主体-关系-值（subject-relation-value）三元组**，并采用双视野（短期/长期）记忆模型。
- **关键技术细节**：
  - **事实提取**：使用轻量级 LLM（如 gpt-4.1-mini，温度 0.1）将对话片段转换为 JSON 数组，每个事实包含 subject、relation、value、confidence、scope。每轮最多提取 10 个事实，编码否定关系（如 no_longer_has、lost）。
  - **CLIPS 规则引擎**：本地运行，使用 CLIPSPy 绑定 CLIPS 6.4。三个核心模板：memory-fact（已有知识）、candidate-fact（候选）、engine-decision（数据库操作）。分为三个按优先级执行的阶段：分类、矛盾检测与真值维护、提升/存储。
  - **数据模型**：三元组额外关联用户、智能体/流程标识、九种语义类别（personal, preference, task, relationship, skill, context, instruction, temporal, other）、memory_type（short_term/long_term）、置信度、访问次数、时间戳、软删除标志。
  - **三阶作用域**：user 作用域（全局用户信息）、agent 作用域（特定智能体状态）、flow 作用域（多智能体工作流上下文）。只有 user 作用域事实注入 LLM prompt，避免污染。
  - **双视野记忆**：短期事实默认存储，访问次数 ≥ 3 提升为长期；短期事实超过 24 小时且访问次数为 0 则标记为不活跃（后台剪枝）。
  - **矛盾检测**：对同一 subject-relation 的不同值进行基于编辑距离的近似匹配（相似度阈值 0.85）。单值关系采用“最新值胜出”策略；多值关系仅撤销具体匹配值。否定关系触发旧事实的撤销。
  - **读取路径**：查询活跃的 user 作用域事实，按 memory_type（长期优先）、访问次数、时间排序，最多选取 30 个，格式化为文本块注入 LLM 系统提示。
  - **写入路径示例**：用户说“我刚搬到多伦多，在iVedha开始新工作”，提取器输出两个事实；CLIPS 引擎分类为 personal，检测到原有居住地/雇主矛盾，撤销旧事实，存储新事实。

- 混合神经符号架构（LLM 提取 + CLIPS 规则推理）为 LLM 智能体记忆提供了**实用、可信、可审计**的路径。
- 相比日志检索、摘要、键值存储，NeuSymMS 通过结构化三元组、显式生命周期规则、矛盾处理、作用域隔离等设计，克服了上下文膨胀、跨实体污染、难以更新等问题。

### 7. 优点

- **架构创新**：LLM 仅负责自然语言理解（提取），符号系统负责确定性逻辑（分类、矛盾处理、生命周期决策），实现了各取所长。
- **低成本**：CLIPS 本地运行零 API 成本，提取使用轻量模型，无需向量搜索或额外评分模型。
- **显式可审计**：CLIPS 规则痕迹可检查，三元组存储便于调试和审计。
- **作用域隔离**：三阶作用域（用户/智能体/流程）有效防止跨实体信息污染，适合多智能体平台。
- **自策展机制**：基于访问次数的提升和时间衰减剪枝，模拟认知心理学中的复述和遗忘曲线。
- **人类参与**：提供 Memory UI 界面，用户可查看、编辑、删除事实，增强透明度和隐私控制。
