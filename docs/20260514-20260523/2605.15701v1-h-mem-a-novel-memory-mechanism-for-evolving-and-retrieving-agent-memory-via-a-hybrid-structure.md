## 摘要
记忆数据在基于大语言模型（LLM）的智能体中无处不在（例如OpenClaw和Manus）。近期的一些工作尝试利用智能体的记忆来提升其在问答（QA）任务上的性能，但它们缺乏一种原则性的机制来有效建模记忆数据随时间演化的过程以及高效检索记忆数据，导致记忆利用率低下。为填补这一空白，我们提出H-Mem，一种通过混合结构实现的新型记忆机制，既能有效建模智能体记忆在长期时间内的演化，又能提供高效的记忆检索方法。具体而言，H-Mem构建了一个时间与语义树结构，使短期记忆数据逐步演化为长期记忆数据，后者提供关于前者的总结信息，同时构建一个知识图谱以捕获记忆实体间的关系。此外，它通过利用树与图结构的混合方式提供了一种有效的记忆检索方法。在三个智能体记忆基准上的大量实验表明，H-Mem在问答任务上达到了最先进的性能。

## Abstract
Memory data are ubiquitous in Large Language Model (LLM)-based agents (e.g., OpenClaw and Manus). A few recent works have attempted to exploit agents'memory for improving their performance on the question-answering (QA) task, but they lack a principled mechanism for effectively modeling how memory data evolves over time and retrieving memory data effectively, leading to poor performance in memory utilization. To fill this gap, we present H-Mem, a novel memory mechanism via a hybrid structure that can not only effectively model the evolution of agent memory over a long period of time, but also provide an efficient memory retrieval approach. Particularly, H-Mem builds a temporal and semantic tree structure that allows the short-term memory data to evolve progressively into long-term memory data, where the latter provides summarized information about the former, while simultaneously constructing a knowledge graph to capture the relationships between entities in memory. Moreover, it offers an effective memory retrieval approach by exploiting the hybrid structure of the tree and graph structures. Extensive experiments on three agent memory benchmarks show that H-Mem achieves state-of-the-art performance on the QA task.

## 1. 核心问题与整体含义（研究动机和背景）

大语言模型（LLM）驱动的智能体（如 OpenClaw、Manus）在交互过程中积累了大量的记忆数据（对话历史、任务执行记录等）。利用这些记忆可以提升智能体对用户偏好的理解、语境连贯性和复杂任务执行能力。然而，现有记忆机制存在两个主要局限：

- **缺乏有效的记忆演化建模**：未能显式考虑时间维度，无法将短期记忆逐步整合为长期记忆（即人类记忆巩固过程）。
- **检索不准确**：基于向量索引的方法无法捕获时间抽象和实体关系依赖；树索引无法处理多跳实体关系；图索引缺少记忆演化机制。单一索引结构难以精准检索相关证据。

因此，需要一种能够同时建模长期记忆演化并支持精准检索的新机制。

## 2. 方法论：核心思想、关键技术细节

### 2.1 整体框架

H-Mem 由两个阶段组成：
- **离线索引（Offline Indexing）**：构建混合结构（树 + 图）。
- **在线检索（Online Retrieval）**：利用混合结构进行智能体记忆检索。

### 2.2 混合结构

#### 树结构（Temporal and Semantic Tree）
- **层次化构建**：树有 L 层，叶子节点存储原始记忆事件（每个事件对应一个细粒度语义主题），上层节点存储下层节点记忆的摘要（consolidated summary）。
- **时间窗口**：每一层有预定义的时间窗口（例如 L1=天，L2=周，L3=月，L4=年）。父节点的时间窗口覆盖子节点窗口。
- **语义合并**：当同一层内两个节点时间窗口接近且语义相似度超过阈值 αl 时，它们共享一个父节点，其摘要由子节点合并生成。阈值从低层到高层递减（如 0.8 → 0.7 → 0.6），以支持更抽象的长期记忆。
- **增量更新**：新记忆事件以自底向上方式插入，根据时间窗口和语义相似度合并到现有簇，生成/更新父节点摘要。

#### 图结构（Knowledge Graph）
- **实体与关系提取**：从原始记忆片段中提取命名实体及关系；通过文本归一化、词干化、模糊匹配进行实体消歧与融合。
- **实体节点与边**：每个实体节点链接到包含它的原始记忆片段；关系作为边插入（重复边合并证据）。
- **实体画像**：为重要实体（如人、组织、地点）维护画像，包含持久和近期记忆信息。

### 2.3 在线检索

#### 步骤1：检索规划（Retrieval Planning）
- 使用 LLM 将原始查询 Q 分解为一系列子查询 {Q_k}，并推断每个子查询的记忆范围（SHORT/LONG/MIXED）以及时间提示。
- 生成每个子查询的检索工作流。

#### 步骤2：证据检索（Evidence Retrieval）
- **图检索**：从子查询中提取实体，通过词汇匹配和向量相似度定位图中的种子实体，然后执行多跳扩展以发现更多相关实体。
- **树检索**：将实体映射到原始记忆片段，进而定位到树中的记忆事件。按记忆范围选择检索层级（SHORT 只搜叶子，LONG 只搜上层摘要，MIXED 两者都搜）。
- **证据评分**：对每条记忆证据 m，综合三个因素：
  - 语义相似度 S(m, Q_k)：余弦相似度。
  - 时间相关性 T(m, Q_k)：基于时间重叠和中心距离的公式（式1）。
  - 记忆鲁棒性 R(m, t)：基于艾宾浩斯遗忘曲线，考虑最近巩固时间和重复提及次数（式2）。
  - 最终得分 F(m, Q_k, t) = θ1·S + θ2·T + θ3·R（式3）。已去重排序后形成证据链。

#### 步骤3：生成过程（Generation Process）
- 对每个子查询基于其证据生成子答案；若子查询有依赖关系，则先处理依赖子查询的答案。
- 最终由 LLM 将所有子答案合成为对原始查询的最终答案。

## 6. 主要结论与发现

1. H-Mem 在所有三个基准上取得了最佳 F1 和 LLM-Judge Accuracy，尤其在多跳和时间类问题提升最大。
2. 混合结构（树 + 图）优于单一索引结构：树负责时间与语义演化，图负责实体中心多跳检索，二者互补。
3. 消融实验显示树结构贡献最大（移除后 F1 降幅最明显），图结构和长期记忆摘要也至关重要。
4. 索引和检索成本虽略高于简单方法但合理，性能增益显著。
