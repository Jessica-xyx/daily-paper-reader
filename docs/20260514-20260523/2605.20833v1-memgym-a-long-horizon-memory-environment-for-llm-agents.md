## 摘要
记忆是LLM智能体在长时任务中运行的核心能力。现有的记忆基准测试主要评估多轮对话场景中个性化信息的保留，忽视了智能体长时间执行过程中发生的动态记忆形成。因此，它们生成的记忆系统难以迁移至诸如编码和网页导航等实际智能体环境。我们提出了MemGym，一个面向智能体记忆的基准测试，它将现有的智能体gym和内部基于记忆的流水线统一在一个记忆推理接口之后。MemGym涵盖五个评估轨道，分为四个智能体机制：工具使用对话（tau2-bench）、多轮深度研究搜索（MEMGYM-DR）、编码（SWE-Gym和MEMGYM-CODEQA）以及计算机使用（WebArena-Infinity）。MemGym报告记忆隔离分数，将记忆性能与推理、检索和工具使用能力解耦，从而可以在没有这些混淆变量的情况下对记忆策略进行排名。我们为MEMGYM-CODEQA和MEMGYM-DR构建的合成流水线具有长度可控、每个阶段都经过消融验证，并且与下游场景紧密对齐。为了使编码环境下的评估在学术上易于处理，我们训练了MemRM，一个轻量级奖励模型（使用QLoRA微调的Qwen3-1.7B），它通过快速标量读取来评分压缩质量，替代完整的Docker部署。

## Abstract
Memory is a central capability for LLM agents operating across long-horizon tasks. Existing memory benchmarks predominantly evaluate retention of personalized information in multi-turn chat scenarios, overlooking the dynamic memory formation that occurs during extended agent execution. Consequently, the memory systems they produce transfer poorly to realistic agentic environments, such as coding and web navigation. We present MemGym, a benchmark for agentic memory that unifies existing agent gyms and in-house memory-grounded pipelines behind one memory-reasoning interface. MemGym spans five evaluation tracks grouped into four agentic regimes: tool-use dialogue (tau2-bench), multi-turn deep-research search (MEMGYM-DR), coding (SWE-Gym and MEMGYM-CODEQA), and computer use (WebArena-Infinity). MemGym reports memory-isolated scores that decouple memory performance from reasoning, retrieval, and tool-use ability, so memory strategies can be ranked without those confounders. Our synthetic pipelines for MEMGYM-CODEQA and MEMGYM-DR are length-controllable, ablation-verified at every stage, and tightly aligned with downstream scenarios. To make evaluation on coding environments academically tractable, we train MemRM, a lightweight reward model (Qwen3-1.7B fine-tuned with QLoRA) that scores compression quality as a fast scalar read in place of full Docker rollouts.

### 1. 论文的核心问题与整体含义（研究动机和背景）

LLM智能体在长期、多步骤任务中，必须持续决定哪些信息需要保留、总结或遗忘。这一过程被称为**智能体执行过程中的记忆形成**。然而，现有的主流记忆基准测试存在以下三个核心障碍：

*   **指标混淆**：现有的长时任务环境（如SWE-Gym、WebArena）仅报告最终的任务成功率，将记忆失败与推理、检索和工具使用错误混为一谈，无法独立评估记忆模块的贡献。
*   **虚假记忆压力**：一些看似对记忆要求高的任务，实际上无需显式的记忆管理也能表现良好，因为关键信息可以从已有资料库中重新推导。
*   **评估成本高昂**：例如，在SWE-Gym上进行一次完整的评估需要Docker基础设施和数十步执行，成本高昂，使得在学术界进行系统性的记忆策略迭代变得不切实际。

因此，论文旨在解决“如何独立、公平、低成本的评估LLM智能体的记忆能力”这一核心问题。

### 2. 论文提出的方法论

#### 2.1 核心思想：记忆隔离的统一评估框架

MemGym的核心思想是创建一个统一的评估框架，将记忆模块与推理模型明确分离。通过**记忆隔离评分（memory-isolated scores）**，算法可以独立比较不同记忆策略的表现，而不会被底层推理、检索或工具使用能力的差异所干扰。

#### 2.2 关键技术细节

1.  **统一记忆基础设施（Unified Memory Infrastructure）**：
    *   **接口**：定义了 `manage_context` 接口，所有记忆策略都通过此接口在调用策略LLM前对输入的上下文进行包装（例如压缩、检索、总结）。
    *   **组件**：包括 `BaseEnvironment`, `BaseMemoryManager`, `BaseAgent`, `BaseRunner` 等基类，所有环境（SWE-Gym, WebArena等）都继承自这些基类，确保评估流程统一。
    *   **评估指标**：核心指标是 **记忆增益（memory gain）**，即“带记忆”运行的得分减去“无记忆”基线运行的得分。

2.  **MemRM：轻量级记忆评估信号**：
    *   **动机**：替换昂贵的Docker rollout，使编码环境的评估在学术上可行。
    *   **构成**：一个基于QLoRA微调的Qwen3-1.7B奖励模型。它接受一个三元组（`上下文之前`、`压缩后的上下文`、`候选动作`），并预测一个标量：`Pr[行为不变 | 压缩]`，即该压缩是否安全（不改变下游行为）。
    *   **训练数据**：通过回放已有轨迹，在压缩事件点上生成 `SAFE`（安全）和 `HARMFUL`（有害）的标签。
    *   **工作流**：在推理时，MemRM可以作为一个快速门控，决定是否接受一个压缩事件，从而将一个数分钟的Docker执行替换为一个亚秒级的模型调用。

#### 2.3 合成数据管道（Synthetic Pipelines）

针对两个内部构建的轨道（MEMGYM-DR 和 MEMGYM-CODEQA），论文设计了可控制的合成数据管道：
*   **模板**：遵循“源数据处理 -> 事实提取（区分‘仅需记忆’和‘可通过检索发现’的事实）-> 注入干扰项 -> 长度缩放 -> 验证器认证”的流程。
*   **虚构化（Fictionalization）**：在MEMGYM-DR中，这一步骤至关重要。它将文档中的真实实体（人名、方法名、数字等）替换为虚构的实体，防止前沿模型仅凭预训练知识就能回答问题，从而确保测试的是记忆而非检索预训练知识。
