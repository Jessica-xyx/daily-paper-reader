## 摘要
为了实现可靠的长期交互，LLM智能体需要一个能够忠实存储、高效检索并对累积对话历史进行深度推理的记忆系统。现有大多数方法采用基于抽取事实的范式：手工制作的静态提示将原始对话压缩为原子事实，然后进行存储、匹配并注入下游推理。然而，这种以事实为中心的设计不可避免地丢失了原始对话中的细粒度细节，并且无法支持对分散孤立事实的深度推理。此外，静态提示无法在不同对话风格中保持一致的抽取粒度。为了解决这些局限性，我们提出TriMem，它维护三种共存的表示粒度，包括：由源标识符锚定的原始对话片段以保证存储保真度，抽取的原子事实以实现高效记忆检索，以及将分散事实聚合成整体语义理解的综合画像以实现深度推理。我们进一步采用基于TextGrad的提示优化，通过响应质量反馈迭代优化抽取和画像提示，实现无需任何参数更新的终身进化。在LoCoMo和PerLTQA上使用多种LLM骨干网络的广泛实验表明，TriMem始终优于强大的记忆基线。代码可在https://TMLR-TriMem.github.io获得。

## Abstract
To enable reliable long-term interaction, LLM agents require a memory system that can faithfully store, efficiently retrieve, and deeply reason over accumulated dialogue history. Most existing methods adopt an extracted fact based paradigm: handcrafted static prompts compress raw dialogues into atomic facts, which are then stored, matched, and injected into downstream reasoning. Nevertheless, such fact-centric designs inevitably discard fine-grained details in original dialogues and fail to support deep reasoning over scattered isolated facts. Moreover, static prompts cannot maintain consistent extraction granularity across diverse dialogue styles. To address these limitations, we propose TriMem, which maintains three coexisting representation granularities, including raw dialogue segments anchored by source identifiers for storage fidelity, extracted atomic facts for efficient memory retrieval, synthesized profiles that aggregate dispersed facts into holistic semantic understanding for deep reasoning. We further adopt TextGrad-based prompt optimization, which iteratively refines extraction and profiling prompts via response quality feedback, achieving lifelong evolution without any parameter updating. Extensive experiments on LoCoMo and PerLTQA across multiple LLM backbones demonstrate that TriMem consistently outperforms strong memory baselines. The code is available at https://TMLR-TriMem.github.io .

## 1. 核心问题与研究动机

现有LLM智能体记忆系统普遍采用“基于抽取事实”的范式：使用固定提示词将原始对话压缩为原子事实，然后存储、检索并用于推理。但这种范式存在三个关键局限：
- **有损存储**：事实抽取必然丢弃原始对话中的细粒度细节（如修饰语“trans”），导致信息不可逆丢失。
- **浅层推理**：仅依赖孤立事实无法支持需要综合理解的推理（如情感推断、行为模式归纳），多证据问题性能显著下降。
- **次优提示**：静态提示词无法适应高度异质的对话风格和内容类别，导致抽取粒度不一致，性能波动。

作者通过实验量化了这些问题：抽取事实比原始对话丢失14.5%的信息；多证据问题F1仅35.8%，而单证据为55.3%；不同对话组间性能波动显著。

## 2. 方法论：TriMem

**核心思想**：维护三种共存的表示粒度，分别服务于存储保真度、检索效率和深度推理，并通过基于反馈的提示优化实现终身演化。

### 关键技术细节

1. **多粒度记忆架构**：
   - **原始对话片段（Raw Dialogue Segments）**：通过源标识符（source_dialogue_ids）将每个记忆条目锚定到原始对话窗口，保证可恢复完整细节。
   - **原子事实（Extracted Facts）**：从滑动窗口中提取结构化信息（损失无关复述、时间、人物、实体等），用于高效相似性检索。
   - **综合画像（Synthesized Profiles）**：按人物聚合分散事实，构建结构化画像（身份、性格、兴趣、价值观等），支持理解式推理。

2. **提示优化**：采用TextGrad（基于文本的“梯度”方法），根据答案质量信号（LLM法官评分）生成自然语言编辑建议，迭代优化抽取提示和画像构建提示，无需更新模型参数。

### 流程简述
- **记忆构建**：滑动窗口分割对话 → 多维信息抽取（含源标识符） → 构建每个人物的综合画像。
- **记忆检索**：分析问题所需信息 → 生成搜索查询 → 相似性匹配原子事实 → 通过标识符获取原始对话和画像。
- **持续进化**：根据失败案例由高级模型（Claude Opus 4.6）分析并优化提示。

## 6. 主要结论与发现

1. **TriMem显著优于所有基线**：在LoCoMo上，平均BLEU/F1提升约4-10个点（如GPT-4.1-mini下BLEU 43.79 vs 最好基线39.97）；在PerLTQA上，四个子任务均领先。
2. **性能与效率平衡**：仅消耗约1.2k tokens，远低于Full-Context（16.8k），接近高效基线（如SimpleMem 587 tokens），但性能大幅提升。
3. **兼容性广**：对高能力模型（GPT系列）和轻量模型（Qwen3-8B, Llama-3.1-8B）均有效；无需模型logits，可适应API-only模型。
4. **组件有效性**：消融实验证明画像和原始对话均不可或缺；搜索查询显著提升检索准确率；4轮提示优化达到最佳性能。
5. **终身演化能力**：TextGrad提示优化可在无参数更新条件下自适应提升抽取和画像质量。

## 7. 优点

- **问题诊断深刻**：在提出方法前系统分析现有范式的三个瓶颈，实验证据充分（检索效率、存储完整性、推理性能对比）。
- **设计创新性**：首次在同一系统中融合三种粒度（原始片段、事实、画像），并建立明确索引关联，克服了单一事实表示的固有缺陷。
- **实用性**：提示优化方法无需模型参数更新，兼容闭源模型；token开销可控，适合实际部署。
- **实验充分**：涵盖多种模型规模、多种数据集、多种基线，消融全面，结论可靠。
